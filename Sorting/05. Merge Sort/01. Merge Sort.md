## Merge Sort

### Intuition (Merge Sort)
Merge sort follows **divide and conquer**:
- Break the array into two halves until each part has only **one element** (already sorted).
- Then **merge** two sorted halves into one sorted array.
- Repeating merges builds the fully sorted array.

Key idea: merging two sorted lists can be done efficiently by comparing their front elements.

### Algorithm Steps
**A) `mergesort(arr, start, end)`**  
1. **Base case:** if `start == end`, the subarray has one element → return.
2. Find `mid = start + (end - start) / 2`.
3. Recursively sort the left half: `mergesort(arr, start, mid)`.
4. Recursively sort the right half: `mergesort(arr, mid+1, end)`.
5. Merge the two sorted halves using `mergeArray(arr, start, mid, end)`.

**B) `mergeArray(arr, start, mid, end)`**  
1. Create a temporary array `temp` to store the merged output.
2. Use two pointers:
   - `left = start` (for left half `start..mid`)
   - `right = mid+1` (for right half `mid+1..end`)
3. While both halves have elements:
   - pick the smaller of `arr[left]` and `arr[right]` and push into `temp`.
4. Copy remaining elements from the left half (if any).
5. Copy remaining elements from the right half (if any).
6. Copy `temp` back into `arr[start..end]`.

``` cpp
#include <bits/stdc++.h>
using namespace std;

// needs all three to know the two sorted parts it must merge
void mergeArray(int arr[], int start, int mid, int end)
{
    vector<int> temp(end - start + 1);

    int left = start;
    int right = mid + 1;
    int index = 0;

    // merge both sorted halves
    while (left <= mid && right <= end)
    {

        if (arr[left] <= arr[right])
        {
            temp[index] = arr[left];
            left++;
        }
        else
        {
            temp[index] = arr[right];
            right++;
        }
        index++;
    }

    // left side remaining
    while (left <= mid)
    {
        temp[index] = arr[left];
        left++;
        index++;
    }

    // right side remaining
    while (right <= end)
    {
        temp[index] = arr[right];
        right++;
        index++;
    }

    // copying temp → original array
    index = 0;
    while (start <= end)
    {
        arr[start] = temp[index];
        start++;
        index++;
    }
}

void mergesort(int arr[], int start, int end)
{
    if (start == end)
        return;

    int mid = start + (end - start) / 2;

    // left side
    mergesort(arr, start, mid);

    // right side
    mergesort(arr, mid + 1, end);

   // needs all three to know the two sorted parts it must merge
    mergeArray(arr, start, mid, end);
}

int main()
{
    int arr[] = {7, 3, 9, 1, 4};
    int n = 5;

    mergesort(arr, 0, n - 1);

    for (int x : arr)
        cout << x << " ";
}
```
## Complexity Analysis (Merge Sort)

Let `n` be the number of elements.

Merge sort always:
- splits the array into halves (divide)
- then merges sorted halves (conquer)
This behavior does **not** depend much on the initial order, so best/average/worst are the same.

---

## Time Complexity

### Why it is `O(n log n)`
- **Splitting levels:** each time we divide by 2, so number of levels is `log2(n)`.
- **Work per level:** at each level, all subarrays together contain `n` elements, and merging across that level processes each element once → `O(n)`.

So total:
\[
O(n) \times O(\log n) = O(n \log n)
\]

### Best Case
**Example:** already sorted  
`[1, 2, 3, 4, 5]`

Even though it’s sorted, merge sort still splits into halves and merges them back.

**Time = O(n log n)**

### Average Case
**Example:** random  
`[7, 3, 9, 1, 4]`

Same number of levels and merging work.

**Time = O(n log n)**

### Worst Case
**Example:** reverse sorted  
`[5, 4, 3, 2, 1]`

Still splits and merges in the same way.

**Time = O(n log n)**

---

## Space Complexity

### Extra Space
- `mergeArray` creates a temporary array `temp` of size `(end - start + 1)`
- At any moment, the largest merge uses `O(n)` extra memory.

So:
- **Auxiliary array space:** `O(n)`
- **Recursion stack depth:** `O(log n)`

Overall extra space:
**Space = O(n)** (dominant term)
