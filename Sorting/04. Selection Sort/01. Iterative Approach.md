## Selection Sort

### Intuition (Selection Sort)
Selection sort works like choosing the smallest item repeatedly:
- Split the array into two parts: **sorted (left)** and **unsorted (right)**.
- In each step, find the **minimum element** from the unsorted part.
- Swap it with the first element of the unsorted part.
- This grows the sorted part one element at a time.

After pass `i`, the element at index `i` is the correct smallest element for that position.

### Algorithm Steps
1. For `i` from `0` to `n-2`:
2. Assume the first element of the unsorted part is the minimum: `mini = i`.
3. Search for the smallest element in the range `i+1` to `n-1`:
   - If `arr[j] < arr[mini]`, update `mini = j`.
4. Swap `arr[i]` with `arr[mini]` (put the minimum at position `i`).
5. Repeat until the array becomes fully sorted.

``` cpp
#include <bits/stdc++.h>
using namespace std;

// Function to perform selection sort
void selection_sort(int arr[], int n) {
    // Traverse the array
    for (int i = 0; i < n - 1; i++) {
        int mini = i; // Assume current index holds the minimum value

        // Find the index of the smallest element in the remaining array
        for (int j = i + 1; j < n; j++) {
            if (arr[j] < arr[mini]) {
                mini = j; // Update mini if smaller element is found
            }
        }

        // Swap the found minimum element with the first element of the unsorted part
        int temp = arr[mini];
        arr[mini] = arr[i];
        arr[i] = temp;
    }

    // Print the sorted array
    cout << "After selection sort: " << "\n";
    for (int i = 0; i < n; i++) {
        cout << arr[i] << " ";
    }
    cout << "\n";
}

int main() {
    // Initialize array
    int arr[] = {13, 46, 24, 52, 20, 9};
    int n = sizeof(arr) / sizeof(arr[0]);

    // Print array before sorting
    cout << "Before selection sort: " << "\n";
    for (int i = 0; i < n; i++) {
        cout << arr[i] << " ";
    }
    cout << "\n";

    // Call selection sort
    selection_sort(arr, n);

    return 0;
}
``` 
## Complexity Analysis (Selection Sort)

Let `n` be the number of elements.

Selection sort always does the same scanning work:
- On pass `i`, it finds the minimum in the subarray `arr[i..n-1]`
- That scan takes `(n - i - 1)` comparisons
So comparisons don’t depend on whether the array is sorted or not.

---

### Time Complexity (Best / Average / Worst)
Total comparisons:
\[
(n-1) + (n-2) + \dots + 1 = \frac{n(n-1)}{2}
\]

So in **all cases**:

**Time = O(n^2)**

#### Best Case Example
Example: `[1, 2, 3, 4, 5]` (already sorted)
- Still compares to find the minimum each pass.
- Swaps: usually `0` effective swaps (or self-swap), but comparisons are still `n(n-1)/2`.

#### Average Case Example
Example: `[13, 46, 24, 52, 20, 9]`
- Same number of comparisons `n(n-1)/2`.
- Swaps: at most `n-1`.

#### Worst Case Example
Example: `[5, 4, 3, 2, 1]` (reverse)
- Same number of comparisons `n(n-1)/2`.
- Swaps: at most `n-1`.

---

### Swap Count (Important Note)
- Selection sort does **at most one swap per pass**.
- Total swaps ≤ `n - 1` in all cases.

So:
- Comparisons: `O(n^2)` always
- Swaps: `O(n)` always

---

### Space Complexity
Uses only a few extra variables (`mini`, `temp`).

**Space = O(1)**
