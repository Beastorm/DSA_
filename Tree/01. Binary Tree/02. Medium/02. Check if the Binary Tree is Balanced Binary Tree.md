## Check if the Binary Tree is Balanced Binary Tree
**Problem Statement:**   
Given a Binary Tree, return true if it is a Balanced Binary Tree, else return false. A Binary Tree is balanced if, for all nodes in the tree, the difference between the left and right subtree heights is not more than 1.

---

>### Examples:
### Example 1:
- **Input:** 
```
        3
       / \
      9  20
        /  \
       15   7

```       
- **Output:** Yes  
 


### Example 2:
- **Input:** 
```
      1
     /
    2
     \
      3

```      
- **Output:** No   


---

## Brute Force Approach


### 1. The Core Idea (Intuition)

A tree is balanced if it doesn't tilt too heavily to one side. Specifically, for **every single node** in the tree, its left arm and right arm must not differ in height by more than $1$.

This approach takes a **Top-Down** strategy (like a manager checking a corporate hierarchy):

1. Stand at the current node (Root) and ask: *"How tall is my left side? How tall is my right side?"*
2. If the difference between them is **greater than 1**, stop immediately — the tree is unbalanced.
3. If they look fine, **pass the buck down**: run the exact same check on the left child and the right child to make sure no imbalance is hiding deeper down.

---

### 2. The Algorithm (How it Works)

The logic relies on two distinct recursive functions working together.

---

#### A. The Helper: `getHeight(root)`

- **Job:** Measures the max depth of a subtree.
- **Logic:** If the node is empty (`nullptr`), its height is $0$. Otherwise, it asks for the height of its left and right children, takes the larger one, and adds $1$ (for itself):

$$\text{Height} = \max(\text{LeftHeight},\ \text{RightHeight}) + 1$$

---

#### B. The Checker: `isBalanced(root)`

For any given node, it checks **three strict conditions**:

| # | Condition | Check |
|---|---|---|
| 1 | **Local Balance** | $\lvert \text{LeftHeight} - \text{RightHeight} \rvert \leq 1$ |
| 2 | **Left Subtree Balance** | Is the entire left side internally balanced? → `isBalanced(root->left)` |
| 3 | **Right Subtree Balance** | Is the entire right side internally balanced? → `isBalanced(root->right)` |

If **all three** conditions are `true`, it returns `true`. If **even one** fails, it returns `false`.

### C++ Code:
``` cpp
#include <bits/stdc++.h>
using namespace std;

// Node structure for the binary tree
struct Node {
    int data;       // Data stored in the node
    Node* left;     // Pointer to the left child
    Node* right;    // Pointer to the right child

    // Constructor to initialize the node with a value
    Node(int val) : data(val), left(nullptr), right(nullptr) {}
};

class Solution {
public:
    // Function to check if a binary tree is balanced
    bool isBalanced(Node* root) {
        // If the tree is empty, it's balanced
        if (root == nullptr) {
            return true;
        }

        // Calculate the height of the left and right subtrees
        int leftHeight = getHeight(root->left);
        int rightHeight = getHeight(root->right);

        // Check if the absolute difference in heights of left and right subtrees is <= 1
        if (abs(leftHeight - rightHeight) <= 1 && isBalanced(root->left) && isBalanced(root->right)) { 
            return true;
        }

        // If any condition fails, the tree is unbalanced
        return false;
    }

    // Function to calculate the height of a subtree
    int getHeight(Node* root) {
        // Base case: if the current node is NULL, return 0 (height of an empty tree)
        if (root == nullptr) {
            return 0;
        }

        // Recursively calculate the height of left and right subtrees
        int leftHeight = getHeight(root->left);
        int rightHeight = getHeight(root->right);

        // Return the maximum height of left and right subtrees plus 1 (to account for the current node)
        return max(leftHeight, rightHeight) + 1;
    }
};

// Main function
int main() {
    // Creating a sample binary tree
    Node* root = new Node(1);
    root->left = new Node(2);
    root->right = new Node(3);
    root->left->left = new Node(4);
    root->left->right = new Node(5);
    root->left->right->right = new Node(6);
    root->left->right->right->right = new Node(7);

    // Creating an instance of the Solution class
    Solution solution;

    // Checking if the tree is balanced
    if (solution.isBalanced(root)) {
        cout << "The tree is balanced." << endl;
    } else {
        cout << "The tree is not balanced." << endl;
    }

    return 0;
}
```

### Complexity Analysis:


- **Time Complexity:** $\mathcal{O}(N^2)$ in the worst case (skewed tree).

- _**The Problem:**_ It is inefficient because it **recalculates heights repeatedly**. When checking node 1, it calculates the height of node 2.
  Then, when it moves down to check node 2, it recalculates the heights of nodes 4 and 5 all over again. This double-work compounds heavily down the tree.

- _**In short**_, the code is inefficient because it **recounts the same nodes over and over again from scratch**:
         Whenever the code checks a node, it forces a walk **all the way down to the bottom leaves** to measure the height.
         When it moves down to the next node, it **completely forgets** the previous count and walks all the way down to the leaves a second time.
         Because the top nodes **don't share their answers** with the bottom nodes, the deepest nodes end up getting recounted hundreds of times as the tree grows — causing a massive amount of  wasted, duplicate work.


- **Space Complexity:** $O(H)$, where H is the height of the tree. This space is used by the recursive call stack of the getHeight function. In the worst case (skewed tree), H = N, and in the best case (balanced tree), H = log N. No additional data structures are used, so auxiliary space remains constant.


---

## Optimal Approach

### 1. Intuition:

Instead of standing at the top and looking down repeatedly, this approach starts at the **very bottom leaves** and works its way **up just once**.

Every node does **two jobs at the same time** as it reports upward:

1. It calculates its **own height**.
2. It checks if its **left and right sides are balanced**.

---

#### The `-1` Emergency Brake:

If any node discovers that its left and right sides differ in height by more than $1$, it immediately **hits the emergency brake** and returns `-1`. The value `-1` doesn't represent a real height — it is a **special signal** meaning *"Unbalanced!"*. Once a `-1` is triggered, it **propagates straight up to the root**, terminating all further unnecessary work.

---

### 2. The Algorithm (How it Works)

The entire logic is bundled inside a single optimized function: `dfsHeight(root)`.

| Step | Action |
|---|---|
| **Base Case** | If a node is empty (`nullptr`), its height is `0`. |
| **Check Left** | Ask the left child for its height. If it returns `-1`, pass `-1` upwards immediately. |
| **Check Right** | Ask the right child for its height. If it returns `-1`, pass `-1` upwards immediately. |
| **Check Current Node** | Compare the two returned heights (see below). |

**At the current node:**

- If $\lvert \text{LeftHeight} - \text{RightHeight} \rvert > 1$ → return `-1` (signal an imbalance).
- Otherwise, everything is fine — pass the **actual height** up to the parent:

$$\text{Height} = \max(\text{LeftHeight},\ \text{RightHeight}) + 1$$

---

Finally, `isBalanced()` simply checks the return value of `dfsHeight(root)`:

- Returns `-1` → tree is **unbalanced** (`false`)
- Returns any valid height → tree is **balanced** (`true`)

### C++ Code:
``` cpp
#include <bits/stdc++.h>
using namespace std;

// Node structure for the binary tree
struct Node {
    int data;       // Data stored in the node
    Node* left;     // Pointer to the left child
    Node* right;    // Pointer to the right child

    // Constructor to initialize the node with a value
    Node(int val) : data(val), left(nullptr), right(nullptr) {}
};

class Solution {

public:
    // Function to check if a binary tree is balanced
    bool isBalanced(Node* root) {
        // Check if the tree's height difference between subtrees is less than 2
        // If not, return false; otherwise, return true
        return dfsHeight(root) != -1;
    }

    // Recursive function to calculate the height of the tree
    int dfsHeight(Node* root) {
        // Base case: if the current node is NULL, return 0 (height of an empty tree)
        if (root == NULL) return 0;

        // Recursively calculate the height of the left subtree
        int leftHeight = dfsHeight(root->left);

        // If the left subtree is unbalanced, propagate the unbalanced status
        if (leftHeight == -1) 
            return -1;

        // Recursively calculate the height of the right subtree
        int rightHeight = dfsHeight(root->right);

        // If the right subtree is unbalanced, propagate the unbalanced status
        if (rightHeight == -1) 
            return -1;

        // Check if the difference in height between left and right subtrees is greater than 1
        // If it's greater, the tree is unbalanced, return -1 to propagate the unbalanced status
        if (abs(leftHeight - rightHeight) > 1)  
            return -1;

        // Return the maximum height of left and right subtrees, adding 1 for the current node
        return max(leftHeight, rightHeight) + 1;
    }
};

// Main function
int main() {
    // Creating a sample binary tree
    Node* root = new Node(1);
    root->left = new Node(2);
    root->right = new Node(3);
    root->left->left = new Node(4);
    root->left->right = new Node(5);
    root->left->right->right = new Node(6);
    root->left->right->right->right = new Node(7);

    // Creating an instance of the Solution class
    Solution solution;

    // Checking if the tree is balanced
    if (solution.isBalanced(root)) {
        cout << "The tree is balanced." << endl;
    } else {
        cout << "The tree is not balanced." << endl;
    }

    return 0;
}

```
### Complexity Analysis:
- **Time Complexity:** $O(N)$, where N is the number of nodes in the Binary Tree. Each node is visited exactly once during the postorder traversal.
- **Space Complexity:** $O(1)$, since no extra data structures are used that grow with input size. However, $O(H)$ auxiliary space is used by the recursion stack, where H is the height of the tree. In the best case (balanced tree), H = log₂N; in the worst case (skewed tree), H = N.
