## Minimum Time Taken to BURN the Binary Tree from a Node

**Problem Statement:**   
Given a target node data and the root of a binary tree. If the target is set on fire, determine the shortest amount of time needed to burn the entire binary tree. It is known that in 1 second, all nodes connected to a given node get burned — that is, its left child, right child, and parent.

---

>### Examples:

**Example 1:**
- **Input:** `root = [1, 2, 3, 4, null, 5, 6, null, 7]`, `target = 1`
- **Output:** `3`
- **Explanation:** Node 1 is set on fire. In 1st second, it burns 2 and 3. In 2nd second it burns 4, 5, 6. In 3rd second it burns 7.
```
           1  ← fire starts here
          / \
         2   3         t=1: 2, 3 burn
        /   / \
       4   5   6       t=2: 4, 5, 6 burn
        \
         7             t=3: 7 burns

  Total time = 3 ✓
```

**Example 2:**
- **Input:** `root = [1, 2, 3, null, 5, null, 4]`, `target = 4`
- **Output:** `4`
- **Explanation:** Node 4 is set on fire. In 1st second, it burns 3. In 2nd second it burns 1. In 3rd second it burns 2. In 4th second it burns 5.
```
       1
      / \
     2   3              Fire starts at 4 (right child of 3)
      \    \
       5    4  ← fire

  t=1: 3 burns  (parent of 4)
  t=2: 1 burns  (parent of 3)
  t=3: 2 burns  (child of 1)
  t=4: 5 burns  (child of 2)

  Total time = 4 ✓
```

---

### Key Insight:

In a binary tree, fire can spread in **three directions**: left child, right child, and parent. Since binary trees don't store parent links, we convert the tree into an **undirected graph** (adjacency list) and run BFS from the target node, counting levels until all nodes are burned.
```
  Binary Tree:              Undirected Graph:

       1                    1 — 2 — 4
      / \                   |       |
     2   3         →        3   5   7
      \   \                 |
       5   4                4 (target)

  Fire spreads 1 hop per second in any direction ✓
```

---

### Algorithm:

**Step 1 — Build Undirected Graph:**
Use DFS to traverse the tree and create bidirectional edges between each node and its children.
```
  buildGraph(1, null):
    graph[1] ↔ graph[2]
    graph[1] ↔ graph[3]
    graph[2] ↔ graph[4]
    graph[3] ↔ graph[5]
    graph[3] ↔ graph[6]
    graph[4] ↔ graph[7]
```

**Step 2 — BFS from Target:**
Start BFS from the target node, spreading fire level by level. Increment time only when new nodes are burned.
```
  BFS from target=1, k=?:

  t=0: queue=[1],       visited={1}
  t=1: burn 2, 3        queue=[2,3],   burned=true → time=1
  t=2: burn 4, 5, 6     queue=[4,5,6], burned=true → time=2
  t=3: burn 7           queue=[7],     burned=true → time=3
  t=4: queue empty      burned=false → stop

  Return time = 3 ✓
```

**Step 3 — Return time** once the queue becomes empty.

---

### C++ Code:
```cpp
#include <bits/stdc++.h>
using namespace std;

struct TreeNode {
    int val;
    TreeNode* left;
    TreeNode* right;
    TreeNode(int x) : val(x), left(NULL), right(NULL) {}
};

class Solution {
public:
    int minTime(TreeNode* root, int target) {
        // Step 1: Build an undirected graph
        unordered_map<int, vector<int>> graph;
        buildGraph(root, nullptr, graph);

        // Step 2: BFS from target
        unordered_set<int> visited;
        queue<int> q;
        q.push(target);
        visited.insert(target);

        int time = 0;

        while (!q.empty()) {
            int size = q.size();
            bool burned = false;

            for (int i = 0; i < size; i++) {
                int node = q.front();
                q.pop();

                // Spread fire to all neighbors
                for (int neighbor : graph[node]) {
                    if (!visited.count(neighbor)) {
                        visited.insert(neighbor);
                        q.push(neighbor);
                        burned = true;
                    }
                }
            }

            // Increment time only if fire spreads
            if (burned) time++;
        }

        return time;
    }

private:
    void buildGraph(TreeNode* node, TreeNode* parent,
                    unordered_map<int, vector<int>>& graph) {
        if (!node) return;

        // Bidirectional edge between parent and child
        if (parent) {
            graph[node->val].push_back(parent->val);
            graph[parent->val].push_back(node->val);
        }

        buildGraph(node->left,  node, graph);
        buildGraph(node->right, node, graph);
    }
};

int main() {
    TreeNode* root = new TreeNode(1);
    root->left = new TreeNode(2);
    root->right = new TreeNode(3);
    root->left->left = new TreeNode(4);
    root->right->left = new TreeNode(5);
    root->right->right = new TreeNode(6);
    root->left->left->right = new TreeNode(7);

    Solution sol;
    int target = 1;

    cout << "Minimum time to burn the tree: "
         << sol.minTime(root, target) << endl;

    return 0;
}
```

---

### Step-by-Step Trace:
```
  Tree:
           1
          / \
         2   3
        /   / \
       4   5   6
        \
         7

  Graph (adjacency list):
  1: [2, 3]
  2: [1, 4]
  3: [1, 5, 6]
  4: [2, 7]
  5: [3]
  6: [3]
  7: [4]

  BFS from target=1:

  Start: queue=[1], visited={1}, time=0

  Round 1:
    Process 1 → neighbors: 2(unvisited), 3(unvisited)
    queue=[2,3], burned=true → time=1

  Round 2:
    Process 2 → neighbors: 1(visited), 4(unvisited)
    Process 3 → neighbors: 1(visited), 5(unvisited), 6(unvisited)
    queue=[4,5,6], burned=true → time=2

  Round 3:
    Process 4 → neighbors: 2(visited), 7(unvisited)
    Process 5 → neighbors: 3(visited)
    Process 6 → neighbors: 3(visited)
    queue=[7], burned=true → time=3

  Round 4:
    Process 7 → neighbors: 4(visited)
    queue=[], burned=false → no increment

  Return time = 3 ✓
```

---

### Edge Cases:
```
  Single node:            Target is a leaf:
       1                         1
                                / \
  target=1                     2   3
  No neighbors → time=0 ✓
                          target=2:
                          t=1: 1 burns
                          t=2: 3 burns
                          time=2 ✓

  Linear (skewed) tree:
  1 → 2 → 3 → 4 → 5

  target=3 (middle):
  t=1: 2, 4 burn
  t=2: 1, 5 burn
  time=2 ✓
```

---

### Complexity Analysis:
- **Time Complexity:** O(N). We perform two full traversals of the tree — a DFS to build the graph (O(N)) and a BFS to simulate fire spreading (O(N)). Total = O(N).
- **Space Complexity:** O(N). The adjacency list stores O(N) edges, the visited set holds up to O(N) nodes, and the BFS queue can hold up to O(N) nodes in the worst case.
