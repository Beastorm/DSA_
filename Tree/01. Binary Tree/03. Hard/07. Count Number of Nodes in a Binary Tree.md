## Count Number of Nodes in a Binary Tree

**Problem Statement:**   
Given a Complete Binary Tree, count and return the number of nodes in the given tree. A Complete Binary Tree is a binary tree in which all levels are completely filled, except possibly for the last level, and all nodes are as left as possible.

---

>### Examples:
**Example 1:**
- **Input:** Binary Tree: `1 2 3 4 5 6`
- **Output:** `6`
```
       1
      / \
     2   3
    / \ /
   4  5 6

  Total nodes = 6 ✓
```

**Example 2:**
- **Input:** Binary Tree: `2 4 3 5 9 8 7 1 6`
- **Output:** `9`
```
           2
          / \
         4   3
        / \ / \
       5  9 8  7
      / \
     1   6

  Total nodes = 9 ✓
```

---

### Key Insight: Complete Binary Tree Property
```
  Perfect Binary Tree (all levels full):
           1
          / \
         2   3          height h = 3
        / \ / \          nodes  = 2^h - 1 = 7
       4  5 6  7

  Complete Binary Tree (last level partially filled, left-biased):
           1
          / \
         2   3
        / \ /
       4  5 6            leftHeight = 3, rightHeight = 2
                         leftH ≠ rightH → NOT perfect → recurse
```

The trick: if `leftHeight == rightHeight`, the subtree is a **perfect binary tree** and we can use `2^h - 1` directly without visiting every node.

---

## Brute Force Approach

### Algorithm:

A brute force approach traverses the tree using in-order (or any) traversal and counts nodes one by one.

- Start by initialising a counter variable at 0.
- Define a recursive inorder function with a base case for null nodes.
- Increment the counter for each visited node.
- Recursively traverse left and right subtrees.
- Return the final count.
```
  Inorder traversal count:

       1
      / \
     2   3
    / \ /
   4  5 6

  Visit: 4 → 2 → 5 → 1 → 6 → 3
  Count: 1    2    3    4    5    6 ✓
```

### C++ Code:
```cpp
#include <bits/stdc++.h>
using namespace std;

struct TreeNode {
    int val;
    TreeNode* left;
    TreeNode* right;
    TreeNode(int x) : val(x), left(nullptr), right(nullptr) {}
};

class Solution {
public:
    // Helper to traverse and count nodes
    void inorder(TreeNode* root, int& count) {
        if (root == NULL) return;
        count++;
        inorder(root->left, count);
        inorder(root->right, count);
    }

    // Count total nodes
    int countNodes(TreeNode* root) {
        if (root == NULL) return 0;
        int count = 0;
        inorder(root, count);
        return count;
    }
};

int main() {
    TreeNode* root = new TreeNode(1);
    root->left = new TreeNode(2);
    root->right = new TreeNode(3);
    root->left->left = new TreeNode(4);
    root->left->right = new TreeNode(5);
    root->right->left = new TreeNode(6);

    Solution sol;
    cout << "Total nodes: " << sol.countNodes(root) << endl;

    return 0;
}
```

### Complexity Analysis:

- **Time Complexity:** O(N), each node is visited exactly once.

- **Space Complexity:** O(N) for the recursive call stack in the worst case (skewed tree). For a balanced tree, the stack depth is O(log₂N).

---

## Optimal Approach

### Algorithm:

We exploit the **Complete Binary Tree** property. At any node:
- Compute `leftHeight` (traverse only left pointers).
- Compute `rightHeight` (traverse only right pointers).
- If `leftHeight == rightHeight` → subtree is a **perfect binary tree** → use formula `2^h - 1`.
- If `leftHeight != rightHeight` → recurse on left and right subtrees.
```
  leftHeight == rightHeight?
         YES                    NO
          ↓                      ↓
  return (1<<lh) - 1    return 1 + count(left) + count(right)
  (perfect BST formula)  (recurse deeper)
```

**Why does this work?**
```
  In a complete binary tree, at any node, either:

  Case 1: Left subtree is PERFECT         Case 2: Right subtree is PERFECT
           *                                       *
          / \                                     / \
         *   *    ← rightH matches leftH         *   *
        / \ / \                                 / \
       *  * *  *   use formula! ✓              *   *   use formula for right! ✓
```

### C++ Code:
```cpp
#include <bits/stdc++.h>
using namespace std;

struct TreeNode {
    int val;
    TreeNode* left;
    TreeNode* right;
    TreeNode(int x) : val(x), left(nullptr), right(nullptr) {}
};

class Solution {
public:
    int countNodes(TreeNode* root) {
        if (root == NULL) return 0;

        // Get leftmost height
        int lh = findHeightLeft(root);
        // Get rightmost height
        int rh = findHeightRight(root);

        // Perfect binary tree: use formula
        if (lh == rh) {
            return (1 << lh) - 1;
        }

        // Not perfect: recurse on subtrees
        return 1 + countNodes(root->left) + countNodes(root->right);
    }

    // Traverse leftmost path
    int findHeightLeft(TreeNode* node) {
        int height = 0;
        while (node) {
            height++;
            node = node->left;
        }
        return height;
    }

    // Traverse rightmost path
    int findHeightRight(TreeNode* node) {
        int height = 0;
        while (node) {
            height++;
            node = node->right;
        }
        return height;
    }
};

int main() {
    TreeNode* root = new TreeNode(1);
    root->left = new TreeNode(2);
    root->right = new TreeNode(3);
    root->left->left = new TreeNode(4);
    root->left->right = new TreeNode(5);
    root->right->left = new TreeNode(6);

    Solution sol;
    cout << "Total nodes: " << sol.countNodes(root) << endl;

    return 0;
}
```

### Complexity Analysis:

- **Time Complexity:** O(log N \* log N). Computing `leftHeight` and `rightHeight` each takes O(log N). Recursive calls are made at most O(log N) times (height of the tree).
- **Space Complexity:** O(log N). The recursion depth equals the height of the complete binary tree, which is always log N.

---

### Step-by-Step Trace:
```
  Tree:
         1
        / \
       2   3
      / \ /
     4  5 6

  countNodes(1):
    lh = 3 (1→2→4), rh = 2 (1→3)
    lh ≠ rh → recurse

    countNodes(2):
      lh = 2 (2→4), rh = 2 (2→5)
      lh == rh → return (1<<2) - 1 = 3 ✓

    countNodes(3):
      lh = 2 (3→6), rh = 1 (3→null)
      lh ≠ rh → recurse

      countNodes(6):
        lh = 1, rh = 1 → return (1<<1)-1 = 1 ✓
      countNodes(null):
        return 0

      return 1 + 1 + 0 = 2 ✓

  return 1 + 3 + 2 = 6 ✓
```

## Comparison of Approaches
```
  Approach       Time          Space       Key Idea
  ──────────────────────────────────────────────────────
  Brute Force    O(N)          O(N)        Visit every node
  Optimal        O(log N       O(log N)    Use perfect BST formula
                  * log N)                 to skip subtrees
```
```
  Nodes visited in optimal:

  Full tree (N=15):                 Brute visits all 15
           1                        Optimal:
          / \                       At root: lh=4, rh=4 → formula → done!
         2   3                      Visits only O(log N) nodes per level
        ...                         = O(log N * log N) total ✓
```
