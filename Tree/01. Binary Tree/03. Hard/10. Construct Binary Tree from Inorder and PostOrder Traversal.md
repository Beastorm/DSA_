## Construct Binary Tree from Inorder and PostOrder Traversal

**Problem Statement:**   
Given the Postorder and Inorder traversal of a Binary Tree, construct the Unique Binary Tree represented by them.

---

>### Examples:

**Example 1:**
- **Input:** `Inorder: [40, 20, 50, 10, 60, 30]`, `Postorder: [40, 50, 20, 60, 30, 10]`
```
  postorder = [40, 50, 20, 60, 30, 10]
                                   ↑
                                root = 10

  inorder   = [40, 20, 50, 10, 60, 30]
                             ↑
                         root at index 3

  Left  of root in inorder: [40, 20, 50] → left subtree
  Right of root in inorder: [60, 30]     → right subtree

  Result:
           10
          /  \
        20    30
       /  \  /
      40  50 60
```

**Example 2:**
- **Input:** `Inorder: [9, 3, 15, 20, 7]`, `Postorder: [9, 15, 7, 20, 3]`
```
  postorder = [9, 15, 7, 20, 3]
                              ↑
                           root = 3

  inorder   = [9, 3, 15, 20, 7]
                  ↑
              root at index 1

  Left:  [9]        → left subtree
  Right: [15, 20, 7] → right subtree

  Result:
         3
        / \
       9   20
          /  \
         15   7
```

---

### Key Insight:
```
  Preorder  [Root, Left, Right]: root at START of array
  Postorder [Left, Right, Root]: root at END   of array ← KEY DIFFERENCE

  postorder = [40, 50, 20, 60, 30, 10]
                                    ↑
                                  ROOT

  inorder   = [40, 20, 50, 10, 60, 30]
                             ↑
                     splits into LEFT and RIGHT

  leftSize = inRoot - inStart = 3 - 0 = 3
  Left  postorder slice: [40, 50, 20]  (ps to ps+leftSize-1)
  Right postorder slice: [60, 30]      (ps+leftSize to pe-1)
```

---

### Comparison: Preorder vs Postorder Construction
```
  Preorder-Inorder:              Postorder-Inorder:
  root = preorder[preStart]      root = postorder[postEnd]
  (first element)                (last element)

  Left:  preStart+1 to           Left:  ps to
         preStart+numsLeft              ps+numsLeft-1
  Right: preStart+numsLeft+1     Right: ps+numsLeft to
         to preEnd                      pe-1
```

---
## Approach
### Algorithm:

- Create a hashmap storing indices of elements in the inorder array for O(1) lookup.
- Define a recursive function with parameters for current ranges in inorder and postorder arrays.
- **Base case:** If `postStart > postEnd` or `inStart > inEnd`, return `null`.
- Current root is `postorder[postEnd]` (last element).
- Find its index in inorder using the hashmap.
- Calculate `leftSubtreeSize = rootIndex - inStart`.
- Recursively build left subtree: `inStart to rootIndex-1`, `postStart to postStart+leftSize-1`.
- Recursively build right subtree: `inRoot+1 to inEnd`, `postStart+leftSize to postEnd-1`.
- Return the constructed root.
```
  Index boundaries at each call:

  build(is, ie, ps, pe):
    root      = postorder[pe]
    inRoot    = hm[root]
    numsLeft  = inRoot - is

    LEFT:  build(is,       inRoot-1,        ps,            ps+numsLeft-1)
    RIGHT: build(inRoot+1, ie,              ps+numsLeft,   pe-1         )
```

---

### C++ Code:
```cpp
#include <bits/stdc++.h>
using namespace std;

struct TreeNode {
    int val;
    TreeNode *left, *right;
    TreeNode(int x) : val(x), left(nullptr), right(nullptr) {}
};

class Solution {
public:
    TreeNode* buildTree(vector<int>& inorder, vector<int>& postorder) {
        if (inorder.size() != postorder.size()) return nullptr;

        // Build hashmap: value → index in inorder
        map<int, int> hm;
        for (int i = 0; i < inorder.size(); i++) {
            hm[inorder[i]] = i;
        }

        return build(inorder,  0, inorder.size() - 1, postorder, 0, postorder.size() - 1, hm);
    }

private:
    TreeNode* build(vector<int>& inorder,  int is, int ie, vector<int>& postorder, int ps, int pe, map<int, int>& hm) {

        // Base case
        if (ps > pe || is > ie) return nullptr;

        // Root is the LAST element in the current postorder range
        TreeNode* root = new TreeNode(postorder[pe]);

        // Find root in inorder
        int inRoot   = hm[postorder[pe]];
        int numsLeft = inRoot - is;

        // Build left subtree
        root->left = build(inorder, is, inRoot - 1, postorder, ps, ps + numsLeft - 1, hm);

        // Build right subtree
        root->right = build(inorder, inRoot + 1, ie, postorder, ps + numsLeft, pe - 1, hm);

        return root;
    }
};

void printInorder(TreeNode* root) {
    if (!root) return;
    printInorder(root->left);
    cout << root->val << " ";
    printInorder(root->right);
}

int main() {
    vector<int> inorder   = {40, 20, 50, 10, 60, 30};
    vector<int> postorder = {40, 50, 20, 60, 30, 10};

    cout << "Inorder Vector:   ";
    for (int x : inorder)   cout << x << " ";
    cout << "\nPostorder Vector: ";
    for (int x : postorder) cout << x << " ";
    cout << endl;

    Solution sol;
    TreeNode* root = sol.buildTree(inorder, postorder);

    cout << "Inorder of Unique Binary Tree Created: ";
    printInorder(root);
    cout << endl;

    return 0;
}
```

---

### Step-by-Step Trace:
```
  inorder   = [40, 20, 50, 10, 60, 30]
  postorder = [40, 50, 20, 60, 30, 10]
  hm        = {40:0, 20:1, 50:2, 10:3, 60:4, 30:5}

  build(in:0-5, post:0-5):
    root = 10, inRoot=3, numsLeft=3
    LEFT:  build(in:0-2, post:0-2)
    RIGHT: build(in:4-5, post:3-4)

  build(in:0-2, post:0-2):
    root = 20, inRoot=1, numsLeft=1
    LEFT:  build(in:0-0, post:0-0) → root=40 (leaf) ✓
    RIGHT: build(in:2-2, post:1-1) → root=50 (leaf) ✓

  build(in:4-5, post:3-4):
    root = 30, inRoot=5, numsLeft=1
    LEFT:  build(in:4-4, post:3-3) → root=60 (leaf) ✓
    RIGHT: build(in:6-5 → invalid) → null

  Final tree:
           10
          /  \
        20    30
       /  \  /
      40  50 60   ✓
```

---

### Edge Cases:
```
  Single node:              Right skewed:
  inorder   = [1]           inorder   = [1, 2, 3]
  postorder = [1]           postorder = [3, 2, 1]

  root=1, no children ✓     root=1 (post[2])
                            inRoot=0, numsLeft=0
                            RIGHT: root=2, right=3
                            Tree: 1→2→3 (right skewed) ✓
```

---

### Complexity Analysis:
- **Time Complexity:** O(N), every node is visited exactly once. The hashmap provides O(1) root lookups in inorder.
- **Space Complexity:** O(N), due to the hashmap storing N entries, plus O(H) for the recursion stack where H is the height of the tree.
```
  Without hashmap: O(N²) — scan inorder each time
  With hashmap:    O(N)  — O(1) lookup per node ✓
```
