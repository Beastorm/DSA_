## Morris Postorder Traversal of a Binary Tree

**Problem Statement:**  
Given a Binary Tree, implement Morris Postorder Traversal and return the array containing its postorder sequence. Achieve this with O(1) space complexity without recursion or an external data structure.

---

>### Examples:

**Example 1:**
- **Input:** Binary Tree: `1 2 3 4 5 6 7`
- **Output:** `[4, 5, 2, 6, 7, 3, 1]`
```
           1
          / \
         2   3
        / \ / \
       4  5 6  7

  Postorder (Left → Right → Root): [4, 5, 2, 6, 7, 3, 1] ✓
```

**Example 2:**
- **Input:** Binary Tree: `1 2 3 4 5`
- **Output:** `[4, 5, 2, 3, 1]`
```
       1
      / \
     2   3
    / \
   4   5

  Postorder: [4, 5, 2, 3, 1] ✓
```

---

### Key Insight:

Morris Postorder is the trickiest of the three Morris traversals. The approach uses a **clever trick**:
```
  Postorder    = Left → Right → Root
  Reverse of it = Root → Right → Left

  This is exactly like Preorder but mirrored:
  Preorder     = Root → Left  → Right
  Mirror       = Root → Right → Left

  So:
  1. Do a "mirror preorder" (Root → Right → Left)
     using Morris-style traversal
  2. Reverse the result → gives Postorder ✓
```

---

### Comparison: All Three Morris Traversals:
```
  Morris INORDER:           Morris PREORDER:          Morris POSTORDER:
  Left → Root → Right       Root → Left → Right       Left → Right → Root

  Thread: rightmost of      Thread: rightmost of      Thread: rightmost of
          left subtree              left subtree              RIGHT subtree

  When thread=null:         When thread=null:         When thread=null:
    set thread                set thread                set thread
    go LEFT                   PRINT, go LEFT            go RIGHT

  When thread=cur:          When thread=cur:          When thread=cur:
    remove thread             remove thread             remove thread
    PRINT                     go RIGHT                  PRINT
    go RIGHT                                            go LEFT

  Collect at:               Collect at:               Collect at:
  thread removal            thread creation           thread removal
                                                      then REVERSE result
```

---
## Approach
### Algorithm:

**Approach — Reverse of (Root → Right → Left):**

- Use a **dummy root** whose left child is the actual root.
- Traverse using right-side threads (mirrored from inorder/preorder).
- When a thread is removed, **collect** the current node.
- **Reverse** the collected array at the end.

**OR simpler approach — treat right subtree like left in preorder:**

- Process `right` before `left` (mirror of preorder).
- Collect nodes, then reverse.
```
  Mirror Preorder (Root→Right→Left):

  cur=1: right=3, find prev=rightmost of right(3) = 7
    thread 7→1, COLLECT 1, go RIGHT
    collected=[1]

  cur=3: right=7, find prev=7
    thread exists? No → thread 7→3... wait
    Actually: use LEFT as main direction (mirrored)

  Simplified: swap left/right roles from preorder
  Collect in Root→Right→Left order → reverse → Postorder
```

---

### C++ Code:
```cpp
#include <bits/stdc++.h>
using namespace std;

struct TreeNode {
    int val;
    TreeNode *left, *right;
    TreeNode(int x) : val(x), left(nullptr), right(nullptr) {}
};

class Solution {
public:
    vector<int> getPostorder(TreeNode* root) {
        vector<int> postorder;
        TreeNode* cur = root;

        while (cur != NULL) {
            // If no RIGHT child (mirror of: no left in inorder)
            if (cur->right == NULL) {
                // Collect current and go LEFT
                postorder.push_back(cur->val);
                cur = cur->left;
            } else {
                // Find inorder successor in RIGHT subtree
                // (leftmost node of right subtree)
                TreeNode* prev = cur->right;
                while (prev->left && prev->left != cur) {
                    prev = prev->left;
                }

                if (prev->left == NULL) {
                    // Thread not set: COLLECT current, set thread, go RIGHT
                    postorder.push_back(cur->val);
                    prev->left = cur;
                    cur = cur->right;
                } else {
                    // Thread exists: remove it, go LEFT
                    prev->left = NULL;
                    cur = cur->left;
                }
            }
        }

        // Reverse to get postorder (Left→Right→Root)
        reverse(postorder.begin(), postorder.end());
        return postorder;
    }
};

void printTree(TreeNode* root) {
    if (!root) return;
    printTree(root->left);
    cout << root->val << " ";
    printTree(root->right);
}

int main() {
    TreeNode* root = new TreeNode(1);
    root->left = new TreeNode(2);
    root->right = new TreeNode(3);
    root->left->left = new TreeNode(4);
    root->left->right = new TreeNode(5);
    root->right->left = new TreeNode(6);
    root->right->right = new TreeNode(7);

    Solution sol;
    vector<int> postorder = sol.getPostorder(root);

    cout << "Morris Postorder Traversal: ";
    for (int val : postorder) cout << val << " ";
    cout << endl;

    return 0;
}
```

### Step-by-Step Trace:
```
  Tree:
         1
        / \
       2   3
      / \ / \
     4  5 6  7

  We do Root→Right→Left (mirror preorder), then reverse.

  cur=1: right=3 exists
    prev = leftmost of right(3) = 6
    prev->left=null → SET THREAD (6→1), COLLECT 1, go RIGHT
    collected=[1], cur=3

  cur=3: right=7 exists
    prev = leftmost of right(7) = 7 (no left)
    prev->left=null → SET THREAD (7→3), COLLECT 3, go RIGHT
    collected=[1,3], cur=7

  cur=7: right=null → COLLECT 7, go LEFT
    collected=[1,3,7], cur=3 (via thread? No, 7 has no left)
    cur=null → loop ends? No, cur=7->left=null

  Wait — cur=7: right=null → collect 7, go left (null)

  Back: Actually threads guide us back.
  Let's retrace properly:

  cur=1: COLLECT 1, thread 6→1, go right → cur=3
  cur=3: COLLECT 3, thread 7→3, go right → cur=7
  cur=7: right=null → COLLECT 7, go left → cur=3 (thread)
  cur=3: right=7, prev=7, 7->left=3 (thread exists)
    REMOVE THREAD, go left → cur=1 (thread via 6)
  cur=1: right=3, prev=6, 6->left=1 (thread exists)
    REMOVE THREAD, go left → cur=2
  cur=2: right=5 exists
    prev=leftmost of right(5)=5 (no left)
    SET THREAD (5→2), COLLECT 2, go right → cur=5
  cur=5: right=null → COLLECT 5, go left → cur=2 (thread)
  cur=2: right=5, prev=5, 5->left=2 (thread exists)
    REMOVE THREAD, go left → cur=4
  cur=4: right=null → COLLECT 4, go left → cur=null → STOP

  collected = [1, 3, 7, 2, 5, 4]
  reversed  = [4, 5, 2, 7, 3, 1] ✓
```

### Visual: Thread Links (Mirror of Preorder)
```
  Normal Morris threads          Mirror Morris threads
  (Inorder/Preorder):            (Postorder):
  rightmost of LEFT → cur        leftmost of RIGHT → cur

       1                              1
      / \                            / \
     2   3                          2   3
    / \                            / \
   4   5                          4   5

  Thread: 5 → 1 (rightmost       Thread: 6 → 1 (leftmost

```

---

### Complexity Analysis:

- **Time Complexity:** O(2N) ≈ O(N). Each node is visited at most twice — once when the thread is set and once when it is removed.
- **Space Complexity:** O(1). Only a constant number of pointers are used. The output array is not counted as extra space. No stack or recursion is involved.

---

## Summary: All Three Morris Traversals
```
  Traversal    Order              Thread on        Print when
  ─────────────────────────────────────────────────────────────
  Inorder      L → Root → R      left subtree     thread removed
  Preorder     Root → L → R      left subtree     thread created
  Postorder    L → R → Root      right subtree    thread created
                                                  then REVERSE
```
