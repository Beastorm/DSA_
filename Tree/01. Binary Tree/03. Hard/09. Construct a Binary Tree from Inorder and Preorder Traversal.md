## Construct a Binary Tree from Inorder and Preorder Traversal

**Problem Statement:**    
Given the Preorder and Inorder traversal of a Binary Tree, construct the Unique Binary Tree represented by them.

---

>### Examples:

**Example 1:**
- **Input:** `preorder = [3, 9, 20, 15, 7]`, `inorder = [9, 3, 15, 20, 7]`
- **Output:** `[3, 9, 20, null, null, 15, 7]`
```
  preorder = [3, 9, 20, 15, 7]
               ↑
             root = 3

  inorder  = [9, 3, 15, 20, 7]
                  ↑
              root at index 1

  Left of root  in inorder: [9]       → left subtree
  Right of root in inorder: [15,20,7] → right subtree

  Result:
         3
        / \
       9   20
          /  \
         15   7
```

**Example 2:**
- **Input:** `preorder = [3, 4, 5, 6, 2, 9]`, `inorder = [5, 4, 6, 3, 2, 9]`
- **Output:** `[3, 4, 2, 5, 6, null, 9]`
```
  preorder = [3, 4, 5, 6, 2, 9]
               ↑
             root = 3

  inorder  = [5, 4, 6, 3, 2, 9]
                        ↑
                    root at index 3

  Left:  [5, 4, 6] → left subtree
  Right: [2, 9]    → right subtree

  Result:
         3
        / \
       4   2
      / \   \
     5   6   9
```

---

### Key Insight:

Two fundamental properties drive this algorithm:
```
  Preorder  [Root, Left, Right]:  first element = ROOT
  Inorder   [Left, Root, Right]:  root splits the array into LEFT and RIGHT subtrees

  preorder = [3,  9,  20, 15,  7]
              ↑
            root

  inorder  = [9,  3,  15, 20,  7]
              ←left→ ↑  ←right→
                    root
                  index=1

  leftSize = rootIndex - inStart = 1 - 0 = 1
  Left  preorder slice: [9]        (preStart+1 to preStart+leftSize)
  Right preorder slice: [20,15,7]  (preStart+leftSize+1 to preEnd)
```

---
## Approach
### Algorithm:

- Store each value's index from the inorder traversal in a hashmap for O(1) lookup.
- Create a recursive function `buildTree` that takes preorder, inorder, and their current index boundaries.
- **Base case:** If boundaries are invalid (`preStart > preEnd` or `inStart > inEnd`), return `NULL`.
- Identify the root as `preorder[preStart]` and find its index in inorder using the hashmap.
- Calculate `leftSubtreeSize = rootIndex - inStart`.
- Recursively build the left and right subtrees using updated index boundaries.
- Return the root with left and right children connected.
```
  Index boundaries for each recursive call:

  build(preStart, preEnd, inStart, inEnd):

    root = preorder[preStart]
    inRoot = inMap[root]
    numsLeft = inRoot - inStart

    LEFT:  build(preStart+1,          preStart+numsLeft, inStart,    inRoot-1)
    RIGHT: build(preStart+numsLeft+1, preEnd,            inRoot+1,   inEnd  )
```

---

### C++ Code:
```cpp
#include <bits/stdc++.h>
using namespace std;

struct TreeNode {
    int val;
    TreeNode *left;
    TreeNode *right;
    TreeNode(int x) : val(x), left(nullptr), right(nullptr) {}
};

class Solution {
public:
    TreeNode* buildTree(vector<int>& preorder, vector<int>& inorder) {
        // Build hashmap: value → index in inorder
        map<int, int> inMap;
        for (int i = 0; i < inorder.size(); i++) {
            inMap[inorder[i]] = i;
        }

        return build(preorder, 0, preorder.size() - 1,
                     inorder,  0, inorder.size()  - 1, inMap);
    }

private:
    TreeNode* build(vector<int>& preorder, int preStart, int preEnd, vector<int>& inorder, int inStart, int inEnd,
map<int, int>& inMap) {

        // Base case
        if (preStart > preEnd || inStart > inEnd) return nullptr;

        // Root is always the first element of the current preorder range
        TreeNode* root = new TreeNode(preorder[preStart]);

        // Find root's position in inorder
        int inRoot   = inMap[root->val];
        int numsLeft = inRoot - inStart;

        // Build left subtree
        root->left = build(preorder, preStart + 1, preStart + numsLeft,
                           inorder, inStart, inRoot - 1,
                           inMap);

        // Build right subtree
        root->right = build(preorder, preStart + numsLeft + 1, preEnd,
                            inorder, inRoot + 1, inEnd,
                            inMap);

        return root;
    }
};

void printInorder(TreeNode* root) {
    if (!root) return;
    printInorder(root->left);
    cout << root->val << " ";
    printInorder(root->right);
}

int main() {
    vector<int> inorder  = {9, 3, 15, 20, 7};
    vector<int> preorder = {3, 9, 20, 15, 7};

    Solution sol;
    TreeNode* root = sol.buildTree(preorder, inorder);

    cout << "Inorder of Unique Binary Tree Created: ";
    printInorder(root);
    cout << endl;

    return 0;
}
```

---

### Step-by-Step Trace:
```
  preorder = [3, 9, 20, 15, 7]
  inorder  = [9, 3, 15, 20, 7]
  inMap    = {9:0, 3:1, 15:2, 20:3, 7:4}

  build(pre:0-4, in:0-4):
    root = 3, inRoot=1, numsLeft=1
    LEFT:  build(pre:1-1, in:0-0)
    RIGHT: build(pre:2-4, in:2-4)

  build(pre:1-1, in:0-0):    → root=9 (leaf) ✓

  build(pre:2-4, in:2-4):
    root = 20, inRoot=3, numsLeft=1
    LEFT:  build(pre:3-3, in:2-2) → root=15 (leaf) ✓
    RIGHT: build(pre:4-4, in:4-4) → root=7  (leaf) ✓

  Final tree:
         3
        / \
       9   20
          /  \
         15   7   ✓
```

---

### Edge Cases:
```
  Single node:              Left skewed:
  preorder = [1]            preorder = [1,2,3]
  inorder  = [1]            inorder  = [3,2,1]

  root = 1, no children     root=1, left subtree=[2,3]
  return node(1) ✓            root=2, left=[3]
                                root=3 (leaf)
                            Tree: 1→2→3 (left skewed) ✓
```

---

### Complexity Analysis:
- **Time Complexity:** O(N), as each node is visited exactly once. The hashmap provides O(1) lookup for root positions in inorder.
- **Space Complexity:** O(N), for the hashmap storing all N inorder indices, plus O(H) for the recursion stack where H is the height of the tree. In the worst case (skewed tree), H = N.
```
  Without hashmap: O(N²) — linear scan each time to find root in inorder
  With hashmap:    O(N)  — O(1) lookup per node ✓
```
