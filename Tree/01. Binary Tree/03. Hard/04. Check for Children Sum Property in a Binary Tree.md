## Check for Children Sum Property in a Binary Tree

**Problem Statement:**   
Given a Binary Tree, convert the value of its nodes to follow the Children Sum Property. The Children Sum Property in a binary tree states that for every node, the sum of its children's values (if they exist) should be equal to the node's value. If a child is missing, its value is 0.

**Note:**
1. The node values can be increased by any positive integer any number of times, but decrementing any node value is not allowed.
2. A value for a NULL node can be assumed as 0.
3. We cannot change the structure of the given binary tree.

---

>### Examples:

**Example 1:**
- **Input:** Binary Tree: `2 35 10 2 3 5 2`
- **Output:** Binary Tree: `45 35 10 30 5 8 2`
- **Explanation:** We cannot decrement the value of the node, but only increment it. We increase values so that each node's value equals the sum of its left and right children.
```
  Before:              After:
       2                    45
      / \                  /  \
    35   10              35    10
    / \ / \              / \  /  \
   2  3 5  2            30  5 8   2

  Verify:
  35 + 10 = 45 ✓
  30 + 5  = 35 ✓
  8  + 2  = 10 ✓
```

**Example 2:**
- **Input:** Binary Tree: `50 7 2 3 5 1 30`
- **Output:** Binary Tree: `50 55 5 86 1 31 30`
- **Explanation:** We modify the tree so that the value of each node equals the sum of its left and right children. Since we cannot decrease values, children are increased to match parent when needed.
```
  Before:              After:
       50                   50
      /  \                 /  \
     7    2              55    5
    / \  / \             / \  / \
   3  5  1 30           86  1 31  30  (wait, let's verify)
                        ← children increased to match the parent ✓
```

---

### Key Insight:

We cannot decrease values, only increase. This creates a challenge:
```
  Problem scenario:
       10             Parent = 10
      /  \            Left = 3, Right = 4
     3    4           Sum = 7 < 10

  We CANNOT reduce parent (10→7) ✗
  We MUST increase a child to match the parent:
    Set left child = 10 (parent's value)
       10
      /  \
    10    4
  Then recurse DOWN, then update parent UP
```

The trick is a **two-pass approach at each node**:
1. **Top-down:** If children sum < parent, push parent's value down to a child.
2. **Recurse** into children.
3. **Bottom-up:** Update parent to equal the sum of (now-updated) children.

---

### Algorithm:

- Only node values can be increased; decreasing is not allowed. Tree structure stays fixed.
- Bottom-up adjustment fails when the child sum is more than the parent, since we can't reduce the parent.
- If children sum is less than the node, we increase children's values recursively to match.
- Use recursion to visit every node and check the Children Sum Property at each level.
- If children ≥ node → update node to match child sum.
- If children < node → update one child to match parent (since we can't decrease).
- After processing both children, update the node value to the final left + right sum.
```
  At each node:

  Step 1 — Compare:
    childSum = left->val + right->val
    if childSum >= node->val:
      node->val = childSum       ← pull up
    else:
      leftChild->val = node->val ← push down

  Step 2 — Recurse:
    changeTree(left)
    changeTree(right)

  Step 3 — Update node:
    node->val = left->val + right->val  ← final correction
```

---

### C++ Code:
```cpp
#include <iostream>
using namespace std;

// TreeNode structure
struct TreeNode {
    int val;
    TreeNode *left;
    TreeNode *right;
    TreeNode(int x) : val(x), left(nullptr), right(nullptr) {}
};

class Solution {
public:
    void changeTree(TreeNode* root) {
        // Base case
        if (root == NULL) return;

        // Step 1: Calculate the sum of children
        int child = 0;
        if (root->left)  child += root->left->val;
        if (root->right) child += root->right->val;

        // Compare and adjust
        if (child >= root->val) {
            // Children sum is enough → update node
            root->val = child;
        } else {
            // Children sum too small → push node value down to a child
            if (root->left)       root->left->val  = root->val;
            else if (root->right) root->right->val = root->val;
        }

        // Step 2: Recurse into children
        changeTree(root->left);
        changeTree(root->right);

        // Step 3: Update node to sum of (now updated) children
        int tot = 0;
        if (root->left)  tot += root->left->val;
        if (root->right) tot += root->right->val;

        if (root->left || root->right) {
            root->val = tot;
        }
    }
};

void inorderTraversal(TreeNode* root) {
    if (root == nullptr) return;
    inorderTraversal(root->left);
    cout << root->val << " ";
    inorderTraversal(root->right);
}

int main() {
    TreeNode* root = new TreeNode(3);
    root->left = new TreeNode(5);
    root->right = new TreeNode(1);
    root->left->left = new TreeNode(6);
    root->left->right = new TreeNode(2);
    root->right->left = new TreeNode(0);
    root->right->right = new TreeNode(8);
    root->left->right->left = new TreeNode(7);
    root->left->right->right = new TreeNode(4);

    Solution sol;

    cout << "Binary Tree before modification: ";
    inorderTraversal(root);
    cout << endl;

    sol.changeTree(root);

    cout << "Binary Tree after Children Sum Property: ";
    inorderTraversal(root);
    cout << endl;

    return 0;
}
```

### Step-by-Step Trace:
```
  Initial tree:
           3
          / \
         5   1
        / \ / \
       6  2 0  8
         / \
        7   4

  Process node(3): child = 5+1 = 6 > 3 → node=6
  Process node(5): child = 6+2 = 8 > 5 → node=8
  Process node(6): leaf → no change
  Process node(2): child = 7+4 = 11 > 2 → node=11
  Process node(7): leaf → no change
  Process node(4): leaf → no change
  Back at node(2): tot = 7+4 = 11 → node=11 ✓
  Back at node(5): tot = 6+11 = 17 → node=17 ✓
  Process node(1): child = 0+8 = 8 > 1 → node=8
  Process node(0): leaf → no change
  Process node(8): leaf → no change
  Back at node(1): tot = 0+8 = 8 → node=8 ✓
  Back at node(3): tot = 17+8 = 25 → node=25 ✓

  Final tree:
           25
          /  \
         17   8
        /  \ / \
       6  11 0  8
         /  \
        7    4
```

### Edge Cases:
```
  Single node (leaf):      Node with one child:
       5                         10
                                /
  → No children, no change     3
  val stays 5 ✓
                           child=3 < 10 → push 10 down
                                10
                               /
                              10
                           recurse → leaf stays 10
                           back: tot=10 → node=10 ✓
```

---

### Complexity Analysis:

- **Time Complexity:** O(N), where N is the number of nodes in the binary tree. The algorithm traverses each node exactly once, performing constant-time operations at each node.
- **Space Complexity:** O(N), where N is the number of nodes in the Binary Tree, due to the recursive call stack, which in the worst case (skewed tree) holds N frames.
