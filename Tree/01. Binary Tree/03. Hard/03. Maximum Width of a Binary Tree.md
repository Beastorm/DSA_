## Maximum Width of a Binary Tree

**Problem Statement:**   
Given a Binary Tree, return its maximum width. The maximum width of a Binary Tree is the maximum diameter among all its levels. The width or diameter of a level is the number of nodes between the leftmost and rightmost nodes.

---

>### Examples:

**Example 1:**
- **Input:** Binary Tree: `1 2 3 5 6 -1 9`
- **Output:** `Maximum Width: 4`
- **Explanation:** Level 3 is the widest level of the Binary Tree, whose end-to-end width is 4, comprising nodes: `{5, 6, null, 9}`.
```
          1                Level 1: width = 1
         / \
        2   3              Level 2: width = 2
       / \    \
      5   6    9           Level 3: width = 4
                           (5 to 9, including null gap)

  Index assignment:
  Node 1 → index 0
  Node 2 → index 1 (2*0+1), Node 3 → index 2 (2*0+2)
  Node 5 → index 3, Node 6 → index 4, Node 9 → index 6

  Width at level 3 = lastIndex - firstIndex + 1 = 6 - 3 + 1 = 4 ✓
```

**Example 2:**
- **Input:** Binary Tree: `1 2 3 5`
- **Output:** `Maximum Width: 2`
- **Explanation:** Level 2 is the widest level of the Binary Tree, whose end-to-end width is comprised of nodes: `{2, 3}`.
```
          1                Level 1: width = 1
         / \
        2   3              Level 2: width = 2
       /
      5                    Level 3: width = 1

  Maximum Width = 2 ✓
```

---

### Key Insight:

Assign indices to nodes as if the tree were a **complete binary tree**:
- Root → index `0`
- Left child of node at index `i` → `2*i + 1`
- Right child of node at index `i` → `2*i + 2`

Width at each level = `lastIndex - firstIndex + 1`
```
  Complete binary tree indexing:

            0
           / \
          1   2
         / \ / \
        3  4 5  6

  Width at level 3 = 6 - 3 + 1 = 4

  Sparse tree:
            0
           / \
          1   2
         /       \
        3          6      ← null nodes in between still count

  Width at level 3 = 6 - 3 + 1 = 4
```

> **Note:** We subtract `minIndex` at each level to prevent integer overflow for deep trees.

---

## Algorithm

- Use level-order traversal (BFS) and track each node's position as if the tree were a complete binary tree.
- For each level in the tree, note the position of the first and last node and compute the width as the difference between these two positions.
- For the next level, assign positions to children. After processing all levels, return the maximum width recorded.
```
  BFS level processing:

  Queue: [(node, index)]

  For each level:
    minIndex = front of queue's index (to normalize and avoid overflow)
    first = first node's relative index
    last  = last node's relative index
    width = last - first + 1
    Push children with indices 2*currIndex+1 and 2*currIndex+2
```

---

### C++ Code:
```cpp
#include <bits/stdc++.h>
using namespace std;

class TreeNode {
public:
    int val;
    TreeNode* left;
    TreeNode* right;

    TreeNode(int x) {
        val = x;
        left = nullptr;
        right = nullptr;
    }
};

class Solution {
public:
    int widthOfBinaryTree(TreeNode* root) {
        // If the tree is empty, return 0
        if (!root) return 0;

        int maxWidth = 0;

        // Queue stores {node, index} pairs
        queue<pair<TreeNode*, int>> q;
        q.push({root, 0});

        while (!q.empty()) {
            int size = q.size();

            // Normalize index to prevent overflow
            int minIndex = q.front().second;
            int first = 0, last = 0;

            for (int i = 0; i < size; i++) {
                int currIndex = q.front().second - minIndex;
                TreeNode* node = q.front().first;
                q.pop();

                // Track first and last index at this level
                if (i == 0) first = currIndex;
                if (i == size - 1) last = currIndex;

                // Push children with calculated indices
                if (node->left)
                    q.push({node->left,  2 * currIndex + 1});
                if (node->right)
                    q.push({node->right, 2 * currIndex + 2});
            }

            // Update maximum width
            maxWidth = max(maxWidth, last - first + 1);
        }

        return maxWidth;
    }
};

int main() {
    TreeNode* root = new TreeNode(1);
    root->left = new TreeNode(3);
    root->right = new TreeNode(2);
    root->left->left = new TreeNode(5);
    root->left->right = new TreeNode(3);
    root->right->right = new TreeNode(9);

    Solution sol;
    cout << "Maximum width: " << sol.widthOfBinaryTree(root) << endl;

    return 0;
}
```

---

### Step-by-Step Trace
```
  Tree:
          1 (idx=0)
         / \
        3   2
       / \    \
      5   3    9

  Level 1:
    Queue: [(1,0)]
    minIndex=0, first=0, last=0
    Push: (3,1), (2,2)
    width = 0 - 0 + 1 = 1, maxWidth = 1

  Level 2:
    Queue: [(3,1), (2,2)]
    minIndex=1
    currIndex of 3 = 1-1 = 0 → first=0
    currIndex of 2 = 2-1 = 1 → last=1
    Push: (5,1), (3,2) from node 3; (9,4) from node 2
    width = 1 - 0 + 1 = 2, maxWidth = 2

  Level 3:
    Queue: [(5,1), (3,2), (9,4)]
    minIndex=1
    currIndex of 5 = 0 → first=0
    currIndex of 3 = 1
    currIndex of 9 = 3 → last=3
    width = 3 - 0 + 1 = 4, maxWidth = 4

  Final answer: 4 ✓
```

---

### Edge Cases:
```
  Single node:             Left skewed:
       1                       1
                              /
  width = 1 ✓               2
                            /
                           3
                       width = 1 at every level ✓

  Perfect binary tree:
           1
          / \
         2   3
        / \ / \
       4  5 6  7
  Level 3 width = 7 - 4 + 1 = 4 ✓
```

---

### Complexity Analysis:

- **Time Complexity:** O(N), each node is processed once in level order traversal.

- **Space Complexity:** O(N), in the worst case, the queue holds all the nodes of the last level, i.e., N/2 nodes.
