## Constructing a unique binary tree

**The core problem:**   
To reconstruct a binary tree exactly, you need to know not just *what the root is*, but *which nodes go left and which go right*.

**The three traversals:**

| Traversal | Pattern | Root position |
|---|---|---|
| Pre-order | Root → Left → Right | **First** element |
| In-order | Left → Root → Right | **Middle** element |
| Post-order | Left → Right → Root | **Last** element |

**Why In-order is the key:**
In-order traversal places the root in the *middle* of the sequence. This means once you identify the root, everything to its left belongs to the left subtree, and everything to its right belongs to the right subtree. No other traversal does this.

**The two valid combinations:**

- **In-order + Pre-order** — Use Pre-order to find the root (first element), then use In-order to split the remaining nodes into left and right subtrees. Repeat recursively.

- **In-order + Post-order** — Use Post-order to find the root (last element), then use In-order to split the remaining nodes. Repeat recursively.

**Why Pre-order + Post-order alone fails:**
Both can identify the root, but neither can tell you whether a child node belongs to the *left* or *right* subtree. Multiple different tree shapes can produce the same Pre-order and Post-order sequences, so the tree cannot be uniquely determined.

> **The rule:** In-order is mandatory. Pair it with either Pre-order or Post-order to uniquely reconstruct a binary tree.

Here's a visual to make this concrete: 
```
           A          ← root
          / \
         B   C
        / \ / \
       D  E F  G

```
In the inorder sequence, **A sits exactly in the middle** — everything to its left (D, B, E) is the left subtree, everything to its right (F, C, G) is the right subtree. That's the splitting power that makes a unique reconstruction possible.
