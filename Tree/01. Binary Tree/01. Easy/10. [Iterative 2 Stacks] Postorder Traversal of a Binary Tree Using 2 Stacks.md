## Iterative Postorder Traversal of a Binary Tree Using 2 Stacks

**Problem Statement:**  
Given the root of a Binary Tree, create a function that performs a postorder traversal using two stacks and returns an array containing the traversal sequence.

---

>### Examples:
### Example 1:
**Input:** Binary Tree:
```
         4
       /   \
      2     5
     /     / 
    3     6   
   / \   / \
  1   9 7   8

```
  
**Output:** [1, 9, 3, 2, 7, 8, 6, 5, 4]   
**Explanation:**  We traverse the binary tree in the order of Left, Right, and then Root recursively. This results in the following traversal:  
1 → 9 → 3 → 2 → 7 → 8 → 6 → 5 → 4.

---

## Approach
### Iterative Post-order Traversal using Two Stacks

> Intuition broken down simply and clearly.

---

### 1. The Core Trick: "Reverse Pre-order"

**Post-order** traversal visits nodes in the order:

$$\text{Left} \rightarrow \text{Right} \rightarrow \text{Root}$$

If you take that exact sequence and **reverse it completely**, it becomes:

$$\text{Root} \rightarrow \text{Right} \rightarrow \text{Left}$$

This looks almost exactly like a **Pre-order** traversal ($\text{Root} \rightarrow \text{Left} \rightarrow \text{Right}$), except we visit the **Right child before the Left child**.

---

### Why Two Stacks?

| Stack | Role |
|---|---|
| `st1` | Handles the **traversal strategy** — walks the tree in $\text{Root} \rightarrow \text{Right} \rightarrow \text{Left}$ order |
| `st2` | Acts as a **storage box / data inverter** — since stacks are LIFO, nodes fed in as $\text{Root} \rightarrow \text{Right} \rightarrow \text{Left}$ automatically pop out in the correct post-order: $\text{Left} \rightarrow \text{Right} \rightarrow \text{Root}$ |

---

### 2. Intuition of the Algorithm:

> Think of `st1` as a **processing conveyor belt** and `st2` as a **stack of plates being prepped for serving**.

---

### Phase 1 — Traversal (`while !st1.empty()`)

1. **Pop** a node from `st1`.
2. **Push** it straight into `st2`.
   - Since the root enters `st2` first, it sinks to the very bottom — guaranteeing it gets printed **last**.
3. **Push its children** onto `st1` — Left child first, then Right child.
   - **Why this order?** Since `st1` is a stack, pushing Left then Right means the **Right child sits on top** and gets popped next. This keeps the processing order as $\text{Root} \rightarrow \text{Right} \rightarrow \text{Left}$.

---

### Phase 2 — Inverting & Printing (`while !st2.empty()`)

Once `st1` is completely empty, `st2` holds every node in **perfectly reversed order**. Pop nodes out of `st2` one by one and push their values into the result vector — that's your post-order output.

---

### Summary:

```
st1 traversal order  →  Root → Right → Left
                                 ↓
                        fed into st2 (LIFO)
                                 ↓
st2 pop order        →  Left → Right → Root  ✓ (Post-order)
```

### C++ Code:
``` cpp
#include <bits/stdc++.h>
using namespace std;

// Node structure for the binary tree
struct Node {
    int data;   // Value of the node
    Node* left; // Pointer to the left child
    Node* right; // Pointer to the right child
    
    // Constructor to initialize the node with a value
    Node(int val) : data(val), left(nullptr), right(nullptr) {}
};

// Function to return the post-order traversal of a binary tree using two stacks
vector<int> postOrder(Node* root) {
    vector<int> postorder; // Vector to store the postorder traversal

    // If the tree is empty, return an empty traversal
    if(root == NULL){
        return postorder;
    }

    // Two stacks for iterative traversal
    stack<Node*> st1, st2;

    // Push the root node onto the first stack
    st1.push(root);

    // Iterative traversal to populate st2 with nodes in postorder
    while(!st1.empty()){
        // Get the top node from st1
        root = st1.top();
        st1.pop();

        // Push the node onto st2
        st2.push(root);

        // Push left child onto st1 if exists
        if(root->left != NULL){
            st1.push(root->left);
        }

        // Push right child onto st1 if exists
        if(root->right != NULL){
            st1.push(root->right);
        }
    }

    // Populate the postorder traversal vector by popping st2
    while(!st2.empty()){
        postorder.push_back(st2.top()->data);
        st2.pop();
    }

    // Return the postorder traversal
    return postorder;
}

// Function to print the elements of a vector
void printVector(const vector<int>& vec) {
    // Iterate through the vector and print each element
    for (int num : vec) {
        cout << num << " ";
    }
    cout << endl;
}

// Main function
int main()
{
    // Creating a sample binary tree
    Node* root = new Node(1);
    root->left = new Node(2);
    root->right = new Node(3);
    root->left->left = new Node(4);
    root->left->right = new Node(5);

    // Getting postorder traversal
    vector<int> result = postOrder(root);

    // Printing the postorder traversal result
    cout << "Postorder traversal: ";
    printVector(result);

    return 0;
}
```

### Complexity Analysis:
- **Time Complexity:** O(N), where N is the number of nodes in the binary tree. Each node is visited once during the traversal.
- **Space Complexity:** O(H), where H is the height of the binary tree. The space is used by the stack to store nodes during traversal.
