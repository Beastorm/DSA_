# Binary Tree Traversals

There are two broad categories of tree traversal: **Depth-First Search (DFS)** and **Breadth-First Search (BFS)**.

---

## 1. Depth-First Search (DFS)
DFS explores as far as possible along each branch before backtracking. There are three common types:

### A. Pre-order Traversal (Root → Left → Right)
Used to create a copy of the tree or to get a prefix expression of an expression tree.
* **Process:** Visit Root, then traverse Left subtree, then traverse Right subtree.
* **Result for diagram below:** `1, 2, 4, 5, 3`

### B. In-order Traversal (Left → Root → Right)
In a **Binary Search Tree (BST)**, this traversal visits nodes in non-decreasing (sorted) order.
* **Process:** Traverse Left, then visit Root, then traverse Right.
* **Result for diagram below:** `4, 2, 5, 1, 3`

### C. Post-order Traversal (Left → Right → Root)
Used to delete the tree or get the postfix expression of an expression tree.
* **Process:** Traverse Left, then traverse Right, then visit Root.
* **Result for diagram below:** `4, 5, 2, 3, 1`



---

## 2. Breadth-First Search (BFS) / Level Order
This visits all nodes at the current depth (level) before moving on to the nodes at the next depth level.

* **Process:** Visit nodes level by level, starting from the root.
* **Implementation:** Usually implemented using a **Queue**.
* **Result for diagram below:** `1, 2, 3, 4, 5`



---

## 3. Visual Example
Consider this tree:
```text
        1 (Root)
       / \
      2   3
     / \
    4   5 (Leaves)

Traversal Type, Order of Nodes
Pre-order,1 → 2 → 4 → 5 → 3
In-order,4 → 2 → 5 → 1 → 3
Post-order,4 → 5 → 2 → 3 → 1
Level Order,1 → 2 → 3 → 4 → 5
```

>### Examples:

### Example 1: The Perfect Binary Tree (7 Nodes)
This tree is perfectly balanced and full.

**Structure:**
```text
        10
      /    \
    20      30
   /  \    /  \
  40  50  60  70
```
**Pre-order (Root-L-R):** 10, 20, 40, 50, 30, 60, 70
Intuition: You "outline" the tree from the left, hitting every node as you first encounter it.  

**In-order (L-Root-R):** 40, 20, 50, 10, 60, 30, 70

Intuition: Imagine projecting all nodes straight down onto a single horizontal line.

**Post-order (L-R-Root):** 40, 50, 20, 60, 70, 30, 10

Intuition: You process all children before their respective parents.

**Level Order:** 10, 20, 30, 40, 50, 60, 70

### Example 2: The Right-Skewed (Degenerate) Tree
This tree mimics a Linked List, which is a common edge case you'll handle in AOSP or native C++ development.

**Structure:**
```
   1
     \
      2
       \
        3
         \
          4
 ```         

**Pre-order:** 1, 2, 3, 4  
**In-order:** 1, 2, 3, 4   
**Post-order:** 4, 3, 2, 1   
**Level Order:** 1, 2, 3, 4   

**Note:** Notice that for a right-skewed tree, Pre-order, In-order, and Level Order all produce the same sequence! 

### Example 3: Deep Left-Heavy Tree
This structure tests your ability to stay deep in the left subtree before touching the right.  

**Structure:**   

```
          A
         / \
        B   C
       /
      D
     / \
    E   F

```
Pre-order	A, B, D, E, F, C   
In-order	E, D, F, B, A, C   
Post-order	E, F, D, B, C, A   
Level Order	A, B, C, D, E, F
