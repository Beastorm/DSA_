## Approach
### The Intuition:
In post-order, you visit the Root only after its Left and Right children are done.

**Go Deep:** Always try to go as far left as possible, pushing every node onto the stack (the "ladder").

**The Look-Back:** When you can't go left anymore, look at the top node of the stack.
- If it has a Right child you haven't visited yet, you must go there first.
- If there is no right child (or you just came back from it), you are finally allowed to "visit" the node and pop it.

### The Algorithm:
- **Push Left:** Keep pushing the current node and move current = current->left until NULL.

- **Peek:** Look at the stack's top node (peekNode).

- **Check Right:** * If peekNode has a right child AND it’s not the one we just finished (lastVisited), set current = peekNode->right and go back to Step 1.

- **Pop & Visit:** * Otherwise, the right side is done. Pop peekNode, add to result, set lastVisited = peekNode, and keep current = NULL (to force another peek/pop from the stack in the next turn).


### C++ Code:
``` cpp
#include <iostream>
#include <vector>
#include <stack>
#include <algorithm>

// Standard Tree Node Definition
struct TreeNode {
    int val;
    TreeNode *left;
    TreeNode *right;
    
    TreeNode(int x) : val(x), left(nullptr), right(nullptr) {}
};

class Solution {
public:
    std::vector<int> postorderTraversal(TreeNode* root) {
        std::vector<int> result;
        if (!root) return result;

        std::stack<TreeNode*> s;
        TreeNode* current = root;
        TreeNode* lastVisited = nullptr;

        while (current != nullptr || !s.empty()) {
            if (current != nullptr) {
                // Descend left, pushing parents to stack
                s.push(current);
                current = current->left;
            } else {
                TreeNode* peekNode = s.top();
                
                // If right child exists and we're coming from the left child, 
                // move to the right child instead of popping the parent.
                if (peekNode->right != nullptr && lastVisited != peekNode->right) {
                    current = peekNode->right;
                } else {
                    // Visit the node (Left and Right are done)
                    result.push_back(peekNode->val);
                    lastVisited = peekNode;
                    s.pop();
                }
            }
        }
        return result;
    }
};

// Helper function to clean up memory (Post-order deletion)
void deleteTree(TreeNode* root) {
    if (!root) return;
    deleteTree(root->left);
    deleteTree(root->right);
    delete root;
}

int main() {
    /* Constructing a sample tree:
              1
             / \
            2   3
           / \
          4   5
    */
    TreeNode* root = new TreeNode(1);
    root->left = new TreeNode(2);
    root->right = new TreeNode(3);
    root->left->left = new TreeNode(4);
    root->left->right = new TreeNode(5);

    Solution sol;
    std::vector<int> path = sol.postorderTraversal(root);

    std::cout << "Post-order Traversal: ";
    for (int val : path) {
        std::cout << val << " ";
    }
    std::cout << std::endl;

    // Output should be: 4 5 2 3 1 

    deleteTree(root);
    return 0;
}

```

### Complexity Analysis:
- **1. Time Complexity:** $O(n)$ 
    - The Logic: Every node in the tree is pushed onto the stack exactly once and popped exactly once.
    - The Nuance: While the while loop runs multiple times, the total number of operations is linear. We visit each edge of the tree at most twice (once going down, once coming up). 
    - Performance: In C++, this is very fast because std::vector::push_back and std::stack::push are $O(1)$ amortized.

- **2. Space Complexity:** $O(h)$
    - The Logic: The stack's size is determined by the maximum depth (height) of the tree.
        - Best Case (Balanced Tree): $O(\log n)$. For a tree with 1 million nodes, the stack only holds ~20 pointers.
        - Worst Case (Skewed Tree): $O(n)$. The stack holds all nodes. 
    - Memory Footprint: Each element in the stack is a pointer (8 bytes on a 64-bit system). 
        - Unlike a Two-Stack solution, which requires $O(n)$ space regardless of tree shape, the One-Stack solution is more memory-efficient for balanced trees.
