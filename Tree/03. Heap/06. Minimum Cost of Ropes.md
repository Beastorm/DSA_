## Minimum Cost of Ropes

**Problem Statement:**   
Given an array of rope lengths, connect all ropes into one single rope. The cost to connect two ropes equals the sum of their lengths. Find the **minimum total cost** to connect all ropes.

---

>### Examples:

**Example 1:**
- **Input:** `[4, 3, 2, 6]`
- **Output:** `29`
```
  Optimal Strategy:
  Step 1: Connect 2+3 = 5,  cost=5,  ropes=[4,5,6]
  Step 2: Connect 4+5 = 9,  cost=9,  ropes=[6,9]
  Step 3: Connect 6+9 = 15, cost=15, ropes=[15]

  Total = 5 + 9 + 15 = 29 ✓

  Suboptimal Strategy:
  Step 1: Connect 4+6 = 10, cost=10, ropes=[3,2,10]
  Step 2: Connect 3+2 = 5,  cost=5,  ropes=[5,10]
  Step 3: Connect 5+10 = 15,cost=15, ropes=[15]

  Total = 10 + 5 + 15 = 30 ✗ (worse!)
```

**Example 2:**
- **Input:** `[1, 2, 3, 4, 5]`
- **Output:** `33`
```
  Step 1: Connect 1+2 = 3,  cost=3,  ropes=[3,3,4,5]
  Step 2: Connect 3+3 = 6,  cost=6,  ropes=[4,5,6]
  Step 3: Connect 4+5 = 9,  cost=9,  ropes=[6,9]
  Step 4: Connect 6+9 = 15, cost=15, ropes=[15]

  Total = 3 + 6 + 9 + 15 = 33 ✓
```

---

## Key Insight — Why Always Pick Two Smallest?
```
  Think of it as a binary tree of connections:

  Rope connected at depth d contributes its length × d to total cost.

  To minimize cost → give shorter ropes larger depth (more additions)
                    give longer ropes smaller depth (fewer additions)

  [1, 2, 3, 4, 5]:

  Optimal tree:          Suboptimal tree:
       15                      15
      /  \                    /  \
     6    9                  1   14
    / \  / \                    /  \
   3  3  4  5                  2   12
  / \                             /  \
 1   2                           3    9
                                     / \
                                    4   5

  1 contributes: 1×3 = 3    vs    1×1 = 1 (seems better?)
  But total = 33             vs    total = 1+2+2+3+4+4+5+5 = ...
  Greedy always gives minimum ✓
```

---
## Approach
### Algorithm:
```
  1. Insert all rope lengths into a Min Heap
  2. While heap has more than 1 element:
       a. Extract two smallest ropes (first, second)
       b. cost = first + second
       c. totalCost += cost
       d. Push cost back into heap
  3. Return totalCost
```
```
  Min Heap trace for [4, 3, 2, 6]:

  Initial heap:    [2, 4, 3, 6]
                    ↑ min

  Step 1: pop 2, pop 3 → combined=5, totalCost=5
          push 5 → heap=[4, 5, 6]

  Step 2: pop 4, pop 5 → combined=9, totalCost=14
          push 9 → heap=[6, 9]

  Step 3: pop 6, pop 9 → combined=15, totalCost=29
          push 15 → heap=[15]

  Heap size=1 → return 29 ✓
```

---

### C++ Code:
```cpp
#include <bits/stdc++.h>
using namespace std;

long long minCostRopes(vector<long long>& arr, int n) {
    // Edge case: 0 or 1 rope
    if (n <= 1) return 0;

    // Build Min Heap from all rope lengths — O(N)
    priority_queue<long long,
                   vector<long long>,
                   greater<long long>> minPQ(arr.begin(), arr.end());

    long long totalCost = 0;

    // Keep combining until one rope remains
    while (minPQ.size() > 1) {
        // Extract two smallest ropes
        long long first  = minPQ.top(); minPQ.pop();
        long long second = minPQ.top(); minPQ.pop();

        // Cost to connect them
        long long cost = first + second;
        totalCost += cost;

        // Push combined rope back
        minPQ.push(cost);
    }

    return totalCost;
}

int main() {
    vector<long long> ropes1 = {4, 3, 2, 6};
    cout << "Min cost: " << minCostRopes(ropes1, ropes1.size()) << endl;
    // Output: 29

    vector<long long> ropes2 = {1, 2, 3, 4, 5};
    cout << "Min cost: " << minCostRopes(ropes2, ropes2.size()) << endl;
    // Output: 33

    vector<long long> ropes3 = {5};
    cout << "Min cost: " << minCostRopes(ropes3, ropes3.size()) << endl;
    // Output: 0 (only one rope, no connection needed)

    vector<long long> ropes4 = {10, 20};
    cout << "Min cost: " << minCostRopes(ropes4, ropes4.size()) << endl;
    // Output: 30

    return 0;
}
```

```
  Input: [1, 2, 3, 4, 5]

  heap = [1, 2, 3, 4, 5]

           1

  ── Iteration 1 ──

           3
          / \

        /

  ── Iteration 2 ──

           4

           6
```

### Visualization as a Tree:
```
  [1, 2, 3, 4, 5] → total cost = 33

  Connection tree:
           15        cost=15 (1 connection)
          /  \
         6    9      cost=6+9=15
        / \  / \
       3  3  4  5    cost=3+3+4+5 (from previous)
      / \
     1   2           cost=1+2=3

  Each rope's contribution to total:
  Rope 1: appears in connections at depth 3 → contributes 1×3 = 3
  Rope 2: appears at depth 3                → contributes 2×3 = 6
  Rope 3: appears at depth 2                → contributes 3×2 = 6
  Rope 4: appears at depth 1                → contributes 4×1 = 4  (wait)
  
  Actually: contribution = rope_length × (number of times it's combined)
  Total = sum of all intermediate combined values = 3+6+9+15 = 33 ✓
```

## Edge Cases
```
  N=0:  return 0  (no ropes)
  N=1:  return 0  (already one rope, no cost)
  N=2:  return a[0]+a[1] (only one connection possible)

  All equal: [5,5,5,5]
    5+5=10 (cost=10), [5,10,10]
    5+10=15 (cost=15), [10,15]
    10+15=25 (cost=25)
    Total = 10+15+25 = 50

  Two very large ropes: [10^9, 10^9]
    Use long long to avoid overflow! ✓
```

---

### Complexity Analysis:

**Time Complexity:** O(N log N)
- Building the Min Heap: O(N)
- Each of the (N-1) iterations: extract twice O(log N) + insert once O(log N) = O(log N)
- Total: O(N) + O(N log N) = **O(N log N)**

**Space Complexity:** O(N) for the heap.
```
  N ropes → N-1 merge operations
  Each merge: O(log N) heap operations
  Total: O(N log N) ✓
```

---

### Why Greedy Works?:
```
  Proof sketch (Exchange Argument):

  Suppose optimal solution does NOT pick two smallest ropes first.
  Then there exist ropes a ≤ b ≤ c where c is picked before b.

  Cost with order (a,c) then (ac, b):  a+c + a+c+b = 2a+2c+b
  Cost with order (a,b) then (ab, c):  a+b + a+b+c = 2a+2b+c

  Since b ≤ c: 2b ≤ 2c → 2a+2b+c ≤ 2a+2c+b

  → Always cheaper to merge two SMALLEST first ✓
  → Min Heap gives optimal greedy choice at each step ✓
```
