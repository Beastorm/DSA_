## Is Binary Tree a Heap?

**Problem Statement:**   
Given a Binary Tree, check whether it satisfies the **Heap property** or not. A Binary Tree is a Heap if it satisfies **both**:
1. **Structural Property:** It must be a **Complete Binary Tree**.
2. **Heap Order Property:** Every node's value must be **greater than or equal to** its children's values (Max Heap property).

---

>### Examples:

**Example 1:**
- **Input:** `root = [97, 46, 37, 12, 3, 7, 31, 6, 9]`
- **Output:** `True`
```
              97
            /    \
          46      37
         /  \    /  \
        12   3  7   31
       / \
      6   9

  Complete Binary Tree? ✓ (all levels filled, last level left-biased)
  Max Heap property?    ✓ (97>46,37 | 46>12,3 | 37>7,31 | 12>6,9)
  Answer: True ✓
```

**Example 2:**
- **Input:** `root = [97, 46, 37, 12, 3, 7, 31, 6, 9, 2]` (with extra node)
- **Output:** `True` (still complete + heap property holds)

**Example 3:**
- **Input:** `root = [1, 2, 3]`
- **Output:** `False` (1 < 2 and 1 < 3 → violates Max Heap property)
```
       1
      / \
     2   3

  Complete? ✓
  Max Heap? ✗ (1 < 2 and 1 < 3)
  Answer: False ✗
```

**Example 4:**
- **Input:** Missing node in middle (not complete)
```
       10
      /  \
     9    8
      \
       7

  Complete? ✗ (9 has no left child but has right child)
  Answer: False ✗
```

---

### Key Insight: Two Conditions to Check:
```
  Condition 1: Complete Binary Tree
  ───────────────────────────────────
  All levels completely filled EXCEPT possibly last level.
  Last level nodes are as LEFT as possible.

  ✓ Complete:         ✗ Not Complete:
       1                    1
      / \                  / \
     2   3                2   3
    / \                    \
   4   5                    5   ← no left child for 2

  Condition 2: Max Heap Order
  ───────────────────────────
  Every node ≥ its children (if they exist).

  ✓ Max Heap:         ✗ Not Max Heap:
       10                    5
      /  \                  / \
     8    9                8   3
    / \                   ↑
   4   7               8 > 5 → violation!
```

---
## Approach
### Algorithm
```
  Step 1: Count total nodes N in the tree.

  Step 2: Check Completeness using index-based approach.
    - Assign indices like array representation:
      root = 0, left = 2i+1, right = 2i+2
    - If any node's index ≥ N → NOT a complete binary tree.

  Step 3: Check Heap Order property.
    - For each node, check if it's ≥ both children.
    - If any violation found → NOT a heap.

  If both checks pass → IS a heap. ✓
```
```
  Index assignment (like array):

         97 (idx=0)
        /    \
      46(1)  37(2)
      /  \   /  \
    12(3) 3(4) 7(5) 31(6)
    / \
  6(7) 9(8)

  N = 9 nodes
  All indices 0-8 < N=9 → Complete ✓
  97>46,37 | 46>12,3 | 37>7,31 | 12>6,9 → Heap ✓
```

---
### Code:
```cpp
#include <bits/stdc++.h>
using namespace std;

struct TreeNode {
    int val;
    TreeNode* left;
    TreeNode* right;
    TreeNode(int x) : val(x), left(nullptr), right(nullptr) {}
};

class Solution {
public:
    // Count total nodes in the tree
    int countNodes(TreeNode* root) {
        if (!root) return 0;
        return 1 + countNodes(root->left) + countNodes(root->right);
    }

    // Check if tree is Complete Binary Tree AND Max Heap
    // index: current node's index in array representation
    // n: total number of nodes
    bool isHeapUtil(TreeNode* root, int index, int n) {
        // Base case: null node is valid
        if (!root) return true;

        // Condition 1: Check completeness
        // If index >= n, this position shouldn't exist
        if (index >= n) return false;

        // Condition 2: Check Max Heap order property
        if (root->left) {
            // Left child index = 2*index + 1
            if (root->left->val > root->val) return false;
        }
        if (root->right) {
            // Right child index = 2*index + 2
            if (root->right->val > root->val) return false;
        }

        // Recursively check left and right subtrees
        return isHeapUtil(root->left,  2 * index + 1, n) &&
               isHeapUtil(root->right, 2 * index + 2, n);
    }

    bool isHeap(TreeNode* root) {
        if (!root) return true;

        int n = countNodes(root);
        return isHeapUtil(root, 0, n);
    }
};

int main() {
    // Example 1: Valid Max Heap
    TreeNode* root1 = new TreeNode(97);
    root1->left  = new TreeNode(46);
    root1->right = new TreeNode(37);
    root1->left->left   = new TreeNode(12);
    root1->left->right  = new TreeNode(3);
    root1->right->left  = new TreeNode(7);
    root1->right->right = new TreeNode(31);
    root1->left->left->left  = new TreeNode(6);
    root1->left->left->right = new TreeNode(9);

    Solution sol;
    cout << "Example 1: " << (sol.isHeap(root1) ? "True" : "False") << endl; // True

    // Example 2: Not a heap (order violation)
    TreeNode* root2 = new TreeNode(5);
    root2->left  = new TreeNode(8);
    root2->right = new TreeNode(3);

    cout << "Example 2: " << (sol.isHeap(root2) ? "True" : "False") << endl; // False

    // Example 3: Not complete
    TreeNode* root3 = new TreeNode(10);
    root3->left  = new TreeNode(9);
    root3->right = new TreeNode(8);
    root3->left->right = new TreeNode(7); // right child without left → not complete

    cout << "Example 3: " << (sol.isHeap(root3) ? "True" : "False") << endl; // False

    return 0;
}
```

### Complexity Analysis:

**Time Complexity:** O(N)
- `countNodes()`: visits each node once → O(N)
- `isHeapUtil()`: visits each node once → O(N)
- Total: **O(N)**

**Space Complexity:** O(H) where H is the height of the tree.
- Recursive call stack depth = height of tree
- For complete binary tree: H = O(log N)
- Total: **O(log N)**
```
  N nodes in complete binary tree:
  Height = floor(log₂N)

  Space = O(log N) for recursion stack ✓

  Much better than O(N) for skewed trees
  because complete binary trees are always balanced!
```

### Summary:
```
  isHeap checks TWO conditions simultaneously:

  ┌─────────────────────────────────────────────────────┐
  │ 1. Completeness: index < N for every node           │
  │    (using array-style index assignment)             │
  │                                                     │
  │ 2. Max Heap Order: every node ≥ its children        │
  │                                                     │
  │ Both must be True → Tree IS a Max Heap              │
  │ Either False      → Tree is NOT a Max Heap          │
  └─────────────────────────────────────────────────────┘
```
