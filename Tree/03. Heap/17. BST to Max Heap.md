## BST to Max Heap

**Problem Statement:**   
Given a Binary Search Tree (BST), convert it into a **Max Heap** such that:
1. The Max Heap must contain **all the elements** of the BST.
2. The **property of BST** is maintained — i.e., for any node, all elements in its left subtree are smaller, and all in the right subtree are larger.
3. The resulting tree should also satisfy the **Max Heap property** — parent ≥ children.

**Special Condition:** The resulting tree should simultaneously be a BST-like structure (all left subtree < root < right subtree) AND a Max Heap (root ≥ children).

> **Simpler Version:** Just convert BST values into a Max Heap array/tree (preserving the structure but rearranging values).

---

>### Examples:

**Example 1:**
- **Input BST:**
```
       4
      / \
     2   6
    / \ / \
   1  3 5  7
```
- **Output Max Heap:**
```
       7
      / \
     3   6
    / \ / \
   1  2 4  5
```
```
  BST Inorder:  [1, 2, 3, 4, 5, 6, 7]  ← sorted ascending
  For Max Heap with BST structure:
  Postorder fill with sorted values (ascending)
  → smallest values go to leaves, largest to root
```

**Example 2:**
- **Input BST:**
```
       8
      / \
     4   10
    / \    \
   2   5    12
```
- **Output Max Heap:**
```
       12
      /  \
     5    10
    / \
   2   4
```

---

### Key Insight:
```
  BST Inorder traversal → sorted ascending order
  [1, 2, 3, 4, 5, 6, 7]

  Max Heap with same structure:
  Postorder traversal fills nodes bottom-up.
  If we fill postorder positions with values in ASCENDING order:
  → Leaves get smallest values
  → Root gets largest value
  → Parent always gets larger value than children ✓

  Why Postorder?
  Postorder visits: Left → Right → Root
  If we assign values in ascending order during postorder:
  → Left subtree (smaller positions) gets smaller values
  → Right subtree gets medium values
  → Root gets largest value → Max Heap ✓
```

---

### Algorithm:
```
  Step 1: Perform INORDER traversal of BST
          → Collect all values in sorted ascending order
          → arr = [1, 2, 3, 4, 5, 6, 7]

  Step 2: Perform POSTORDER traversal of the tree
          → At each node (visited in postorder), assign
            next value from the sorted array
          → Smallest values fill leaves first
          → Root gets the largest value

  Result: Tree has Max Heap property ✓
```
```
  BST:
         4 (0)
        / \
       2   6
      / \ / \
     1  3 5  7

  Inorder  (sorted): [1, 2, 3, 4, 5, 6, 7]
  Postorder order:   [1, 3, 2, 5, 7, 6, 4]
                      ↑ positions visited in postorder

  Fill postorder positions with sorted array (ascending):
  Position 1→ value 1
  Position 3→ value 2
  Position 2→ value 3
  Position 5→ value 4
  Position 7→ value 5
  Position 6→ value 6
  Position 4→ value 7

  Result tree (node at position = assigned value):
         7
        / \
       3   6
      / \ / \
     1  2 4  5

  Verify Max Heap:
  7 > 3, 6 ✓
  3 > 1, 2 ✓
  6 > 4, 5 ✓
```

### Code:
```cpp
#include <bits/stdc++.h>
using namespace std;

struct TreeNode {
    int val;
    TreeNode* left;
    TreeNode* right;
    TreeNode(int x) : val(x), left(nullptr), right(nullptr) {}
};

class Solution {
private:
    vector<int> sorted; // Inorder traversal (sorted)
    int idx;            // Index for filling values

    // Step 1: Inorder traversal → sorted array
    void inorder(TreeNode* root) {
        if (!root) return;
        inorder(root->left);
        sorted.push_back(root->val);
        inorder(root->right);
    }

    // Step 2: Postorder traversal → fill values from sorted array
    void postorder(TreeNode* root) {
        if (!root) return;
        postorder(root->left);
        postorder(root->right);
        root->val = sorted[idx++]; // Assign next sorted value
    }

public:
    void convertToMaxHeap(TreeNode* root) {
        // Step 1: Get all values in sorted order
        inorder(root);

        // Step 2: Fill postorder positions with sorted values
        idx = 0;
        postorder(root);
    }
};

// Print level-order (BFS) to verify heap structure
void levelOrder(TreeNode* root) {
    if (!root) return;
    queue<TreeNode*> q;
    q.push(root);
    while (!q.empty()) {
        TreeNode* node = q.front(); q.pop();
        cout << node->val << " ";
        if (node->left)  q.push(node->left);
        if (node->right) q.push(node->right);
    }
    cout << endl;
}

// Print inorder to verify values preserved
void inorderPrint(TreeNode* root) {
    if (!root) return;
    inorderPrint(root->left);
    cout << root->val << " ";
    inorderPrint(root->right);
}

int main() {
    // Construct BST: [4, 2, 6, 1, 3, 5, 7]
    TreeNode* root = new TreeNode(4);
    root->left  = new TreeNode(2);
    root->right = new TreeNode(6);
    root->left->left   = new TreeNode(1);
    root->left->right  = new TreeNode(3);
    root->right->left  = new TreeNode(5);
    root->right->right = new TreeNode(7);

    cout << "BST Inorder (before): ";
    inorderPrint(root);
    cout << endl;

    Solution sol;
    sol.convertToMaxHeap(root);

    cout << "After conversion:" << endl;
    cout << "Level Order: ";
    levelOrder(root);
    cout << "Inorder: ";
    inorderPrint(root);
    cout << endl;

    return 0;
}
```

### Step-by-Step Trace:
```
  BST:
         4
        / \
       2   6
      / \ / \
     1  3 5  7

  ── Step 1: Inorder Traversal ──
  Visit: 1 → 2 → 3 → 4 → 5 → 6 → 7
  sorted = [1, 2, 3, 4, 5, 6, 7]
  idx = 0

  ── Step 2: Postorder Traversal + Fill ──

  Postorder visits: Left → Right → Root

  postorder(4):
    postorder(2):
      postorder(1):
        postorder(null): return
        postorder(null): return
        node(1).val = sorted[0] = 1, idx=1
      postorder(3):
        postorder(null): return
        postorder(null): return
        node(3).val = sorted[1] = 2, idx=2
      node(2).val = sorted[2] = 3, idx=3
    postorder(6):
      postorder(5):
        postorder(null): return
        postorder(null): return
        node(5).val = sorted[3] = 4, idx=4
      postorder(7):
        postorder(null): return
        postorder(null): return
        node(7).val = sorted[4] = 5, idx=5
      node(6).val = sorted[5] = 6, idx=6
    node(4).val = sorted[6] = 7, idx=7

  ── Result ──
  Node positions and their new values:
  Old: 4→7, 2→3, 6→6, 1→1, 3→2, 5→4, 7→5

  Tree:
         7         ← was 4, now 7 (largest)
        / \
       3   6       ← was 2→3, was 6→6
      / \ / \
     1  2 4  5     ← was 1→1, 3→2, 5→4, 7→5

  Level Order: [7, 3, 6, 1, 2, 4, 5] ✓

  Verify Max Heap:
  7 > 3 ✓, 7 > 6 ✓
  3 > 1 ✓, 3 > 2 ✓
  6 > 4 ✓, 6 > 5 ✓
  All good! ✓
```

---

### Why This Works?
```
  Postorder traversal processes nodes BOTTOM-UP:
  Leaves first → then internal nodes → root last

  Assigning values in ASCENDING order (smallest first):
  → Leaves get smallest values ✓
  → Each parent is processed AFTER its children
  → Parent gets a larger value than both children ✓
  → Root (last processed) gets the LARGEST value ✓

  This guarantees:
  parent_value > left_child_value  (parent assigned later = larger val)
  parent_value > right_child_value (parent assigned later = larger val)
  → Max Heap property holds for ALL nodes ✓
```
```
  Postorder order:  L R Root
  Values assigned:  ↑ ↑  ↑
                  small  large

  Children always get smaller values than parent
  → Max Heap guaranteed ✓
```

### Visualization: Value Assignment:
```
  sorted = [1, 2, 3, 4, 5, 6, 7]
            ↑  ↑  ↑  ↑  ↑  ↑  ↑
            0  1  2  3  4  5  6  (indices)

  Postorder assignment sequence:
  Node 1 (leaf) → sorted[0] = 1
  Node 3 (leaf) → sorted[1] = 2
  Node 2        → sorted[2] = 3  (gets bigger than its children 1,2)
  Node 5 (leaf) → sorted[3] = 4
  Node 7 (leaf) → sorted[4] = 5
  Node 6        → sorted[5] = 6  (gets bigger than its children 4,5)
  Node 4 (root) → sorted[6] = 7  (gets biggest of all)

  Final:
  7 at root > 3 (left) and 6 (right) ✓
  3 > 1 (left) and 2 (right) ✓
  6 > 4 (left) and 5 (right) ✓
```

### Edge Cases:
```
  Single node:          Only left children (skewed):
  BST: [5]              BST: 3→2→1 (left skewed)
  sorted=[5]            sorted=[1,2,3]
  postorder fills 5     postorder: 1→2→3
  Result: [5] ✓         Result: 3→2→1 (same, valid heap) ✓

  Two nodes:            Already a max heap:
  BST:  2               BST:  7
       /                     / \
      1                     5   6
  sorted=[1,2]          sorted=[5,6,7]
  postorder: 1→2        postorder fills: 5→6→7
  Result: 2 ✓           Result: 7 ✓
           \                   / \
            1                 5   6
  Wait — structure unchanged, values reassigned
```

### Complexity Analysis:

**Time Complexity:** O(N)
- Inorder traversal: O(N)
- Postorder traversal + fill: O(N)
- Total: **O(N)**

**Space Complexity:** O(N)
- `sorted` array stores N values: O(N)
- Recursion stack (height of BST): O(H) = O(log N) for balanced BST, O(N) worst case
- Total: **O(N)**
```
  N = number of nodes in BST

  Two linear passes over the tree:
  Pass 1 (Inorder):   O(N) — collect sorted values
  Pass 2 (Postorder): O(N) — fill values back

  Space: O(N) array + O(H) stack
  For balanced BST: O(N) + O(log N) = O(N) ✓
```
