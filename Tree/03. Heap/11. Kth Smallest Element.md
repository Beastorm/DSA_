## Kth Smallest Element

**Problem Statement:**  
Given an array of `N` elements and an integer `K`, find the **Kth smallest element** in the array.

---

>### Examples:

**Example 1:**
- **Input:** `arr = [7, 10, 4, 3, 20, 15]`, `K = 3`
- **Output:** `7`
```
  Sorted: [3, 4, 7, 10, 15, 20]
                ↑
          3rd smallest = 7 ✓
```

**Example 2:**
- **Input:** `arr = [7, 10, 4, 20, 15]`, `K = 4`
- **Output:** `15`
```
  Sorted: [4, 7, 10, 15, 20]
                     ↑
          4th smallest = 15 ✓
```

**Example 3:**
- **Input:** `arr = [1]`, `K = 1`
- **Output:** `1`

---

## Approaches Overview
```
  Method 1: Sort           → O(N log N)  simple, wasteful
  Method 2: Max Heap       → O(N log K)  optimal for streams
  Method 3: Min Heap       → O(N + K log N)
  Method 4: QuickSelect    → O(N) average
```

---

## Method 1: Sorting (Brute Force)

### Algorithm
Sort the array and return the element at index `K-1`.
```
  arr = [7, 10, 4, 3, 20, 15], K=3

  Sort → [3, 4, 7, 10, 15, 20]
  Return arr[K-1] = arr[2] = 7 ✓
```

### Code:
```cpp
int kthSmallest_sort(vector<int>& arr, int k) {
    sort(arr.begin(), arr.end());
    return arr[k - 1];
}
```

**Time:** O(N log N) | **Space:** O(1)

## Method 2: Max Heap of Size K (Optimal)

### Key Insight:
```
  Maintain a Max Heap of size K.
  The top of the heap = Kth smallest element!

  Why?
  If the heap has exactly K elements and the top is the max,
  then all K elements are the K smallest seen so far,
  and the largest among them = Kth smallest ✓

  For each new element:
  - If smaller than heap top → it belongs in K smallest
    → pop top (too large), push new element
  - If larger than heap top → not in K smallest, skip
```

### Algorithm
```
  1. Build Max Heap with first K elements
  2. For each remaining element x:
       If x < heap.top():
         heap.pop()
         heap.push(x)
  3. Return heap.top() → Kth smallest
```
```
  arr = [7, 10, 4, 3, 20, 15], K=3

  Process first K=3 elements:
  push 7  → heap=[7]
  push 10 → heap=[10, 7]
  push 4  → heap=[10, 7, 4]
        10
       /  \
      7    4

  Process remaining elements:
  x=3:  3 < top(10) → pop 10, push 3 → heap=[7, 3, 4]
        7
       / \
      3   4

  x=20: 20 > top(7) → skip

  x=15: 15 > top(7) → skip

  heap.top() = 7 → 3rd smallest = 7 ✓
```

### Code:
```cpp
#include <bits/stdc++.h>
using namespace std;

int kthSmallest(vector<int>& arr, int k) {
    // Max Heap of size K
    priority_queue<int> maxPQ;

    for (int x : arr) {
        maxPQ.push(x);

        // Keep heap size = K
        if (maxPQ.size() > k) {
            maxPQ.pop(); // Remove largest → maintains K smallest
        }
    }

    // Top of max heap = Kth smallest
    return maxPQ.top();
}

int main() {
    vector<int> arr1 = {7, 10, 4, 3, 20, 15};
    cout << kthSmallest(arr1, 3) << endl; // 7

    vector<int> arr2 = {7, 10, 4, 20, 15};
    cout << kthSmallest(arr2, 4) << endl; // 15

    vector<int> arr3 = {12, 3, 5, 7, 19};
    cout << kthSmallest(arr3, 2) << endl; // 5

    return 0;
}
```

### Step-by-Step Trace (Method 2)
```
  arr = [7, 10, 4, 3, 20, 15], K = 3

  ── Process each element ──

  x=7:  push 7 → heap=[7], size=1 ≤ 3
        heap:   7

  x=10: push 10 → heap=[10,7], size=2 ≤ 3
        heap:   10
               /
              7

  x=4:  push 4 → heap=[10,7,4], size=3 ≤ 3
        heap:   10
               /  \
              7    4

  x=3:  push 3 → heap=[10,7,4,3], size=4 > 3
        pop 10 → heap=[7,3,4], size=3
        heap:   7
               / \
              3   4

  x=20: push 20 → heap=[20,3,4,7], size=4 > 3
        pop 20 → heap=[7,3,4], size=3
        (20 > 7 so pushing and immediately popping = no change)

        Optimization: if x > heap.top(), just skip!
        heap:   7
               / \
              3   4

  x=15: push 15 → size=4 > 3
        pop 15 → heap=[7,3,4], size=3
        heap unchanged.

  ── Answer ──
  heap.top() = 7 → 3rd smallest = 7 ✓
```

## Method 3: Min Heap

### Algorithm:
```
  1. Build Min Heap from all N elements — O(N)
  2. Call pop() exactly K times
  3. The Kth pop gives Kth smallest element
```
```
  arr = [7, 10, 4, 3, 20, 15], K=3

  Build Min Heap:
        3
       / \
      7   4
     / \ /
    10 20 15

  Pop 1: pop 3  → 1st smallest
  Pop 2: pop 4  → 2nd smallest
  Pop 3: pop 7  → 3rd smallest = 7 ✓
```

### Code:
```cpp
int kthSmallest_minHeap(vector<int>& arr, int k) {
    // Build Min Heap — O(N)
    priority_queue<int, vector<int>, greater<int>>
        minPQ(arr.begin(), arr.end());

    // Pop K-1 times
    for (int i = 0; i < k - 1; i++) {
        minPQ.pop();
    }

    return minPQ.top(); // Kth smallest
}
```

**Time:** O(N + K log N) | **Space:** O(N)

## Method 4: QuickSelect (Most Optimal Average Case)

### Algorithm:
```
  Like QuickSort but only recurse on ONE side.
  Average case: O(N), Worst case: O(N²)

  1. Pick a pivot (usually last element)
  2. Partition: elements < pivot go left, > pivot go right
  3. If pivot is at index K-1 → found it!
  4. If K-1 < pivot index → recurse LEFT
  5. If K-1 > pivot index → recurse RIGHT
```

### Code:
```cpp
int partition(vector<int>& arr, int left, int right) {
    int pivot = arr[right];
    int i = left - 1;

    for (int j = left; j < right; j++) {
        if (arr[j] <= pivot) {
            i++;
            swap(arr[i], arr[j]);
        }
    }
    swap(arr[i + 1], arr[right]);
    return i + 1;
}

int quickSelect(vector<int>& arr, int left, int right, int k) {
    if (left == right) return arr[left];

    int pivotIdx = partition(arr, left, right);

    if (pivotIdx == k)       return arr[pivotIdx]; // Found!
    else if (pivotIdx > k)   return quickSelect(arr, left, pivotIdx - 1, k);
    else                     return quickSelect(arr, pivotIdx + 1, right, k);
}

int kthSmallest_quickSelect(vector<int>& arr, int k) {
    return quickSelect(arr, 0, arr.size() - 1, k - 1);
}
```

**Time:** O(N) average, O(N²) worst | **Space:** O(log N) recursion stack

## Comparison of All Methods:
```
  Method          Time           Space    Notes
  ──────────────────────────────────────────────────────────
  Sort            O(N log N)     O(1)     Simple, modifies array
  Max Heap (K)    O(N log K)     O(K)     Best for data streams
  Min Heap (N)    O(N + K logN)  O(N)     Good when K is large
  QuickSelect     O(N) avg       O(log N) Best average case

  When to use which:
  ┌─────────────────────────────────────────────────────┐
  │ Data stream / online   → Max Heap of size K         │
  │ K is small             → Min Heap (pop K times)     │
  │ K ≈ N/2 (median)       → QuickSelect                │
  │ Simple / one-time      → Sort                       │
  └─────────────────────────────────────────────────────┘
```

  Result = 7 ✓

---

## Complexity Summary:
```
  Method 2 (Max Heap) is preferred when:
  → Elements arrive as a stream
  → Memory is limited (only O(K) space needed)
  → K << N

  O(N log K) vs O(N log N):
  K=10,  N=10^6: log K = 3.3,  log N = 20  → 6× faster
  K=100, N=10^6: log K = 6.6,  log N = 20  → 3× faster
  K=N:           log K = log N → same as sorting
```
