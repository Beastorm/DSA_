## Construct Target Array With Multiple Sums

**Problem Statement:**   
You are given an array `target` of `n` integers. You start with an array `A` of `n` ones, and in one step you may replace any element of `A` with the sum of all elements of `A`.

Return `true` if it is possible to construct the `target` array from `A`, otherwise return `false`.

### Key Insight Рђћ Work Backwards!
```
  Forward: [1,1,...,1] Рєњ target (hard to simulate)
  Backward: target Рєњ [1,1,...,1] (easier!)

  In each REVERSE step:
  - The largest element was created last.
  - Before that step, it had value = max - rest_sum
    (where rest_sum = total_sum - max)

  More specifically:
  If current max = M and rest_sum = S (sum of others):
  Before this step, max was M - S (subtracting rest once)
  But M could have been the sum many times:
  prev = M mod S  (if prev == 0, it was actually S, but handle edge case)

  Stop conditions (return false):
  1. max РЅц 0 after substitution
  2. rest_sum == 0 but max РЅа 1
  3. max mod rest_sum == max (no progress possible)

  Stop (return true):
  All elements become 1.
```

---

### Algorithm:
```
  1. Build Max Heap from target array
  2. Compute total sum
  3. While max > 1:
       a. max = heap.top(); pop
       b. rest = totalSum - max
       c. If rest == 1: return true (can always reduce)
       d. If max <= rest: return false (max can't be reduced further)
       e. prev = max mod rest (previous value before last step)
       f. If prev == 0: prev = rest (edge case)
       g. If prev >= max: return false (no progress)
       h. totalSum = rest + prev
       i. Push prev back to heap
  4. Return true (all 1s reached)
```

### Code:
```cpp
#include <bits/stdc++.h>
using namespace std;

bool isPossible(vector<int>& target) {
    int n = target.size();

    // Edge case: single element
    if (n == 1) return target[0] == 1;

    // Build Max Heap and compute total sum
    priority_queue<long long> maxPQ;
    long long totalSum = 0;

    for (int x : target) {
        maxPQ.push(x);
        totalSum += x;
    }

    while (maxPQ.top() > 1) {
        long long maxVal = maxPQ.top();
        maxPQ.pop();

        long long rest = totalSum - maxVal;

        // If rest is 0 or 1, we can always reduce
        if (rest <= 1) {
            return rest == 1; // rest==0 means only one element
        }

        // Compute previous value before the last step
        long long prev = maxVal % rest;

        // If prev == 0, the previous value was rest itself
        // (can't have 0 in array, so it wraps to rest)
        if (prev == 0) prev = rest;

        // If no progress made, impossible
        if (prev >= maxVal) return false;

        // Update total sum and push prev back
        totalSum = rest + prev;
        maxPQ.push(prev);
    }

    return true;
}

int main() {
    vector<int> t1 = {9, 3, 5};
    cout << "target=[9,3,5]:     " << (isPossible(t1) ? "true" : "false") << endl;

    vector<int> t2 = {1, 1, 1, 2};
    cout << "target=[1,1,1,2]:   " << (isPossible(t2) ? "true" : "false") << endl;

    vector<int> t3 = {8, 5};
    cout << "target=[8,5]:       " << (isPossible(t3) ? "true" : "false") << endl;

    vector<int> t4 = {1, 1000000000};
    cout << "target=[1,10^9]:    " << (isPossible(t4) ? "true" : "false") << endl;

    vector<int> t5 = {2, 900000002};
    cout << "target=[2,900000002]:" << (isPossible(t5) ? "true" : "false") << endl;

    return 0;
}
```

### Why Use Modulo?
```
  Without modulo (naive subtraction):
  target=[1, 1000000000]:
  1000000000 Рєњ 999999999 Рєњ ... Рєњ 1
  = 10^9 iterations Рєњ TLE!

  With modulo:
  max mod rest = max - k├Ќrest (for max k such that result > 0)
  = We skip k intermediate steps at once!
  1000000000 mod 1 = 0 Рєњ prev = rest = 1
  Done in 1 step! РюЊ

  Example: max=100, rest=7
  Naive: 100 Рєњ 93 Рєњ 86 Рєњ 79 Рєњ 72 Рєњ 65 Рєњ 58 Рєњ 51 Рєњ 44 Рєњ 37 Рєњ 30 Рєњ 23 Рєњ 16 Рєњ 9 Рєњ 2
  14 steps!

  Modulo: 100 mod 7 = 2 Рєњ prev=2 Рєњ 1 step! РюЊ
```

### Edge Cases & Special Conditions:
```
  Single element n=1:
  target=[1] Рєњ already all 1s Рєњ true
  target=[5] Рєњ can't reduce without other elements Рєњ false

  All ones already:
  target=[1,1,1] Рєњ true immediately

  max РЅц rest:
  target=[3,4]: max=4, rest=3
  4 mod 3 = 1 Рєњ prev=1, push 1 Рєњ [3,1] sum=4
  max=3, rest=1 Рєњ rest==1 Рєњ return true РюЊ

  prev >= max (no progress):
  target=[1,1]: max=1 Рєњ return true immediately

  rest=0 (only one element type situation):
  If rest=0, can't have gotten there Рєњ return false
```

### Complexity Analysis:

**Time Complexity:** O(N log N + total_steps ├Ќ log N)

Where `total_steps` depends on the values. With modulo optimization:
- Each step at least halves the max value (when rest РЅц max/2).
- Total steps РЅѕ O(log(max_value) ├Ќ N) in most cases.
- For edge cases like `[1, 10^9]`: O(log(10^9)) = O(30) steps.

Without modulo: O(max_value) steps possible Рєњ **TLE**.
With modulo: O(log(max_value)) steps Рєњ **fast** РюЊ

**Space Complexity:** O(N) for the Max Heap.
```
  Key: modulo turns potential O(10^9) iterations
  into O(log(10^9)) РЅѕ 30 iterations РюЊ

  This is why the Max Heap + modulo approach is essential
  for this problem!
```
