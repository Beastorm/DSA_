## Profit Maximisation

**Problem Statement:**   
Given `N` rows of seats in a stadium and an integer `B` (number of seats to be sold). Each row has a certain number of available seats. The profit from selling a seat in a row = **current number of available seats in that row**. After a seat is sold, the available count in that row decreases by 1.

Sell exactly `B` seats to **maximise total profit**. Return the maximum profit.

---

>### Examples:

**Example 1:**
- **Input:** `A = [4, 3, 2]`, `B = 5`
- **Output:** `14`
```
  Initial rows: [4, 3, 2]

  Sale 1: Sell from row with 4 seats → profit=4, row becomes 3 → [3,3,2]
  Sale 2: Sell from row with 3 seats → profit=3, row becomes 2 → [3,2,2]  (pick either 3)
  Sale 3: Sell from row with 3 seats → profit=3, row becomes 2 → [2,2,2]
  Sale 4: Sell from row with 2 seats → profit=2, row becomes 1 → [2,2,1]
  Sale 5: Sell from row with 2 seats → profit=2, row becomes 1 → [2,1,1]

  Total = 4 + 3 + 3 + 2 + 2 = 14 ✓
```

**Example 2:**
- **Input:** `A = [1, 2, 3, 4, 5]`, `B = 6`
- **Output:** `27`
```
  Initial rows: [1, 2, 3, 4, 5]

  Sale 1: row=5, profit=5 → [1,2,3,4,4]
  Sale 2: row=4, profit=4 → [1,2,3,4,3]  (pick either 4)
  Sale 3: row=4, profit=4 → [1,2,3,3,3]
  Sale 4: row=3, profit=3 → [1,2,3,3,2]  (pick any 3)
  Sale 5: row=3, profit=3 → [1,2,3,2,2]
  Sale 6: row=3, profit=3 → [1,2,2,2,2]

  Total = 5+4+4+3+3+3 = 22

  Hmm let me recheck...
  5+4+4+3+3+3 = 22
```

---

### Key Insight:
```
  To maximize profit → always sell from the row with
  MOST available seats (greedy choice).

  Why? Selling from row with 5 seats gives profit 5.
  Selling from row with 3 seats gives profit 3.
  Always picking maximum guarantees optimal total. ✓

  → MAX HEAP perfectly models this:
    top()  = row with most seats (max profit)
    pop()  = remove that row
    push() = add back row with one less seat
```

---
## Approach
### Algorithm
```
  1. Build Max Heap from array A
  2. totalProfit = 0
  3. Repeat B times:
       a. If heap.top() == 0 → break (no seats left)
       b. top = heap.top(); heap.pop()
       c. totalProfit += top
       d. heap.push(top - 1)      ← one seat sold, row reduces
  4. Return totalProfit
```
```
  Max Heap trace for [4, 3, 2], B=5:

  Initial heap = [4, 3, 2]
        4
       / \
      3   2

  Sale 1: pop 4, push 3 → heap=[3,3,2], profit=4
  Sale 2: pop 3, push 2 → heap=[3,2,2], profit=7
  Sale 3: pop 3, push 2 → heap=[2,2,2], profit=10
  Sale 4: pop 2, push 1 → heap=[2,2,1], profit=12
  Sale 5: pop 2, push 1 → heap=[2,1,1], profit=14

  Return 14 ✓
```

---

### C++ Code:
```cpp
#include <bits/stdc++.h>
using namespace std;

int profitMaximisation(vector<int>& A, int B) {
    // Edge cases
    if (A.empty() || B == 0) return 0;

    // Build Max Heap — O(N)
    priority_queue<int> maxPQ(A.begin(), A.end());

    long long totalProfit = 0;

    for (int i = 0; i < B; i++) {
        // If no seats left in any row
        if (maxPQ.top() == 0) break;

        // Pick row with most seats
        int top = maxPQ.top();
        maxPQ.pop();

        // Profit = number of available seats in this row
        totalProfit += top;

        // One seat sold → row has one less seat
        maxPQ.push(top - 1);
    }

    return (int)totalProfit;
}

int main() {
    vector<int> A1 = {4, 3, 2};
    cout << profitMaximisation(A1, 5) << endl; // 14

    vector<int> A2 = {1, 2, 3, 4, 5};
    cout << profitMaximisation(A2, 6) << endl;

    vector<int> A3 = {10};
    cout << profitMaximisation(A3, 3) << endl; // 10+9+8=27

    vector<int> A4 = {0, 0, 0};
    cout << profitMaximisation(A4, 5) << endl; // 0

    return 0;
}
```

### Why Greedy Works?
```
  Greedy claim: Always sell from the row with the most seats.

  Proof (exchange argument):
  Suppose optimal picks row with i seats when row j > i exists.

  Current profit: i
  After swap (pick j instead): j

  Since j > i → j profit > i profit
  → Swapping to pick j is always at least as good

  → Greedy (always pick max) is optimal ✓
```

### Edge Cases:
```
  B = 0:                 All rows = 0:
  No sales               [0, 0, 0], B=5
  Return 0 ✓             top=0 → break immediately
                         Return 0 ✓

  B > total seats:       Single row:
  [2, 1], B=10           [5], B=3
  sell 2→1→1→0→0         sell 5→4→3
  Then break at 0        profit = 5+4+3 = 12 ✓
  Total = 2+1+1 = 4 ✓

  Large values:
  [10^5], B=10^5
  profit = 10^5 + (10^5-1) + ... + 1
         = 10^5 × (10^5+1) / 2
  Use long long! ✓
```

### Complexity Analysis:

**Time Complexity:** O(N + B log N)
- Building Max Heap: O(N)
- B iterations, each with 1 pop + 1 push: O(B log N)
- Total: **O(N + B log N)**

**Space Complexity:** O(N) for the heap.
```
  N = number of rows
  B = number of seats to sell

  If B >> N: time dominated by O(B log N)
  If B << N: time dominated by O(N) build

  Worst case: B = sum of all seats = O(N × maxSeats)
  → O(N × maxSeats × log N)
  Use early break when top = 0 to avoid unnecessary iterations ✓
```
