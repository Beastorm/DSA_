## Sum of Elements Between K1 and K2 Smallest Elements

**Problem Statement:**   
Given an array of integers and two values `K1` and `K2`, find the **sum of all elements** that lie strictly between the K1th smallest and K2th smallest elements.

---

>### Examples:

**Example 1:**
- **Input:** `arr = [2, 5, 3, 1, 4]`, `K1 = 2`, `K2 = 4`
- **Output:** `3`
```
  Sorted: [1, 2, 3, 4, 5]
               ↑     ↑
          K1=2nd    K2=4th
          smallest  smallest
            = 2       = 4

  Elements BETWEEN 2 and 4 (exclusive): [3]
  Sum = 3 ✓
```

**Example 2:**
- **Input:** `arr = [20, 8, 22, 4, 12, 10, 14]`, `K1 = 3`, `K2 = 6`
- **Output:** `26`
```
  Sorted: [4, 8, 10, 12, 14, 20, 22]
                ↑            ↑
           K1=3rd          K2=6th
           smallest        smallest
             = 10            = 20

  Elements between 10 and 20 (exclusive): [12, 14]
  Sum = 12 + 14 = 26 ✓
```

---

## Key Insight
```
  We need:
  1. K1th smallest element → lower bound (exclusive)
  2. K2th smallest element → upper bound (exclusive)
  3. Sum all elements strictly between these two bounds

  Elements to include: arr[i] where K1th_small < arr[i] < K2th_small

  Two approaches:
  ┌───────────────────────────────────────────────────┐
  │ Method 1: Two Max Heaps of size K1 and K2         │
  │ Method 2: One Max Heap of size K2, smart summing  │
  └───────────────────────────────────────────────────┘
```

---

## Method 1: Two Max Heaps

### Algorithm:
```
  Step 1: Find K1th smallest using Max Heap of size K1
          → top() = K1th smallest

  Step 2: Find K2th smallest using Max Heap of size K2
          → top() = K2th smallest

  Step 3: Sum all elements x where:
          K1th_smallest < x < K2th_smallest
```
```
  arr = [20, 8, 22, 4, 12, 10, 14], K1=3, K2=6

  ── Find K1th (3rd) smallest ──
  Max Heap of size K1=3:
  Process: 20,8,22,4,12,10,14
  After:   heap = {10,12,14}?

  Let's trace properly:
  push 20 → [20]
  push 8  → [20,8]
  push 22 → [22,8,20]  size=3
  push 4  → size=4>3, pop 22 → [20,8,4]
  push 12 → size=4>3, pop 20 → [12,8,4]
  push 10 → size=4>3, pop 12 → [10,8,4]
  push 14 → size=4>3, pop 14? wait top=10
          → push 14 → [14,10,8,4], pop 14? No top=14>K1?

  Hmm: if 14 > top(10) → pop 10, push 14? No...
  For Kth SMALLEST we use MAX heap and pop if size > K

  push 20 → heap=[20]
  push 8  → heap=[20,8]
  push 22 → heap=[22,20,8]  size=3=K1, no pop
  push 4  → size=4>3, pop top=22 → heap=[20,8,4]
  push 12 → size=4>3, pop top=20 → heap=[12,8,4]
  push 10 → size=4>3, pop top=12 → heap=[10,8,4]
  push 14 → size=4>3, pop top=14? top=10, 14>10
            push 14 → [14,10,8,4], pop 14 → [10,8,4]
            Wait: we always pop after push if size > K
            push 14 → heap=[14,10,8,4], pop top=14 → heap=[10,8,4]

  K1th (3rd) smallest = top = 10 ✓  (sorted: 4,8,10,12,14,20,22)

  ── Find K2th (6th) smallest ──
  Max Heap of size K2=6:
  push 20 → [20]
  push 8  → [20,8]
  push 22 → [22,20,8]
  push 4  → [22,20,8,4]
  push 12 → [22,20,12,8,4]
  push 10 → [22,20,12,10,8,4]  size=6=K2, no pop
  push 14 → size=7>6, pop top=22 → [20,14,12,10,8,4]

  K2th (6th) smallest = top = 20 ✓

  ── Sum between 10 and 20 (exclusive) ──
  Traverse arr: 20(no), 8(no), 22(no), 4(no), 12(yes), 10(no), 14(yes)
  Sum = 12 + 14 = 26 ✓
```

### Code:
```cpp
#include <bits/stdc++.h>
using namespace std;

// Find Kth smallest using Max Heap of size K
int kthSmallest(vector<int>& arr, int k) {
    priority_queue<int> maxPQ; // Max Heap

    for (int x : arr) {
        maxPQ.push(x);
        if (maxPQ.size() > k) {
            maxPQ.pop();
        }
    }
    return maxPQ.top();
}

// Method 1: Two Max Heaps
int sumBetweenK1K2_method1(vector<int>& arr, int k1, int k2) {
    // Step 1: Find K1th and K2th smallest
    int k1Small = kthSmallest(arr, k1);
    int k2Small = kthSmallest(arr, k2);

    // Step 2: Sum elements strictly between them
    int sum = 0;
    for (int x : arr) {
        if (x > k1Small && x < k2Small) {
            sum += x;
        }
    }

    return sum;
}

int main() {
    vector<int> arr1 = {2, 5, 3, 1, 4};
    cout << sumBetweenK1K2_method1(arr1, 2, 4) << endl; // 3

    vector<int> arr2 = {20, 8, 22, 4, 12, 10, 14};
    cout << sumBetweenK1K2_method1(arr2, 3, 6) << endl; // 26

    return 0;
}
```

### Complexity Analysis:

- **Time:** O(N log K2) for both heap operations + O(N) for sum = **O(N log K2)**

- **Space:** O(K2) for the larger heap.

## Method 2: Single Max Heap of Size K2

### Key Insight:
```
  Build ONE Max Heap of size K2.
  This heap stores exactly the K2 smallest elements.

  Among these K2 elements:
  - K1th smallest is somewhere inside
  - K2th smallest = heap.top()
  - Elements between K1th and K2th smallest
    = elements in heap that are > K1th smallest
      AND < K2th smallest (= heap.top())

  Wait — smarter approach:
  Pop K2th smallest (top) → now heap has K2-1 elements
  Then pop elements until we pop K1th smallest
  Sum everything we pop between (exclusive of both)
```

### Algorithm:
```
  1. Build Max Heap of size K2
     → stores K2 smallest elements
     → top = K2th smallest

  2. Pop top (K2th smallest) — don't add to sum

  3. Pop next (K2-1)th smallest — ADD to sum
     ...
     Pop (K1+1)th smallest — ADD to sum

  4. Pop next (K1th smallest) — don't add to sum

  Sum = sum of elements at positions K1+1 to K2-1
```
```
  arr = [20, 8, 22, 4, 12, 10, 14], K1=3, K2=6

  Build Max Heap of size K2=6:
  After processing all: heap = {4, 8, 10, 12, 14, 20}
  (6 smallest elements, max=20 at top)

        20
       /  \
      14   12
     / \ /
    8  4 10

  Pop 1: pop 20 (K2th = 6th smallest) → don't add, sum=0
  Pop 2: pop 14 (5th smallest) → ADD, sum=14
  Pop 3: pop 12 (4th smallest) → ADD, sum=26
  Pop 4: pop 10 (3rd=K1th smallest) → don't add

  Total sum = 14 + 12 = 26 ✓
```

### Code:
```cpp
#include <bits/stdc++.h>
using namespace std;

// Method 2: Single Max Heap
int sumBetweenK1K2_method2(vector<int>& arr, int k1, int k2) {
    // Build Max Heap of size K2 (K2 smallest elements)
    priority_queue<int> maxPQ;

    for (int x : arr) {
        maxPQ.push(x);
        if (maxPQ.size() > k2) {
            maxPQ.pop();
        }
    }

    // Pop K2th smallest (don't count)
    maxPQ.pop();

    // Sum elements between K1th and K2th (exclusive)
    int sum = 0;
    int count = k2 - k1 - 1; // Number of elements to sum

    for (int i = 0; i < count; i++) {
        sum += maxPQ.top();
        maxPQ.pop();
    }

    // Next pop would be K1th smallest — don't count

    return sum;
}

int main() {
    vector<int> arr1 = {2, 5, 3, 1, 4};
    cout << sumBetweenK1K2_method2(arr1, 2, 4) << endl; // 3

    vector<int> arr2 = {20, 8, 22, 4, 12, 10, 14};
    cout << sumBetweenK1K2_method2(arr2, 3, 6) << endl; // 26

    return 0;
}
```

---

### Step-by-Step Trace (Method 2):
```
  arr = [2, 5, 3, 1, 4], K1=2, K2=4
  Sorted: [1, 2, 3, 4, 5]
               ↑     ↑
           2nd=2   4th=4
  Answer: sum of elements between 2 and 4 = 3

  ── Build Max Heap of size K2=4 ──
  push 2 → [2]
  push 5 → [5,2]
  push 3 → [5,3,2]
  push 1 → [5,3,2,1]  size=4=K2, no pop
  push 4 → size=5>4, pop top=5 → [4,3,2,1]

  Heap = {4,3,2,1} = 4 smallest elements
        4
       / \
      3   2
     /
    1

  ── Pop K2th (4th) smallest ──
  pop 4 → heap = {3,2,1}   (don't add to sum)

  ── Sum elements at positions K1+1 to K2-1 ──
  count = K2 - K1 - 1 = 4 - 2 - 1 = 1

  i=0: pop 3, sum += 3 → sum=3

  ── K1th (2nd) smallest would be next ──
  (don't pop/count)

  Return sum = 3 ✓
```

---

## Comparison of Both Methods
```
  Method 1 (Two Max Heaps):          Method 2 (One Max Heap):
  ─────────────────────────          ────────────────────────
  Build heap of K1 → O(N log K1)    Build heap of K2 → O(N log K2)
  Build heap of K2 → O(N log K2)    Pop K2-K1-1 times → O(K2 log K2)
  Linear scan      → O(N)
                                     Total: O(N log K2)
  Total: O(N log K2)

  Space: O(K1 + K2)                 Space: O(K2)

  ┌────────────────────────────────────────────────────────┐
  │  Method 1: Simpler to understand, two passes           │
  │  Method 2: Less space, cleaner — preferred             │
  └────────────────────────────────────────────────────────┘
```

---

## Edge Cases:
```
  K2 - K1 = 1:              No elements between adjacent Kth
  arr=[1,2,3,4,5],K1=2,K2=3
  2nd small=2, 3rd small=3
  No element strictly between 2 and 3
  Sum = 0 ✓

  Duplicates:                Only unique values between bounds counted
  arr=[1,2,2,3,4], K1=1,K2=4
  1st=1, 4th=3
  Between: [2,2] → sum=4

  Large K gap:               Many elements between bounds
  arr=[1..10], K1=1, K2=10
  Between 1 and 10: [2,3,4,5,6,7,8,9]
  Sum = 44
```

---

## Complexity Summary:
```
  Both Methods: O(N log K2) time

  Method 1 space: O(K1 + K2)
  Method 2 space: O(K2)        ← preferred

  For large arrays, both are efficient as K2 << N typically.
```
