## Magician and Chocolates

**Problem Statement:**   
Given `N` bags of chocolates with `A[i]` chocolates in each bag. Each second, the magician:
1. Picks the bag with the **most chocolates**.
2. Eats **all** chocolates from that bag.
3. Puts back a new bag with **floor(A[i] / 2)** chocolates.

Find the **total chocolates eaten** in `k` seconds. Return answer modulo `10^9 + 7`.

---

>### Examples:

**Example 1:**
- **Input:** `A = [6, 5]`, `k = 3`
- **Output:** `14`
```
  Second 1: Pick 6, eat 6, put back 3  → bags=[5,3], eaten=6
  Second 2: Pick 5, eat 5, put back 2  → bags=[3,2], eaten=11
  Second 3: Pick 3, eat 3, put back 1  → bags=[2,1], eaten=14

  Total = 14 ✓
```

**Example 2:**
- **Input:** `A = [2, 4, 6, 8, 10]`, `k = 6`
- **Output:** `33`
```
  Second 1: Pick 10, eat 10, put back 5 → [8,6,5,4,2], eaten=10
  Second 2: Pick 8,  eat 8,  put back 4 → [6,5,4,4,2], eaten=18
  Second 3: Pick 6,  eat 6,  put back 3 → [5,4,4,3,2], eaten=24
  Second 4: Pick 5,  eat 5,  put back 2 → [4,4,3,2,2], eaten=29
  Second 5: Pick 4,  eat 4,  put back 2 → [4,3,2,2,2], eaten=33
  Second 6: Pick 4,  eat 4,  put back 2 → [3,2,2,2,2], eaten=37

  Hmm let me recount... total after 6 = 37
```

---

### Key Insight:
```
  At each step → always eat from MAXIMUM bag
  After eating → new bag = floor(original / 2)

  This screams MAX HEAP:
  - top() gives maximum in O(1)
  - pop() removes it in O(log N)
  - push() adds new bag in O(log N)

  Termination: if top() == 0, eating gives 0 chocolates → stop early
```

---
## Approach
### Algorithm:
```
  1. Build Max Heap from array A
  2. totalEaten = 0
  3. Repeat k times:
       a. If heap.top() == 0 → break (no chocolates left)
       b. top = heap.top(); heap.pop()
       c. totalEaten = (totalEaten + top) % MOD
       d. heap.push(top / 2)
  4. Return totalEaten
```
```
  Max Heap trace for [6, 5], k=3:

  Initial:   heap = [6, 5]
             6
            /
           5

  k=1: eat 6, push 3 → heap=[5,3], eaten=6
  k=2: eat 5, push 2 → heap=[3,2], eaten=11
  k=3: eat 3, push 1 → heap=[2,1], eaten=14

  Return 14 ✓
```

---

### C++ Code:
```cpp
#include <bits/stdc++.h>
using namespace std;

const int MOD = 1e9 + 7;

int nChocolates(int k, vector<int>& A) {
    // Edge case
    if (A.empty() || k == 0) return 0;

    // Build Max Heap — O(N)
    priority_queue<int> maxPQ(A.begin(), A.end());

    long long totalEaten = 0;

    for (int i = 0; i < k; i++) {
        // If max bag is empty, no more chocolates to eat
        if (maxPQ.top() == 0) break;

        // Pick the largest bag
        int top = maxPQ.top();
        maxPQ.pop();

        // Eat all chocolates from that bag
        totalEaten = (totalEaten + top) % MOD;

        // Put back bag with floor(top/2) chocolates
        maxPQ.push(top / 2);
    }

    return (int)totalEaten;
}

int main() {
    vector<int> A1 = {6, 5};
    cout << nChocolates(3, A1) << endl; // 14

    vector<int> A2 = {2, 4, 6, 8, 10};
    cout << nChocolates(6, A2) << endl;

    vector<int> A3 = {1000000000};
    cout << nChocolates(30, A3) << endl; // Large values

    vector<int> A4 = {0, 0, 0};
    cout << nChocolates(3, A4) << endl; // 0 (all zero)

    return 0;
}
```

```
  ── Second 1 ──

  top = 5 (≠ 0), eat 5
  totalEaten = 11

  top = 3 (≠ 0), eat 3
  totalEaten = 14

  heap = [2, 1]

```

---

### When Does Eating Stop Early?
```
  If all bags become 0, no point continuing:

  A = [1, 1], k = 10

  k=1: eat 1, push 0 → heap=[1,0], eaten=1
  k=2: eat 1, push 0 → heap=[0,0], eaten=2
  k=3: top=0 → BREAK

  Return 2 (not 10 iterations needed) ✓

  Large bag eventually reaches 0:
  A = [8], k = 10

  k=1: eat 8, push 4 → eaten=8
  k=2: eat 4, push 2 → eaten=12
  k=3: eat 2, push 1 → eaten=14
  k=4: eat 1, push 0 → eaten=15
  k=5: top=0 → BREAK

  Return 15
```

---

### Handling Large Values (MOD):
```
  Why MOD = 10^9 + 7?
  If A[i] = 10^9 and k = 10^6:
  totalEaten can overflow int and even long long!

  Solution: take mod at each step
  totalEaten = (totalEaten + top) % MOD

  Example:
  A = [10^9], k = 5

  k=1: eat 10^9,       push 5×10^8,  eaten = 10^9
  k=2: eat 5×10^8,     push 25×10^7, eaten = 1.5×10^9 > INT_MAX!
                                             ↑ needs long long + MOD
```

---

### Edge Cases:
```
  k = 0:          No eating → return 0
  All zeros:      A=[0,0,0] → top=0 → break immediately → return 0
  Single bag:     A=[8], k=3 → eat 8,4,2 → return 14
  k > possible:   Eventually all bags become 0 → break early
  Large values:   Use long long + MOD for totalEaten
```

---

### Visualization of Chocolate Decay:
```
  Starting with 64 chocolates:

  Round:  1    2    3    4    5    6    7
  Value: 64 → 32 → 16 → 8  → 4  → 2  → 1  → 0

  Total eaten = 64+32+16+8+4+2+1 = 127

  In general, for value V:
  Total = V + V/2 + V/4 + ... (geometric series)
        ≈ 2V  (approximately)
        = V × (1 - (1/2)^k) / (1 - 1/2) for k iterations
```

---

### Complexity Analysis:

**Time Complexity:** O(N + K log N)
- Building Max Heap: O(N)
- K iterations, each with pop + push: O(K log N)
- Total: **O(N + K log N)**

**Space Complexity:** O(N) for the heap.
```
  N = number of bags
  K = number of seconds

  If K >> N: dominated by O(K log N)
  If K << N: dominated by O(N) build time
```
