## Merge Two Binary Max Heaps

**Problem Statement:**  
Given two binary Max Heaps represented as arrays, merge them into a **single Max Heap** and return the resulting array.

---

>### Examples:

**Example 1:**
- **Input:** `a = [10, 5, 6, 2]`, `b = [12, 7, 9]`
- **Output:** `[12, 10, 9, 2, 5, 6, 7]`
```
  Heap a:          Heap b:
       10               12
      /  \             /  \
     5    6           7    9
    /
   2

  Merged Max Heap:
         12
        /  \
       10   9
      /  \ /
     2   5 6
    /
   7
```

**Example 2:**
- **Input:** `a = [10, 5, 6, 2]`, `b = [12, 7, 9, 1, 10, 15, 20]`
- **Output:** `[20, 10, 15, 2, 5, 12, 9, 1, ...]`

---

### Key Insight:
```
  Naive approach:
  1. Concatenate both arrays → O(N+M)
  2. Sort combined array    → O((N+M) log(N+M))
  Result: valid Max Heap? Only if we heapify!

  Better approach:
  1. Concatenate both arrays         → O(N+M)
  2. Build Max Heap using Heapify    → O(N+M)
  Total: O(N+M) ← optimal!

  Why not just merge and sort?
  Sorting is O((N+M) log(N+M)) — overkill.
  Heapify only needs O(N+M)! ✓
```

---
## Approach
### Algorithm:
```
  1. Create merged array = a + b  (concatenate)
  2. Apply Build Heap (bottom-up heapify) on merged array
     Start from last internal node = (size/2 - 1) down to 0
  3. Return merged array

  This is identical to Method 2 (Build Heap) from earlier!
```
```
  a = [10, 5, 6, 2], b = [12, 7, 9]

  Step 1: Concatenate
  merged = [10, 5, 6, 2, 12, 7, 9]
  size = 7

  Step 2: Build Max Heap (bottom-up)
  Last internal node = (7/2)-1 = 2

  Tree before heapify:
          10  (0)
         /  \
        5    6  (1,2)
       / \ / \
      2 12 7  9  (3,4,5,6)

  Heapify(2): node=6, children=7,9
    max(6,7,9)=9 → SWAP(6,9)
          10
         /  \
        5    9
       / \ / \
      2 12 7  6

  Heapify(1): node=5, children=2,12
    max(5,2,12)=12 → SWAP(5,12)
          10
         /  \
        12   9
       / \ / \
      2  5 7  6
    Heapify(4): node=5, children=none (leaf) → stop

  Heapify(0): node=10, children=12,9
    max(10,12,9)=12 → SWAP(10,12)
          12
         /  \
        10   9
       / \ / \
      2  5 7  6
    Heapify(1): node=10, children=2,5
    max(10,2,5)=10 → no swap

  Final Max Heap: [12, 10, 9, 2, 5, 7, 6] ✓
```

---

### Code:
```cpp
#include <bits/stdc++.h>
using namespace std;

// Heapify down (Max Heap)
void heapify(vector<int>& arr, int n, int i) {
    int largest = i;
    int left    = 2 * i + 1;
    int right   = 2 * i + 2;

    if (left  < n && arr[left]  > arr[largest]) largest = left;
    if (right < n && arr[right] > arr[largest]) largest = right;

    if (largest != i) {
        swap(arr[i], arr[largest]);
        heapify(arr, n, largest);
    }
}

// Merge two Max Heaps
vector<int> mergeHeaps(vector<int>& a, vector<int>& b) {
    // Step 1: Concatenate both arrays
    vector<int> merged;
    merged.reserve(a.size() + b.size());

    for (int x : a) merged.push_back(x);
    for (int x : b) merged.push_back(x);

    int n = merged.size();

    // Step 2: Build Max Heap using bottom-up heapify — O(N+M)
    for (int i = n / 2 - 1; i >= 0; i--) {
        heapify(merged, n, i);
    }

    return merged;
}

void printArray(vector<int>& arr) {
    for (int x : arr) cout << x << " ";
    cout << endl;
}

int main() {
    vector<int> a1 = {10, 5, 6, 2};
    vector<int> b1 = {12, 7, 9};
    vector<int> result1 = mergeHeaps(a1, b1);
    cout << "Merged Heap 1: ";
    printArray(result1); // [12, 10, 9, 2, 5, 7, 6]

    vector<int> a2 = {10, 5, 6, 2};
    vector<int> b2 = {12, 7, 9, 1, 10, 15, 20};
    vector<int> result2 = mergeHeaps(a2, b2);
    cout << "Merged Heap 2: ";
    printArray(result2);

    return 0;
}
```

### Detailed Step-by-Step Trace:
```
  a = [10, 5, 6, 2], b = [12, 7, 9]

  ── Step 1: Concatenate ──
  merged = [10, 5, 6, 2, 12, 7, 9]
  indices:  [0, 1, 2, 3,  4, 5, 6]

  Tree representation:
          10  (idx 0)
         /  \
        5    6  (idx 1, 2)
       / \ / \
      2 12 7  9  (idx 3,4,5,6)

  ── Step 2: Build Max Heap (bottom-up) ──
  n = 7, start from i = n/2-1 = 2

  ─ Heapify(i=2) ─
  node = arr[2] = 6
  left  = arr[5] = 7
  right = arr[6] = 9
  largest = 6 (idx 2)
  7 > 6 → largest = 5
  9 > 7 → largest = 6
  largest(6) ≠ i(2) → SWAP arr[2] and arr[6]
  merged = [10, 5, 9, 2, 12, 7, 6]

          10
         /  \
        5    9  ← was 6, now 9
       / \ / \
      2 12 7  6  ← was 9, now 6

  Heapify(6): left=13, right=14 → out of bounds → stop

  ─ Heapify(i=1) ─
  node = arr[1] = 5
  left  = arr[3] = 2
  right = arr[4] = 12
  12 > 5 → largest = 4
  largest(4) ≠ i(1) → SWAP arr[1] and arr[4]
  merged = [10, 12, 9, 2, 5, 7, 6]

          10
         /  \
        12   9  ← 5 and 12 swapped
       / \ / \
      2  5 7  6

  Heapify(4): node=5, left=9,right=10 → out of bounds → stop

  ─ Heapify(i=0) ─
  node = arr[0] = 10
  left  = arr[1] = 12
  right = arr[2] = 9
  12 > 10 → largest = 1
  9 < 12  → largest stays 1
  largest(1) ≠ i(0) → SWAP arr[0] and arr[1]
  merged = [12, 10, 9, 2, 5, 7, 6]

          12  ← new root
         /  \
        10   9
       / \ / \
      2  5 7  6

  Heapify(1): node=10, left=arr[3]=2, right=arr[4]=5
  max(10,2,5)=10 → no swap → stop

  ── Final Merged Max Heap ──
  [12, 10, 9, 2, 5, 7, 6] ✓

  Verification:
  12 > 10, 9 ✓    (root > children)
  10 > 2, 5  ✓    (node > children)
  9 > 7, 6   ✓    (node > children)
```

### Why Not Just Merge and Sort?
```
  Sort approach:
  [10, 5, 6, 2, 12, 7, 9] → sort desc → [12, 10, 9, 7, 6, 5, 2]
  This IS a valid Max Heap, but:
  Time = O((N+M) log(N+M)) ← extra log factor!

  Build Heap approach:
  Concatenate + Heapify = O(N+M) + O(N+M) = O(N+M) ✓
  Faster by a log factor!

  Comparison:
  N=M=10^6:
  Sort:      ~4×10^7 operations
  BuildHeap: ~2×10^6 operations  ← 20× faster!
```

---

### Edge Cases:
```
  One heap empty:            Both heaps empty:
  a=[], b=[5,3,8]            a=[], b=[]
  merged=[5,3,8]             merged=[]
  Already a valid heap ✓     Return [] ✓

  Single element:            All equal elements:
  a=[10], b=[5]              a=[3,3], b=[3,3]
  merged=[10,5]              merged=[3,3,3,3]
  Heapify → [10,5] ✓         All valid ✓

  One much larger:
  a=[1,2,3], b=[100,99,98]
  merged after heapify: [100,99,98,1,2,3] or similar valid Max Heap
```

---

### Complexity Analysis:

**Time Complexity:** O(N + M)
- Concatenation: O(N + M)
- Build Max Heap (bottom-up heapify): O(N + M)
- Total: **O(N + M)**

**Space Complexity:** O(N + M) for the merged array.
```
  N = size of heap a
  M = size of heap b

  Why O(N+M) and not O((N+M) log(N+M))?
  Bottom-up Build Heap is always O(size) — not O(size × log(size))
  This is the key advantage over sorting! ✓

  Proof sketch:
  Nodes at height h do ≤ h comparisons
  Few nodes near root, many near leaves
  Total work = Σ h × (N+M)/2^h = O(N+M) ✓
```
