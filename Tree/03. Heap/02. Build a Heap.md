## Method 1 — Insert One by One: O(N log N)

### Idea

Insert elements **one by one** using `Insert()`. Each insertion bubbles the new element **up** to restore the heap property.

### Step-by-Step Example

Build a Min-Heap from `[4, 1, 2, 6, 7, 3, 8, 5]`:

```
Insert 4  →  [4]
Insert 1  →  [4, 1]  →  1 < parent(4)  →  SWAP  →  [1, 4]
Insert 2  →  [1, 4, 2]  →  2 > parent(1)  →  no swap
Insert 6  →  [1, 4, 2, 6]  →  no swap
Insert 7  →  [1, 4, 2, 6, 7]  →  no swap
Insert 3  →  [1, 4, 2, 6, 7, 3]  →  3 > parent(2)  →  no swap
Insert 8  →  [1, 4, 2, 6, 7, 3, 8]  →  no swap
Insert 5  →  [1, 4, 2, 6, 7, 3, 8, 5]  →  no swap

Final Min-Heap: [1, 4, 2, 6, 7, 3, 8, 5] ✓
```

### C++ Code

```cpp
#include <bits/stdc++.h>
using namespace std;

class BinaryHeap {
public:
    int capacity, size, *arr;

    BinaryHeap(int cap) { capacity = cap; size = 0; arr = new int[cap]; }

    int parent(int i) { return (i - 1) / 2; }
    int left(int i)   { return 2 * i + 1;   }
    int right(int i)  { return 2 * i + 2;   }

    void Insert(int x) {
        if (size == capacity) { cout << "Overflow\n"; return; }
        arr[size] = x;
        int k = size++;

        // Bubble up
        while (k != 0 && arr[parent(k)] > arr[k]) {
            swap(arr[parent(k)], arr[k]);
            k = parent(k);
        }
    }

    void buildHeap(int* input, int n) {
        for (int i = 0; i < n; i++) Insert(input[i]);
    }

    void print() {
        for (int i = 0; i < size; i++) cout << arr[i] << " ";
        cout << "\n";
    }
};

int main() {
    int arr[] = {4, 1, 2, 6, 7, 3, 8, 5};
    BinaryHeap h(8);
    h.buildHeap(arr, 8);
    cout << "Heap (Method 1): ";
    h.print();
}
```

### Complexity Analysis

Each `Insert()` takes **O(log N)** — the new element bubbles up at most the full height of the tree.

Work per level (bottom-up):

| Level (bottom-up) | Nodes | Work per node | Total work |
|-------------------|-------|---------------|------------|
| 0 (leaves) | $n/2$ | $\log n$ | $\dfrac{n}{2} \log n$ |
| 1 | $n/4$ | $\log n - 1$ | $\dfrac{n}{4}(\log n - 1)$ |
| 2 | $n/8$ | $\log n - 2$ | $\dfrac{n}{8}(\log n - 2)$ |
| $\vdots$ | $\vdots$ | $\vdots$ | $\vdots$ |
| $\log n$ (root) | $1$ | $1$ | $1$ |

**Total work:**

$$T(n) = \frac{n}{2}\log n + \frac{n}{4}(\log n - 1) + \frac{n}{8}(\log n - 2) + \cdots$$

Separate into two parts:

$$T(n) = n\log n\!\left(\frac{1}{2} + \frac{1}{4} + \frac{1}{8} + \cdots\right) - n\!\left(\frac{1}{4} + \frac{2}{8} + \frac{3}{16} + \cdots\right)$$

The first series $\displaystyle\sum_{k=1}^{\infty}\frac{1}{2^k} = 1$, and the second converges to a constant. Therefore:

$$\boxed{T(n) = O(n \log n)}$$

## Method 2 — Bottom-Up Heapify: O(N)

### Idea

Copy **all elements** into the array first, then call `Heapify()` **bottom-up** starting from the last internal node down to the root.

```
Key insight:
  Leaf nodes (indices n/2 … n-1)  → already valid heaps (no children)
  Internal nodes (indices 0 … n/2-1) → need heapifying

  Start at last internal node = index (n/2 − 1)
  Work upward toward the root
  When a node is heapified, its subtrees are already valid ✓
```

### C++ Code

```cpp
#include <bits/stdc++.h>
using namespace std;

class BinaryHeap {
public:
    int capacity, size, *arr;

    BinaryHeap(int cap) { capacity = cap; size = 0; arr = new int[cap]; }

    int left(int i)  { return 2 * i + 1; }
    int right(int i) { return 2 * i + 2; }

    void Heapify(int i) {
        int smallest = i;
        int l = left(i), r = right(i);
        if (l < size && arr[l] < arr[smallest]) smallest = l;
        if (r < size && arr[r] < arr[smallest]) smallest = r;
        if (smallest != i) {
            swap(arr[i], arr[smallest]);
            Heapify(smallest);   // sift down recursively
        }
    }

    void buildHeap(int* input, int n) {
        for (int i = 0; i < n; i++) arr[i] = input[i];
        size = n;

        // Heapify every internal node from bottom to root
        for (int i = (n / 2) - 1; i >= 0; i--)
            Heapify(i);
    }

    void print() {
        for (int i = 0; i < size; i++) cout << arr[i] << " ";
        cout << "\n";
    }
};

int main() {
    int arr[] = {4, 1, 2, 6, 7, 3, 8, 5};
    BinaryHeap h(8);
    h.buildHeap(arr, 8);
    cout << "Heap (Method 2): ";
    h.print();
}
```

### Complexity Analysis

#### Time Complexity

A node at height $h$ does **at most $h$ swaps** during `Heapify`.
The number of nodes at height $h$ in a complete binary tree is $\approx \dfrac{n}{2^{h+1}}$.

**Work per level (bottom-up):**

| Level (bottom-up) | Nodes | Work per node | Total work |
|-------------------|-------|---------------|------------|
| 0 (leaves) | $n/2$ | $0$ | $0$ |
| 1 | $n/4$ | $1$ | $\dfrac{n}{4} \times 1$ |
| 2 | $n/8$ | $2$ | $\dfrac{n}{8} \times 2$ |
| 3 | $n/16$ | $3$ | $\dfrac{n}{16} \times 3$ |
| $\vdots$ | $\vdots$ | $\vdots$ | $\vdots$ |
| $\log n$ (root) | $1$ | $\log n$ | $\log n$ |

**Step 1 — Write out the total work:**

Leaves do zero work, so the sum starts from level 1:

$$T(n) = \frac{n}{4} \times 1 + \frac{n}{8} \times 2 + \frac{n}{16} \times 3 + \cdots + 1 \times \log n$$

**Step 2 — Factor out $n$:**

$$T(n) = n \left[ \frac{1}{4} + \frac{2}{8} + \frac{3}{16} + \frac{4}{32} + \cdots \right]$$

$$T(n) = n \sum_{h=1}^{\log n} \frac{h}{2^{h+1}} = \frac{n}{2} \sum_{h=1}^{\infty} \frac{h}{2^{h}}$$

**Step 3 — Evaluate the series using the identity:**

For $|x| < 1$, we have $\displaystyle\sum_{h=1}^{\infty} h \cdot x^h = \frac{x}{(1-x)^2}$

Substitute $x = \dfrac{1}{2}$:

$$\sum_{h=1}^{\infty} \frac{h}{2^h} = \frac{\frac{1}{2}}{\left(1 - \frac{1}{2}\right)^2} = \frac{\frac{1}{2}}{\frac{1}{4}} = 2$$

**Step 4 — Substitute back:**

$$T(n) = \frac{n}{2} \times 2 = n$$

$$\boxed{T(n) = O(n)}$$

> **Intuition:** Leaf nodes ($n/2$ of them) do **zero work**. Nodes near the root do $O(\log n)$ work, but there are very few of them. The weighted sum converges to a constant — so total work is linear.

#### Space Complexity

| Component | Space | Reason |
|-----------|-------|--------|
| Array | $O(n)$ | Stores all $n$ elements |
| Recursion stack (`Heapify`) | $O(\log n)$ | `Heapify` recurses at most $h = \log n$ levels deep |
| **Total** | $O(n)$ | Array dominates |

## Comparison

| | Method 1 — Insert One by One | Method 2 — Bottom-Up Heapify |
|-|------------------------------|------------------------------|
| **Time complexity** | $O(n \log n)$ | $O(n)$ ✅ more optimal |
| **Space complexity** | $O(n)$ | $O(n)$ array + $O(\log n)$ stack |
| **Direction** | Each element bubbles **up** | Internal nodes sift **down** |
| **Elements processed** | All $n$ elements | Only internal nodes ($n/2$) |
| **Analogy** | Sort $n$ cards one at a time into a pile | Rearrange all cards already on the table in one pass |

### Why is Method 2 faster?

```
Method 1:  Every element may bubble up the full height O(log N)
           N elements × O(log N) = O(N log N)

Method 2:  Leaf nodes (N/2 of them) do ZERO work
           Only nodes near the root do O(log N) work
           But there are very few of them (just 1 at the top!)
           Mathematical sum of the series  →  O(N)
```

### When to use which?

| Situation | Use |
|-----------|-----|
| Elements arrive as a stream (online) | Method 1 |
| Need dynamic insertions | Method 1 |
| All elements known upfront | Method 2 |
| Building from an existing array | Method 2 |
| Used inside HeapSort | Method 2 |
