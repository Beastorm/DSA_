## Building a Heap — Two Methods


## Method 1: Insert One by One (Naive)- Step Up

### Idea:

Insert elements into the heap **one by one** using the `Insert()` operation. Each insertion bubbles up the new element to maintain the heap property.
```
  Build heap from: [4, 1, 2, 6, 7, 3, 8, 5]

  Insert 4:  [4]
  Insert 1:  [4, 1] → 1 < parent(4) → SWAP → [1, 4]
  Insert 2:  [1, 4, 2] → 2 < parent(1) → no swap
  Insert 6:  [1, 4, 2, 6] → 6 > parent(4) → no swap
  Insert 7:  [1, 4, 2, 6, 7] → no swap
  Insert 3:  [1, 4, 2, 6, 7, 3] → 3 < parent(2) → SWAP
             [1, 4, 2, 6, 7, 3] → 2 < parent? no
  Insert 8:  [1, 4, 2, 6, 7, 3, 8] → no swap
  Insert 5:  [1, 4, 2, 6, 7, 3, 8, 5] → 5 < parent? no

  Final Min Heap: [1, 4, 2, 6, 7, 3, 8, 5] ✓
```

### C++ Code:
```cpp
#include <bits/stdc++.h>
using namespace std;

class BinaryHeap {
public:
    int capacity, size;
    int* arr;

    BinaryHeap(int cap) {
        capacity = cap;
        size = 0;
        arr = new int[capacity];
    }

    int parent(int i) { return (i - 1) / 2; }
    int left(int i)   { return 2 * i + 1;   }
    int right(int i)  { return 2 * i + 2;   }

    // Insert and bubble up
    void Insert(int x) {
        if (size == capacity) {
            cout << "Overflow" << endl;
            return;
        }
        arr[size] = x;
        int k = size;
        size++;

        // Bubble up
        while (k != 0 && arr[parent(k)] > arr[k]) {
            swap(arr[parent(k)], arr[k]);
            k = parent(k);
        }
    }

    // Method 1: Build heap by inserting one by one
    void buildHeapMethod1(int* input, int n) {
        for (int i = 0; i < n; i++) {
            Insert(input[i]);
        }
    }

    void print() {
        for (int i = 0; i < size; i++)
            cout << arr[i] << " ";
        cout << endl;
    }
};

int main() {
    int arr[] = {4, 1, 2, 6, 7, 3, 8, 5};
    int n = 8;

    BinaryHeap h(n);
    h.buildHeapMethod1(arr, n);

    cout << "Heap (Method 1): ";
    h.print();

    return 0;
}
```

### Complexity Analysis:

**Time Complexity:** O(N log N)
- Each `Insert()` takes O(log N) in the worst case.
- We call it N times → O(N log N) total.

**Space Complexity:** O(N) for the array.

# Complexity Calculations (Bottom)

Here is the step-by-step breakdown of the time complexity calculation shown at the bottom of the board:

**Step 1 — Initial Summation:**

$$
\frac{n}{2} \times \log n + \frac{n}{4} \times (\log n - 1) + \frac{n}{8} \times (\log n - 2)
$$

**Step 2 — Separating the Terms:**

$$
\left( \frac{n}{2} \times \log n + \frac{n}{4} \times \log n + \frac{n}{8} \times \log n + \dots \right) - \left( \frac{n}{4} + \frac{2n}{8} + \frac{3n}{16} + \dots \right)
$$

**Step 3 — Factoring out $n \log n$ and $n$:**

$$
n \log n \left( \frac{1}{2} + \frac{1}{4} + \frac{1}{8} + \frac{1}{16} + \dots \right) - n \left( \frac{1}{4} + \frac{2}{8} + \frac{3}{16} + \dots \right)
$$

**Step 4 — Final Result:**

$$
n \log n
$$

---

### Summary:

| Level (bottom-up) | Nodes at Level | Work per Node | Total Work |
|---|---|---|---|
| 0 (leaves) | $n/2$ | $\log n$ | $\frac{n}{2} \log n$ |
| 1 | $n/4$ | $\log n - 1$ | $\frac{n}{4} (\log n - 1)$ |
| 2 | $n/8$ | $\log n - 2$ | $\frac{n}{8} (\log n - 2)$ |
| $\dots$ | $\dots$ | $\dots$ | $\dots$ |
| $\log n$ (root) | $1$ | $1$ | $1$ |

The infinite geometric series $\frac{1}{2} + \frac{1}{4} + \frac{1}{8} + \dots = 1$, and the subtracted series converges to a constant. Therefore, the overall time complexity simplifies to:

$$
\boxed{O(n \log n)}
$$

## Method 2: Build Heap Using Heapify (Optimized) Step-Down

### Idea:

Copy all elements into the array first, then apply `Heapify()` **bottom-up** starting from the last non-leaf node up to the root. Leaf nodes are already valid heaps (size 1), so we only need to heapify internal nodes.
```
  Key insight:
  - Leaf nodes: indices n/2 to n-1  (no children, already valid)
  - Internal nodes: indices 0 to n/2-1  (need heapifying)
  - Start from LAST internal node = index (n/2 - 1)
  - Heapify from bottom-up → children are already valid when parent is processed
```

### C++ Code:
```cpp
#include <bits/stdc++.h>
using namespace std;

class BinaryHeap {
public:
    int capacity, size;
    int* arr;

    BinaryHeap(int cap) {
        capacity = cap;
        size = 0;
        arr = new int[capacity];
    }

    int parent(int i) { return (i - 1) / 2; }
    int left(int i)   { return 2 * i + 1;   }
    int right(int i)  { return 2 * i + 2;   }

    // Sift down to fix heap at index ind
    void Heapify(int ind) {
        int li = left(ind);
        int ri = right(ind);
        int smallest = ind;

        if (li < size && arr[li] < arr[smallest]) smallest = li;
        if (ri < size && arr[ri] < arr[smallest]) smallest = ri;

        if (smallest != ind) {
            swap(arr[ind], arr[smallest]);
            Heapify(smallest);
        }
    }

    // Method 2: Build heap using bottom-up heapify
    void buildHeapMethod2(int* input, int n) {
        // Copy all elements into array
        for (int i = 0; i < n; i++) {
            arr[i] = input[i];
        }
        size = n;

        // Start from last internal node, heapify upward
        // Last internal node index = (n/2) - 1
        for (int i = (n / 2) - 1; i >= 0; i--) {
            Heapify(i);
        }
    }

    void print() {
        for (int i = 0; i < size; i++)
            cout << arr[i] << " ";
        cout << endl;
    }
};

int main() {
    int arr[] = {4, 1, 2, 6, 7, 3, 8, 5};
    int n = 8;

    BinaryHeap h(n);
    h.buildHeapMethod2(arr, n);

    cout << "Heap (Method 2): ";
    h.print();

    return 0;
}
```

### Complexity Analysis:

- **Time Complexity:** O(N)
```
  Why O(N) and not O(N log N)?

  Nodes at height h do at most h swaps during Heapify.
  Number of nodes at height h ≈ N / 2^(h+1)

  Total work = Σ (h * N/2^(h+1)) for h=0 to log N
             = N * Σ (h / 2^h)
             = N * 2         (geometric series sum = 2)
             = O(N) ✓

  Most nodes are LEAVES (height 0) → do no work
  Few nodes near the root (high height) → do more work
  But there are very few of them!
```

- **Space Complexity:** O(N) for the array, O(log N) for the recursion stack in Heapify.

The series $\sum_{k=0}^{\infty} \frac{k}{2^{k+2}}$ converges to a constant, so the total work is bounded by:

$$
n \times \text{constant} = \boxed{O(n)}
$$

### Comparison of Both Methods:
```
  Method 1: Insert one by one     Method 2: Bottom-up Heapify
  ────────────────────────────    ────────────────────────────
  Time: O(N log N)                Time: O(N)  ← More Optimal
  Space: O(N)                     Space: O(N)

  How: Insert + bubble UP         How: Copy all + Heapify DOWN
  Direction: Bottom → Top         Direction: Top ← Bottom
             (each element)                  (internal nodes only)

  Analogy: Sort N cards one       Analogy: Rearrange N cards
           at a time into a                already on table
           sorted pile                     in one pass
```
```
  Why Method 2 is faster:

  Method 1: Every element bubbles up to height O(log N)
            N elements × O(log N) = O(N log N)

  Method 2: Leaf nodes (N/2) do ZERO work
            Nodes near the root (very few) do O(log N) work
            Total: mathematical sum = O(N)
```

---

### When to Use Which?
```
  Use Method 1 (Insert one by one):
  - When elements arrive online (stream)
  - When you need to insert dynamically
  - Simpler to implement

  Use Method 2 (Bottom-up Heapify):
  - When all elements are known upfront
  - When building heap from an existing array
  - More efficient: O(N) vs O(N log N)
  - Used internally by HeapSort
```
