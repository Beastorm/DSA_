## Kth Smallest Element in a Matrix

**Problem Statement:**    
Given an `N × N` matrix where each row and each column is sorted in ascending order, find the **Kth smallest element** in the matrix.

---

>### Examples:

**Example 1:**
- **Input:**
```
  matrix = [[1,  5,  9],
             [10, 11, 13],
             [12, 13, 15]], K = 8
```
- **Output:** `13`
```
  All elements sorted: [1, 5, 9, 10, 11, 12, 13, 13, 15]
                                              ↑
                                         8th = 13 ✓
```

**Example 2:**
- **Input:**
```
  matrix = [[1, 2],
             [1, 3]], K = 2
```
- **Output:** `1`
```
  All elements sorted: [1, 1, 2, 3]
                            ↑
                       2nd = 1 ✓
```

---

## Approach 1: Brute Force (Collect + Sort)

### Algorithm:
```
  1. Collect all N×N elements into array
  2. Sort the array
  3. Return element at index K-1
```

### Code
```cpp
int kthSmallest_brute(vector<vector<int>>& matrix, int k) {
    vector<int> arr;

    for (auto& row : matrix)
        for (int x : row)
            arr.push_back(x);

    sort(arr.begin(), arr.end());
    return arr[k - 1];
}
```
```
  matrix = [[1,5,9],[10,11,13],[12,13,15]]
  arr = [1,5,9,10,11,13,12,13,15]
  sorted = [1,5,9,10,11,12,13,13,15]
  arr[K-1] = arr[7] = 13 ✓
```

**Time:** O(N² log N²) = O(N² log N) | **Space:** O(N²)

## Approach 2: Max Heap of Size K

### Algorithm:
```
  Maintain a Max Heap of size K.
  For each element: push, if size > K → pop.
  top() = Kth smallest.
```

### Code:
```cpp
int kthSmallest_maxHeap(vector<vector<int>>& matrix, int k) {
    priority_queue<int> maxPQ; // Max Heap

    for (auto& row : matrix) {
        for (int x : row) {
            maxPQ.push(x);
            if (maxPQ.size() > k) {
                maxPQ.pop();
            }
        }
    }

    return maxPQ.top();
}
```
```
  K=3, matrix = [[1,5,9],[10,11,13],[12,13,15]]

  Push 1  → heap=[1]
  Push 5  → heap=[5,1]
  Push 9  → heap=[9,5,1]          size=3=K
  Push 10 → size=4>3, pop 10      heap=[9,5,1] (10>top? yes pop 10)

  Wait: we push 10, then pop top=10? No:
  push 10 → [10,9,5,1]? size=4>3 → pop top=10 → [9,5,1]? No...
  Actually push then pop top:
  push 10 → heap=[10,9,5,1] → pop 10 → heap=[9,5,1]

  Hmm: 10 > 9 (current max) so popping 10 keeps [9,5,1]
  But that's wrong! 10 is 4th smallest, should not be in K=3 smallest.

  Actually for Kth SMALLEST using Max Heap of size K:
  We keep the K SMALLEST elements.
  Max of those K = Kth smallest.

  push 1  → [1]
  push 5  → [5,1]
  push 9  → [9,5,1]
  push 10 → [10,9,5,1], size=4>K=3, pop top=10 → [9,5,1]
  push 11 → [11,9,5,1], pop 11 → [9,5,1]
  push 13 → pop 13 → [9,5,1]
  push 12 → pop 12 → [9,5,1]
  push 13 → pop 13 → [9,5,1]
  push 15 → pop 15 → [9,5,1]

  top = 9 = 3rd smallest ✓

  For K=8:
  After all pushes, heap stores 8 smallest: {1,5,9,10,11,12,13,13}
  top = 13 = 8th smallest ✓
```

**Time:** O(N² log K) | **Space:** O(K)

## Approach 3: Min Heap (Merge K Sorted Lists)

### Algorithm:
```
  Treat each ROW as a sorted list.
  Use Min Heap to merge K sorted lists.
  Pop K times → Kth smallest.

  Each heap entry: {value, row, col}
  Start: push first element of each row.
  Each pop: push next element from same row.
```

### Code:
```cpp
int kthSmallest_minHeap(vector<vector<int>>& matrix, int k) {
    int n = matrix.size();

    // Min Heap: {value, row, col}
    priority_queue<tuple<int,int,int>,
                   vector<tuple<int,int,int>>,
                   greater<>> minPQ;

    // Push first element of each row
    for (int i = 0; i < n; i++) {
        minPQ.push({matrix[i][0], i, 0});
    }

    int result = 0;

    // Pop K times
    for (int i = 0; i < k; i++) {
        auto [val, row, col] = minPQ.top();
        minPQ.pop();
        result = val;

        // Push next element from same row
        if (col + 1 < n) {
            minPQ.push({matrix[row][col + 1], row, col + 1});
        }
    }

    return result;
}
```
```
  matrix = [[1,5,9],[10,11,13],[12,13,15]], K=4

  Initial heap (first of each row):
  {1,0,0}, {10,1,0}, {12,2,0}
       1
      / \
    10   12

  Pop 1: {1,0,0} → result=1, push {5,0,1}
  heap: {5,0,1}, {10,1,0}, {12,2,0}

  Pop 2: {5,0,1} → result=5, push {9,0,2}
  heap: {9,0,2}, {10,1,0}, {12,2,0}

  Pop 3: {9,0,2} → result=9, no next in row 0
  heap: {10,1,0}, {12,2,0}

  Pop 4: {10,1,0} → result=10, push {11,1,1}
  heap: {11,1,1}, {12,2,0}

  Return result = 10 = 4th smallest ✓

  Sorted: [1,5,9,10,11,12,13,13,15]
               ↑ 4th = 10 ✓
```

**Time:** O(K log N) | **Space:** O(N)

## Approach 4: Binary Search on Value Range

### Algorithm:
```
  The answer lies between matrix[0][0] (min) and matrix[N-1][N-1] (max).
  Binary search on the VALUE range.

  For a mid value, count how many elements ≤ mid.
  If count < K → answer is in upper half.
  If count ≥ K → answer might be mid or lower.

  Count elements ≤ mid efficiently:
  Start from bottom-left corner.
  If matrix[row][col] ≤ mid → all elements in this column above are ≤ mid
    → count += (row+1), move right.
  Else → move up.
```

### Code:
```cpp
// Count elements ≤ mid in sorted matrix
int countLessEqual(vector<vector<int>>& matrix, int mid) {
    int n = matrix.size();
    int count = 0;
    int row = n - 1, col = 0; // Start from bottom-left

    while (row >= 0 && col < n) {
        if (matrix[row][col] <= mid) {
            count += row + 1; // All elements in this column up to row
            col++;            // Move right
        } else {
            row--;            // Move up
        }
    }
    return count;
}

int kthSmallest_binarySearch(vector<vector<int>>& matrix, int k) {
    int n = matrix.size();
    int lo = matrix[0][0];
    int hi = matrix[n-1][n-1];

    while (lo < hi) {
        int mid = lo + (hi - lo) / 2;
        int count = countLessEqual(matrix, mid);

        if (count < k) {
            lo = mid + 1; // Need more elements → go right
        } else {
            hi = mid;     // mid could be answer → go left
        }
    }

    return lo; // lo == hi = Kth smallest
}
```
```
  matrix = [[1,5,9],[10,11,13],[12,13,15]], K=8

  lo=1, hi=15

  Iteration 1: mid=8
  Count ≤ 8: start(2,0)
    matrix[2][0]=12 > 8 → row-- → (1,0)
    matrix[1][0]=10 > 8 → row-- → (0,0)
    matrix[0][0]=1 ≤ 8  → count+=1, col++ → (0,1)
    matrix[0][1]=5 ≤ 8  → count+=1, col++ → (0,2)
    matrix[0][2]=9 > 8  → row-- → (-1,2) stop
  count=2 < K=8 → lo=9

  Iteration 2: lo=9, hi=15, mid=12
  Count ≤ 12:
    matrix[2][0]=12 ≤ 12 → count+=3, col++ → (2,1)
    matrix[2][1]=13 > 12 → row-- → (1,1)
    matrix[1][1]=11 ≤ 12 → count+=2, col++ → (1,2)
    matrix[1][2]=13 > 12 → row-- → (0,2)
    matrix[0][2]=9 ≤ 12  → count+=1, col++ → done
  count=6 < K=8 → lo=13

  Iteration 3: lo=13, hi=15, mid=14
  Count ≤ 14:
    matrix[2][0]=12 ≤ 14 → count+=3, col++
    matrix[2][1]=13 ≤ 14 → count+=3, col++
    matrix[2][2]=15 > 14 → row--
    matrix[1][2]=13 ≤ 14 → count+=2, col++ → done
  count=8 ≥ K=8 → hi=14

  Iteration 4: lo=13, hi=14, mid=13
  Count ≤ 13: = 8 ≥ K=8 → hi=13

  lo=hi=13 → return 13 ✓
```

**Time:** O(N log(max-min)) | **Space:** O(1)

## Approach 5: Nth Root / STL Built-in

### Algorithm (Using nth_element):
```
  C++ STL nth_element rearranges array so that:
  - Element at Kth position is what it would be if sorted
  - Elements before it are ≤ it (not necessarily sorted)
  Average O(N²) time.
```

### Code:
```cpp
int kthSmallest_nth(vector<vector<int>>& matrix, int k) {
    vector<int> arr;

    for (auto& row : matrix)
        for (int x : row)
            arr.push_back(x);

    // Rearrange so K-1 position has the Kth smallest
    nth_element(arr.begin(), arr.begin() + k - 1, arr.end());

    return arr[k - 1];
}
```

**Time:** O(N²) average | **Space:** O(N²)

## Complete Code (All Approaches)
```cpp
#include <bits/stdc++.h>
using namespace std;

class Solution {
public:
    // Approach 1: Sort
    int approach1(vector<vector<int>>& matrix, int k) {
        vector<int> arr;
        for (auto& row : matrix)
            for (int x : row) arr.push_back(x);
        sort(arr.begin(), arr.end());
        return arr[k - 1];
    }

    // Approach 2: Max Heap of size K
    int approach2(vector<vector<int>>& matrix, int k) {
        priority_queue<int> maxPQ;
        for (auto& row : matrix) {
            for (int x : row) {
                maxPQ.push(x);
                if (maxPQ.size() > k) maxPQ.pop();
            }
        }
        return maxPQ.top();
    }

    // Approach 3: Min Heap (merge rows)
    int approach3(vector<vector<int>>& matrix, int k) {
        int n = matrix.size();
        priority_queue<tuple<int,int,int>,
                       vector<tuple<int,int,int>>,
                       greater<>> minPQ;

        for (int i = 0; i < n; i++)
            minPQ.push({matrix[i][0], i, 0});

        int result = 0;
        for (int i = 0; i < k; i++) {
            auto [val, row, col] = minPQ.top(); minPQ.pop();
            result = val;
            if (col + 1 < n)
                minPQ.push({matrix[row][col+1], row, col+1});
        }
        return result;
    }

    // Approach 4: Binary Search
    int countLessEqual(vector<vector<int>>& matrix, int mid) {
        int n = matrix.size(), count = 0;
        int row = n-1, col = 0;
        while (row >= 0 && col < n) {
            if (matrix[row][col] <= mid) { count += row+1; col++; }
            else row--;
        }
        return count;
    }

    int approach4(vector<vector<int>>& matrix, int k) {
        int n = matrix.size();
        int lo = matrix[0][0], hi = matrix[n-1][n-1];
        while (lo < hi) {
            int mid = lo + (hi-lo)/2;
            if (countLessEqual(matrix, mid) < k) lo = mid+1;
            else hi = mid;
        }
        return lo;
    }

    // Approach 5: nth_element
    int approach5(vector<vector<int>>& matrix, int k) {
        vector<int> arr;
        for (auto& row : matrix)
            for (int x : row) arr.push_back(x);
        nth_element(arr.begin(), arr.begin()+k-1, arr.end());
        return arr[k-1];
    }
};

int main() {
    vector<vector<int>> matrix = {
        {1,  5,  9},
        {10, 11, 13},
        {12, 13, 15}
    };
    int k = 8;

    Solution sol;
    cout << "Approach 1 (Sort):         " << sol.approach1(matrix, k) << endl;
    cout << "Approach 2 (Max Heap K):   " << sol.approach2(matrix, k) << endl;
    cout << "Approach 3 (Min Heap):     " << sol.approach3(matrix, k) << endl;
    cout << "Approach 4 (Binary Search):" << sol.approach4(matrix, k) << endl;
    cout << "Approach 5 (nth_element):  " << sol.approach5(matrix, k) << endl;

    return 0;
}
```

## Complexity Comparison:
```
  Approach           Time              Space    Notes
  ─────────────────────────────────────────────────────────────
  1. Sort            O(N² log N)       O(N²)    Simplest
  2. Max Heap K      O(N² log K)       O(K)     Good when K small
  3. Min Heap N rows O(K log N)        O(N)     Best when K small
  4. Binary Search   O(N log(max-min)) O(1)     Best space, fast
  5. nth_element     O(N²) avg         O(N²)    STL convenience

  ┌─────────────────────────────────────────────────────────┐
  │  Best Overall: Binary Search → O(N log V), O(1) space   │
  │  Best for small K: Min Heap → O(K log N)                │
  │  Simplest: Sort → O(N² log N)                           │
  └─────────────────────────────────────────────────────────┘
```
