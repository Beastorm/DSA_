## Kth Largest Element in an Array

**Problem Statement:**  
Given an integer array `nums` and an integer `k`, return the **Kth largest element** in the array. Note: it is the Kth largest element in sorted order, not the Kth distinct element.

---

>### Examples:

**Example 1:**
- **Input:** `nums = [3, 2, 1, 5, 6, 4]`, `k = 2`
- **Output:** `5`
```
  Sorted (desc): [6, 5, 4, 3, 2, 1]
                      ↑
                 2nd largest = 5 ✓
```

**Example 2:**
- **Input:** `nums = [3, 2, 3, 1, 2, 4, 5, 5, 6]`, `k = 4`
- **Output:** `4`
```
  Sorted (desc): [6, 5, 5, 4, 3, 3, 2, 2, 1]
                           ↑
                  4th largest = 4 ✓
```

**Example 3:**
- **Input:** `nums = [1]`, `k = 1`
- **Output:** `1`

---

### Key Insight: Kth Largest = (N-K+1)th Smallest
```
  arr = [3, 2, 1, 5, 6, 4], K=2

  Sorted ascending:  [1, 2, 3, 4, 5, 6]
  Sorted descending: [6, 5, 4, 3, 2, 1]
                          ↑
  Kth largest (K=2) = 5
  = (N-K+1)th smallest = (6-2+1)=5th smallest = 5 ✓
```

---

### Approaches Overview
```
  Method 1: Sort             → O(N log N)      simple
  Method 2: Min Heap (K)     → O(N log K)      optimal for streams
  Method 3: Max Heap         → O(N + K log N)
  Method 4: QuickSelect      → O(N) average
```

---

## Method 1: Sorting (Brute Force)

### Algorithm:
Sort in descending order, return element at index `K-1`.
```
  arr = [3, 2, 1, 5, 6, 4], K=2

  Sort desc → [6, 5, 4, 3, 2, 1]
  Return arr[K-1] = arr[1] = 5 ✓
```

### Code:
```cpp
int findKthLargest_sort(vector<int>& nums, int k) {
    sort(nums.begin(), nums.end(), greater<int>());
    return nums[k - 1];

    // OR sort ascending and return nums[n-k]
    // sort(nums.begin(), nums.end());
    // return nums[nums.size() - k];
}
```

**Time:** O(N log N) | **Space:** O(1)

## Method 2: Min Heap of Size K (Optimal)

### Key Insight:
```
  Maintain a Min Heap of size K.
  The top of the heap = Kth largest element!

  Why?
  If heap has exactly K elements and top is the min,
  then all K elements are the K largest seen so far,
  and the smallest among them = Kth largest ✓

  For each new element x:
  - If x > heap.top() → x is larger than current Kth largest
    → pop top (too small), push x
  - If x ≤ heap.top() → x is not in K largest, skip
```

### Algorithm:
```
  1. Build Min Heap with first K elements
  2. For each remaining element x:
       If x > heap.top():
         heap.pop()
         heap.push(x)
  3. Return heap.top() → Kth largest
```
```
  arr = [3, 2, 1, 5, 6, 4], K=2

  Process first K=2 elements:
  push 3 → heap=[3]
  push 2 → heap=[2, 3]
       2
        \
         3

  Process remaining:
  x=1: 1 < top(2) → skip
  x=5: 5 > top(2) → pop 2, push 5 → heap=[3,5]
       3
        \
         5
  x=6: 6 > top(3) → pop 3, push 6 → heap=[5,6]
       5
        \
         6
  x=4: 4 < top(5) → skip

  heap.top() = 5 → 2nd largest = 5 ✓
```

### Code:
```cpp
#include <bits/stdc++.h>
using namespace std;

int findKthLargest(vector<int>& nums, int k) {
    // Min Heap of size K
    priority_queue<int, vector<int>, greater<int>> minPQ;

    for (int x : nums) {
        minPQ.push(x);

        // Keep only K largest elements
        if (minPQ.size() > k) {
            minPQ.pop(); // Remove smallest → maintains K largest
        }
    }

    // Top of min heap = Kth largest
    return minPQ.top();
}

int main() {
    vector<int> arr1 = {3, 2, 1, 5, 6, 4};
    cout << findKthLargest(arr1, 2) << endl; // 5

    vector<int> arr2 = {3, 2, 3, 1, 2, 4, 5, 5, 6};
    cout << findKthLargest(arr2, 4) << endl; // 4

    vector<int> arr3 = {1};
    cout << findKthLargest(arr3, 1) << endl; // 1

    vector<int> arr4 = {-1, -1};
    cout << findKthLargest(arr4, 2) << endl; // -1

    return 0;
}
```

## Method 3: Max Heap

### Algorithm:
```
  1. Build Max Heap from all N elements — O(N)
  2. Call pop() exactly K times
  3. The Kth pop gives Kth largest element
```
```
  arr = [3, 2, 1, 5, 6, 4], K=2

  Build Max Heap:
        6
       / \
      5   4
     / \
    2   3
   /
  1

  Pop 1: pop 6 → 1st largest
  Pop 2: pop 5 → 2nd largest = 5 ✓
```

### Code:
```cpp
int findKthLargest_maxHeap(vector<int>& nums, int k) {
    // Build Max Heap — O(N)
    priority_queue<int> maxPQ(nums.begin(), nums.end());

    // Pop K-1 times
    for (int i = 0; i < k - 1; i++) {
        maxPQ.pop();
    }

    return maxPQ.top(); // Kth largest
}
```

**Time:** O(N + K log N) | **Space:** O(N)

## Method 4: QuickSelect

### Algorithm:
```
  Kth largest = (N-K)th index in ascending sorted array.
  Use partition like QuickSort, but only recurse on one side.
```

### Code:
```cpp
int partition(vector<int>& nums, int left, int right) {
    int pivot = nums[right];
    int i = left - 1;

    for (int j = left; j < right; j++) {
        if (nums[j] <= pivot) {
            swap(nums[++i], nums[j]);
        }
    }
    swap(nums[i + 1], nums[right]);
    return i + 1;
}

int quickSelect(vector<int>& nums, int left, int right, int k) {
    if (left == right) return nums[left];

    int pivotIdx = partition(nums, left, right);

    if (pivotIdx == k)      return nums[pivotIdx];
    else if (pivotIdx < k)  return quickSelect(nums, pivotIdx + 1, right, k);
    else                    return quickSelect(nums, left, pivotIdx - 1, k);
}

int findKthLargest_quickSelect(vector<int>& nums, int k) {
    int n = nums.size();
    // Kth largest = element at index (n-k) in sorted ascending order
    return quickSelect(nums, 0, n - 1, n - k);
}
```

**Time:** O(N) average, O(N²) worst | **Space:** O(log N)

---

### Kth Smallest vs Kth Largest — Quick Comparison:
```
  Kth SMALLEST                  Kth LARGEST
  ─────────────────────         ─────────────────────
  Use MAX Heap of size K        Use MIN Heap of size K
  Push if x < top               Push if x > top
  Pop if size > K               Pop if size > K
  top() = Kth smallest          top() = Kth largest

  arr=[3,2,1,5,6,4], K=2:

  Max Heap for Kth smallest:    Min Heap for Kth largest:
  Maintain {3,4} → top=4? No   Maintain {5,6} → top=5
  Maintain {3,4} → No           top() = 5 = 2nd largest ✓
```

---

### Edge Cases:
```
  K = 1:          Return maximum element
  K = N:          Return minimum element
  Duplicates:     [5,5,5,5], K=2 → 5
  Negatives:      [-1,-2,-3,-4], K=2 → -2
  Single elem:    [42], K=1 → 42
  K = N/2:        Median-like element
```

---

### Complexity Summary:
```
  Method          Time           Space    Best For
  ──────────────────────────────────────────────────────────
  Sort            O(N log N)     O(1)     Simple cases
  Min Heap (K)    O(N log K)     O(K)     Streams, K << N
  Max Heap (N)    O(N + K logN)  O(N)     Small K
  QuickSelect     O(N) avg       O(log N) Best average case

  Recommended: Min Heap of size K
  → O(N log K) time
  → O(K) space
  → Works on data streams
  → When K << N, much better than O(N log N)
```
```
  Why Min Heap for Kth LARGEST?

  Intuition: "Keep the K largest, discard the rest"
  The MINIMUM of the K largest = Kth largest

       Min Heap of size K:
       top = min of K largest = Kth largest ✓

  Why Max Heap for Kth SMALLEST?

  Intuition: "Keep the K smallest, discard the rest"
  The MAXIMUM of the K smallest = Kth smallest

       Max Heap of size K:
       top = max of K smallest = Kth smallest ✓
```
