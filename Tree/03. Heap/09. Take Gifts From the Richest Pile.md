## Take Gifts From the Richest Pile

**Problem Statement:**   
You are given an integer array `gifts` representing piles of gifts and an integer `k`. Every second:
1. Choose the pile with the **most gifts**.
2. Leave behind `floor(√gifts[i])` gifts.
3. Repeat for `k` seconds.

Return the **total number of gifts** remaining after `k` seconds.

---

>### Examples:

**Example 1:**
- **Input:** `gifts = [25, 64, 9, 4, 100]`, `k = 4`
- **Output:** `29`
```
  Initial: [25, 64, 9, 4, 100]

  Second 1: max=100, floor(√100)=10  → [25, 64, 9, 4, 10]
  Second 2: max=64,  floor(√64)=8   → [25, 8,  9, 4, 10]
  Second 3: max=25,  floor(√25)=5   → [5,  8,  9, 4, 10]
  Second 4: max=10,  floor(√10)=3   → [5,  8,  9, 4, 3]

  Sum = 5 + 8 + 9 + 4 + 3 = 29 ✓
```

**Example 2:**
- **Input:** `gifts = [1, 1, 1, 1]`, `k = 4`
- **Output:** `4`
```
  Initial: [1, 1, 1, 1]

  Second 1: max=1, floor(√1)=1 → [1, 1, 1, 1] (no change!)
  Second 2: max=1, floor(√1)=1 → [1, 1, 1, 1]
  Second 3: max=1, floor(√1)=1 → [1, 1, 1, 1]
  Second 4: max=1, floor(√1)=1 → [1, 1, 1, 1]

  Sum = 4 ✓ (once all piles = 1, taking doesn't reduce them)
```

**Example 3:**
- **Input:** `gifts = [9]`, `k = 3`
- **Output:** `1`
```
  Second 1: max=9, floor(√9)=3   → [3]
  Second 2: max=3, floor(√3)=1   → [1]
  Second 3: max=1, floor(√1)=1   → [1] (no further reduction)

  Sum = 1 ✓
```

---

### Key Insight:
```
  At each step → always pick RICHEST pile (maximum)
  → Replace it with floor(√max)
  → MAX HEAP is perfect!

  floor(√n) grows slowly:
  √100 = 10   (took 90 gifts)
  √64  = 8    (took 56 gifts)
  √10  ≈ 3    (took 7 gifts)
  √3   ≈ 1    (took 2 gifts)
  √1   = 1    (no more reduction — stable!)
```

---
## Approach
### Algorithm:
```
  1. Build Max Heap from the gifts array
  2. Repeat k times:
       a. top = heap.top(); heap.pop()
       b. heap.push(floor(√top))
  3. Sum all remaining elements in the heap
  4. Return total sum
```
```
  Max Heap trace for [25, 64, 9, 4, 100], k=4:

  Initial: heap = [100, 64, 25, 9, 4]
        100
       /   \
      64    25
     / \
    9   4

  k=1: pop 100, push floor(√100)=10
  heap = [64, 25, 10, 9, 4]

  k=2: pop 64, push floor(√64)=8
  heap = [25, 10, 8, 9, 4]

  k=3: pop 25, push floor(√25)=5
  heap = [10, 9, 8, 5, 4]

  k=4: pop 10, push floor(√10)=3
  heap = [9, 5, 8, 3, 4]

  Sum = 9+8+5+4+3 = 29 ✓
```

---

### C++ Code:
```cpp
#include <bits/stdc++.h>
using namespace std;

long long pickGifts(vector<int>& gifts, int k) {
    // Build Max Heap — O(N)
    priority_queue<int> maxPQ(gifts.begin(), gifts.end());

    // Perform k operations
    for (int i = 0; i < k; i++) {
        int richest = maxPQ.top();
        maxPQ.pop();

        // Leave floor(sqrt(richest)) gifts
        maxPQ.push((int)sqrt(richest));
    }

    // Sum all remaining gifts
    long long total = 0;
    while (!maxPQ.empty()) {
        total += maxPQ.top();
        maxPQ.pop();
    }

    return total;
}

int main() {
    vector<int> gifts1 = {25, 64, 9, 4, 100};
    cout << pickGifts(gifts1, 4) << endl; // 29

    vector<int> gifts2 = {1, 1, 1, 1};
    cout << pickGifts(gifts2, 4) << endl; // 4

    vector<int> gifts3 = {9};
    cout << pickGifts(gifts3, 3) << endl; // 1

    vector<int> gifts4 = {1000000000};
    cout << pickGifts(gifts4, 1) << endl; // floor(√10^9) = 31622

    return 0;
}
```

### Decay Rate Visualization:
```
  How fast does a pile shrink?

  Start: 1,000,000,000
  k=1:   31,622       (√10^9)
  k=2:   177          (√31622)
  k=3:   13           (√177)
  k=4:   3            (√13)
  k=5:   1            (√3)
  k=6:   1            (stable — √1 = 1, no change)

  Large piles shrink VERY fast!
  Any pile reaches 1 in about log(log(N)) steps

  Once all piles = 1:
  → sqrt(1) = 1 → no reduction → sum stays constant
```

---

### Edge Cases:
```
  k = 0:               k > needed:
  No operations        Piles reach 1, then stable
  Return sum(gifts)    Return N (if all piles become 1)

  All piles = 1:       Single pile:
  [1,1,1,1], k=100    [16], k=2
  → No change          k=1: pop 16, push 4 → [4]
  → Return 4 ✓         k=2: pop 4,  push 2 → [2]
                        Sum = 2 ✓

  Large values:
  [10^9], k=1
  floor(√10^9) = floor(31622.77) = 31622
  Return 31622 ✓
```

---

### Optimization: Early Termination
```cpp
// If max pile is already 1, no further reduction possible
for (int i = 0; i < k; i++) {
    if (maxPQ.top() <= 1) break; // All piles are 1, stable state

    int richest = maxPQ.top();
    maxPQ.pop();
    maxPQ.push((int)sqrt(richest));
}
```
```
  This helps when k is very large but piles reduce quickly:
  [100], k = 1000000

  Without early termination: 1M heap operations
  With early termination:    ~5 operations (100→10→3→1→stop) ✓
```

---

### Complexity Analysis:

**Time Complexity:** O(N + K log N)
- Building Max Heap: O(N)
- K iterations, each with 1 pop + 1 push: O(K log N)
- Summing N elements: O(N)
- Total: **O(N + K log N)**

**Space Complexity:** O(N) for the heap.
```
  N = number of piles
  K = number of seconds

  With early termination (pile reaches 1):
  Effective K ≈ log(log(max_value)) per pile
  In practice, much faster than K iterations ✓
```

### Summary:
```
  Problem Pattern:
  ┌──────────────────────────────────────────────────┐
  │  Repeat K times:                                 │
  │    Pick maximum → transform it → put back        │
  │  → MAX HEAP is the natural fit                   │
  └──────────────────────────────────────────────────┘

  Similar problems use the same pattern:
  - Magician & Chocolates  (eat all, put back half)
  - Last Stone Weight      (smash two heaviest)
  - Take Gifts             (replace max with sqrt)

  Key: top() + pop() + push() = O(log N) per step
```
