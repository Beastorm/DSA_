## Height of Heap

**Problem Statement:**   
Given the number of nodes `N` in a **Complete Binary Tree** (heap), find the height of the heap.

---

### Key Property:

A heap is always a **Complete Binary Tree**. In a complete binary tree:
```
  Level 0 (root):  1 node      = 2⁰
  Level 1:         2 nodes     = 2¹
  Level 2:         4 nodes     = 2²
  Level h:         2^h nodes   = 2^h

  Total nodes in perfect tree of height h:
  N = 2⁰ + 2¹ + 2² + ... + 2^h = 2^(h+1) - 1

  Therefore:
  Height = ⌊log₂(N)⌋
```

---

>### Examples:

**Example 1:** N = 6
```
         *          level 0
        / \
       *   *         level 1
      / \ /
     *  * *          level 2

  height = floor(log2(6)) = floor(2.58) = 2 ✓
```

**Example 2:** N = 7
```
         *          level 0
        / \
       *   *         level 1
      / \ / \
     *  * *  *       level 2

  height = floor(log2(7)) = floor(2.80) = 2 ✓
```

**Example 3:** N = 8
```
         *           level 0
        / \
       *   *          level 1
      / \ / \
     *  * *  *        level 2
    /
   *                  level 3

  height = floor(log2(8)) = floor(3.0) = 3 ✓
```

### Pattern Table:
```
  N     Binary of N   floor(log₂N)   Height
  ──────────────────────────────────────────
  1     0001           0              0
  2     0010           1              1
  3     0011           1              1
  4     0100           2              2
  5     0101           2              2
  6     0110           2              2
  7     0111           2              2
  8     1000           3              3
  15    1111           3              3
  16    10000          4              4

  Observation: Height increases by 1 every time N
               crosses a power of 2.
```

---

## Alternative Formula
```
  Height = number of bits in binary representation of N, minus 1
         = floor(log₂(N))

  N=5 → binary = 101 → 3 bits → height = 3-1 = 2 ✓
  N=8 → binary = 1000 → 4 bits → height = 4-1 = 3 ✓
```

---

### C++ Code:
```cpp
#include <bits/stdc++.h>
using namespace std;

// Method 1: Using log2
int heightLog(int n) {
    return (int)log2(n);
}

// Method 2: Counting levels manually
int heightLoop(int n) {
    int height = 0;
    while (n > 1) {
        n /= 2;  // Go up one level
        height++;
    }
    return height;
}

// Method 3: Bit manipulation
// Height = position of most significant bit - 1
int heightBit(int n) {
    return 31 - __builtin_clz(n);
    // __builtin_clz = count leading zeros
    // 31 - leading zeros = position of MSB
}

int main() {
    int testCases[] = {1, 2, 3, 4, 6, 7, 8, 15, 16};

    cout << "N\tlog2\tloop\tbit" << endl;
    cout << "─────────────────────" << endl;
    for (int n : testCases) {
        cout << n << "\t"
             << heightLog(n)  << "\t"
             << heightLoop(n) << "\t"
             << heightBit(n)  << endl;
    }

    return 0;
}
```

### Visual Height vs N:
```
  Height
    4 |                          ****...
    3 |              ********
    2 |      ****
    1 |  **
    0 | *
      └─────────────────────────────── N
       1 2 3 4 5 6 7 8 9 ... 15 16

  Height jumps at powers of 2:
  N=1 → h=0
  N=2 → h=1  (jumped at N=2  = 2¹)
  N=4 → h=2  (jumped at N=4  = 2²)
  N=8 → h=3  (jumped at N=8  = 2³)
  N=16→ h=4  (jumped at N=16 = 2⁴)
```

---

### Complexity Analysis:
```
  Method          Time    Space   Notes
  ──────────────────────────────────────────────
  log2(n)         O(1)    O(1)    Floating point
  Loop (n/=2)     O(logN) O(1)    Integer only
  Bit (__clz)     O(1)    O(1)    Fastest, hardware
```

> For competitive programming, `(int)log2(n)` or `31 - __builtin_clz(n)` is preferred for O(1) solution.
