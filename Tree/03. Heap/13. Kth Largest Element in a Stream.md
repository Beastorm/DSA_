## Kth Largest Element in a Stream

**Problem Statement:**   
Design a class to find the **Kth largest element in a stream** of numbers. The class should:
- Be initialized with `k` and an initial array `nums`.
- Support `add(val)` which adds `val` to the stream and returns the **current Kth largest** element.

---

>### Examples:

**Example 1:**
- **Input:** `k = 3`, `nums = [4, 5, 8, 2]`
- Operations: `add(3)`, `add(5)`, `add(10)`, `add(9)`, `add(4)`
- **Output:** `[4, 5, 5, 8, 8]`
```
  Initial nums = [4, 5, 8, 2], K = 3
  Sorted desc: [8, 5, 4, 2] → 3rd largest = 4

  add(3):  stream=[8,5,4,3,2]  → 3rd largest = 4
  add(5):  stream=[8,5,5,4,3,2]→ 3rd largest = 5
  add(10): stream=[10,8,5,5,4] → 3rd largest = 5
  add(9):  stream=[10,9,8,5,5] → 3rd largest = 8
  add(4):  stream=[10,9,8,5,5] → 3rd largest = 8

  Output: [4, 5, 5, 8, 8] ✓
```

---

### Key Insight:
```
  The Kth largest in a stream changes as new elements arrive.
  We need to efficiently:
    1. Add a new element
    2. Query the Kth largest

  Solution: Maintain a MIN HEAP of size K.
  → top() of heap = Kth largest element ✓

  Why?
  Min Heap of size K stores the K largest elements seen so far.
  The smallest of these K largest = Kth largest.
  → top() gives answer in O(1)!

  Stream: [10, 8, 5, 3, 1], K=3
  K largest = {10, 8, 5} → min of these = 5 = 3rd largest ✓
```

---
## Approach
### Algorithm:
```
  Constructor(k, nums):
    1. Store k
    2. Build Min Heap from nums
    3. While heap.size() > k: pop() ← keep only K largest

  add(val):
    1. Push val into heap
    2. If heap.size() > k: pop() ← remove smallest (not in K largest)
    3. Return heap.top() ← Kth largest
```
```
  K=3, nums=[4,5,8,2]

  Build heap: [4,5,8,2]
  While size > 3: pop min(2) → heap=[4,5,8]
       4
      / \
     5   8

  add(3): push 3 → heap=[3,4,5,8], size=4 > 3
          pop 3 → heap=[4,5,8]
          return top=4 ✓

  add(5): push 5 → heap=[4,5,5,8], size=4 > 3
          pop 4 → heap=[5,5,8]
          return top=5 ✓

  add(10): push 10 → heap=[5,5,8,10], size=4 > 3
           pop 5 → heap=[5,8,10]
           return top=5 ✓

  add(9): push 9 → heap=[5,8,9,10], size=4 > 3
          pop 5 → heap=[8,9,10]
          return top=8 ✓

  add(4): push 4 → heap=[4,8,9,10], size=4 > 3
          pop 4 → heap=[8,9,10]
          return top=8 ✓

  Output: [4, 5, 5, 8, 8] ✓
```

### Code:
```cpp
#include <bits/stdc++.h>
using namespace std;

class KthLargest {
private:
    int k;
    priority_queue<int, vector<int>, greater<int>> minPQ; // Min Heap

public:
    KthLargest(int k, vector<int>& nums) {
        this->k = k;

        // Add all initial elements
        for (int x : nums) {
            minPQ.push(x);
        }

        // Keep only K largest elements
        while (minPQ.size() > k) {
            minPQ.pop();
        }
    }

    int add(int val) {
        // Add new element to stream
        minPQ.push(val);

        // If we have more than K elements, remove smallest
        if (minPQ.size() > k) {
            minPQ.pop();
        }

        // Top of min heap = Kth largest
        return minPQ.top();
    }
};

int main() {
    vector<int> nums = {4, 5, 8, 2};
    KthLargest* obj = new KthLargest(3, nums);

    cout << obj->add(3)  << endl; // 4
    cout << obj->add(5)  << endl; // 5
    cout << obj->add(10) << endl; // 5
    cout << obj->add(9)  << endl; // 8
    cout << obj->add(4)  << endl; // 8

    return 0;
}
```

### Why Min Heap and NOT Max Heap?
```
  MAX HEAP approach (wrong for this problem):
  → Stores all elements
  → Need to traverse K elements to find Kth largest
  → O(K) per query — slow for large K

  MIN HEAP of size K (correct):
  → Stores exactly K largest elements
  → top() = smallest of K largest = Kth largest
  → O(1) per query, O(log K) per add

  Example: K=3, stream=[10,9,8,7,6,5]
  Min Heap stores: {8,9,10}
  top() = 8 = 3rd largest ✓ (O(1) query!)
```

---

### Handling Edge Cases:
```
  Initial nums is empty:         K > initial array size:
  nums=[], K=1                   nums=[3], K=3
  heap=[]                        heap=[3] (size<K, that's ok)

  add(1): push 1 → heap=[1]      add(5): push 5 → heap=[3,5]
  return 1 ✓                     add(2): push 2 → heap=[2,3,5]
                                  return top=2 = 3rd largest ✓

  New element smaller than all:  New element largest seen:
  K=2, heap={5,6}                K=2, heap={5,6}
  add(1): push 1 → {1,5,6}      add(10): push 10 → {5,6,10}
  pop 1 → {5,6}                  pop 5 → {6,10}
  return 5 ✓ (no change)         return 6 ✓
```

### Comparison: Static Array vs Stream:
```
  Static Array (find once):      Stream (find continuously):
  ─────────────────────────      ──────────────────────────
  Sort → O(N log N)              Can't sort (stream is infinite)
  Max Heap → O(N + K logN)       Must update with each new element
  QuickSelect → O(N) avg         Must answer each add() in O(log K)

  Solution for stream:           Min Heap of size K:
  → add() in O(log K)              push: O(log K)
  → query in O(1)                  pop:  O(log K)
                                   top:  O(1)     ← Kth largest
```

---

### Complexity Analysis:

**Constructor:**
- Building heap from N elements: O(N log N)
- Trimming to size K: O(N log K)
- Total: **O(N log N)**

**add(val):**
- Push: O(log K)
- Pop (if size > K): O(log K)
- Total: **O(log K)**

**Space Complexity:** O(K) — heap stores at most K elements at any time.
```
  Why O(log K) not O(log N)?
  Heap size is always ≤ K, never grows beyond K.
  All heap operations are on a heap of size K.
  → O(log K) per add() ✓

  This is the KEY advantage over a full heap:
  K << N → log K << log N → much faster per query!
```
