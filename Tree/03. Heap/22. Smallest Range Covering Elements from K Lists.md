## Smallest Range Covering Elements from K Lists

**Problem Statement:**   
You have `K` sorted lists of integers. Find the **smallest range** `[a, b]` such that there is at least one number from each of the K lists in the range `[a, b]`.

If there are multiple smallest ranges, return the one with the **smallest left bound**.

---

>### Examples:

**Example 1:**
- **Input:**
```
  nums = [[4,10,15,24,26],
           [0,9,12,20],
           [5,18,22,30]]
```
- **Output:** `[20, 24]`
```
  [20,24] contains:
  List 0: 24 ✓
  List 1: 20 ✓
  List 2: 22 ✓
  Range = 24-20 = 4

  Other candidates: [4,9]: 4,9,5 → wait, no 9 in list2
  [4,5]: no element from list1
  [20,24] is optimal ✓
```

**Example 2:**
- **Input:**
```
  nums = [[1,2,3],
           [1,2,3],
           [1,2,3]]
```
- **Output:** `[1, 1]`
```
  [1,1]: contains 1 from each list ✓
  Range = 0 (smallest possible)
```

**Example 3:**
- **Input:**
```
  nums = [[10,10],
           [11,11]]
```
- **Output:** `[10, 11]`

---

### Key Insight:
```
  We need a window [min, max] that covers at least
  one element from EACH of the K lists.

  Strategy:
  - Use Min Heap to track the MINIMUM of current elements.
  - Track the MAXIMUM of current elements.
  - Range = [heap.top(), currentMax]
  - Pop minimum (it's the bottleneck), push next from same list.
  - This shrinks the minimum end while max is tracked.

  Why this works:
  - We always have exactly K elements (one per list).
  - Min = heap.top(), Max = tracked variable.
  - Pop min → new element from same list → updates min and possibly max.
  - Range might shrink or shift.
  - Track the smallest range seen.
```

---
## Approach
### Algorithm:
```
  1. Initialize:
     - Push nums[i][0] for all i into Min Heap as {val, listIdx, elemIdx}
     - currentMax = max of all first elements
     - bestRange = {-INF, INF} (infinite range initially)

  2. While all lists still have elements:
     a. min = heap.top() (popped)
     b. range = [min, currentMax]
     c. If range smaller than bestRange → update bestRange
     d. If min's list has next element:
           Push next element, update currentMax if needed
        Else:
           STOP (one list exhausted → can't cover all K lists)

  3. Return bestRange
```

  heap:  0(L1), 4(L0), 5(L2)

### Code:
```cpp
#include <bits/stdc++.h>
using namespace std;

vector<int> smallestRange(vector<vector<int>>& nums) {
    int K = nums.size();

    // Min Heap: {value, list_index, element_index}
    priority_queue<tuple<int,int,int>,
                   vector<tuple<int,int,int>>,
                   greater<>> minPQ;

    int currentMax = INT_MIN;

    // Initialize: push first element of each list
    for (int i = 0; i < K; i++) {
        minPQ.push({nums[i][0], i, 0});
        currentMax = max(currentMax, nums[i][0]);
    }

    // Best range: [rangeStart, rangeEnd]
    int rangeStart = 0, rangeEnd = INT_MAX;

    while (!minPQ.empty()) {
        auto [minVal, listIdx, elemIdx] = minPQ.top();
        minPQ.pop();

        // Current range = [minVal, currentMax]
        if (currentMax - minVal < rangeEnd - rangeStart) {
            rangeStart = minVal;
            rangeEnd   = currentMax;
        }

        // If list is exhausted, we can't cover all K lists
        if (elemIdx + 1 == nums[listIdx].size()) break;

        // Push next element from same list
        int nextVal = nums[listIdx][elemIdx + 1];
        minPQ.push({nextVal, listIdx, elemIdx + 1});
        currentMax = max(currentMax, nextVal);
    }

    return {rangeStart, rangeEnd};
}

int main() {
    vector<vector<int>> nums1 = {
        {4, 10, 15, 24, 26},
        {0, 9, 12, 20},
        {5, 18, 22, 30}
    };
    vector<int> res1 = smallestRange(nums1);
    cout << "Example 1: [" << res1[0] << ", " << res1[1] << "]" << endl;
    // Output: [20, 24]

    vector<vector<int>> nums2 = {
        {1, 2, 3},
        {1, 2, 3},
        {1, 2, 3}
    };
    vector<int> res2 = smallestRange(nums2);
    cout << "Example 2: [" << res2[0] << ", " << res2[1] << "]" << endl;
    // Output: [1, 1]

    vector<vector<int>> nums3 = {
        {10, 10},
        {11, 11}
    };
    vector<int> res3 = smallestRange(nums3);
    cout << "Example 3: [" << res3[0] << ", " << res3[1] << "]" << endl;
    // Output: [10, 11]

    return 0;
}
```

## Why Stop When a List is Exhausted?
```
  When a list's last element is popped:
  → No more elements from that list to push
  → Future windows can't include that list
  → Can't form a valid range covering ALL K lists
  → Stop immediately ✓

  The current window [minVal, currentMax] at that moment
  is the last valid window checked.
```

## Edge Cases
```
  All lists same elements:     K=1:
  [[1,2], [1,2], [1,2]]        [[5,10,15]]
  Init max=1, push {1,0,0}×3   Push {5,0,0}
  pop 1(L0), range=[1,1] ✓     pop {5}, range=[5,5]
                                No next? No, push 10
                                Continues until exhausted
                                bestRange=[5,5] ✓ (1 element from 1 list)

  Single element per list:     Already in order:
  [[1],[3],[2]]                 [[1,4],[2,5],[3,6]]
  push {1,3,2}, max=3          Init: {1,0,0},{2,1,0},{3,2,0}
  pop 1, range=[1,3] → done    max=3, range=[1,3]
  L0 exhausted → return [1,3]  → keeps improving as elements advance
```

### Complexity Analysis:

**Time Complexity:** O(NK log K)
- Total elements across all lists: N × K
- Each element: 1 push + 1 pop = O(log K) (heap size always = K)
- Total: **O(NK log K)**

**Space Complexity:** O(K)
- Min Heap always holds exactly K elements (one per list).
- Total: **O(K)**
```
  N = average elements per list
  K = number of lists
  Total elements = NK

  Heap operations: NK × O(log K) = O(NK log K)
  Space: O(K) for heap ← independent of N! ✓

  Compare with brute force (check all pairs):
  O(NK)² = O(N²K²) → much worse for large inputs
```
