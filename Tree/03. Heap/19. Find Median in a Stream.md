## Find Median in a Stream

**Problem Statement:**   
Design a data structure that supports:
- `addNum(num)` — Add a number from the data stream to the data structure.
- `findMedian()` — Return the median of all elements so far.

The **median** is the middle value in a sorted list:
- If even number of elements → average of two middle values.
- If odd number of elements → middle value.

---

>### Examples:

**Example 1:**
- `addNum(1)` → median = `1.0`
- `addNum(2)` → median = `1.5`
- `addNum(3)` → median = `2.0`
```
  After 1:    [1]         → median = 1.0
  After 1,2:  [1,2]       → median = (1+2)/2 = 1.5
  After 1,2,3:[1,2,3]     → median = 2.0
```

**Example 2:**
- `addNum(6)` → `6.0`
- `addNum(2)` → `4.0`
- `addNum(1)` → `2.0`
- `addNum(5)` → `3.5`
- `addNum(4)` → `4.0`
```
  [6]         → 6.0
  [2,6]       → (2+6)/2 = 4.0
  [1,2,6]     → 2.0
  [1,2,5,6]   → (2+5)/2 = 3.5
  [1,2,4,5,6] → 4.0
```

---

## Key Insight
```
  Naive: Store all elements, sort each time → O(N log N) per query.

  Better: Use TWO HEAPS to split elements around the median:

  ┌─────────────────────────────────────────────────────┐
  │  Left half  → MAX HEAP (top = largest of left half) │
  │  Right half → MIN HEAP (top = smallest of right)    │
  │                                                      │
  │  Max Heap top ← MEDIAN → Min Heap top               │
  └─────────────────────────────────────────────────────┘

  Stream: [1, 2, 3, 4, 5]

  MAX HEAP    MIN HEAP
  [1, 2]      [3, 4, 5]
     ↑              ↑
  top=2         top=3

  Odd count:  median = minHeap.top() = 3
  Even count: median = (maxHeap.top() + minHeap.top()) / 2
```

---

### Invariants to Maintain:
```
  1. All elements in maxHeap ≤ all elements in minHeap
     (left half always ≤ right half)

  2. Size balance:
     |maxHeap.size() - minHeap.size()| ≤ 1

     If total is ODD:  minHeap has one more element
                       → median = minHeap.top()
     If total is EVEN: both heaps equal size
                       → median = (maxHeap.top() + minHeap.top()) / 2
```

---

### Algorithm for addNum(num):
```
  Step 1: Decide which heap to add to:
    If num ≤ maxHeap.top() → push to maxHeap (left half)
    Else                   → push to minHeap (right half)

  Step 2: Rebalance if sizes differ by more than 1:
    If maxHeap.size() > minHeap.size() + 1:
      → move maxHeap.top() to minHeap
    If minHeap.size() > maxHeap.size():
      → move minHeap.top() to maxHeap

  Step 3: findMedian():
    If equal sizes → (maxHeap.top() + minHeap.top()) / 2.0
    Else           → minHeap.top() (has one more)
```

---

### Code:
```cpp
#include <bits/stdc++.h>
using namespace std;

class MedianFinder {
private:
    priority_queue<int> maxPQ;                              // Left half (Max Heap)
    priority_queue<int, vector<int>, greater<int>> minPQ;  // Right half (Min Heap)

public:
    MedianFinder() {}

    void addNum(int num) {
        // Step 1: Add to appropriate heap
        if (maxPQ.empty() || num <= maxPQ.top()) {
            maxPQ.push(num); // Belongs to left half
        } else {
            minPQ.push(num); // Belongs to right half
        }

        // Step 2: Rebalance heaps
        // maxHeap can have at most 1 more than minHeap
        if (maxPQ.size() > minPQ.size() + 1) {
            // Move top of maxHeap to minHeap
            minPQ.push(maxPQ.top());
            maxPQ.pop();
        } else if (minPQ.size() > maxPQ.size()) {
            // Move top of minHeap to maxHeap
            maxPQ.push(minPQ.top());
            minPQ.pop();
        }
    }

    double findMedian() {
        if (maxPQ.size() == minPQ.size()) {
            // Even count: average of two middles
            return (maxPQ.top() + minPQ.top()) / 2.0;
        } else {
            // Odd count: maxHeap has one more
            return maxPQ.top();
        }
    }
};

int main() {
    MedianFinder mf;

    vector<int> stream = {1, 2, 3, 4, 5};

    for (int num : stream) {
        mf.addNum(num);
        cout << "Add " << num << " → Median = "
             << mf.findMedian() << endl;
    }

    cout << endl;

    MedianFinder mf2;
    vector<int> stream2 = {6, 2, 1, 5, 4};
    for (int num : stream2) {
        mf2.addNum(num);
        cout << "Add " << num << " → Median = "
             << mf2.findMedian() << endl;
    }

    return 0;
}
```

### Heap State After Each Addition:
```
  Num    maxPQ (left)    minPQ (right)    Median
  ──────────────────────────────────────────────
  1      [1]             []               1.0
  2      [1]             [2]              1.5
  3      [2,1]           [3]              2.0
  4      [2,1]           [3,4]            2.5
  5      [3,2,1]         [4,5]            3.0
  6      [3,2,1]         [4,5,6]          3.5
  7      [4,3,2,1]       [5,6,7]          4.0

  maxPQ always has ≤ 1 more element than minPQ ✓
  maxPQ.top() ≤ minPQ.top() always ✓
```

---

### Edge Cases:
```
  Single element:            Negative numbers:
  addNum(5)                  addNum(-5), addNum(-3)
  maxPQ=[5], minPQ=[]        sorted: [-5,-3]
  median = 5.0 ✓             median = (-5+-3)/2 = -4.0 ✓

  Duplicates:                Large stream:
  addNum(3),addNum(3)        Same O(log N) per operation
  sorted: [3,3]              regardless of stream size ✓
  median = 3.0 ✓
```

---

### Why Two Heaps?
```
  To find median we need the MIDDLE element(s).

  Option 1: Sorted array → O(1) median, O(N) insert (shift)
  Option 2: BST          → O(log N) insert, O(log N) median
  Option 3: Two Heaps    → O(log N) insert, O(1) median ← BEST

  Two Heaps logic:
  MAX HEAP stores lower half  → top = largest of lower half
  MIN HEAP stores upper half  → top = smallest of upper half

  These two tops are the "middle" elements!
  → Median computed in O(1) from just two values ✓
```

---

### Complexity Analysis:
```
  Operation      Time        Space
  ──────────────────────────────────────────────────
  addNum()       O(log N)    O(1) per call
  findMedian()   O(1)        O(1)
  Total space:   O(N)        (storing all N elements)

  N = number of elements added so far

  addNum breakdown:
  - Push to heap:    O(log N)
  - Pop + push rebalance: O(log N)
  Total per addNum: O(log N) ✓

  findMedian:
  - Just access top() of both heaps: O(1) ✓

  Compare with naive:
  Naive sort: O(N log N) per findMedian
  Two Heaps:  O(1) per findMedian ← much better!
```
