## 1. What is a Priority Queue?

A Priority Queue processes elements by **priority**, not arrival order. The highest-priority element is always dequeued first.

```
Normal Queue (FIFO):          Priority Queue:
[cold, flu, heart attack]     [heart attack, flu, cold]
 ↑ processed first             ↑ processed first
 (arrival order)               (priority order)
```

> Think of an emergency room — patients are treated by severity, not the time they walked in.

If two elements share the same priority, insertion order is used as a tiebreaker.

## 2. Binary Heap

A Binary Heap is a Binary Tree satisfying **two conditions**:

1. **Complete Binary Tree** — all levels fully filled except the last, which fills left to right
2. **Heap Property** — every parent satisfies the ordering rule with its children

### Complete Binary Tree

```
  Complete ✓                Not Complete ✗
       1                          1
      / \                        / \
     2   3                      2   3
    / \ /                      /     \
   4  5 6                     4       6
                               (node 5 missing)
```

### Heap Property

```
  Min Heap                       Max Heap
  Parent ≤ Children              Parent ≥ Children

         1  ← minimum                  9  ← maximum
        / \                           / \
       3   2                         7   8
      / \ / \                       / \ / \
     6  5 4  8                     1  2 3  4

  Root = smallest element        Root = largest element
```

## 3. Array Representation

A Binary Heap is stored as a flat array — no pointers needed.

```
  Tree:                   Array:
         1                [1, 3, 2, 6, 5, 4, 8]
        / \                0  1  2  3  4  5  6
       3   2
      / \ / \
     6  5 4  8
```

### Index Relationships

| Relation | Formula |
|----------|---------|
| Left child of `i` | `2*i + 1` |
| Right child of `i` | `2*i + 2` |
| Parent of `i` | `(i-1) / 2` |

**Example — node at index 1 (val = 3):**

```
  Left  child → 2*1+1 = 3  (val = 6)
  Right child → 2*1+2 = 4  (val = 5)
  Parent      → (1-1)/2 = 0 (val = 1)
```

## 4. Step Up vs Step Down

These are the two fundamental fix-up operations. Every heap operation is built on one of them.

### ⬆️ Step Up (Bubble Up)

**Triggered by:** Insertion — new node placed at the bottom may violate heap order going upward.

**Direction:** Child → Parent

**Rule:** Compare with parent; if out of order, swap. Repeat until satisfied.

```
Insert 15 into a Max-Heap:

        10
       /  \
      8    6
     /
    15  ← just inserted

15 > 8 (parent) → swap

        10
       /  \
     15    6
     /
    8

15 > 10 (parent) → swap

        15
       /  \
     10    6
     /
    8

✓ Done — 15 reaches the root
```

---

### ⬇️ Step Down (Bubble Down / Heapify)

**Triggered by:** Deletion — last node placed at root may violate heap order going downward.

**Direction:** Parent → Children

**Rule:** Compare with both children; swap with the **larger child** (max-heap) or **smaller child** (min-heap). Repeat until satisfied.

```
Delete root from a Max-Heap:

        15
       /  \
     10    6
     /
    8

Step 1 — remove 15, move last node (8) to root:

        8
       /  \
     10    6

Step 2 — 8 < 10 (larger child) → swap

        10
       /  \
      8    6

✓ Done
```

---

### Step Up vs Step Down — Quick Reference

| | Step Up | Step Down |
|-|---------|-----------|
| **Triggered by** | Insertion | Deletion (of root) |
| **Moves** | New node upward | Replacement node downward |
| **Compares with** | Parent | Both children |
| **Swaps with** | Parent (if wrong order) | Larger child (max-heap) / Smaller child (min-heap) |
| **Time Complexity** | O(log n) | O(log n) |

> **Memory trick:** Insert → go **Up** · Delete → go **Down**

## 5. Operations on Min Heap

### Insert() — O(log N)

Add key at the end, then **bubble up**.

```
Insert 2 into [1, 3, 4, 6, 5]:

Step 1 — add at end:
[1, 3, 4, 6, 5, 2]
              ↑ index 5, parent = (5-1)/2 = 2 (val=4)

Step 2 — 2 < 4 → swap:
[1, 3, 2, 6, 5, 4]
        ↑ index 2, parent = (2-1)/2 = 0 (val=1)

Step 3 — 2 > 1 → stop:
[1, 3, 2, 6, 5, 4] ✓
```

---

### Heapify() — O(log N)

Fix a violation at index `i` by **sifting down**. Assumes all subtrees below are already valid heaps.

```
Heapify at index 0 (val=8) in [8, 3, 4, 6, 5]:

Step 1 — min among 8, left(3), right(4) → min=3 at index 1 → swap:
[3, 8, 4, 6, 5]

Step 2 — heapify at index 1 (val=8):
min among 8, left(6), right(5) → min=5 at index 4 → swap:
[3, 5, 4, 6, 8]

Step 3 — index 4 has no children → stop ✓
```

---

### getMin() — O(1)

```
[1, 3, 2, 6, 5, 4]
 ↑
 return arr[0] = 1 ✓
```

Root is always the minimum — direct access, no traversal.

---

### ExtractMin() — O(log N)

Remove root, replace with last element, then heapify down.

```
ExtractMin from [1, 3, 2, 6, 5, 4]:

Step 1 — save min = 1
Step 2 — move last element to root, shrink size:
         [4, 3, 2, 6, 5]
Step 3 — Heapify(0):
         [2, 3, 4, 6, 5] ✓

Return 1
```

---

### Decreasekey(i, val) — O(log N)

Update `arr[i]` to a smaller value, then **bubble up** to restore order.

```
Decreasekey(3, -2) in [1, 3, 2, 6, 5]:
(index 3: 6 → -2)

Step 1 — set arr[3] = -2:
[1, 3, 2, -2, 5]

Step 2 — -2 < parent arr[1]=3 → swap:
[1, -2, 2, 3, 5]

Step 3 — -2 < parent arr[0]=1 → swap:
[-2, 1, 2, 3, 5] ✓
```

---

### Delete(i) — O(log N)

Combine Decreasekey + ExtractMin.

```
Delete(2) from [1, 3, 2, 6, 5]:

Step 1 — Decreasekey(2, INT_MIN) → bubbles to root:
[INT_MIN, 3, 1, 6, 5]

Step 2 — ExtractMin() removes INT_MIN:
[1, 3, 5, 6] ✓
```

## 6. C++ Implementation
 **Min Heap**
```cpp
#include <bits/stdc++.h>
using namespace std;

class BinaryHeap {
public:
    int capacity, size, *arr;

    BinaryHeap(int cap) {
        capacity = cap;
        size     = 0;
        arr      = new int[capacity];
    }

    int parent(int i) { return (i - 1) / 2; }
    int left(int i)   { return 2 * i + 1;   }
    int right(int i)  { return 2 * i + 2;   }

    void Insert(int x) {
        if (size == capacity) { cout << "Overflow\n"; return; }
        arr[size] = x;
        int k = size++;

        // Bubble up
        while (k != 0 && arr[parent(k)] > arr[k]) {
            swap(arr[parent(k)], arr[k]);
            k = parent(k);
        }
    }

    void Heapify(int i) {
        int smallest = i;
        int l = left(i), r = right(i);

        if (l < size && arr[l] < arr[smallest]) smallest = l;
        if (r < size && arr[r] < arr[smallest]) smallest = r;

        if (smallest != i) {
            swap(arr[i], arr[smallest]);
            Heapify(smallest);   // recurse down
        }
    }

    int getMin() { return arr[0]; }

    int ExtractMin() {
        if (size <= 0) return INT_MAX;
        if (size == 1) { size--; return arr[0]; }

        int mini = arr[0];
        arr[0] = arr[--size];
        Heapify(0);
        return mini;
    }

    void Decreasekey(int i, int val) {
        arr[i] = val;
        while (i != 0 && arr[parent(i)] > arr[i]) {
            swap(arr[parent(i)], arr[i]);
            i = parent(i);
        }
    }

    void Delete(int i) {
        Decreasekey(i, INT_MIN);
        ExtractMin();
    }

    void print() {
        for (int i = 0; i < size; i++) cout << arr[i] << " ";
        cout << "\n";
    }
};

int main() {
    BinaryHeap h(20);
    h.Insert(4); h.Insert(1); h.Insert(2);
    h.Insert(6); h.Insert(7); h.Insert(3);
    h.Insert(8); h.Insert(5);

    cout << h.getMin() << "\n";   // 1

    h.Insert(-1);
    cout << h.getMin() << "\n";   // -1

    h.Decreasekey(3, -2);
    cout << h.getMin() << "\n";   // -2

    h.ExtractMin();
    cout << h.getMin() << "\n";   // -1

    h.Delete(0);
    cout << h.getMin() << "\n";   // next min
}
```

## 7. Complexity Summary

| Operation | Time | Why |
|-----------|------|-----|
| `Insert()` | O(log N) | Bubble up ≤ H levels |
| `Heapify()` | O(log N) | Sift down ≤ H levels |
| `getMin()` | O(1) | Root is always minimum |
| `ExtractMin()` | O(log N) | Replace root + heapify |
| `Decreasekey()` | O(log N) | Bubble up ≤ H levels |
| `Delete()` | O(log N) | Decreasekey + ExtractMin |

> H = height = **log N** for a complete binary tree

## 8. Min Heap vs Max Heap

| | Min Heap | Max Heap |
|-|----------|----------|
| **Heap property** | Parent ≤ children | Parent ≥ children |
| **Root** | Smallest element | Largest element |
| **Swap condition** | `arr[parent] > arr[k]` | `arr[parent] < arr[k]` |
| **Common uses** | Dijkstra, Prim's, task scheduling | HeapSort, K largest elements |
