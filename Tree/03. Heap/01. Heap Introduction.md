## Introduction to Priority Queues using Binary Heaps

### What is a Priority Queue?

A Priority Queue is a special type of queue where each element is assigned a priority, and instead of being processed in the order they arrive (like a normal queue), the element with the **highest priority is always processed first**.

If two elements have the same priority, they are handled based on their insertion order.
```
  Normal Queue (FIFO):         Priority Queue:
  [cold, flu, heart attack]    [heart attack, flu, cold]
   ↑ processed first            ↑ processed first
   (arrival order)              (priority order)
```

> Think of it like an emergency room — patients are treated by severity, not arrival time.

---

### Binary Heap:

A Binary Heap is a Binary Tree that satisfies **two conditions**:

1. It must be a **Complete Binary Tree**
2. It must satisfy the **Heap Property**

### Complete Binary Tree:

All levels are completely filled except the last level, and the last level is filled **from left to right**.
```
  Complete Binary Tree ✓        Not Complete ✗
         1                           1
        / \                         / \
       2   3                       2   3
      / \ /                       /     \
     4  5 6                      4       6
                                 (missing node 5)
```

### Heap Property:
```
  Min Heap:                      Max Heap:
  Parent ≤ Children              Parent ≥ Children

         1  ← minimum                  9  ← maximum
        / \                           / \
       3   2                         7   8
      / \ / \                       / \ / \
     6  5 4  8                     1  2 3  4

  Root = smallest element        Root = largest element
```

---

### Array Representation:

A Binary Heap is stored as an array.
```
  Tree:                  Array:
         1               [1, 3, 2, 6, 5, 4, 8]
        / \               0  1  2  3  4  5  6
       3   2
      / \ / \
     6  5 4  8

  Index Relationships:
  ┌────────────────────────────────────┐
  │ Node at i  → Left  child: 2*i + 1 │
  │             → Right child: 2*i + 2 │
  │             → Parent:     (i-1)/2  │
  └────────────────────────────────────┘

  Example: Node at index 1 (val=3)
    Left  child: 2*1+1 = 3 (val=6)
    Right child: 2*1+2 = 4 (val=5)
    Parent:      (1-1)/2 = 0 (val=1)
```

## Operations on Min Heap

### 1. Insert() — O(log N)

Insert key at the end, then **bubble up** (sift up) to fix heap property.
```
  Insert 2 into: [1, 3, 4, 6, 5]

  Step 1: Add at end
  [1, 3, 4, 6, 5, 2]
                  ↑ index 5, parent = (5-1)/2 = 2 (val=4)

  Step 2: 2 < 4 → SWAP
  [1, 3, 2, 6, 5, 4]
          ↑ index 2, parent = (2-1)/2 = 0 (val=1)

  Step 3: 2 > 1 → STOP
  [1, 3, 2, 6, 5, 4] ✓
```

### 2. Heapify() — O(log N)

Fix a violation at index `i` by **sifting down**. Assumes all subtrees are valid heaps.
```
  Heapify at index 0 (val=8) in [8, 3, 4, 6, 5]:

  Step 1: Find min among 8, left(3), right(4) → min=3 at index 1
  SWAP index 0 and 1:
  [3, 8, 4, 6, 5]

  Step 2: Heapify at index 1 (val=8)
  Find min among 8, left(6), right(5) → min=5 at index 4
  SWAP:
  [3, 5, 4, 6, 8]

  Step 3: Index 4 has no children → STOP ✓
```

### 3. getMin() — O(1)

Simply return `arr[0]` — the root is always the minimum.
```
  [1, 3, 2, 6, 5, 4]
   ↑
  return 1 ✓
```

### 4. ExtractMin() — O(log N)

Remove the root, replace with last element, then heapify.
```
  ExtractMin from [1, 3, 2, 6, 5, 4]:

  Step 1: Save min = 1
  Step 2: Copy last element to root, decrease size
          [4, 3, 2, 6, 5]
  Step 3: Heapify(0)
          [2, 3, 4, 6, 5] ✓

  Return 1
```

### 5. Decreasekey(i, val) — O(log N)

Update value at index `i` to `val` (val < current), then **bubble up**.
```
  Decreasekey(3, -2) in [1, 3, 2, 6, 5]:
  (index 3, val=6 → -2)

  Step 1: arr[3] = -2
          [1, 3, 2, -2, 5]

  Step 2: -2 < parent(3)=arr[1]=3 → SWAP
          [1, -2, 2, 3, 5]

  Step 3: -2 < parent(1)=arr[0]=1 → SWAP
          [-2, 1, 2, 3, 5] ✓
```

### 6. Delete(i) — O(log N)

Delete node at index `i` using a combination of Decreasekey + ExtractMin.
```
  Delete(2) from [1, 3, 2, 6, 5]:

  Step 1: Decreasekey(2, INT_MIN)
          [INT_MIN, 3, 1, 6, 5] → bubbles up →
          [INT_MIN, 3, 1, 6, 5] → reaches root

  Step 2: ExtractMin() removes INT_MIN
          [1, 3, 5, 6] ✓
```

### C++ Code:
```cpp
#include <bits/stdc++.h>
using namespace std;

class BinaryHeap {
public:
    int capacity;
    int size;
    int* arr;

    BinaryHeap(int cap) {
        capacity = cap;
        size = 0;
        arr = new int[capacity];
    }

    int parent(int i) { return (i - 1) / 2; }
    int left(int i)   { return 2 * i + 1;   }
    int right(int i)  { return 2 * i + 2;   }

    // Insert a new key
    void Insert(int x) {
        if (size == capacity) {
            cout << "Binary Heap Overflow" << endl;
            return;
        }
        arr[size] = x;
        int k = size;
        size++;

        // Bubble up
        while (k != 0 && arr[parent(k)] > arr[k]) {
            swap(arr[parent(k)], arr[k]);
            k = parent(k);
        }
    }

    // Fix heap violation at index ind (sift down)
    void Heapify(int ind) {
        int ri = right(ind);
        int li = left(ind);
        int smallest = ind;

        if (li < size && arr[li] < arr[smallest]) smallest = li;
        if (ri < size && arr[ri] < arr[smallest]) smallest = ri;

        if (smallest != ind) {
            swap(arr[ind], arr[smallest]);
            Heapify(smallest);
        }
    }

    // Return minimum element
    int getMin() {
        return arr[0];
    }

    // Remove and return minimum element
    int ExtractMin() {
        if (size <= 0) return INT_MAX;
        if (size == 1) { size--; return arr[0]; }

        int mini = arr[0];
        arr[0] = arr[size - 1];
        size--;
        Heapify(0);
        return mini;
    }

    // Decrease value at index i to val
    void Decreasekey(int i, int val) {
        arr[i] = val;
        while (i != 0 && arr[parent(i)] > arr[i]) {
            swap(arr[parent(i)], arr[i]);
            i = parent(i);
        }
    }

    // Delete element at index i
    void Delete(int i) {
        Decreasekey(i, INT_MIN);
        ExtractMin();
    }

    void print() {
        for (int i = 0; i < size; i++)
            cout << arr[i] << " ";
        cout << endl;
    }
};

int main() {
    BinaryHeap h(20);
    h.Insert(4); h.Insert(1); h.Insert(2);
    h.Insert(6); h.Insert(7); h.Insert(3);
    h.Insert(8); h.Insert(5);

    cout << "Min value is " << h.getMin() << endl;      // 1

    h.Insert(-1);
    cout << "Min value is " << h.getMin() << endl;      // -1

    h.Decreasekey(3, -2);
    cout << "Min value is " << h.getMin() << endl;      // -2

    h.ExtractMin();
    cout << "Min value is " << h.getMin() << endl;      // -1

    h.Delete(0);
    cout << "Min value is " << h.getMin() << endl;      // next min

    return 0;
}
```

### Time Complexity Summary:
```
  Operation       Time        Why
  ─────────────────────────────────────────────────────
  Insert()        O(log N)    Bubble up at most H levels
  Heapify()       O(log N)    Sift down at most H levels
  getMin()        O(1)        Root is always minimum
  ExtractMin()    O(log N)    Replace root + heapify
  Decreasekey()   O(log N)    Bubble up at most H levels
  Delete()        O(log N)    Decreasekey + ExtractMin

  Where H = height = log N for a complete binary tree
```

### Min Heap vs Max Heap
```
  Min Heap                       Max Heap
  ──────────────────────         ──────────────────────
  Parent ≤ children              Parent ≥ children
  Root = minimum                 Root = maximum
  arr[parent] > arr[k]           arr[parent] < arr[k]
  → SWAP condition               → SWAP condition
  Used for: Dijkstra,            Used for: HeapSort,
  Prim's, task scheduling        finding K largest
```
