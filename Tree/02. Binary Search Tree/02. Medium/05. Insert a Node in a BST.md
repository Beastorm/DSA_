## Insert a Node in a BST

**Problem Statement:**   
Inserting a node in a BST follows the same logic as searching. We traverse the tree comparing values to find the correct position where the new node should be placed, maintaining the BST property.
```
  Insert 5 into this BST:

       8                    8
      / \                  / \
     3   10      →        3   10
    / \    \             / \    \
   1   6    14          1   6    14
                           /
                          5
```

---

>### Examples:

**Example 1:** Insert `5` into BST
```
  Before:                After:
       8                    8
      / \                  / \
     3   10               3   10
    / \    \             / \    \
   1   6    14          1   6    14
                           /
                          5

  Traversal:
  8  → 5 < 8,  go LEFT
  3  → 5 > 3,  go RIGHT
  6  → 5 < 6,  go LEFT
  null → INSERT here ✓
```

**Example 2:** Insert `11` into BST
```
  Before:                After:
       8                    8
      / \                  / \
     3   10               3   10
           \                 /  \
            14              11   14

  Traversal:
  8  → 11 > 8,  go RIGHT
  10 → 11 > 10, go RIGHT
  14 → 11 < 14, go LEFT
  null → INSERT here ✓
```

**Example 3:** Insert `1` into empty BST
```
  Before:      After:
   null           1

  Tree is empty → new node becomes root ✓
```

---
---

### Edge Cases:
```
  Insert into single node BST:      Insert duplicate (not allowed):
         5                                   5
                                            / \
  Insert 3:      Insert 8:                3     8
         5              5
        /                \             Insert 5 again →
       3                  8            Typically ignored or
                                       handled per requirement
```

---

## Approach
### Algorithm:

- If the tree is empty, the new node becomes the root.
- Start at the root and traverse the tree:
  - If the key is **less than** the current node, move to the **left** child.
  - If the key is **greater than** the current node, move to the **right** child.
- When a `null` position is found, insert the new node there.
- Return the root of the modified tree.
```
        START
          |
      root == null?
       /         \
     YES          NO
      |            |
  return        key < root?
  new node      /        \
              YES          NO
               |            |
          insert in      insert in
          left subtree   right subtree
```

---

### C++ Code:
```cpp
#include <bits/stdc++.h>
using namespace std;

// Definition of TreeNode
struct TreeNode {
    int val;
    TreeNode* left;
    TreeNode* right;

    TreeNode(int data) {
        val = data;
        left = right = nullptr;
    }
};

class Solution {
public:
    // Function to insert a node in BST
    TreeNode* insertIntoBST(TreeNode* root, int key) {
        // If tree is empty, new node is the root
        if (root == nullptr) {
            return new TreeNode(key);
        }

        // Use a pointer to traverse without losing root
        TreeNode* curr = root;

        while (true) {
            // Key goes into left subtree
            if (key < curr->val) {
                if (curr->left == nullptr) {
                    // Found the position, insert here
                    curr->left = new TreeNode(key);
                    break;
                }
                // Move left
                curr = curr->left;
            }
            // Key goes into right subtree
            else {
                if (curr->right == nullptr) {
                    // Found the position, insert here
                    curr->right = new TreeNode(key);
                    break;
                }
                // Move right
                curr = curr->right;
            }
        }

        // Return original root
        return root;
    }
};

// Helper to print inorder traversal
void inorder(TreeNode* root) {
    if (!root) return;
    inorder(root->left);
    cout << root->val << " ";
    inorder(root->right);
}

// Driver code
int main() {
    TreeNode* root = new TreeNode(8);
    root->left = new TreeNode(3);
    root->right = new TreeNode(10);
    root->left->left = new TreeNode(1);
    root->left->right = new TreeNode(6);
    root->right->right = new TreeNode(14);

    cout << "Inorder before insertion: ";
    inorder(root);
    cout << endl;

    Solution obj;
    root = obj.insertIntoBST(root, 5);

    cout << "Inorder after inserting 5: ";
    inorder(root);
    cout << endl;

    return 0;
}
```

### Complexity Analysis:

- **Time Complexity:** O(H), where H is the height of the BST.
  - Balanced BST: O(log N)
  - Skewed BST: O(N)
  ```
    Balanced BST             Skewed BST
         8                       1
        / \                       \
       3   10                      3
      /      \                      \
     1        14                     8
                                      \
    Insert ~ log(N) steps             10
                                    Insert ~ N steps
  ```

- **Space Complexity:** O(1), the iterative approach uses only a constant amount of extra space. No recursion stack is involved.
