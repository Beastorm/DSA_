## Merge 2 BSTs

**Problem Statement:**  
Given two BSTs, return the elements of the merged BSTs in sorted form.

---

>### Examples:

**Example 1:**
- **Input:** `root1 = [5, 3, 6, 2, 4]`, `root2 = [2, 1, 3, Null, Null, Null, 7, 6]`
- **Output:** `[1, 2, 2, 3, 3, 4, 5, 6, 6, 7]`
```
  BST 1:              BST 2:
       5                   2
      / \                 / \
     3   6               1   3
    / \                       \
   2   4                       7
                              /
                             6

  Inorder BST1: [2, 3, 4, 5, 6]
  Inorder BST2: [1, 2, 3, 6, 7]
  Merged:       [1, 2, 2, 3, 3, 4, 5, 6, 6, 7] ✓
```

**Example 2:**
- **Input:** `root1 = [12, 9, Null, 6, 11]`, `root2 = [8, 5, 10, 2]`
- **Output:** `[2, 5, 6, 8, 9, 10, 11, 12]`
```
  BST 1:              BST 2:
      12                   8
      /                   / \
     9                   5   10
    / \                 /
   6   11              2

  Inorder BST1: [6, 9, 11, 12]
  Inorder BST2: [2, 5, 8, 10]
  Merged:       [2, 5, 6, 8, 9, 10, 11, 12] ✓
```

---

### Key Insight:

The **inorder traversal** of a BST always gives a **sorted array**. We can exploit this property to merge two BSTs efficiently.
```
  Approach 1 (Brute Force):
  Collect all elements → Sort

  Approach 2 (Optimal):
  Inorder BST1 → sorted array1
  Inorder BST2 → sorted array2
  Merge two sorted arrays (two-pointer) → result

  Like merging two sorted arrays in merge sort!

  arr1: [2, 3, 4, 5]     arr2: [1, 2, 3, 7]
   i=0                    j=0

  2 > 1 → take 1, j++     → [1]
  2 = 2 → take 2, i++     → [1, 2]
  3 > 2 → take 2, j++     → [1, 2, 2]
  3 = 3 → take 3, i++     → [1, 2, 2, 3]
  ... and so on
```

---

## Brute Force Approach

### Algorithm:

- Traverse both BSTs using any traversal method and collect all elements in a single list.
- Insert elements from both trees into this list without worrying about order during traversal.
- Sort the list using a sorting algorithm.
- Return the sorted list as the merged elements of both BSTs.
```
  BST1 traversal: [2, 3, 4, 5, 6]
  BST2 traversal: [1, 3, 2, 7, 6]
  Combined list:  [2, 3, 4, 5, 6, 1, 3, 2, 7, 6]
  After sort:     [1, 2, 2, 3, 3, 4, 5, 6, 6, 7] ✓
```

### C++ Code:
```cpp
#include <bits/stdc++.h>
using namespace std;

// Structure for a BST node
struct Node {
    int data;
    Node* left;
    Node* right;
    Node(int val) : data(val), left(NULL), right(NULL) {}
};

class Solution {
public:
    // Function to traverse a BST and store elements in list
    void traverse(Node* root, vector<int>& elements) {
        if (!root) return;
        traverse(root->left, elements);
        elements.push_back(root->data);
        traverse(root->right, elements);
    }

    // Function to merge elements of two BSTs
    vector<int> mergeBSTs(Node* root1, Node* root2) {
        vector<int> elements;
        // Traverse first BST and collect elements
        traverse(root1, elements);
        // Traverse second BST and collect elements
        traverse(root2, elements);
        // Sort the collected elements
        sort(elements.begin(), elements.end());
        return elements;
    }
};

int main() {
    Node* root1 = new Node(2);
    root1->left = new Node(1);
    root1->right = new Node(4);

    Node* root2 = new Node(3);
    root2->left = new Node(0);
    root2->right = new Node(5);

    Solution sol;
    vector<int> result = sol.mergeBSTs(root1, root2);

    for (int val : result) {
        cout << val << " ";
    }
    return 0;
}
```

### Complexity Analysis:

- **Time Complexity:** O((N + M) \* log(N + M)), we traverse both BSTs and sort all the elements.

- **Space Complexity:** O(N + M), additional space required for storing elements of the two BSTs.

---

## Optimal Approach

### Algorithm:

The inorder traversal of any BST returns elements in sorted order. We can use this property to optimize our solution by traversing both BSTs and storing their elements in separate sorted lists, then merging them using a two-pointer approach.

- Traverse both BSTs using inorder traversal and collect elements in separate lists.
- Create an additional list to store the elements of the two sorted lists.
- Merge the sorted lists using a two-pointer approach.
- Return the sorted list as the merged elements of both BSTs.
```
  Two-pointer merge:

  arr1: [1, 3, 5]     i →
  arr2: [2, 4, 6]     j →
  result: []

  Step 1: arr1[0]=1 < arr2[0]=2 → take 1, i++   → [1]
  Step 2: arr1[1]=3 > arr2[0]=2 → take 2, j++   → [1, 2]
  Step 3: arr1[1]=3 < arr2[1]=4 → take 3, i++   → [1, 2, 3]
  Step 4: arr1[2]=5 > arr2[1]=4 → take 4, j++   → [1, 2, 3, 4]
  Step 5: arr1[2]=5 < arr2[2]=6 → take 5, i++   → [1, 2, 3, 4, 5]
  Step 6: i exhausted → append remaining [6]     → [1, 2, 3, 4, 5, 6]
```

### C++ Code:
```cpp
#include <bits/stdc++.h>
using namespace std;

// Structure for a binary tree node
struct Node {
    int data;
    Node* left;
    Node* right;
    Node(int val) : data(val), left(NULL), right(NULL) {}
};

class Solution {
public:
    // Function to perform inorder traversal and store elements
    void inorderTraversal(Node* root, vector<int>& arr) {
        if (!root) return;
        inorderTraversal(root->left, arr);
        arr.push_back(root->data);
        inorderTraversal(root->right, arr);
    }

    // Function to merge two sorted arrays using two pointers
    vector<int> mergeArrays(vector<int>& arr1, vector<int>& arr2) {
        vector<int> merged;
        int i = 0, j = 0;

        // Merge until one array ends
        while (i < arr1.size() && j < arr2.size()) {
            if (arr1[i] < arr2[j]) merged.push_back(arr1[i++]);
            else merged.push_back(arr2[j++]);
        }

        // Add remaining elements from arr1
        while (i < arr1.size()) merged.push_back(arr1[i++]);

        // Add remaining elements from arr2
        while (j < arr2.size()) merged.push_back(arr2[j++]);

        return merged;
    }

    // Function to merge two BSTs in sorted order
    vector<int> mergeBSTs(Node* root1, Node* root2) {
        vector<int> arr1, arr2;
        // Inorder traversal of both BSTs
        inorderTraversal(root1, arr1);
        inorderTraversal(root2, arr2);
        // Merge the two sorted arrays
        return mergeArrays(arr1, arr2);
    }
};

int main() {
    Node* root1 = new Node(3);
    root1->left = new Node(1);
    root1->right = new Node(5);

    Node* root2 = new Node(4);
    root2->left = new Node(2);
    root2->right = new Node(6);

    Solution sol;
    vector<int> result = sol.mergeBSTs(root1, root2);

    for (int val : result) cout << val << " ";
    return 0;
}
```

### Complexity Analysis:

- **Time Complexity:** O(N + M), we traverse both BSTs and merge two sorted lists using the two-pointer technique.

- **Space Complexity:** O(N + M), additional space required for storing elements of the two BSTs.

---

## Comparison of Approaches
```
  Approach       Time                 Space      Key Idea
  ────────────────────────────────────────────────────────
  Brute Force    O((N+M) * log(N+M))  O(N+M)     Collect all + sort
  Optimal        O(N+M)               O(N+M)     Inorder + two-pointer merge
```
