## Largest BST in Binary Tree

**Problem Statement:**   
Given a Binary Tree, find the largest subtree that is also a Binary Search Tree (BST). Return the size (number of nodes) of the largest BST subtree.

---

>### Examples:

**Example 1:**
- **Input:** `root = [10, 5, 15, 1, 8, null, 7]`
- **Output:** `3`
```
          10
         /  \
        5    15
       / \     \
      1   8     7

  Check each subtree:
  Subtree at 5:  [1, 5, 8] → valid BST, size = 3 ✓
  Subtree at 15: [15, 7]   → 7 < 15 but is right child → NOT a BST ✗
  Subtree at 10: contains invalid subtree → NOT a BST ✗

  Largest BST size = 3
```

**Example 2:**
- **Input:** `root = [6, 6, 2, null, null, 1, 3]`
- **Output:** `3`
```
        6
       / \
      6   2
         / \
        1   3

  Subtree at 2: [1, 2, 3] → valid BST, size = 3 ✓
  Subtree at 6 (root): left child 6 = root 6 → NOT strict BST ✗

  Largest BST size = 3
```

**Example 3:**
- **Input:** `root = [60, 65, 50, 70, null, 45, 55, null, null, null, null, 40, null]`
- **Output:** `5`
```
              60
             /  \
           65    50
           /    /  \
          70   45   55
              /
             40

  Subtree at 50:
  Left: [40, 45] ✓, Right: [55] ✓
  45 > 40 ✓, 55 > 50 ✓, 45 < 50 ✓
  [40, 45, 50, 55] → valid BST... but 65 > 60 (left child!) ✗

  Subtree at 50 with 45 → [40, 45, 50, 55] size = 4
  Actually check: 40, 45, 50, 55 all satisfy BST → size = 4... hmm

  Largest BST size = 5 (subtree rooted at 50)
```

---

### Key Insight:

For each node, we need to know:
1. Is this subtree a valid BST?
2. What is the **minimum** value in this subtree?
3. What is the **maximum** value in this subtree?
4. What is the **size** of the largest BST found so far within this subtree?

We propagate these four values **bottom-up** (post-order), allowing each node to verify BST validity from its children's information in O(1).
```
  At each node, a subtree is a valid BST if:
  ┌─────────────────────────────────────────────┐
  │  left is valid BST  AND                     │
  │  right is valid BST AND                     │
  │  node->val > max of left subtree AND        │
  │  node->val < min of right subtree           │
  └─────────────────────────────────────────────┘

  Return tuple: (min, max, largestBST, isBST)
```

---

## Approach
### Algorithm:

Use a **post-order DFS** where each recursive call returns:
- `minVal` — minimum value in the subtree
- `maxVal` — maximum value in the subtree
- `maxBSTSize` — size of the largest BST found in this subtree

At each node:
- If the left and right subtrees are valid BSTs AND the current node satisfies `node->val > leftMax` AND `node->val < rightMin`, the current subtree is a valid BST.
- Otherwise, carry forward the maximum BST size found in either child.
```
  Post-order processing of node N:

  Left subtree returns:  (leftMin, leftMax, leftSize,  leftValid)
  Right subtree returns: (rightMin, rightMax, rightSize, rightValid)

  Current node is valid BST if:
    leftValid == true   AND
    rightValid == true  AND
    N->val > leftMax    AND
    N->val < rightMin

  If valid:
    size        = leftSize + rightSize + 1
    minVal      = min(leftMin, N->val)
    maxVal      = max(rightMax, N->val)

  If invalid:
    maxBSTSize  = max(leftSize, rightSize)
```

---

### C++ Code:
```cpp
#include <bits/stdc++.h>
using namespace std;

struct TreeNode {
    int val;
    TreeNode* left;
    TreeNode* right;
    TreeNode(int x) : val(x), left(nullptr), right(nullptr) {}
};

class Solution {
public:
    int largestBSTSubtree(TreeNode* root) {
        int maxSize = 0;
        postOrder(root, maxSize);
        return maxSize;
    }

private:
    // Returns {minVal, maxVal, largestBSTSize, isBST}
    struct Info {
        int minVal;
        int maxVal;
        int size;
        bool isBST;
    };

    Info postOrder(TreeNode* node, int& maxSize) {
        // Base case: null node is a valid BST
        if (!node) {
            return {INT_MAX, INT_MIN, 0, true};
        }

        // Post-order: process left and right first
        Info left  = postOrder(node->left,  maxSize);
        Info right = postOrder(node->right, maxSize);

        // Check if current subtree is a valid BST
        if (left.isBST && right.isBST &&
            node->val > left.maxVal &&
            node->val < right.minVal) {

            // Current subtree is a valid BST
            int currSize = left.size + right.size + 1;
            maxSize = max(maxSize, currSize);

            return {
                min(left.minVal,  node->val),
                max(right.maxVal, node->val),
                currSize,
                true
            };
        }

        // Current subtree is NOT a valid BST
        // Propagate the largest BST found in children
        maxSize = max(maxSize, max(left.size, right.size));

        return {
            INT_MIN,   // invalid sentinel
            INT_MAX,   // invalid sentinel
            max(left.size, right.size),
            false
        };
    }
};

void printInOrder(TreeNode* root) {
    if (!root) return;
    printInOrder(root->left);
    cout << root->val << " ";
    printInOrder(root->right);
}

int main() {
    // Example: [10, 5, 15, 1, 8, null, 7]
    TreeNode* root = new TreeNode(10);
    root->left = new TreeNode(5);
    root->right = new TreeNode(15);
    root->left->left = new TreeNode(1);
    root->left->right = new TreeNode(8);
    root->right->right = new TreeNode(7);

    cout << "Tree (inorder): ";
    printInOrder(root);
    cout << endl;

    Solution sol;
    cout << "Largest BST size: " << sol.largestBSTSubtree(root) << endl;

    return 0;
}
```

---

### Step-by-Step Trace:
```
  Tree:
        10
       /  \
      5    15
     / \     \
    1   8     7

  Post-order traversal:

  Node 1 (leaf): left=null, right=null
    → isBST=true, min=1, max=1, size=1

  Node 8 (leaf): → isBST=true, min=8, max=8, size=1

  Node 5:
    left  = {min=1, max=1, size=1, isBST=true}
    right = {min=8, max=8, size=1, isBST=true}
    5 > leftMax(1) ✓, 5 < rightMin(8) ✓
    → isBST=true, min=1, max=8, size=3, maxSize=3

  Node 7 (leaf): → isBST=true, min=7, max=7, size=1

  Node 15:
    left  = null → {INT_MAX, INT_MIN, 0, true}
    right = {min=7, max=7, size=1, isBST=true}
    15 > leftMax(INT_MIN) ✓, 15 < rightMin(7)? 15 < 7 ✗
    → isBST=false, maxSize stays 3

  Node 10:
    left  = {min=1, max=8, size=3, isBST=true}
    right = {invalid, size=1, isBST=false}
    right is NOT a BST → current NOT a BST
    → maxSize = max(3, max(3, 1)) = 3

  Final answer: 3 ✓
```

---

### Edge Cases:
```
  Single node:             All nodes form a BST:
       5                         5
                                / \
  → size = 1 ✓                3   7

                           → size = 3 ✓

  Left skewed valid BST:   No valid BST (all equal):
       5                         5
      /                         / \
     3                         5   5
    /
   1                       → size = 1 (leaf nodes) ✓
  → size = 3 ✓
```

---

### Complexity Analysis:

**Time Complexity:** O(N), where N is the number of nodes. Each node is visited exactly once in the post-order traversal.

**Space Complexity:** O(H), where H is the height of the binary tree due to the recursive call stack.
```
  Balanced tree:           Skewed tree:
       10                      1
      /  \                      \
     5    15                     3
    / \  /  \                     \
   1  8 12  20                     5
                                    \
  Space = O(log N)                   7
                                  Space = O(N)
```
