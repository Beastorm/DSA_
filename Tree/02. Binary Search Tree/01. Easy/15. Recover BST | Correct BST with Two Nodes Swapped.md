## Recover BST | Correct BST with Two Nodes Swapped

**Problem Statement:**  
You are given the root of a Binary Search Tree (BST) where exactly two nodes were swapped by mistake. Recover the tree without changing its structure (restore the original BST).

---

>### Examples:

**Example 1:**
- **Input:** `root = [1, 3, null, null, 2]`
- **Output:** `[3, 1, null, null, 2]`
```
  Swapped BST:           Recovered BST:
       1                      3
      /                      /
     3                      1
      \                      \
       2                      2

  Inorder of swapped:  [3, 1, 2]  ← not sorted, 3 and 1 are swapped
  Inorder of correct:  [1, 2, 3]  ✓
```

**Example 2:**
- **Input:** `root = [3, 1, 4, null, null, 2]`
- **Output:** `[2, 1, 4, null, null, 3]`
```
  Swapped BST:           Recovered BST:
       3                      2
      / \                    / \
     1   4                  1   4
        /                      /
       2                       3

  Inorder of swapped: [1, 3, 2, 4]  ← 3 and 2 are swapped
  Inorder of correct: [1, 2, 3, 4]  ✓
```

---

### Key Insight:

The inorder traversal of a correct BST is always **strictly increasing**. If two nodes are swapped, there will be **one or two violations** (places where `prev > current`) in the inorder sequence.
```
  Case 1: Adjacent nodes swapped
  Inorder: [1, 2, 3, 4, 5]  →  [1, 2, 4, 3, 5]
                                       ↑
                                  1 violation: (4,3)
  first = 4, second = 3

  Case 2: Non-adjacent nodes swapped
  Inorder: [1, 2, 3, 4, 5]  →  [1, 4, 3, 2, 5]
                                    ↑     ↑
                               2 violations: (4,3) and (2, next)
  first = 4, second = 2

  Fix: swap values of first and second
```

---

## Approach 1: Brute Force (Using Extra Space)

### Algorithm:

- Perform an inorder traversal and store all node values in a sorted array.
- Sort the collected values.
- Perform another inorder traversal and assign values back from the sorted array.
```
  Step 1 — Collect inorder:
  Swapped BST inorder: [1, 3, 2, 4]

  Step 2 — Sort:
  Sorted:              [1, 2, 3, 4]

  Step 3 — Reassign values back to nodes in inorder:
  Node 1 ← 1
  Node 3 ← 2  (was 3, now corrected)
  Node 2 ← 3  (was 2, now corrected)
  Node 4 ← 4
```

### C++ Code:
```cpp
#include <bits/stdc++.h>
using namespace std;

struct TreeNode {
    int val;
    TreeNode* left;
    TreeNode* right;
    TreeNode(int x) : val(x), left(nullptr), right(nullptr) {}
};

class Solution {
public:
    void recoverTree(TreeNode* root) {
        vector<int> vals;
        // Step 1: collect all values via inorder
        collectInorder(root, vals);
        // Step 2: sort values
        sort(vals.begin(), vals.end());
        // Step 3: reassign sorted values back
        int idx = 0;
        assignInorder(root, vals, idx);
    }

private:
    void collectInorder(TreeNode* node, vector<int>& vals) {
        if (!node) return;
        collectInorder(node->left, vals);
        vals.push_back(node->val);
        collectInorder(node->right, vals);
    }

    void assignInorder(TreeNode* node, vector<int>& vals, int& idx) {
        if (!node) return;
        assignInorder(node->left, vals, idx);
        node->val = vals[idx++];
        assignInorder(node->right, vals, idx);
    }
};

void printInOrder(TreeNode* root) {
    if (!root) return;
    printInOrder(root->left);
    cout << root->val << " ";
    printInOrder(root->right);
}

int main() {
    TreeNode* root = new TreeNode(3);
    root->left = new TreeNode(1);
    root->right = new TreeNode(4);
    root->right->left = new TreeNode(2);

    cout << "Before recovery (inorder): ";
    printInOrder(root);
    cout << endl;

    Solution sol;
    sol.recoverTree(root);

    cout << "After recovery (inorder): ";
    printInOrder(root);
    cout << endl;

    return 0;
}
```

### Complexity Analysis:
- **Time Complexity:** O(N log N), O(N) for inorder traversal + O(N log N) for sorting + O(N) for reassigning.
- **Space Complexity:** O(N), to store all node values in the array.

## Approach 2: Optimal (O(1) Space — Morris Traversal)

### Algorithm:

Use Morris Inorder Traversal to detect the two swapped nodes **without extra space**.

While doing inorder traversal, track:
- `first` — first node where violation occurs (prev > current)
- `second` — second node in the violation
- `prev` — previously visited node
```
  Violation detection:

  Inorder: [1, 4, 3, 2, 5]
              ↑  ↑  ↑
           prev curr

  At (4,3): prev(4) > curr(3) → first = prev(4), second = curr(3)
  At (3,2): prev(3) > curr(2) → second updated = curr(2)

  After full traversal: first=4, second=2 → swap their values ✓
```

### C++ Code:
```cpp
#include <bits/stdc++.h>
using namespace std;

struct TreeNode {
    int val;
    TreeNode* left;
    TreeNode* right;
    TreeNode(int x) : val(x), left(nullptr), right(nullptr) {}
};

class Solution {
public:
    TreeNode* first = nullptr;
    TreeNode* second = nullptr;
    TreeNode* prev = nullptr;

    void recoverTree(TreeNode* root) {
        // Find the two swapped nodes using inorder traversal
        inorder(root);
        // Swap their values to fix the BST
        if (first && second) {
            swap(first->val, second->val);
        }
    }

private:
    void inorder(TreeNode* node) {
        if (!node) return;

        // Traverse left subtree
        inorder(node->left);

        // Detect violation
        if (prev != nullptr && prev->val > node->val) {
            // First violation: record first as prev
            if (first == nullptr) {
                first = prev;
            }
            // Always update second to current node
            second = node;
        }

        // Update prev to current node
        prev = node;

        // Traverse right subtree
        inorder(node->right);
    }
};

void printInOrder(TreeNode* root) {
    if (!root) return;
    printInOrder(root->left);
    cout << root->val << " ";
    printInOrder(root->right);
}

int main() {
    TreeNode* root = new TreeNode(3);
    root->left = new TreeNode(1);
    root->right = new TreeNode(4);
    root->right->left = new TreeNode(2);

    cout << "Before recovery (inorder): ";
    printInOrder(root);
    cout << endl;

    Solution sol;
    sol.recoverTree(root);

    cout << "After recovery (inorder): ";
    printInOrder(root);
    cout << endl;

    return 0;
}
```

---

### Step-by-Step Trace (Optimal)
```
  Swapped BST:
       3
      / \
     1   4
        /
       2

  Inorder traversal visits: 1 → 3 → 2 → 4

  Visit 1: prev=null → no violation, prev=1
  Visit 3: prev(1) < curr(3) → no violation, prev=3
  Visit 2: prev(3) > curr(2) → VIOLATION!
            first=null → first=3, second=2, prev=2
  Visit 4: prev(2) < curr(4) → no violation, prev=4

  Result: first=3, second=2 → swap(3, 2)

  Recovered inorder: [1, 2, 3, 4] ✓
```

---

### Two Types of Violations
```
  Case 1 — Adjacent swap (1 violation):
  Inorder: [1, 2, 4, 3, 5]
                  ↑
             prev(4) > curr(3) → first=4, second=3
  Swap 4 and 3 → [1, 2, 3, 4, 5] ✓

  Case 2 — Non-adjacent swap (2 violations):
  Inorder: [1, 5, 3, 4, 2]
               ↑        ↑
  First:  prev(5) > curr(3) → first=5, second=3
  Second: prev(4) > curr(2) → second updated=2
  Swap 5 and 2 → [1, 2, 3, 4, 5] ✓
```

---

### Complexity Analysis:

- **Time Complexity:** O(N), where N is the number of nodes. Each node is visited exactly once during inorder traversal.

- **Space Complexity:** O(H), where H is the height of the BST due to the recursive call stack.
- Balanced BST: O(log N)
- Skewed BST: O(N)

---

## Comparison of Approaches
```
  Approach          Time        Space      Key Idea
  ────────────────────────────────────────────────────────
  Brute Force       O(N log N)  O(N)       Collect → sort → reassign
  Optimal           O(N)        O(H)       Track violations in inorder
```
