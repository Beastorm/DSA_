## Introduction to Binary Search Trees

### Introduction:

Binary Search Trees, popularly known as BST, are a category of Binary Trees used to optimize the operation of searching for an element among the Tree Nodes in a Binary Tree.
```
            8
          /   \
        3       10
       / \        \
      1   6        14
         / \       /
        4   7    13
```

As we can visualize from the above Tree, the right subtree has all the elements greater than the root element, and the left subtree has all the elements less than the root element. A point to note here is that every subtree is itself a Binary Search Tree, as it contains a subtree whose root node is less than the value of the current node and another subtree whose root node is greater than the value of the current node.

For every given node in a Binary Search Tree, if a left or a right child exists, then:
```
Left Child < Node < Right Child
```

**For example**, at root node `8`:
```
    3 < 8 < 10   ✓
```

### Need for Binary Search Tree:

After understanding how a Binary Search Tree is represented, let us now understand why there is a need to use a Binary Search Tree instead of a simple Binary Tree.

**BST vs Skewed Binary Tree height comparison:**
```
   BST (balanced)         Skewed Binary Tree
        8                       1
       / \                       \
      3   10                      2
     / \    \                      \
    1   6   14                      3
                                     \
   Height ~ log(N)                    4
                                   Height ~ N
```

So, generally in a BST, the maximum height in almost all cases is kept in order of `log(2N)` where `N` = number of nodes, which is in contrast to the plain Binary Tree whose maximum height can reach the order of `N` when the tree is skewed.

This particularly makes the time of searching for a given node in a Binary Search Tree a lot less than searching in a simple Binary Tree.

**Search example — looking for value `4` in the BST:**
```
            8          → 4 < 8, go LEFT
          /
        3              → 4 > 3, go RIGHT
          \
           6           → 4 < 6, go LEFT
           /
          4            ✓ FOUND in 3 steps!
```

For example, if we have to search for an element, we can directly compare it to the root node value of the BST:
- If the value matches, we have found the element.
- If the value is greater than the root node, we move on to searching that element in the right subtree because the right subtree has all the node values greater than the root value.
- The search process is terminated for the left subtree, as it would only search for those nodes that have values less than the root.

> **Note:** This is very similar to the Binary Search operation, which we perform in Arrays to optimize the time taken to search for an element using Linear Search. Hence, this category of trees is named Binary Search Trees.
