## Check if a Tree is a BST or Not

**Problem Statement:**   
Given the root of a binary tree, determine whether it is a valid Binary Search Tree (BST) or not. A valid BST must satisfy:
- The left subtree of a node contains only nodes with values **less than** the node's value.
- The right subtree of a node contains only nodes with values **greater than** the node's value.
- Both the left and right subtrees must also be valid BSTs.

---

>#### Examples:

**Example 1:**
- **Input:** `root = [2, 1, 3]`
- **Output:** `True`
```
       2
      / \
     1   3

  Left subtree:  1 < 2 ✓
  Right subtree: 3 > 2 ✓
  Valid BST ✓
```

**Example 2:**
- **Input:** `root = [5, 1, 4, null, null, 3, 6]`
- **Output:** `False`
```
       5
      / \
     1   4
        / \
       3   6

  Right child = 4, but 4 < 5 ✗
  NOT a valid BST ✗
```

**Example 3:**
- **Input:** `root = [10, 5, 15, null, null, 6, 20]`
- **Output:** `False`
```
        10
       /  \
      5    15
          /  \
         6   20

  Node 6 is in the right subtree of 10,
  but 6 < 10 → violates BST property ✗
  NOT a valid BST ✗
```

---

### Key Insight:

Simply checking `left < node < right` at each node is **not enough**. We must ensure every node falls within a valid **range** inherited from its ancestors.
```
  Wrong approach (only local check):     Correct approach (range check):

         10                                      10
        /  \                                    /  \
       5    15          →               [−∞,10) 5   15 (10,+∞]
           /  \                                    /    \
          6   20                          (10,15) 6     20 (15,+∞]
                                          6 < 10 → INVALID ✗
```

---

## Brute Force Approach

### Algorithm:

- Perform an in-order traversal of the binary tree and store all node values in an array.
- In a valid BST, the inorder traversal always produces a **strictly increasing** sequence.
- Check if the resulting array is strictly sorted in ascending order.
- If yes, the tree is a valid BST; otherwise, it is not.
```
  Inorder of valid BST:          Inorder of invalid BST:
       5                                5
      / \                              / \
     3   7                            3   4
    / \                              / \
   2   4                            2   6

  Result: [2, 3, 4, 5, 7]         Result: [2, 3, 6, 5, 4]
  Strictly increasing ✓           Not strictly increasing ✗
```

### C++ Code:
```cpp
#include <bits/stdc++.h>
using namespace std;

// Definition of TreeNode
struct TreeNode {
    int val;
    TreeNode* left;
    TreeNode* right;
    TreeNode(int data) : val(data), left(nullptr), right(nullptr) {}
};

class Solution {
public:
    // Helper to collect inorder traversal
    void inorder(TreeNode* node, vector<int>& values) {
        if (!node) return;
        inorder(node->left, values);
        values.push_back(node->val);
        inorder(node->right, values);
    }

    // Check if tree is valid BST
    bool isValidBST(TreeNode* root) {
        vector<int> values;
        inorder(root, values);

        // Check if inorder is strictly increasing
        for (int i = 1; i < values.size(); i++) {
            if (values[i] <= values[i - 1]) {
                return false;
            }
        }
        return true;
    }
};

// Driver code
int main() {
    TreeNode* root = new TreeNode(5);
    root->left = new TreeNode(3);
    root->right = new TreeNode(7);
    root->left->left = new TreeNode(2);
    root->left->right = new TreeNode(4);

    Solution obj;
    cout << (obj.isValidBST(root) ? "Valid BST" : "Not a valid BST") << endl;

    return 0;
}
```

### Complexity Analysis:

- **Time Complexity:** O(N), where N is the number of nodes. Each node is visited once during inorder traversal.

- **Space Complexity:** O(N), to store all node values in the array.

---

## Optimal Approach

### Algorithm:

- Use a recursive function that passes a valid **range** `[minVal, maxVal]` for each node.
- At the root, the range is `(−∞, +∞)`.
- For every node, check if its value lies strictly within its valid range.
- When moving to the **left child**, update the upper bound to the current node's value.
- When moving to the **right child**, update the lower bound to the current node's value.
- If any node violates its range, return `false`.
```
  Validate [10, 5, 15, 6, 20]:

         10  range(-∞, +∞)    → 10 in range ✓
        /  \
       5    15
  (-∞,10)  (10,+∞)           → 5 ✓, 15 ✓
           /  \
          6   20
       (10,15) (15,+∞)        → 6 NOT in (10,15) ✗ → return false
```

### C++ Code:
```cpp
#include <bits/stdc++.h>
using namespace std;

// Definition of TreeNode
struct TreeNode {
    int val;
    TreeNode* left;
    TreeNode* right;
    TreeNode(int data) : val(data), left(nullptr), right(nullptr) {}
};

class Solution {
public:
    // Helper function with valid range for each node
    bool validate(TreeNode* node, long minVal, long maxVal) {
        // Empty node is valid
        if (node == nullptr) return true;

        // Check if current node violates the range
        if (node->val <= minVal || node->val >= maxVal) {
            return false;
        }

        // Validate left subtree (upper bound = current val)
        // Validate right subtree (lower bound = current val)
        return validate(node->left, minVal, node->val) &&
               validate(node->right, node->val, maxVal);
    }

    // Main function to check if BST is valid
    bool isValidBST(TreeNode* root) {
        return validate(root, LONG_MIN, LONG_MAX);
    }
};

// Driver code
int main() {
    // Valid BST
    TreeNode* root1 = new TreeNode(5);
    root1->left = new TreeNode(3);
    root1->right = new TreeNode(7);
    root1->left->left = new TreeNode(2);
    root1->left->right = new TreeNode(4);

    // Invalid BST
    TreeNode* root2 = new TreeNode(5);
    root2->left = new TreeNode(1);
    root2->right = new TreeNode(4);
    root2->right->left = new TreeNode(3);
    root2->right->right = new TreeNode(6);

    Solution obj;
    cout << "Tree 1: " << (obj.isValidBST(root1) ? "Valid BST" : "Not a valid BST") << endl;
    cout << "Tree 2: " << (obj.isValidBST(root2) ? "Valid BST" : "Not a valid BST") << endl;

    return 0;
}
```

---

### Edge Cases:
```
  Single node:        All left skewed:       Duplicate values:
       5                    5                      5
                           /                      / \
  Valid BST ✓             3                      5   5
                         /
                        1                   5 <= 5 → NOT valid ✗
                    Valid BST ✓
```

---

### Complexity Analysis:

- **Time Complexity:** O(N), where N is the number of nodes in the binary tree. Each node is visited exactly once during the recursive traversal.

- **Space Complexity:** O(H), where H is the height of the binary tree due to the recursive call stack.
```
  Balanced BST             Skewed BST
       5                       1
      / \                       \
     3   7                       3
    / \   \                       \
   2   4   8                       5
                                    \
  Space ~ O(log N)                   7
                                  Space ~ O(N)
```
