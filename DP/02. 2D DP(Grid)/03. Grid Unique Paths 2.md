## Grid Unique Paths 2 

**Problem Statement:**  
Given an `m x n` 2D array named `matrix`, where each cell contains either `0` or `1`.

- `0` represents a free cell  
- `1` represents a blocked cell  

Return the number of **unique ways** to go from the **top-left cell** (`matrix[0][0]`) to the **bottom-right cell** (`matrix[m-1][n-1]`).

A path **cannot pass through blocked cells**.

**Movement is allowed only in two directions:**
- Right
- Bottom

---

>### Examples:

### Example 1:

**Input:**
```text
matrix = [[0, 0, 0],
          [0, 1, 0],
          [0, 0, 0]]
```
**Output:** 2   
**Explanation:** The two possible paths to reach the bottom-right cell are:
```
down → down → right → right

right → right → down → down
```
Thus, the total number of unique paths is 2.

---

### Example 2
**Input:**  
```text
matrix = [[0, 0, 0],
          [0, 0, 1],
          [0, 1, 0]]
 ```
**Output:**  0
**Explanation:**
There is no valid path to reach the bottom-right cell because all possible routes are blocked by cells with value 1.

---

## Approach - Recursive
### Intuition:

- You start from the **top-left cell (0,0)** and want to reach the **bottom-right cell (m-1, n-1)**.
- You can move only **right** or **down**.
- If a cell contains `1`, it is **blocked**, and no path can go through it.
- From any cell `(i, j)`, the total number of paths is the sum of:
  - Paths from the cell **below** `(i+1, j)`
  - Paths from the cell **to the right** `(i, j+1)`
- Reaching the destination contributes **one valid path**.


### Algorithm:

1. Start recursion from cell `(0,0)`.
2. If the current cell is **out of bounds** or **blocked**, return `0`.
3. If the destination cell `(m-1, n-1)` is reached, return `1`.
4. Recursively compute:
   - Number of paths by moving **down**
   - Number of paths by moving **right**
5. Return the sum of both values.

---

**Key Idea:**  
Blocked cells stop paths, and each valid path is built by moving only right or down.



### C++ Code:
``` cpp
#include <bits/stdc++.h>
using namespace std;

// Recursive function to count unique paths
int countPaths(int row, int col, int totalRows, int totalCols,
               vector<vector<int>>& matrix) {

    // Out of bounds
    if (row >= totalRows || col >= totalCols)
        return 0;

    // Blocked cell
    if (matrix[row][col] == 1)
        return 0;

    // Destination reached
    if (row == totalRows - 1 && col == totalCols - 1)
        return 1;

    // Move down and move right
    int pathsFromDown = countPaths(row + 1, col, totalRows, totalCols, matrix);
    int pathsFromRight = countPaths(row, col + 1, totalRows, totalCols, matrix);

    return pathsFromDown + pathsFromRight;
}

// Wrapper function
int uniquePathsWithObstacles(vector<vector<int>>& matrix) {
    int totalRows = matrix.size();
    int totalCols = matrix[0].size();

    // If start or destination is blocked
    if (matrix[0][0] == 1 || matrix[totalRows - 1][totalCols - 1] == 1)
        return 0;

    return countPaths(0, 0, totalRows, totalCols, matrix);
}

int main() {
    vector<vector<int>> matrix = {
        {0, 0, 0},
        {0, 1, 0},
        {0, 0, 0}
    };

    cout << uniquePathsWithObstacles(matrix); // Output: 2
    return 0;
}
```
### Complexity Analysis:

- **Time Complexity:**  
  O(2^(m+n)), where `m` is the number of rows and `n` is the number of columns.  
  - Reason: From each cell, the recursion explores two possibilities (right and down), leading to an exponential number of calls.

- **Space Complexity:**  
  O(m + n), due to the recursion stack depth in the worst case (moving only right and down).

## Approach - Memoization

### Algorithm:

1. **Declare a DP array**  
   - Create a 2D array `dp[m][n]`, where `m` and `n` are the maximum values that indices `i` and `j` can take.  
   - `dp[i][j]` stores the total number of ways to reach cell `(i, j)` from `(0,0)`.  
   - Initialize all values in `dp` with `-1` to indicate unsolved subproblems.

2. **Use stored results for overlapping subproblems**  
   - Whenever a subproblem `(i, j)` is encountered, check if `dp[i][j] != -1`.  
   - If true, return the stored value directly to avoid recomputation.

3. **Solve unsolved subproblems**  
   - If `dp[i][j] == -1`, compute the number of ways to reach `(i, j)` using recursion.  
   - Store the computed value in `dp[i][j]` for future reference.

4. **Return the final result**  
   - The value stored in `dp[m-1][n-1]` gives the total number of unique paths to the destination.

**Key Benefit:**  
Memoization avoids repeated calculations of overlapping subproblems, reducing the time complexity from exponential to polynomial.

### C++ Code:

``` cpp
#include <bits/stdc++.h>
using namespace std;

class Solution {
private:

    //Function to solve the problem using memoization
    int func(int i, int j, vector<vector<int>>& matrix, vector<vector<int>> &dp){
        // Base cases
        if (i < 0 || j < 0 || matrix[i][j] == 1) return 0;
        else if(i == 0 && j == 0) return 1;
        
        // If the result is already computed, return it
        if(dp[i][j] != -1) return dp[i][j];

        /* Calculate the number of ways by
        moving up and left recursively.*/
        int up = func(i - 1, j, matrix, dp);
        int left = func(i, j - 1, matrix, dp);

        // Return the total ways
        return dp[i][j] = up + left;
    }
    
public:

    /*Function to find all unique paths to reach
    matrix[m-1][n-1] form matrix[0][0] with obstacles*/
    int uniquePathsWithObstacles(vector<vector<int>>& matrix) {
        int m = matrix.size();
        int n = matrix[0].size();
        
        // Initialize DP table to memoize results
        vector<vector<int>> dp(m, vector<int>(n, -1)); 
        
        //Return the total number of paths
        return func(m-1, n-1, matrix, dp);
    }
};

int main() {
    vector<vector<int>> maze{
        {0, 0, 0},
        {0, 1, 0},
        {0, 0, 0}
    };
    
    //Create an instance of the Solution class
    Solution sol;

    cout << "Number of paths with obstacles: " << sol.uniquePathsWithObstacles(maze) << endl;
    return 0;
}
```
### Complexity Analysis:

- **Time Complexity:**  
  O(m * n), where `m` is the number of rows and `n` is the number of columns.  
  - Reason: Each cell in the `m x n` DP array is computed once, and each computation takes constant time.

- **Space Complexity:**  
  O((m - 1) + (n - 1)) + O(m * n)  
  - `O((m - 1) + (n - 1))` → recursion stack space (maximum path length).  
  - `O(m * n)` → space used by the DP array to store results of subproblems.

## Approach - Tabulation

### Algorithm:

1. **Declare a DP array**  
   - Create a 2D array `dp[m][n]`, where `m` and `n` are the dimensions of the matrix.  
   - `dp[i][j]` represents the total number of ways to reach cell `(i, j)` from the starting cell `(0, 0)`.

2. **Handle the base case**  
   - If the starting cell is not blocked, set:
     ```
     dp[0][0] = 1
     ```
   - If `matrix[0][0]` is blocked, no path is possible.

3. **Iterate through the grid**  
   - Use two nested loops:
     - Outer loop for rows `i` from `0` to `m-1`
     - Inner loop for columns `j` from `0` to `n-1`

4. **Handle blocked cells**  
   - If `matrix[i][j] == 1`, set:
     ```
     dp[i][j] = 0
     ```
     This indicates that no path can pass through this cell.

5. **Compute number of paths for free cells**  
   - If the cell is not blocked:
     - Add paths from the top cell if it exists:
       ```
       if i > 0: dp[i][j] += dp[i-1][j]
       ```
     - Add paths from the left cell if it exists:
       ```
       if j > 0: dp[i][j] += dp[i][j-1]
       ```

6. **Return the final result**  
   - After filling the DP table, `dp[m-1][n-1]` contains the total number of unique paths to reach the bottom-right corner.  
   - Return this value as the answer.

### C++ Code:
``` cpp
#include <bits/stdc++.h>
using namespace std;

class Solution {
private:
    // Function to solve the problem using tabulation
    int func(int m, int n, vector<vector<int>>& matrix, vector<vector<int>>& dp) {
        for (int i = 0; i < m; i++) {
            for (int j = 0; j < n; j++) {
                
                // Base conditions
                if (matrix[i][j] == 1) {
                    /* If there's an obstacle at the 
                    cell, no paths can pass through it*/
                    dp[i][j] = 0;
                    continue;
                }
                if (i == 0 && j == 0) {
                    /* If we are at the starting 
                    point, there is one path to it*/
                    dp[i][j] = 1;
                    continue;
                }

                int up = 0;
                int left = 0;

                /* Check if we can move up and left 
                (if not at the edge of the maze)*/
                if (i > 0)
                    up = dp[i - 1][j];
                if (j > 0)
                    left = dp[i][j - 1];

                /* Total number of paths to reach (i, j)
                is the sum of paths from above and left*/
                dp[i][j] = up + left;
            }
        }

        /* The final result is stored in dp[m-1][n-1],
        which represents the destination*/
        return dp[m - 1][n - 1];
    }

public:
    /* Function to find all unique paths to reach 
    matrix[m-1][n-1] from matrix[0][0] with obstacles*/
    int uniquePathsWithObstacles(vector<vector<int>>& matrix) {
        int m = matrix.size();   
        int n = matrix[0].size(); 

        // Initialize DP table to memoize results
        vector<vector<int>> dp(m, vector<int>(n, 0));

        // Return the total number of paths
        return func(m, n, matrix, dp);
    }
};

int main() {
    vector<vector<int>> maze{
        {0, 0, 0},
        {0, 1, 0},
        {0, 0, 0}
    };

    // Create an instance of the Solution class
    Solution sol;

    cout << "Number of paths with obstacles: " << sol.uniquePathsWithObstacles(maze) << endl;
    return 0;
}
```
### Complexity Analysis:

- **Time Complexity:**  
  O(m * n), where `m` is the number of rows and `n` is the number of columns.  
  - Reason: Each cell of the `m x n` DP table is computed exactly once, and each computation takes constant time.

- **Space Complexity:**  
  O(m * n), where `m` is the number of rows and `n` is the number of columns.  
  - Reason: A 2D DP array is used to store intermediate results for all grid cells.


---

##  Approach - Space Optimized

### Algorithm:

1. Observe that in the tabulation approach, the value of `dp[i][j]` depends only on:
   - The previous row: `dp[i-1][j]`
   - The current row: `dp[i][j-1]`  
   Hence, we can optimize space by storing only **two rows** instead of the full `m x n` DP array.

2. Initialize a **dummy array** `prev` with size `n` (number of columns) filled with 0.  
   - This represents the previous row of the DP table.

3. Iterate through each row of the grid:
   - Use a temporary array `temp` to store the values of the **current row**.
   - Compute `temp[j] = prev[j] + temp[j-1]` for each cell `(i,j)` (as in the tabulation formula).

4. After processing the current row, **update `prev = temp`**.  
   - This ensures `prev` always contains the previous row for the next iteration.

5. After all rows are processed, the value `prev[n-1]` contains the **final result**, i.e., the total number of unique paths to reach the bottom-right corner of the grid.

### C++ Code:
``` cpp
#include <bits/stdc++.h>
using namespace std;

class Solution {
private:
    // Function to solve the problem using tabulation
    int func(int m, int n, vector<vector<int>>& matrix) {
        
        vector<int> prev(n, 0), curr(n, 0);
        
        for (int i = 0; i < m; i++) {
            for (int j = 0; j < n; j++) {
                
                // Base conditions
                if (matrix[i][j] == 1) {
                    /* If there's an obstacle at the 
                    cell, no paths can pass through it*/
                    curr[j] = 0;
                    continue;
                }
                if (i == 0 && j == 0) {
                    /* If we are at the starting 
                    point, there is one path to it*/
                    curr[j] = 1;
                    continue;
                }

                int up = 0;
                int left = 0;

                /* Check if we can move up and left 
                (if not at the edge of the maze)*/
                if (i > 0)
                    up = prev[j];
                if (j > 0)
                    left = curr[j - 1];

                /* Total number of paths to reach (i, j)
                is the sum of paths from above and left*/
                curr[j] = up + left;
            }
            
            prev = curr;
        }

        /* The final result is stored in dp[m-1][n-1],
        which represents the destination*/
        return prev[n-1];
    }

public:
    /* Function to find all unique paths to reach 
    matrix[m-1][n-1] from matrix[0][0] with obstacles*/
    int uniquePathsWithObstacles(vector<vector<int>>& matrix) {
        int m = matrix.size();   
        int n = matrix[0].size(); 

        // Return the total number of paths
        return func(m, n, matrix);
    }
};

int main() {
    vector<vector<int>> maze{
        {0, 0, 0},
        {0, 1, 0},
        {0, 0, 0}
    };

    // Create an instance of the Solution class
    Solution sol;

    cout << "Number of paths with obstacles: " << sol.uniquePathsWithObstacles(maze) << endl;
    return 0;
}
```
### Complexity Analysis:

- **Time Complexity:** O(m * n)  
  We iterate through each cell of the grid once, so the total time is proportional to the number of cells.

- **Space Complexity:** O(n)  
  Only two 1D arrays of size `n` (number of columns) are used to store intermediate results, instead of a full `m x n` DP array.
