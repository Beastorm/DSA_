## Dijkstra's Algorithm — Using Priority Queue

### Problem Statement:

Given a **weighted, undirected, connected graph** with `V` vertices and `E` edges, find the **shortest distance from source `S` to all other vertices**. Graph has no negative weight cycles.

---

## Examples

**Example 1:**
- **Input:** `V=2, E=1`, edge: `0─9─1`, src=0
- **Output:** `[0, 9]`
```
  0 ──9── 1

  Dist[0]=0 (source), Dist[1]=9 ✓
```

**Example 2:**
- **Input:** `V=3, E=3`, edges: `0─6─2, 0─3─1, 1─1─2`, src=2
- **Output:** `[4, 3, 0]`
```
       6
  0 ──────── 2 (src)
  |         /
  3       1
  |      /
  1 ────

  0→2: direct=6, via 1: 1+3=4 → Dist[0]=4 ✓
  1→2: dist=1+? wait...
  Path 2→1→0: 1+3=4
  Dist: [4, 1, 0]? Let me recheck
  2→1=1, 1→0=3: Dist[0]=4, Dist[1]=1, Dist[2]=0 → [4,1,0]
```

---

## Key Insight
```
  Problem: Find shortest path in WEIGHTED graph.
  BFS works for UNIT weights only.
  For weighted: use DIJKSTRA with MIN-HEAP!

  Core idea:
  Always process the node with the SMALLEST current distance.
  → Greedy approach: once a node is processed, its distance is FINAL.

  Min-heap (priority queue) efficiently gives us the
  minimum distance node in O(log V) time.

  Relaxation:
  If dist[u] + weight(u→v) < dist[v]:
      Update dist[v] ← better path found!
      Push (dist[v], v) to heap
```

---

## Algorithm
```
  dijkstra(V, adj, src):

  1. dist[] = all infinity
     dist[src] = 0

  2. Min-heap pq = {(0, src)}

  3. While pq not empty:
       (d, node) = pq.top(); pop

       If d > dist[node]: SKIP  ← outdated entry
       (node already processed with better dist)

       For each (neighbor, wt) in adj[node]:
           If dist[node] + wt < dist[neighbor]:
               dist[neighbor] = dist[node] + wt
               pq.push({dist[neighbor], neighbor})

  4. Return dist[]
```
```
  Why skip outdated entries?
  We might push (5, node_A) then find shorter (3, node_A).
  Both are in heap. When we pop (3, node_A): process it.
  Later pop (5, node_A): 5 > dist[A]=3 → SKIP! ✓
  This "lazy deletion" avoids complex heap updates.
```

## Code
```cpp
#include <bits/stdc++.h>
using namespace std;

class Solution {
public:
    vector<int> dijkstra(int V, vector<vector<pair<int,int>>>& adj, int src) {

        // Step 1: Initialize distances
        vector<int> dist(V, 1e9);
        dist[src] = 0;

        // Min-heap: {distance, node}
        priority_queue<pair<int,int>,
                       vector<pair<int,int>>,
                       greater<pair<int,int>>> pq;

        // Step 2: Push source
        pq.push({0, src});

        // Step 3: Process until heap empty
        while (!pq.empty()) {
            int d    = pq.top().first;
            int node = pq.top().second;
            pq.pop();

            // Skip outdated (stale) entries
            if (d > dist[node]) continue;

            // Relax all edges from this node
            for (auto [neighbor, wt] : adj[node]) {
                if (dist[node] + wt < dist[neighbor]) {
                    dist[neighbor] = dist[node] + wt;
                    pq.push({dist[neighbor], neighbor});
                }
            }
        }

        return dist;
    }
};

int main() {
    int V = 5;
    vector<vector<pair<int,int>>> adj(V);

    // Add undirected weighted edges {neighbor, weight}
    adj[0].push_back({1, 2}); adj[1].push_back({0, 2});
    adj[0].push_back({2, 4}); adj[2].push_back({0, 4});
    adj[1].push_back({2, 1}); adj[2].push_back({1, 1});
    adj[1].push_back({3, 7}); adj[3].push_back({1, 7});
    adj[2].push_back({4, 3}); adj[4].push_back({2, 3});
    adj[3].push_back({4, 2}); adj[4].push_back({3, 2});

    Solution obj;
    vector<int> distances = obj.dijkstra(V, adj, 0);

    cout << "Shortest distances from node 0:" << endl;
    for (int i = 0; i < V; i++) {
        cout << "To " << i << ": " << distances[i] << endl;
    }

    return 0;
}
```

## Why Not BFS for Weighted Graphs?
```
  BFS (unit weight):               Dijkstra (weighted):
  0 ─1─ 1 ─1─ 2                   0 ─1─ 1 ─100─ 2
                                   └──────────2──────┘

  BFS: dist[2]=2 (via 1,2)         BFS: dist[2]=2 (via 1,2) ✗
  Correct ✓                        Should be: dist[2]=2 via direct!

  Wait: 0─2─2 has dist=2
  0─1─1─100─2 has dist=101
  BFS would say dist=2 (2 hops) ✗ (wrong!)
  Dijkstra: dist[2]=2 (direct) ✓

  BFS counts HOPS, not WEIGHTS → fails for weighted graphs
  Dijkstra respects WEIGHTS → correct! ✓
```

---

## Edge Cases
```
  Source is destination:       Disconnected node:
  dist[src] = 0 ✓              Unreachable node stays ∞
                               → replace with -1 if needed

  Multiple paths:              V=1, no edges:
  0─5─2─3─3                   dist=[0] ✓
  0─────7─3

  5+3=8 vs 7 → dist[3]=7 ✓

  Negative weights:            → Use Bellman-Ford instead!
  Dijkstra doesn't work ✗      Dijkstra assumes dist only
                               increases → fails for neg wt
```

## Complexity Analysis

**Time Complexity:** O((V + E) log V)
- Each vertex extracted from heap once: O(V log V)
- Each edge relaxation potentially pushes to heap: O(E log V)
- Total: **O((V + E) log V)**

**Space Complexity:** O(V + E)
- Adjacency list: O(V + E)
- Distance array: O(V)
- Priority queue: O(V + E) worst case (stale entries)
- Total: **O(V + E)**
```
  Comparison:
  Algorithm        Graph Type    Time Complexity
  ──────────────────────────────────────────────
  BFS              Unit weight   O(V + E)
  Topo Sort        DAG           O(V + E)
  Dijkstra         No neg wts    O((V+E) log V)
  Bellman-Ford     Any           O(VE)

  Dijkstra is optimal for non-negative weighted graphs ✓
```
