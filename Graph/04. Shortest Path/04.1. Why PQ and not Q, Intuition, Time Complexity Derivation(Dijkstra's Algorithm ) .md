## 1. Why Priority Queue and NOT Normal Queue?

### ❌ Problem with Regular Queue (FIFO)

A normal queue explores nodes in insertion order — not by distance.

**Example scenario:**

```
Node 3 reached with distance 3  →  enqueue (dist=3, node=3)
Node 3 reached again with distance 7  →  enqueue (dist=7, node=3)
```

Since FIFO doesn't care about distance, both entries are processed in order.
The **worse path (dist=7)** wastes time exploring unnecessary edges.

This leads to:
- Many **redundant entries** in the queue
- **Unnecessary path exploration**
- **Much higher** time complexity

### ✅ Why Priority Queue (Min-Heap) Works

A min-heap always gives you the **node with the smallest distance first** — a greedy choice.

**Key insight:**
> Once a node is popped from the PQ with distance `d`, that `d` **is already optimal**.
> Any later entry for the same node with a larger distance can be **safely ignored**.

**Benefits:**
- Minimises redundant work
- Marks optimal distances **early**
- Significantly reduces total iterations

### 🔑 Key Takeaway

| Approach | Strategy | Efficiency |
|----------|----------|------------|
| Normal Queue | Brute-force (FIFO) | ❌ Wasteful |
| Priority Queue | Greedy (min-dist first) | ✅ Optimal |

## 2. Intuition Behind Dijkstra with PQ

Dijkstra is a **greedy algorithm**.

**Core principle:**
> *"Always process the node with the smallest known distance first."*

Because of this ordering:
- The **first time** a node is popped from the PQ → its distance is **guaranteed optimal**.
- Later pops of the same node (with larger distances) → **skip them**.

```
Pop (dist=3, node=3) → process, relax neighbours ✅
Pop (dist=7, node=3) → dist[3] is already 3, so 7 > 3 → SKIP ❌
```

### PQ vs Set

| Data Structure | Behaviour | Note |
|----------------|-----------|------|
| `priority_queue` | Lazy deletion — stale entries stay but are skipped | Simpler to implement ✅ |
| `set<pair<int,int>>` | Active deletion — erase outdated entries | Better space, slightly complex |

> **Greedy ordering + non-negative edge weights** = Dijkstra is **always correct**.

## 3. Time Complexity Derivation

### Code Structure

```cpp
while (!pq.empty()) {
    auto [dist, node] = pq.top(); pq.pop();          // Extract-Min → O(log H)

    for (auto [adjNode, wt] : adj[node]) {
        if (dist + wt < distance[adjNode]) {
            distance[adjNode] = dist + wt;
            pq.push({distance[adjNode], adjNode});    // Insert → O(log H)
        }
    }
}
```

### Step-by-Step Breakdown

| Operation | Count | Cost Each | Total |
|-----------|-------|-----------|-------|
| `pq.pop()` (Extract-Min) | At most **E** times (each edge can push once) | `O(log H)` | `O(E log H)` |
| `pq.push()` (Insert) | At most **E** times | `O(log H)` | `O(E log H)` |
| Adjacency list traversal | All edges = **E** | `O(1)` | `O(E)` |

> **H = Heap size**. In the worst case (every edge causes a push), H ≈ E.

### Simplifying `log H`

```
H ≤ E  (at most one entry per edge)
log(E) = log(V²) = 2 · log(V)   [for dense graph where E ≈ V²]
→  log(E) = O(log V)
```

### Final Formula

```
Time = O(E log H)
     = O(E log E)
     = O(E log V)      ← since log E = O(log V)
```

Combined with the `V` pop operations:

```
Time = O((V + E) log V)
```

Since `E ≥ V` in a connected graph, the `E log V` term dominates:

$$\boxed{O(E \log V)}$$

### 📌 Important Notes

- This applies to the **binary heap** implementation (standard PQ).
- Multiple entries for the same node in PQ are **allowed** — stale ones are skipped.
- With a **Fibonacci heap** (decrease-key), complexity improves to `O(E + V log V)` — but rarely used in practice.

## 4. Summary Table

| Aspect | Normal Queue (Q) | Priority Queue (PQ) |
|--------|-----------------|---------------------|
| Ordering | FIFO (insertion order) | Min-distance first |
| Strategy | Brute-force | Greedy |
| Redundant work | Very high | Minimal |
| Time complexity | Much worse | **O(E log V)** |
| Correctness | ✅ (but inefficient) | ✅ (and efficient) |
| Implementation | Simple | Simple (with lazy skip) |

---

> 💡 **One-line summary:** PQ gives Dijkstra its greedy power — always pick the cheapest node, mark it optimal, skip everything worse.
