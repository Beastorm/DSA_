## Number of Ways to Arrive at Destination

### Problem Statement:

You are in a city with `n` intersections numbered from `0` to `n-1` with **bi-directional roads**. The graph is connected and has at most one road between any two intersections.

Given an integer `n` and a 2D array `roads` where `roads[i] = [ui, vi, timei]` means there's a road between intersections `ui` and `vi` that takes `timei` minutes.

**Find:** In how many ways can you travel from intersection `0` to intersection `n-1` in the **shortest amount of time**.

Return the answer modulo `10^9 + 7`.

---

## Key Insight

```
This problem combines TWO challenges:

1. Find shortest distance (Dijkstra's)
2. Count number of shortest paths (DP on DAG)

Standard Dijkstra:
→ Finds shortest distance to all nodes ✓
→ But doesn't track HOW MANY paths! ✗

Modified Dijkstra with Counting:
→ Maintains dist[v] = shortest distance to v
→ ALSO maintains ways[v] = number of shortest paths to v
→ Updates both simultaneously! ✓

Key insight:
When relaxing edge u → v:
1. If dis + wt < dist[v]:
   → Found NEW shorter path
   → ways[v] = ways[u] (inherit from u)
   
2. If dis + wt == dist[v]:
   → Found ANOTHER path with same length
   → ways[v] += ways[u] (add more ways)

This counts ALL shortest paths! ✓
```

---

## Algorithm

```
1. Build adjacency list from edges (bi-directional):
   adj[u] = {(v, time), ...}
   adj[v] = {(u, time), ...}

2. Initialize:
   - Priority Queue (min-heap): {dist, node}
   - dist[0...n-1] = infinity
   - ways[0...n-1] = 0
   - dist[src] = 0
   - ways[src] = 1 (one way to reach source)

3. Push source: {0, src} into PQ

4. Modified Dijkstra:
   While PQ is not empty:
     a. Pop {dis, node} with minimum distance
     
     b. For each neighbor (adjNode, edW):
        newDist = dis + edW
        
        Case 1: newDist < dist[adjNode]
          → Found SHORTER path
          → dist[adjNode] = newDist
          → ways[adjNode] = ways[node] (inherit)
          → Push {newDist, adjNode} into PQ
        
        Case 2: newDist == dist[adjNode]
          → Found ANOTHER shortest path
          → ways[adjNode] += ways[node] (mod 10^9+7)
          → NO push (already in PQ with same dist)
        
        Case 3: newDist > dist[adjNode]
          → Longer path, ignore

5. Return ways[n-1] % MOD
```

```
Why does counting work?

Mathematical principle:
If node u has k shortest paths from source,
and edge u → v is part of a shortest path to v,
then v inherits those k paths!

Example:
Source → u has 3 shortest paths
u → v is on shortest path to v
→ v also has (at least) 3 shortest paths ✓

If multiple nodes lead to v with same shortest distance:
Source → u1 (2 paths) → v
Source → u2 (3 paths) → v
→ v has 2 + 3 = 5 shortest paths! ✓

This is dynamic programming on the shortest path DAG!
```

## Code

```cpp
#include <bits/stdc++.h>
using namespace std;

class Solution {
public:
    int countPaths(int n, vector<vector<int>> &roads) {
        
        // Build adjacency list (bi-directional)
        vector<pair<int, int>> adj[n];
        for (auto it : roads) {
            adj[it[0]].push_back({it[1], it[2]});
            adj[it[1]].push_back({it[0], it[2]});
        }

        // Min-heap: {distance, node}
        priority_queue<pair<long long, int>,
                       vector<pair<long long, int>>,
                       greater<pair<long long, int>>> pq;

        // Distance and ways arrays
        vector<long long> dist(n, LLONG_MAX);
        vector<int> ways(n, 0);

        // Initialize source
        dist[0] = 0;
        ways[0] = 1;
        pq.push({0, 0});

        int mod = (int)(1e9 + 7);

        // Modified Dijkstra
        while (!pq.empty()) {
            long long dis = pq.top().first;
            int node = pq.top().second;
            pq.pop();

            // Skip outdated entries
            if (dis > dist[node]) continue;

            // Explore neighbors
            for (auto it : adj[node]) {
                int adjNode = it.first;
                long long edW = it.second;

                // Case 1: Found shorter path
                if (dis + edW < dist[adjNode]) {
                    dist[adjNode] = dis + edW;
                    pq.push({dis + edW, adjNode});
                    ways[adjNode] = ways[node];
                }
                // Case 2: Found another shortest path
                else if (dis + edW == dist[adjNode]) {
                    ways[adjNode] = (ways[adjNode] + ways[node]) % mod;
                }
            }
        }

        return ways[n - 1] % mod;
    }
};

int main() {
    int n = 7;
    vector<vector<int>> edges = {
        {0, 6, 7}, {0, 1, 2}, {1, 2, 3}, {1, 3, 3}, {6, 3, 3},
        {3, 5, 1}, {6, 5, 1}, {2, 5, 1}, {0, 4, 5}, {4, 6, 2}
    };

    Solution obj;
    int ans = obj.countPaths(n, edges);

    cout << "Number of shortest paths: " << ans << endl;

    return 0;
}
```

## Why Modulo 10^9 + 7?

```
Number of paths can grow exponentially!

Example: Grid graph
n × n grid, shortest path from (0,0) to (n,n)
Number of shortest paths = C(2n, n) ← Binomial coefficient

For n=50:
C(100, 50) ≈ 10^29 → Exceeds long long! ✗

Solution: Take modulo at each step
ways[adjNode] = (ways[adjNode] + ways[node]) % mod

Properties of modulo:
(a + b) % m = ((a % m) + (b % m)) % m
→ Safe to apply modulo incrementally ✓

Final answer:
return ways[n-1] % mod;

Even though we've been applying modulo throughout,
applying it one more time ensures correctness ✓
```

## Edge Cases

```
Single node:                Direct edge to destination:
n = 1                       0 ──5──→ n-1
ways[0] = 1                 Only 1 path
Return: 1 ✓                 ways[n-1] = 1 ✓

Multiple edges same length: Disconnected graph:
0 ──2──→ 1                  0 ──→ 1    2 ──→ 3
|        |                  (destination unreachable)
└──2──→ ─┘                  dist[n-1] = ∞
Both paths same length      ways[n-1] = 0
ways[1] = 2 ✓               Return: 0 ✓

All paths same length:      Very long path:
Diamond graph:              Chain: 0→1→2→...→n-1
  0                         Each edge weight = 1
 / \                        Only 1 path
1   2                       ways[n-1] = 1 ✓
 \ /
  3
ways[3] = 2 ✓

Large answer:               Zero-weight edges:
Exponential paths           Allowed (non-negative)
Need modulo ✓               Doesn't break Dijkstra ✓
```

## Why `long long` for Distance?

```
Code uses:
vector<long long> dist(n, LLONG_MAX);
pair<long long, int> in priority queue

Why long long instead of int?

Edge weights can be large (up to 10^5)
Number of edges can be large (up to 10^5)
Maximum distance = sum of all edges

Worst case:
100,000 edges × 100,000 weight = 10^10
→ Exceeds INT_MAX (≈ 2 × 10^9) ✗
→ Requires long long ✓

But ways[] can be int:
ways is always taken modulo 10^9+7
→ Never exceeds 10^9+7
→ Fits in int ✓

Type safety:
long long dis = pq.top().first;
Ensures no overflow in distance calculations ✓
```

## Difference from Standard Dijkstra

```
Standard Dijkstra:

while (!pq.empty()) {
    auto [dis, node] = pq.top();
    pq.pop();
    
    for (auto [adjNode, edW] : adj[node]) {
        if (dis + edW < dist[adjNode]) {
            dist[adjNode] = dis + edW;
            pq.push({dist[adjNode], adjNode});
        }
    }
}

Modified Dijkstra (counting paths):

while (!pq.empty()) {
    auto [dis, node] = pq.top();
    pq.pop();
    
    for (auto [adjNode, edW] : adj[node]) {
        // Shorter path found
        if (dis + edW < dist[adjNode]) {
            dist[adjNode] = dis + edW;
            pq.push({dist[adjNode], adjNode});
            ways[adjNode] = ways[node];  ← NEW!
        }
        // Same distance, another path
        else if (dis + edW == dist[adjNode]) {  ← NEW!
            ways[adjNode] = (ways[adjNode] + ways[node]) % mod;
        }
    }
}

Key additions:
1. ways[] array to count paths
2. Update ways[] when relaxing
3. Add to ways[] when same distance found
```

## Complexity Analysis

**Time Complexity:** `O(E × log V)`
- Building adjacency list: O(E)
- Modified Dijkstra:
  - Each vertex processed at most once: O(V)
  - Each edge relaxed at most once: O(E)
  - Each PQ operation: O(log V)
- Total: **O(E × log V)**

**Space Complexity:** `O(V + E)`
- Adjacency list: O(E)
- Distance array: O(V)
- Ways array: O(V)
- Priority queue: O(V) at most
- Total: **O(V + E)**

```
Where V = n (number of nodes)
      E = |roads| (number of edges)

Compared to standard Dijkstra:
→ Same time complexity ✓
→ Only adds O(V) for ways array
→ Minimal overhead for counting!

The elegance:
We count paths "for free" during Dijkstra
without any asymptotic slowdown! ✓
```

## Common Mistakes

```
❌ Mistake 1: Not using modulo
ways[adjNode] += ways[node];  // Can overflow!

✓ Correct:
ways[adjNode] = (ways[adjNode] + ways[node]) % mod;

❌ Mistake 2: Using int for distance
vector<int> dist(n, INT_MAX);  // Can overflow!

✓ Correct:
vector<long long> dist(n, LLONG_MAX);

❌ Mistake 3: Forgetting initialization
ways[src] = 0;  // WRONG! Should be 1

✓ Correct:
ways[src] = 1;  // One way to reach source

❌ Mistake 4: Updating ways on longer path
if (dis + edW <= dist[adjNode]) {  // WRONG!
    ways[adjNode] += ways[node];
}

✓ Correct:
Only update on EQUAL or SHORTER path

❌ Mistake 5: Not handling bi-directional edges
adj[u].push_back({v, w});  // Missing reverse!

✓ Correct:
adj[u].push_back({v, w});
adj[v].push_back({u, w});  // Bi-directional!
```

## When to Use This Pattern

```
✓ Use "Dijkstra + Path Counting" when:
  → Need to count NUMBER of shortest paths
  → Non-negative edge weights
  → Single source, single/multiple destinations
  → "How many ways to achieve optimal" type questions

✗ Don't use when:
  → Only need shortest distance → Standard Dijkstra
  → Negative weights → Bellman-Ford
  → All pairs shortest paths → Floyd-Warshall
  → Just need ANY shortest path → Standard Dijkstra

Similar problems:
→ Number of Ways to Arrive at Destination (this problem) ✓
→ Number of Shortest Paths in Grid
→ Count Unique Paths with Maximum Score
→ All Paths from Source Lead to Destination

Key pattern:
"Count ways to achieve shortest/optimal" = Dijkstra + DP! ✓
```
