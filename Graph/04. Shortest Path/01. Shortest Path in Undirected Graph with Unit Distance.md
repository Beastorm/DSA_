## Shortest Path in Undirected Graph with Unit Distance

### Problem Statement:

Given an undirected graph with unit weights, find the **shortest path from the source node to all other nodes**. If a node is unreachable, return `-1` for that node. Source vertex is `0`.

---

## Examples

**Example 1:**
- **Input:** `n=9, m=10`, edges=`[[0,1],[0,3],[3,4],[4,5],[5,6],[1,2],[2,6],[6,7],[7,8],[6,8]]`, src=0
- **Output:** `0 1 2 1 2 3 3 4 4`
```
  Graph:
  0 ─── 1 ─── 2
  |           |
  3 ─── 4 ─── 5 ─── 6 ─── 7
                     |     |
                     └─────8

  Dist: 0=0, 1=1, 2=2, 3=1, 4=2, 5=3, 6=3, 7=4, 8=4 ✓
```

**Example 2:**
- **Input:** `n=8`, src=0
- **Output:** `0 1 2 1 2 3 3 2`

---

## Key Insight
```
  All edges have UNIT WEIGHT = 1.

  Why BFS works perfectly here?
  BFS explores nodes LEVEL BY LEVEL:
  Level 0: source (dist=0)
  Level 1: all nodes 1 hop away (dist=1)
  Level 2: all nodes 2 hops away (dist=2)
  ...

  First time BFS reaches a node = SHORTEST path!
  (No need for Dijkstra's min-heap with unit weights)

  Weighted graph (different weights) → Dijkstra
  Unit weight graph                  → BFS ✓
```

---

## Algorithm
```
  1. Build adjacency list from edges (undirected)

  2. Initialize dist[] = infinity for all nodes
     dist[src] = 0

  3. BFS from src:
     Push src to queue
     While queue not empty:
         node = queue.front(); pop
         For each neighbor of node:
             If dist[node] + 1 < dist[neighbor]:
                 dist[neighbor] = dist[node] + 1
                 push neighbor to queue

  4. Build result:
     If dist[i] != infinity → ans[i] = dist[i]
     Else → ans[i] = -1 (unreachable)

  5. Return ans
```
```
  Why check dist[node]+1 < dist[neighbor]?
  Ensures we only update if shorter path found.
  For unit weight undirected graph + BFS:
  Each node is processed only ONCE in practice
  (first visit = shortest path for unit weights)
```

## Code
```cpp
#include <bits/stdc++.h>
using namespace std;

class Solution {
public:
    vector<int> shortestPath(vector<vector<int>>& edges, int N, int M, int src) {

        // Step 1: Build undirected adjacency list
        vector<int> adj[N];
        for (auto& edge : edges) {
            adj[edge[0]].push_back(edge[1]);
            adj[edge[1]].push_back(edge[0]); // Undirected
        }

        // Step 2: Initialize distances to infinity
        int dist[N];
        for (int i = 0; i < N; i++) dist[i] = 1e9;
        dist[src] = 0;

        // Step 3: BFS from source
        queue<int> q;
        q.push(src);

        while (!q.empty()) {
            int node = q.front();
            q.pop();

            for (int neighbor : adj[node]) {
                // Found shorter path to neighbor
                if (dist[node] + 1 < dist[neighbor]) {
                    dist[neighbor] = dist[node] + 1;
                    q.push(neighbor);
                }
            }
        }

        // Step 4: Build result (-1 for unreachable)
        vector<int> ans(N, -1);
        for (int i = 0; i < N; i++) {
            if (dist[i] != 1e9) {
                ans[i] = dist[i];
            }
        }

        return ans;
    }
};

int main() {
    int N = 9, M = 10;
    vector<vector<int>> edges = {
        {0,1},{0,3},{3,4},{4,5},
        {5,6},{1,2},{2,6},{6,7},
        {7,8},{6,8}
    };

    Solution obj;
    vector<int> ans = obj.shortestPath(edges, N, M, 0);

    cout << "Shortest distances from node 0: ";
    for (int x : ans) cout << x << " ";
    cout << endl;

    return 0;
}
```

## Why Not Dijkstra Here?
```
  Dijkstra uses MIN-HEAP to always process the shortest node.
  For UNIT WEIGHTS, BFS's QUEUE acts like a perfect min-heap:
  → All nodes at distance d are processed before distance d+1
  → No need for sorting/heap!

  Unit weight: BFS = O(N + M)
  Non-unit:    Dijkstra = O((N + M) log N)

  BFS is FASTER for unit weight graphs! ✓
```

---

## Handling Unreachable Nodes
```
  Disconnected graph:
  0 ─── 1 ─── 2      3 ─── 4  (separate component)

  BFS from 0 never reaches 3,4.
  dist[3]=∞, dist[4]=∞

  Final result:
  ans[3] = -1  (stays -1, unreachable)
  ans[4] = -1  (stays -1, unreachable)

  Code: if (dist[i] != 1e9) ans[i] = dist[i]; ✓
```

---

## Edge Cases
```
  Source has no edges:         All nodes connected:
  src=0, adj[0]=[]             dist[] all updated ≠ ∞
  dist[0]=0, rest=-1           No -1 in result ✓

  Single node:                 Linear chain 0─1─2─3─4:
  N=1, no edges                dist=[0,1,2,3,4] ✓
  ans=[0] ✓
```

## Complexity Analysis

**Time Complexity:** O(N + M)
- Building adjacency list: O(M)
- BFS traversal: O(N + 2M) ← each node once, each edge twice (undirected)
- Building result: O(N)
- Total: **O(N + M)**

**Space Complexity:** O(N + M)
- Adjacency list: O(N + 2M)
- Distance array: O(N)
- BFS queue: O(N)
- Result array: O(N)
- Total: **O(N + M)**
```
  N = vertices, M = edges

  BFS = optimal for unit weight shortest path
  No priority queue needed! ✓
  Simpler and faster than Dijkstra for this case ✓
```
