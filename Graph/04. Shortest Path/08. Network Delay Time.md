## Network Delay Time

### Problem Statement:

You are given a **directed weighted graph** representing a communication network with `n` nodes, numbered from `1` to `n`. The graph is provided as an edge list `times`, where each record is of the form `(ui, vi, wi)`:
- `ui` is the source node of the directed edge
- `vi` is the target node of the directed edge  
- `wi` is the time it takes for a signal to travel from `ui` to `vi`

A signal is injected at node `k` at time `0`. The signal propagates along directed edges, and each node immediately retransmits to all outgoing neighbors.

**Return** the minimum time required for **every node** to receive the signal. If some node is unreachable, return `-1`.

---

## Key Insight

```
What does "network delay time" mean?

Signal broadcast from source k:
→ Travels to ALL reachable nodes simultaneously
→ Each path takes its own time
→ "Network delay" = time when LAST node receives signal
→ = MAXIMUM of shortest times to each node

This is a classic shortest path problem!
→ Find shortest time from k to ALL nodes
→ Return the MAXIMUM of those times

If any node unreachable:
→ Its "shortest time" = infinity
→ Return -1

Algorithm: Dijkstra's Algorithm
→ Finds shortest path from source to ALL nodes
→ Perfect for this problem! ✓
```

---

## Algorithm

```
1. Build adjacency list from edge list:
   adj[u] = {(v, weight), ...}

2. Initialize:
   - Priority Queue (min-heap): {time, node}
   - Distance array: dist[1...n] = infinity
   - dist[k] = 0 (source)

3. Push source into PQ: {0, k}

4. Dijkstra traversal:
   While PQ is not empty:
     a. Pop {time, node} with minimum time
     
     b. For each neighbor (nbr, wt) of node:
        newTime = time + wt
        
        If newTime < dist[nbr]:
          → Update: dist[nbr] = newTime
          → Push: {newTime, nbr} into PQ

5. After Dijkstra:
   Find maximum value in dist[1...n]
   
   If max == infinity: return -1 (unreachable node)
   Else: return max (time for last node to receive signal)
```

```
Why Dijkstra works perfectly here:

Dijkstra guarantees:
→ After completion, dist[v] = shortest path from source to v

Signal propagation:
→ Signal reaches each node via shortest path
→ dist[v] = time when node v receives signal

Network delay:
→ Time for ALL nodes to receive signal
→ = max(dist[1], dist[2], ..., dist[n])
→ Because last node determines total delay! ✓

Edge weights are non-negative (times ≥ 0)
→ Dijkstra is optimal! ✓
```

## Code

```cpp
#include <bits/stdc++.h>
using namespace std;

class Solution {
public:
    int networkDelayTime(vector<vector<int>>& times, int n, int k) {
        // Build adjacency list (1-indexed)
        vector<vector<pair<int, int>>> adj(n + 1);
        for (auto& time : times) {
            int u = time[0], v = time[1], w = time[2];
            adj[u].push_back({v, w});
        }

        // Min-heap: {time, node}
        priority_queue<pair<int, int>, 
                       vector<pair<int, int>>, 
                       greater<>> pq;
        pq.push({0, k});

        // Distance array (1-indexed)
        vector<int> dist(n + 1, INT_MAX);
        dist[k] = 0;

        // Dijkstra's algorithm
        while (!pq.empty()) {
            auto [time, node] = pq.top();
            pq.pop();

            // Skip if already processed with better time
            if (time > dist[node]) continue;

            // Explore neighbors
            for (auto& [nbr, wt] : adj[node]) {
                if (dist[nbr] > time + wt) {
                    dist[nbr] = time + wt;
                    pq.push({dist[nbr], nbr});
                }
            }
        }

        // Find maximum time (excluding index 0)
        int ans = *max_element(dist.begin() + 1, dist.end());
        
        return (ans == INT_MAX) ? -1 : ans;
    }
};

int main() {
    Solution sol;
    vector<vector<int>> times = {{2,1,1}, {2,3,1}, {3,4,1}};
    int n = 4, k = 2;
    
    cout << "Network delay time: " << sol.networkDelayTime(times, n, k) << endl;
    
    return 0;
}
```

## Why Maximum of Shortest Paths?

```
Signal broadcast analogy:

Radio station at node k broadcasts signal.
Signal travels ALL paths SIMULTANEOUSLY.
Each path has its own travel time.

Question: When does EVERYONE hear the signal?
Answer: When the LAST person hears it!

Mathematical formulation:
Let dist[v] = shortest time for signal to reach v

All nodes receive signal when:
→ Node 1 receives at time dist[1]
→ Node 2 receives at time dist[2]
→ ...
→ Node n receives at time dist[n]

"Everyone received" = max(dist[1], ..., dist[n])

Example:
dist = [1, 0, 1, 2]
       ↑        ↑
    reached   reached
    at t=1    at t=2

Node 4 is LAST to receive → delays entire network
Network delay = 2 ✓
```

## Handling Unreachable Nodes

```
Example:
times = [[1,2,1]], n = 3, k = 1

Graph:
1 ──1──→ 2    3 (isolated)

After Dijkstra:
dist = [_, ∞, 0, 1, ∞]
              1  2  3
              ↑     ↑
           source  unreachable!

max(dist[1], dist[2], dist[3]) = max(0, 1, ∞) = ∞

Since max == ∞:
→ At least one node is unreachable
→ Signal cannot reach all nodes
→ Return -1 ✗

Code handles this:
int ans = *max_element(dist.begin() + 1, dist.end());
return (ans == INT_MAX) ? -1 : ans;
```

## Edge Cases

```
Single node network:          Source is isolated:
n = 1, k = 1                  No outgoing edges from k
times = []                    Other nodes unreachable
Output: 0                     Output: -1
(source already has signal)

All nodes directly connected: Parallel edges:
times = [[1,2,5], [1,3,3]]   times = [[1,2,5], [1,2,3]]
k = 1, n = 3                 Multiple edges to same node
dist = [0, 5, 3]             Dijkstra picks shorter one
Output: 5 ✓                  dist[2] = min(5, 3) = 3 ✓

Cycle in graph:               Self-loop:
times = [[1,2,1], [2,1,1]]   times = [[1,1,1]]
Signal can loop forever       Self-edge (u → u)
But Dijkstra ignores cycles   Doesn't improve dist[u]
once shortest found ✓         Safely ignored ✓

Zero-weight edges:            Large graph:
times = [[1,2,0], [2,3,0]]   n = 100, many edges
Signal instant travel         Dijkstra handles O(E log V) ✓
dist = [0, 0, 0]
Output: 0 ✓
```

## 1-Indexed vs 0-Indexed

```
Problem uses 1-indexed nodes (1 to n).

In code:
→ Adjacency list: vector<...> adj(n + 1);
→ Distance array: vector<int> dist(n + 1, INT_MAX);
→ Index 0 is unused

When finding max:
→ Skip index 0: max_element(dist.begin() + 1, dist.end())
→ Only consider dist[1] to dist[n]

Alternative (0-indexed):
→ Convert: node_0indexed = node_1indexed - 1
→ adj and dist have size n
→ No +1 offset needed
→ Slightly cleaner but requires conversion

Both approaches work, just be consistent! ✓
```

## Dijkstra Optimization: Skip Outdated

```
Code includes important optimization:

if (time > dist[node]) continue;

Why is this needed?

Same node can be pushed multiple times:
→ First push: {5, node}
→ Later, better path found: {3, node}
→ Both are in priority queue!

When we pop {5, node}:
→ dist[node] is already 3 (updated earlier)
→ time=5 > dist[node]=3
→ This entry is OUTDATED!
→ Skip it to avoid redundant work ✓

Without this check:
→ Algorithm still correct
→ But does unnecessary work
→ Processes same node multiple times

With this check:
→ Each node processed at most once
→ Optimal time complexity O((V+E) log V) ✓
```

## Priority Queue Details

```
Min-heap declaration in C++:

priority_queue<pair<int, int>, 
               vector<pair<int, int>>, 
               greater<>> pq;

Components:
1. pair<int, int>: element type {time, node}
2. vector<pair<int, int>>: underlying container
3. greater<>: comparison (min-heap instead of max-heap)

Default priority_queue:
→ Max-heap (largest first)
→ WRONG for Dijkstra!

With greater<>:
→ Min-heap (smallest first) ✓
→ Always process node with shortest time

Alternative: Use negative values with max-heap
pq.push({-time, node});  // Negate to simulate min-heap
int time = -pq.top().first;  // Negate back

But explicit min-heap is cleaner! ✓
```

## Complexity Analysis

**Time Complexity:** `O((V + E) × log V)`
- Building adjacency list: O(E)
- Dijkstra's algorithm:
  - Each vertex added to PQ at most once: O(V)
  - Each edge relaxed at most once: O(E)
  - Each PQ operation: O(log V)
- Finding maximum: O(V)
- Total: **O((V + E) × log V)**

**Space Complexity:** `O(V + E)`
- Adjacency list: O(E)
- Distance array: O(V)
- Priority queue: O(V) at most
- Total: **O(V + E)**

```
Where V = n (number of nodes)
      E = |times| (number of edges)

For dense graph (E ≈ V²):
Time: O(V² log V)
Space: O(V²)

For sparse graph (E ≈ V):
Time: O(V log V)
Space: O(V)

Dijkstra is efficient for this problem! ✓
```

## Comparison with Other Approaches

```
Approach              Time              Space     When to Use
──────────────────────────────────────────────────────────────────────
Dijkstra (PQ)         O((V+E) log V)    O(V+E)    Non-negative weights ✓
Dijkstra (Set)        O((V+E) log V)    O(V+E)    Same, can erase ✓
Bellman-Ford          O(V × E)          O(V)      Negative weights
Floyd-Warshall        O(V³)             O(V²)     All pairs shortest
BFS                   O(V + E)          O(V+E)    Unweighted only

For this problem:
→ Weights are non-negative ✓
→ Single source (node k) ✓
→ Need shortest to ALL nodes ✓
→ Dijkstra is the optimal choice! ✓

BFS won't work:
→ Edges have different weights
→ BFS assumes all edges = 1
→ Would give wrong answer ✗
```

## When to Use This Pattern

```
✓ Use Dijkstra for "time to reach all nodes" when:
  → Single source, multiple destinations
  → Non-negative edge weights
  → Need shortest path to ALL nodes
  → Return depends on ALL shortest paths (max, sum, etc.)

✗ Don't use when:
  → Negative weights exist → Bellman-Ford
  → Unweighted graph → BFS (faster)
  → All pairs needed → Floyd-Warshall
  → Only one destination → Can optimize Dijkstra

Similar problems:
→ Network Delay Time (this problem) ✓
→ Cheapest Flights (with K stops) → Modified BFS
→ Path with Minimum Effort → Modified Dijkstra
→ Shortest Path in Grid → BFS or Dijkstra

Key insight:
"Minimum time for ALL to receive" = max(shortest paths) ✓
```
