## 1. Problem Statement

You are a hiker preparing for an upcoming hike. Given `heights`, a 2D array of size `rows Г— columns`, where `heights[row][col]` represents the height of cell `(row, col)`. You start at the top-left cell `(0, 0)` and want to reach the bottom-right cell `(rows-1, columns-1)`.

You can move **up, down, left, or right**. Find a route that requires the **minimum effort**.

**Effort Definition:** A route's effort is the **maximum absolute difference** in heights between two consecutive cells of the route.

**Input Format:**

- `heights` = `rows Г— columns` integer matrix

**Output:**

- Minimum effort required to reach bottom-right from top-left

## Key Insight

```
  Why Modified Dijkstra?

  Standard shortest path: minimize SUM of edge weights

  This problem: minimize MAXIMUM edge weight in path

  Example:
  Path A: edges [1, 5, 1] вҶ’ sum=7, max=5
  Path B: edges [3, 3, 3] вҶ’ sum=9, max=3

  Path B is BETTER! (smaller max despite larger sum)
  Standard Dijkstra won't work directly!

  Modified Dijkstra approach:
  вҶ’ Instead of tracking cumulative distance
  вҶ’ Track MAXIMUM effort seen so far on the path
  вҶ’ Use priority queue to process paths with smaller max effort first

  Edge weight = |height[current] - height[neighbor]|
  Path effort = max(current_effort, edge_weight)

  Priority queue ensures we explore paths with
  smallest maximum effort first вҶ’ optimal! вң“
```

## Intuition

```
  Imagine hiking across hills where your fatigue is determined not by
  the total distance you walk, but by the steepest single climb.

  1. You start at the trailhead with zero effort.
  2. Every step between two cells has a difficulty equal to the
     height difference.
  3. Your overall effort for the hike is the hardest single step.
  4. You want to choose a route where this hardest step is as easy
     as possible.

  Why modified Dijkstra works:
  We always expand the path with the current smallest maximum effort.
  When we first reach the destination, no other unexplored path can
  have a smaller maximum effort, because the priority queue would have
  processed it first. This greedy choice is safe for "minimize maximum"
  problems on non-negative edge weights.
```

## Algorithm

```
  minimumEffort(heights):

  1. Create priority queue (min-heap):
     Stores {effort, {row, col}}
     Smallest effort is processed first

  2. Initialize:
     - effort[n][m] matrix = infinity
     - effort[0][0] = 0 (source)
     - Push {0, {0, 0}} into priority queue

  3. While priority queue is not empty:
       a. Pop {diff, {row, col}} with minimum effort

       b. If reached destination (n-1, m-1):
          вҶ’ RETURN diff (this is minimum effort!) вң“

       c. For each neighbor (newr, newc) in 4 directions:
          - Check bounds: 0 вүӨ newr < n, 0 вүӨ newc < m
          - Calculate edge effort:
            edgeEffort = |heights[row][col] - heights[newr][newc]|

          - Calculate path effort:
            newEffort = max(diff, edgeEffort)
            вҶ‘ KEY: Take maximum of current path effort
                  and new edge effort!

          - If newEffort < effort[newr][newc]:
            вҶ’ Update: effort[newr][newc] = newEffort
            вҶ’ Push {newEffort, {newr, newc}} into queue

  4. Return 0 (unreachable, though shouldn't happen)
```

```
  Why max(diff, edgeEffort)?

  Path effort = MAXIMUM edge in the entire path

  Current path has max effort = diff
  New edge adds effort = edgeEffort
  New path's max effort = max(diff, edgeEffort)

  Example:
  Path so far: [1вҶ’3вҶ’5], efforts [2, 2], max=2
  New edge: 5вҶ’3, effort |5-3|=2
  New path effort = max(2, 2) = 2 вң“

  If edge was 5вҶ’8, effort=3:
  New path effort = max(2, 3) = 3 вҶҗ path gets worse!
```

### Why Early Exit Works

```
  When we pop {effort, (n-1, m-1)} from priority queue:
  Guarantee: This is the MINIMUM maximum effort!

  Why?
  вҶ’ Priority queue processes smallest efforts first
  вҶ’ All unprocessed paths have effort вүҘ current
  вҶ’ No future path can improve destination's effort
  вҶ’ Safe to return immediately! вң“

  Example:
  pq = [{2,(2,2)}, {3,(2,2)}, {6,(1,1)}]
        вҶ‘ This is optimal!

  Pop {2,(2,2)} вҶ’ destination reached вҶ’ return 2
  No need to check {3,(2,2)} or {6,(1,1)} вң“

  Optimization saves unnecessary exploration! вң“
```

## Code

```cpp
#include <bits/stdc++.h>
using namespace std;

class Solution {
public:
    int MinimumEffort(vector<vector<int>>& heights) {

        // Min-heap: {effort, {row, col}}
        priority_queue<pair<int, pair<int, int>>,
                       vector<pair<int, pair<int, int>>>,
                       greater<pair<int, pair<int, int>>>> pq;

        int n = heights.size();
        int m = heights[0].size();

        // Effort matrix (infinity initially)
        vector<vector<int>> effort(n, vector<int>(m, 1e9));
        effort[0][0] = 0;

        pq.push({0, {0, 0}});

        // 4 directions: up, right, down, left
        int dr[] = {-1, 0, 1, 0};
        int dc[] = {0, 1, 0, -1};

        while (!pq.empty()) {
            auto it = pq.top();
            pq.pop();

            int diff = it.first;
            int row = it.second.first;
            int col = it.second.second;

            // Reached destination?
            if (row == n - 1 && col == m - 1)
                return diff;

            // Explore 4 neighbors
            for (int i = 0; i < 4; i++) {
                int newr = row + dr[i];
                int newc = col + dc[i];

                // Check bounds
                if (newr >= 0 && newc >= 0 && newr < n && newc < m) {

                    // Calculate new effort
                    int newEffort = max(abs(heights[row][col] - heights[newr][newc]), diff);

                    // If better path found
                    if (newEffort < effort[newr][newc]) {
                        effort[newr][newc] = newEffort;
                        pq.push({newEffort, {newr, newc}});
                    }
                }
            }
        }

        return 0; // Unreachable
    }
};

int main() {
    vector<vector<int>> heights = {
        {1, 2, 2},
        {3, 8, 2},
        {5, 3, 5}
    };

    Solution obj;
    int ans = obj.MinimumEffort(heights);

    cout << "Minimum effort: " << ans << endl;

    return 0;
}
```

## Complexity Analysis

**Time Complexity:** O(4 Г— N Г— M Г— log(N Г— M)) = O(N Г— M Г— log(N Г— M))

- Each cell processed at most once: O(N Г— M)
- For each cell, explore 4 neighbors: O(4)
- Each priority queue operation: O(log(N Г— M))
- Total: **O(N Г— M Г— log(N Г— M))**

**Space Complexity:** O(N Г— M)

- Effort matrix: O(N Г— M)
- Priority queue: O(N Г— M) in worst case
- Total: **O(N Г— M)**

## Interview Questions

```
  Q1: Why can't we use standard Dijkstra for this problem?

  A: Standard Dijkstra minimizes the SUM of edge weights.
     This problem requires minimizing the MAXIMUM edge weight
     along the path. We need to modify the relaxation step to
     track the running maximum instead of cumulative sum.

  в”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җ

  Q2: Why does modified Dijkstra work for "minimize maximum"?

  A: The priority queue always extracts the path with the smallest
     current maximum effort. When we first reach the destination,
     no unexplored path can have a smaller maximum effort because
     the queue would have processed it first. This greedy choice
     is optimal for non-negative edge weights.

  в”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җ

  Q3: Why use max(diff, edgeEffort) instead of sum?

  A: Because the problem defines effort as the maximum absolute
     height difference in the entire route, not the total.

     The running maximum correctly captures: "the hardest step
     encountered so far on this path."

  в”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җ

  Q4: Can we use BFS instead of modified Dijkstra?

  A: No. BFS finds the path with the minimum number of steps,
     not the minimum maximum edge weight. A longer path with
     smaller individual steps could have a lower effort than a
     shorter path with one very steep step.

  в”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җ

  Q5: Why is early exit valid when we pop the destination?

  A: The priority queue always pops the smallest effort first.
     The first time the destination is popped, its effort is the
     minimum possible. Any other path to destination in the queue
     has effort greater than or equal to the popped value.

  в”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җ

  Q6: What is the difference between "minimize sum" and
      "minimize maximum" path problems?

  A: Minimize sum: total cost of all edges matters.
     Example: shortest path in weighted graph.

     Minimize maximum: only the largest edge matters.
     Example: this problem, minimizing the steepest climb.

     The relaxation differs:
     - Sum: newDist = dist[u] + weight
     - Max: newEffort = max(effort[u], weight)

  в”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җ

  Q7: Can we solve this with binary search?

  A: Yes. Binary search on the answer (effort value).

     For each candidate effort E, check if a path exists using
     only edges with effort вүӨ E. If yes, try a smaller E;
     if no, try a larger E.

     Time: O(NГ—M Г— log(max_height_diff))
     Space: O(NГ—M)

  в”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җ

  Q8: When should we prefer binary search over modified Dijkstra?

  A: Both work. Modified Dijkstra is usually more intuitive.
     Binary search can be simpler if the reachability check
     (BFS/DFS with effort constraint) is straightforward.

     Modified Dijkstra tends to be more efficient in practice
     for grid problems.

  в”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җ

  Q9: Can modified Dijkstra handle negative edge weights?

  A: No. The greedy "finalize minimum" property relies on
     non-negative edge weights. Negative weights could create
     paths with smaller maximum effort after a node is processed.

  в”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җ

  Q10: What are common bugs in this implementation?

  A: Common bugs and fixes:

      Bug 1: Using sum instead of max in relaxation
      вҶ’ newEffort = max(abs_diff, diff)

      Bug 2: Returning diff before checking destination
      вҶ’ Check if (row == n-1 && col == m-1) return diff FIRST

      Bug 3: Not initializing effort[0][0] = 0
      вҶ’ effort[0][0] = 0 before pushing to queue

      Bug 4: Bounds check order issues
      вҶ’ Use: newr >= 0 && newc >= 0 && newr < n && newc < m

      Bug 5: Using BFS queue instead of priority queue
      вҶ’ Use min-heap for "minimize maximum" problems

  в”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җ

  Q11: What other problems use the "minimize maximum" pattern?

  A: Similar problems:
      - Swim in Rising Water
      - Path With Minimum Time
      - Minimum Maximum Pair Sum
      - Cheapest Flights Within K Stops (different constraint)

     Key phrase to recognize: "minimum maximum" or
     "minimize the maximum" along a path.

  в”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җ

  Q12: How does the effort matrix differ from a distance matrix?

  A: A distance matrix stores cumulative sum of weights.
     An effort matrix stores the running maximum edge weight.

     Both are updated when a better value is found, but the
     comparison metric differs (sum vs max).

  в”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җ

  Q13: What if movement in all 8 directions is allowed?

  A: Use 8 direction arrays and include diagonal neighbors.
     The algorithm remains the same; only the neighbor
     exploration changes.

     int dr[] = {-1, -1, -1, 0, 0, 1, 1, 1};
     int dc[] = {-1, 0, 1, -1, 1, -1, 0, 1};

  в”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җ

  Q14: Why is the time complexity O(NГ—M log(NГ—M)) and not O(NГ—M)?

  A: Because we use a priority queue. Each cell can be inserted
     into the queue, and each insertion/extraction costs O(log V)
     where V = NГ—M (total cells).

     Even though each cell might be updated multiple times, the
     total complexity remains O(NГ—M log(NГ—M)).

  в”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җ

  Q15: How would you reconstruct the actual path, not just the effort?

  A: Maintain a parent matrix parent[n][m]. When updating
     effort[newr][newc], set parent[newr][newc] = {row, col}.

     After reaching the destination, trace back from destination
     to source using parent pointers to reconstruct the path.
```
