
## Dijkstra's Algorithm - Using Set : G-33

### Problem Statement:

Given a **weighted, undirected, and connected graph** of `V` vertices and an adjacency list `adj` where `adj[i]` is a list of lists containing two integers — the first integer `j` denotes an edge between `i` and `j`, the second integer is the weight of that edge. Given source vertex `S`, find the shortest distance from `S` to all other vertices. Return `-1` for unreachable vertices.

> **Note:** The graph doesn't contain any negative weight cycle.

---

## Examples

**Example 1:**
- **Input:** `V=2`, `S=0`, adj=`{{{1,9}}, {{0,9}}}`
- **Output:** `0 9`

```
Graph:
0 ──9──→ 1

Dist: 0=0, 1=9 ✓
```

**Example 2:**
- **Input:** `V=3`, `E=3`, `S=2`, adj=`{{{1,1},{2,6}}, {{2,3},{0,1}}, {{1,3},{0,6}}}`
- **Output:** `4 3 0`

```
Graph:
     1
0 ←────→ 1
|         |
6↕        ↕3
|         |
└────→ 2 ←┘

Source = 2
2→1: dist=3, 2→1→0: dist=3+1=4, 2→0: dist=6 (worse)

Dist: 0=4, 1=3, 2=0 ✓
```

---

## Key Insight

```
Why Set instead of Priority Queue?

Priority Queue (min-heap):
→ Cannot delete/update existing entries!
→ Old stale {dist, node} pairs remain in heap.
→ Leads to processing outdated entries.

Set (ordered):
→ CAN erase old {dist, node} pair! ✓
→ Insert new {updated_dist, node}.
→ Always processes the true minimum distance.
→ Avoids redundant processing!

Set advantage:
If node reached via dist=9 first, then dist=4 later:
  Priority Queue: both {9,node} and {4,node} exist
  Set: erase {9,node}, insert {4,node} → cleaner! ✓

Both give O(E log V) but set avoids extra iterations.
```

---

## Algorithm

```
1. Build weighted adjacency list
   adj[u] = {{v, weight}, ...}

2. Initialize dist[] = infinity for all nodes
   dist[source] = 0

3. Insert {0, source} into the set

4. While set is not empty:
   a. Extract {dist, node} with SMALLEST distance (set.begin())
   b. Erase it from the set
   c. For each neighbor (adjNode, edgeWt) of node:
      If dist[node] + edgeWt < dist[adjNode]:
         → Erase old {dist[adjNode], adjNode} from set (if exists)
         → Update dist[adjNode] = dist[node] + edgeWt
         → Insert {dist[adjNode], adjNode} into set

5. Return dist[]
```

```
Why does greedy extraction of minimum work?

Dijkstra's greedy property:
The node with the smallest dist in the set
has its FINAL shortest distance!

Why? All remaining nodes have dist >= current min.
No future relaxation can improve current min.
→ Safe to finalize and process it! ✓

⚠️ This breaks with NEGATIVE weights!
A negative edge could later reduce an already-finalized dist.
```

## Code

```cpp
#include <bits/stdc++.h>
using namespace std;

class Solution {
public:
    vector<int> dijkstra(int V, vector<vector<int>> adj[], int S) {
        // Set stores {distance, node} in sorted order
        // Smallest distance is always at begin()
        set<pair<int, int>> st;

        // Initialize all distances as infinity
        vector<int> dist(V, 1e9);

        // Source distance = 0
        st.insert({0, S});
        dist[S] = 0;

        // Process until set is empty
        while (!st.empty()) {
            // Extract node with minimum distance
            auto it = *(st.begin());
            int dis = it.first;
            int node = it.second;
            st.erase(it);

            // Explore all neighbors
            for (auto it : adj[node]) {
                int adjNode = it[0];  // neighbor
                int edgW = it[1];     // edge weight

                // If shorter path found
                if (dis + edgW < dist[adjNode]) {
                    // Erase old entry if it exists
                    if (dist[adjNode] != 1e9)
                        st.erase({dist[adjNode], adjNode});

                    // Update distance
                    dist[adjNode] = dis + edgW;

                    // Insert updated entry
                    st.insert({dist[adjNode], adjNode});
                }
            }
        }

        return dist;
    }
};

int main() {
    int V = 3, E = 3, S = 2;
    vector<vector<int>> adj[V];

    adj[0].push_back({1, 1});
    adj[0].push_back({2, 6});
    adj[1].push_back({2, 3});
    adj[1].push_back({0, 1});
    adj[2].push_back({1, 3});
    adj[2].push_back({0, 6});

    Solution obj;
    vector<int> res = obj.dijkstra(V, adj, S);

    cout << "Shortest distances: ";
    for (int i = 0; i < V; i++)
        cout << res[i] << " ";
    cout << endl;

    return 0;
}
```

## Step-by-Step Trace

```
V=3, S=2
Edges: 0↔1(1), 0↔2(6), 1↔2(3)

── Step 1: Adjacency List ──
adj[0] = {(1,1), (2,6)}
adj[1] = {(2,3), (0,1)}
adj[2] = {(1,3), (0,6)}

── Step 2: Init ──
dist = [∞, ∞, 0]
set  = {(0, 2)}

── Step 3: Process ──

Iteration 1: Extract (0, 2) → node=2, dis=0
  set = {}
  neighbor 1: 0+3=3 < ∞ → dist[1]=3, insert (3,1)
  neighbor 0: 0+6=6 < ∞ → dist[0]=6, insert (6,0)
  dist = [6, 3, 0]
  set  = {(3, 1), (6, 0)}

Iteration 2: Extract (3, 1) → node=1, dis=3
  set = {(6, 0)}
  neighbor 2: 3+3=6 > dist[2]=0 → skip
  neighbor 0: 3+1=4 < dist[0]=6 → erase (6,0), dist[0]=4, insert (4,0)
  dist = [4, 3, 0]
  set  = {(4, 0)}

Iteration 3: Extract (4, 0) → node=0, dis=4
  set = {}
  neighbor 1: 4+1=5 > dist[1]=3 → skip
  neighbor 2: 4+6=10 > dist[2]=0 → skip
  dist unchanged

Set empty → DONE!

Final: dist = [4, 3, 0] ✓
```

## Why Set Can Erase But PQ Cannot

```
Priority Queue (min-heap):
┌───────────────────────────────────┐
│ Internal array: [(4,0), (6,0)]   │
│ Can only access TOP element.     │
│ Cannot search & delete (6,0)!    │
│ → Stale entries accumulate.      │
└───────────────────────────────────┘

Set (balanced BST - typically Red-Black Tree):
┌───────────────────────────────────┐
│        (4, 0)                     │
│       /     \                     │
│    ...       ...                  │
│ Can search ANY element in O(logN)│
│ Can erase ANY element in O(logN) │
│ → Always clean, no stale entries │
└───────────────────────────────────┘
```

## Set vs Priority Queue Comparison

```
Property            Set                    Priority Queue
──────────────────────────────────────────────────────────────
Data Structure      Balanced BST           Binary Heap
Extract min         O(log V) ✓             O(log V) ✓
Insert              O(log V) ✓             O(log V) ✓
Erase specific      O(log V) ✓             NOT possible ✗
Stale entries       None ✓                 Can accumulate ✗
Time Complexity     O(E log V)             O(E log V)
Practical speed     Slightly slower         Slightly faster
                    (BST overhead)          (simpler structure)

When to use Set?
→ When you want cleaner code with no stale entries.
→ Easier to debug and reason about.

When to use PQ?
→ When raw speed matters (lower constant factor).
→ Just skip stale entries with a visited check.
```

## Edge Cases

```
Single node graph:           Disconnected components:
V=1, S=0                    Nodes not reachable from S
dist = [0]                  dist stays ∞ → return -1 ✓

All edges same weight:       Self-loops:
Reduces to BFS essentially  Edge u→u with weight w
Dijkstra still works ✓      dist[u]+w > dist[u]=0 → skip ✓

Multiple edges between       Large weights:
same pair of nodes:          Works fine, just larger values
Dijkstra naturally picks     in the set ✓
the shortest one ✓

⚠️ NEGATIVE WEIGHTS:
Dijkstra FAILS! Use Bellman-Ford instead.
Greedy "finalize min" assumption breaks
when negative edges can reduce finalized distances.
```

---

## Why Dijkstra Fails with Negative Weights

```
Example:
0 ──5──→ 1 ──(-3)──→ 2
0 ──4──→ 2

Dijkstra processes:
1. Extract (0, 0): relax → dist[1]=5, dist[2]=4
2. Extract (4, 2): dist[2]=4 FINALIZED ✗
3. Extract (5, 1): relax → dist[2]=5+(-3)=2 < 4

But dist[2] was already finalized as 4!
Actual shortest: 0→1→2 = 5+(-3) = 2

Dijkstra gives 4, correct answer is 2 → WRONG! ✗

Solution: Use Bellman-Ford for negative weights.
```

---

## Complexity Analysis

**Time Complexity:** `O(E log V)`
- Each edge is relaxed at most once: O(E)
- Each relaxation involves set insert/erase: O(log V)
- Total: **O(E log V)**

**Space Complexity:** `O(V + E)`
- Adjacency list: O(V + E)
- Distance array: O(V)
- Set: O(V) at most
- Total: **O(V + E)**

```
Comparison across algorithms:
Algorithm          Time              Space     Neg Weights?
─────────────────────────────────────────────────────────────
Dijkstra (Set)     O(E log V)        O(V+E)   NO ✗
Dijkstra (PQ)      O(E log V)        O(V+E)   NO ✗
Bellman-Ford        O(V * E)          O(V)     YES ✓
DAG (Topo Sort)    O(V + E)          O(V+E)   YES ✓ (DAG only)

For non-negative weighted graphs:
Dijkstra is optimal! ✓
```
