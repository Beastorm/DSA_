## Bellman Ford Algorithm

### Problem Statement:

Given a **weighted, directed and connected graph** of `V` vertices and `E` edges, find the **shortest distance** of all vertices from source vertex `S`.

> **Note:** If the graph contains a **negative cycle**, return an array consisting of only `-1`.

## Key Insight

```
Why Bellman-Ford when we have Dijkstra?

Dijkstra's Algorithm:
‚Üí Works with non-negative weights only ‚úì
‚Üí Fails with negative weights ‚úó
‚Üí Cannot detect negative cycles ‚úó
‚Üí Faster: O(E log V) with priority queue

Bellman-Ford Algorithm:
‚Üí Works with negative weights ‚úì
‚Üí Can detect negative cycles ‚úì
‚Üí Slower: O(V √ó E)
‚Üí Simpler implementation

When to use Bellman-Ford:
1. Graph has negative edge weights
2. Need to detect negative cycles
3. Simpler code preferred (no PQ needed)

Negative cycle example:
A ‚Üí B (weight = 1)
B ‚Üí C (weight = -3)
C ‚Üí A (weight = 1)
Total cycle weight = 1 + (-3) + 1 = -1 < 0

Going around this cycle repeatedly:
‚Üí Distance keeps decreasing
‚Üí No shortest path exists!
‚Üí Must detect and report this! ‚úì
```

## Algorithm

```
Bellman-Ford Algorithm:

1. Initialize:
   - dist[V] = infinity for all vertices
   - dist[S] = 0 (source)

2. Relax all edges V-1 times:
   for i = 0 to V-2:
     for each edge (u, v, wt):
       if dist[u] + wt < dist[v]:
         dist[v] = dist[u] + wt
   
   Why V-1 iterations?
   ‚Üí In a graph with V vertices
   ‚Üí Longest simple path has at most V-1 edges
   ‚Üí After V-1 relaxations, all shortest paths found! ‚úì

3. Check for negative cycle (V-th relaxation):
   for each edge (u, v, wt):
     if dist[u] + wt < dist[v]:
       ‚Üí Negative cycle detected!
       ‚Üí return [-1]
   
   If any distance still improves after V-1 iterations:
   ‚Üí Must be due to negative cycle
   ‚Üí No finite shortest paths exist!

4. Return dist[] if no negative cycle
```

```
Intuition: Why V-1 iterations?

Example path: S ‚Üí A ‚Üí B ‚Üí C ‚Üí D (4 edges, 5 vertices)

Iteration 1: S‚ÜíA distance finalized
Iteration 2: S‚ÜíA‚ÜíB distance finalized
Iteration 3: S‚ÜíA‚ÜíB‚ÜíC distance finalized
Iteration 4: S‚ÜíA‚ÜíB‚ÜíC‚ÜíD distance finalized

Maximum edges in simple path = V-1
‚Üí Need V-1 iterations to propagate distances! ‚úì

V-th iteration (negative cycle check):
If distances still improve:
‚Üí Improvement comes from cycle
‚Üí Cycle must be negative (else no improvement)
‚Üí Negative cycle detected! ‚úì
```

## Code

```cpp
#include <bits/stdc++.h>
using namespace std;

class Solution {
public:
    vector<int> bellman_ford(int V, vector<vector<int>>& edges, int S) {
        
        // Initialize distances
        vector<int> dist(V, 1e8);
        dist[S] = 0;
        
        // Relax all edges V-1 times
        for (int i = 0; i < V - 1; i++) {
            for (auto it : edges) {
                int u = it[0];   // source
                int v = it[1];   // destination
                int wt = it[2];  // weight
                
                // Relaxation step
                if (dist[u] != 1e8 && dist[u] + wt < dist[v]) {
                    dist[v] = dist[u] + wt;
                }
            }
        }
        
        // V-th relaxation: Check for negative cycle
        for (auto it : edges) {
            int u = it[0];
            int v = it[1];
            int wt = it[2];
            
            // If still relaxable, negative cycle exists
            if (dist[u] != 1e8 && dist[u] + wt < dist[v]) {
                return {-1};  // Negative cycle detected
            }
        }
        
        return dist;
    }
};

int main() {
    int V = 6;
    vector<vector<int>> edges = {
        {3, 2, 6},
        {5, 3, 1},
        {0, 1, 5},
        {1, 5, -3},
        {1, 2, -2},
        {3, 4, -2},
        {2, 4, 3}
    };
    
    int S = 0;
    Solution obj;
    vector<int> dist = obj.bellman_ford(V, edges, S);
    
    cout << "Shortest distances: ";
    for (int d : dist) {
        cout << d << " ";
    }
    cout << endl;
    
    return 0;
}
```

## Why Check dist[u] != 1e8?

```
Code line:

   ‚Üë Why this check?

Reason: Prevent updating from unreachable nodes

Edge u ‚Üí v with weight -10

1e8 + (-10) = 99999990 < dist[v]=1e8? NO
Still works, but...

dist[u] = 1e8, wt = -2e8
dist[u] + wt = -1e8 < dist[v]=1e8? YES ‚úó
WRONG update from unreachable node!

With check:
if (dist[u] != 1e8 && ...) 
‚Üí Only relax from REACHABLE nodes ‚úì
‚Üí Correct semantics
‚Üí Prevents integer overflow issues

Best practice: Always check reachability! ‚úì
```

## Why V-1 Iterations Suffice

```
Mathematical proof:

Lemma: After k iterations of Bellman-Ford,
       dist[v] = shortest path using ‚â§ k edges

Proof by induction:

Base case (k=0):
  Only dist[S] = 0, all others = ‚àû
  Correct! Source has 0-edge path to itself.

Inductive step:
  Assume true for k iterations.
  In iteration k+1:
    For edge u‚Üív:
      dist[v] = min(dist[v], dist[u] + wt)
    
    By induction, dist[u] = shortest using ‚â§ k edges
    So dist[u] + wt = shortest to v using ‚â§ k+1 edges
    Update captures this! ‚úì

After V-1 iterations:
  dist[v] = shortest path using ‚â§ V-1 edges
  
  But shortest simple path has ‚â§ V-1 edges!
  (V vertices, V-1 edges max in simple path)
  
  Therefore, dist[v] = actual shortest distance! ‚úì

V-th iteration:
  If any update happens:
  ‚Üí Using V edges in path
  ‚Üí Must revisit some vertex (only V vertices)
  ‚Üí Must be a cycle
  ‚Üí If improvement happens, cycle is negative! ‚úó
```

## Complexity Analysis

**Time Complexity:** `O(V √ó E)`
- V-1 iterations: O(V)
- Each iteration processes all E edges: O(E)
- Negative cycle check: O(E)
- Total: **O(V √ó E)**

**Space Complexity:** `O(V)`
- Distance array: O(V)
- No additional data structures needed
- Total: **O(V)**

```
Detailed analysis:

Best case: O(V √ó E)
  Even if distances stabilize early,
  we still do V-1 iterations (no early exit)

Worst case: O(V √ó E)
  All iterations make updates

Average case: O(V √ó E)
  Same (no optimization in standard version)

Optimization possible:
  Track if any update happened in iteration
  If no update, can exit early! ‚úì
  
  bool updated = false;
  for each edge: if (relax) updated = true;
  if (!updated) break;

Space is minimal:
  Only O(V) for dist array
  No queue, no heap, no visited array! ‚úì

Comparison with other algorithms:
Algorithm          Time            Space     Neg Weights?
‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ

Floyd-Warshall     O(V¬≥)           O(V¬≤)     YES ‚úì
BFS (unweighted)   O(V+E)          O(V)      N/A
```

## Optimization: Early Termination

```
Standard Bellman-Ford:
Always does V-1 iterations, even if done early

Optimized version:

for (int i = 0; i < V - 1; i++) {
    bool updated = false;  ‚Üê Track updates
    
    for (auto it : edges) {
        int u = it[0], v = it[1], wt = it[2];
        if (dist[u] != 1e8 && dist[u] + wt < dist[v]) {
            dist[v] = dist[u] + wt;
            updated = true;  ‚Üê Mark update
        }
    }
    
    if (!updated) break;  ‚Üê Early exit! ‚úì
}

Benefits:
‚Üí If no update in iteration i, won't be any in i+1
‚Üí Can stop early and save time! ‚úì

Example:
V = 100, but shortest paths found in 5 iterations
‚Üí Standard: 99 iterations
‚Üí Optimized: 5 iterations ‚úì
‚Üí Huge speedup in practice!

Still O(V√óE) worst case, but much faster average! ‚úì
```

## When to Use Bellman-Ford

```
‚úì Use Bellman-Ford when:
  ‚Üí Graph has negative edge weights
  ‚Üí Need to detect negative cycles
  ‚Üí Simpler code preferred (no PQ)
  ‚Üí Distributed algorithms (easy to parallelize)
  ‚Üí Arbitrage detection in finance

‚úó Don't use when:
  ‚Üí All weights non-negative ‚Üí Use Dijkstra (faster)
  ‚Üí Need all-pairs shortest paths ‚Üí Floyd-Warshall
  ‚Üí Unweighted graph ‚Üí Use BFS (faster)

Real-world applications:
‚Üí Currency arbitrage detection (negative cycle = profit)
‚Üí Network routing with link costs
‚Üí Game theory (Nash equilibrium)
‚Üí Distance vector routing protocols (BGP, RIP)

Similar problems:
‚Üí Detect Negative Cycle
‚Üí Shortest Path Faster Algorithm (SPFA)
‚Üí Network Delay with negative delays
‚Üí Cheapest Flights (with negative discounts)

Key insight:
"Negative weights or cycle detection" = Bellman-Ford! ‚úì
```
