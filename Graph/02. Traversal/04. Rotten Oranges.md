## Rotten Oranges: Min Time to Rot All Oranges (BFS)

**Problem Statement:**   
Given an `n × m` grid where:
- `2` = Rotten orange
- `1` = Fresh orange
- `0` = Empty cell

Every minute, a fresh orange adjacent (4-directionally) to a rotten orange becomes rotten. Return the **minimum minutes** required to rot all fresh oranges. If impossible, return `-1`.

---

>### Examples:

**Example 1:**
- **Input:** `grid=[[2,1,1],[0,1,1],[1,0,1]]`
- **Output:** `-1`
```
  2 1 1
  0 1 1
  1 0 1

  Orange at (2,0) is blocked by 0s → can never be reached!
  Return -1 ✗
```

**Example 2:**
- **Input:** `grid=[[2,1,1],[1,1,0],[0,1,1]]`
- **Output:** `4`
```
  Initial:     t=1:       t=2:       t=3:       t=4:
  2 1 1        2 2 1      2 2 2      2 2 2      2 2 2
  1 1 0   →    2 1 0  →   2 2 0  →   2 2 0  →   2 2 0
  0 1 1        0 1 1      0 1 1      0 2 1      0 2 2

  All fresh oranges rotted in 4 minutes ✓
```

---

### Key Insight:
```
  This is a MULTI-SOURCE BFS problem!

  Multiple rotten oranges start rotting simultaneously.
  BFS naturally handles "spreading level by level":
  Level 0: all initially rotten oranges
  Level 1: fresh oranges adjacent to level 0 (t=1)
  Level 2: fresh oranges adjacent to level 1 (t=2)
  ...

  Multi-source BFS: push ALL rotten oranges into
  queue initially, then process level by level.

  This gives MINIMUM time (BFS = shortest path)!

  Single-source (wrong):      Multi-source (correct):
  Start from one rotten       Start ALL rotten simultaneously
  → misses concurrent rotting → models actual simultaneous spread
```

---
## Approach
### Algorithm:
```
  1. Count total oranges (fresh + rotten) → tot
  2. Push ALL rotten oranges (value=2) into the queue
  3. BFS level by level:
       Each level = 1 minute passing
       Process all rotten in the queue:
           Check 4 neighbors
           Fresh neighbor? → mark rotten, push to queue
       If queue still has elements → days++

  4. If cnt (oranges we processed) == tot:
       return days  ← all rotted
     Else:
       return -1    ← some fresh unreachable
```
```
  Variables:
  tot  = total oranges in grid (fresh + rotten)
  cnt  = oranges processed in BFS (all become rotten)
  days = time elapsed

  Key check: tot == cnt → all oranges rotted ✓
             tot != cnt → some fresh stuck → -1
```

---

### Code:
```cpp
#include <bits/stdc++.h>
using namespace std;

int orangesRotting(vector<vector<int>>& grid) {
    if (grid.empty()) return 0;

    int m = grid.size();
    int n = grid[0].size();

    int days = 0; // Time elapsed
    int tot  = 0; // Total oranges
    int cnt  = 0; // Oranges processed

    queue<pair<int,int>> rotten;

    // Step 1: Count all oranges, push rotten ones to the queue
    for (int i = 0; i < m; i++) {
        for (int j = 0; j < n; j++) {
            if (grid[i][j] != 0) tot++;        // Count non-empty
            if (grid[i][j] == 2) rotten.push({i, j}); // Push rotten
        }
    }

    // 4 directions: right, left, down, up
    int dx[] = { 0,  0, 1, -1};
    int dy[] = { 1, -1, 0,  0};

    // Step 2: Multi-source BFS
    while (!rotten.empty()) {
        int k = rotten.size(); // Number of rotten at this minute
        cnt += k;              // Count them as processed

        // Process all rotten oranges at this time step
        while (k--) {
            int x = rotten.front().first;
            int y = rotten.front().second;
            rotten.pop();

            // Check all 4 neighbors
            for (int i = 0; i < 4; i++) {
                int nx = x + dx[i];
                int ny = y + dy[i];

                // Skip: out of bounds or not fresh
                if (nx < 0 || ny < 0 || nx >= m || ny >= n
                    || grid[nx][ny] != 1)
                    continue;

                // Rot this fresh orange
                grid[nx][ny] = 2;
                rotten.push({nx, ny});
            }
        }

        // If the queue still has oranges, another minute passed
        if (!rotten.empty()) days++;
    }

    // Step 3: Check if all oranges are rotted
    return (tot == cnt) ? days : -1;
}

int main() {
    vector<vector<int>> grid = {{2,1,1},{1,1,0},{0,1,1}};
    cout << "Minimum Minutes: " << orangesRotting(grid) << endl; // 4

    vector<vector<int>> grid2 = {{2,1,1},{0,1,1},{1,0,1}};
    cout << "Minimum Minutes: " << orangesRotting(grid2) << endl; // -1

    return 0;
}
```

### Why BFS and Not DFS?
```
  DFS (wrong for min time):   BFS (correct):
  Goes deep one path first    Spreads all directions equally
  Doesn't model simultaneous  Models simultaneous rotting ✓
  spreading correctly

  BFS level = 1 minute of simultaneous rotting
  → Naturally gives MINIMUM time ✓

  Example: Two rotten sources spread simultaneously
  BFS handles both at same time ✓
  DFS would handle them sequentially ✗
```

---

### The -1 Case:
```
  grid = [[2,1,1],
           [0,1,1],
           [1,0,1]]

  2 1 1          Rotten can reach:
  0 1 1    →     (0,0)→(0,1)→(0,2)
  1 0 1          (0,0)→(0,1)→(1,1)→(1,2)

  But (2,0) is blocked by zeros!
  0s at (1,0) and (2,1) cut off (2,0).

  After BFS: tot=7 (7 oranges), cnt=6 (only 6 processed)
  tot(7) ≠ cnt(6) → return -1 ✓
```

---

### Edge Cases:
```
  No fresh oranges:          No rotten oranges:
  grid=[[2,2,0]]             grid=[[1,1,0]]

  tot=2(rotten), cnt=2       queue empty initially
  No fresh → all processed   tot=2(fresh), cnt=0
  return 0 ✓                 tot≠cnt → return -1 ✓

  All empty:                 Single rotten, no fresh:
  grid=[[0,0,0]]             grid=[[2]]

  tot=0, cnt=0               tot=1, cnt=1
  0==0 → return 0 ✓          return 0 ✓ (0 minutes needed)
```

### Complexity Analysis:

**Time Complexity:** O(N × M)
- Each cell is visited at most once in BFS.
- 4 directions checked per cell: O(4 × N × M) = O(N × M)
- Total: **O(N × M)**

**Space Complexity:** O(N × M)
- The queue can hold at most N × M oranges (all cells rotten).
- Total: **O(N × M)**
```
  N = rows, M = cols

  BFS ensures each cell is processed once → O(NM)
  No repeated processing → optimal! ✓

  For n×n grid: O(n²) time and space.
```
