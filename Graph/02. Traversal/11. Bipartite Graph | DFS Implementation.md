## Bipartite Graph | DFS Implementation

**Problem Statement:**   
Given an adjacency list of a graph with `V` vertices (0-based index), check whether the graph is **bipartite** or not.

A graph is bipartite if we can color all vertices using **2 colors** such that **no two adjacent vertices have the same color**.

---

>### Examples:

**Example 1:**
- **Input:** Graph with no odd cycle
- **Output:** `1` (Bipartite)
```
  1 ─── 2
  |     |
  4 ─── 3

  Color:  1=0, 2=1, 3=0, 4=1
  No adjacent pair has the same color → Bipartite ✓
```

**Example 2:**
- **Input:** Graph with odd cycle (triangle)
- **Output:** `0` (Not Bipartite)
```
  1 ─── 2
   \   /
    \ /
     3

  Color: 1=0, 2=1, 3=?
  3 is adjacent to both 1(color=0) and 2(color=1)
  → Can't assign any valid color → Not Bipartite ✗
```

---

### Key Insight:
```
  Bipartite ↔ 2-colorable ↔ No odd-length cycle

  Types of graphs:
  Linear graph (path):    Always bipartite ✓
  Even cycle (4,6,8...):  Always bipartite ✓
  Odd cycle (3,5,7...):   NEVER bipartite ✗

  Example:
  Even cycle:        Odd cycle:
  0 ─── 1            0 ─── 1
  |     |             \   /
  3 ─── 2              \ /
                        2
  0:red, 1:blue         0:red, 1:blue, 2:???
  2:red, 3:blue         2 adj to 0(red) and 1(blue)
  Valid! ✓              No color works! ✗

  DFS Strategy:
  Color the starting node with 0.
  Color each neighbor with the opposite color (1-col).
  If neighbor already colored same as current → NOT bipartite!
```

---
## Appraoch
### Algorithm:
```
  color[] = all -1 (uncolored)

  For each vertex i (0 to V-1):
      If color[i] == -1:           ← unvisited
          If dfs(i, color=0) returns false:
              return false

  return true

  dfs(node, col):
      color[node] = col

      For each neighbor of node:
          If color[neighbor] == -1:     ← uncolored
              If dfs(neighbor, 1-col) == false:
                  return false          ← propagate failure

          Else if color[neighbor] == col:  ← SAME color!
              return false              ← CYCLE detected!

      return true
```

---

### Code:
```cpp
#include <bits/stdc++.h>
using namespace std;

class Solution {
private:
    // DFS to try coloring the graph with 2 colors
    bool dfs(int node, int col, int color[], vector<int> adj[]) {
        // Assign color to current node
        color[node] = col;

        // Traverse all adjacent nodes
        for (int neighbor : adj[node]) {
            if (color[neighbor] == -1) {
                // Uncolored → assign opposite color
                if (dfs(neighbor, !col, color, adj) == false)
                    return false;
            }
            else if (color[neighbor] == col) {
                // Same color as current → ODD CYCLE → not bipartite
                return false;
            }
            // Different color → fine, continue
        }

        return true; // This component is bipartite so far
    }

public:
    bool isBipartite(int V, vector<int> adj[]) {
        int color[V];
        for (int i = 0; i < V; i++) color[i] = -1; // Uncolored

        // Handle disconnected components
        for (int i = 0; i < V; i++) {
            if (color[i] == -1) {
                if (dfs(i, 0, color, adj) == false)
                    return false;
            }
        }

        return true;
    }
};

void addEdge(vector<int> adj[], int u, int v) {
    adj[u].push_back(v);
    adj[v].push_back(u);
}

int main() {
    // Example 1: Bipartite (square)
    int V1 = 4;
    vector<int> adj1[V1];
    addEdge(adj1, 0, 1); addEdge(adj1, 1, 2);
    addEdge(adj1, 2, 3); addEdge(adj1, 3, 0);

    Solution sol;
    cout << "Example 1: " << (sol.isBipartite(V1, adj1) ? "Bipartite" : "Not Bipartite") << endl;

    // Example 2: Not bipartite (triangle)
    int V2 = 3;
    vector<int> adj2[V2];
    addEdge(adj2, 0, 1); addEdge(adj2, 1, 2); addEdge(adj2, 2, 0);

    cout << "Example 2: " << (sol.isBipartite(V2, adj2) ? "Bipartite" : "Not Bipartite") << endl;

    return 0;
}
```

## Step-by-Step Trace

### Bipartite Case (Square: 0─1─2─3─0):
```
  color = [-1,-1,-1,-1]

  i=0: color[0]==-1 → dfs(0, col=0)

  dfs(0, 0):
    color[0]=0
    neighbors: [1, 3]

    neighbor 1: color=-1 → dfs(1, col=1)
      dfs(1, 1):
        color[1]=1
        neighbors: [0, 2]
        neighbor 0: color=0 ≠ 1 → OK (different color)
        neighbor 2: color=-1 → dfs(2, col=0)
          dfs(2, 0):
            color[2]=0
            neighbors: [1, 3]
            neighbor 1: color=1 ≠ 0 → OK
            neighbor 3: color=-1 → dfs(3, col=1)
              dfs(3, 1):
                color[3]=1
                neighbors: [2, 0]
                neighbor 2: color=0 ≠ 1 → OK
                neighbor 0: color=0 ≠ 1 → OK
                return true ✓
            return true ✓
          return true ✓
        return true ✓
      return true ✓

    neighbor 3: color=1 ≠ 0 → OK
    return true ✓

  color = [0, 1, 0, 1]
  return true → BIPARTITE ✓

  Visual coloring:
  0(red) ─── 1(blue)
  |              |
  3(blue) ─── 2(red)

  No adjacent pair has the same color ✓
```

### Not Bipartite Case (Triangle: 0─1─2─0):
```
  color = [-1,-1,-1]

  i=0: dfs(0, col=0)

  dfs(0, 0):
    color[0]=0
    neighbor 1: color=-1 → dfs(1, col=1)
      dfs(1, 1):
        color[1]=1
        neighbor 0: color=0 ≠ 1 → OK
        neighbor 2: color=-1 → dfs(2, col=0)
          dfs(2, 0):
            color[2]=0
            neighbor 1: color=1 ≠ 0 → OK
            neighbor 0: color=0 == 0 → SAME COLOR! return false ✗

        dfs(2) returned false → return false
    dfs(1) returned false → return false
  dfs(0) returned false

  return false → NOT BIPARTITE ✓

  Attempted coloring:
  0(red) ─── 1(blue)
        \   /
         \ /
         2(red) ← adj to 0(red) = CONFLICT! ✗
```

---

### BFS vs DFS for Bipartite:
```
  Both work! Same logic, different traversal:

  BFS version:                    DFS version:
  Use queue                       Use recursion
  Assign color when pushing       Assign color at function entry
  Check neighbor: same? → false   Check neighbor: same? → false

  BFS: color[node] = 0            DFS: color[node] = col
       while queue:                    for each neighbor:
           pop node                        if uncolored: recurse
           for each neighbor:              if same color: return false
               if uncolored: push
               if same color: false
```

---

### Edge Cases:
```
  Single node:              Two nodes, one edge:
  V=1, no edges             0 ─── 1

  color[0]=-1 → dfs(0,0)   dfs(0,0): color 0=0
  No neighbors              neighbor 1: dfs(1,1): color 1=1
  return true ✓             neighbor 0: color=0≠1 → OK
                            return true → BIPARTITE ✓

  Disconnected graph:        Self-loop:
  0─1   2─3─4               adj[0]=[0]

  Check each component:      dfs(0,0):
  Component {0,1}: bipartite   neighbor 0: color=0==0 → false!
  Component {2,3,4}: check...  return false (self-loop = odd cycle)
  Both must be bipartite ✓
```

### Complexity Analysis:

**Time Complexity:** O(V + E)
- Each vertex visited once: O(V)
- Each edge explored twice (undirected): O(E)
- Total: **O(V + E)**

**Space Complexity:** O(V + E)
- Adjacency list: O(V + E)
- Color array: O(V)
- Recursion stack: O(V) worst case
- Total: **O(V + E)**
```
  Key formula:
  Bipartite ↔ 2-colorable ↔ No odd cycle
  DFS assigns colors and detects conflicts.
  Conflict = odd cycle = not bipartite.
```
