## Detect Cycle in an Undirected Graph (DFS)

**Problem Statement:**\   
Given an undirected graph with `V` vertices and `E` edges, check whether it contains any cycle or not using **DFS**.

---

>### Examples:

**Example 1:**
- **Input:** `V=8, E=7`
- **Output:** `False`
```
  1 ─── 2 ─── 3
        |
        4 ─── 7
        |
        5     6     8

  DFS explores all paths, never finds a visited
  non-parent node → No cycle ✓
```

**Example 2:**
- **Input:** `V=8, E=6`
- **Output:** `True`
```
  4 ─── 5 ─── 6
  └───────────┘  ← cycle: 4→5→6→4

  DFS from 4 → 5 → 6 → sees 4 (visited, not parent 5) → CYCLE! ✓
```

---

### Key Insight:
```
  Same logic as BFS cycle detection, but uses DFS:

  At each node, for each neighbor:
  ┌──────────────────────────────────────────────┐
  │  Neighbor NOT visited?  → recurse (DFS)      │
  │  Neighbor IS visited AND ≠ parent? → CYCLE!  │
  │  Neighbor IS visited AND == parent? → SKIP   │
  └──────────────────────────────────────────────┘

  WHY check parent?
  Undirected edge A─B: both A→B and B→A exist in adj list.
  When at B (came from A), A appears as neighbor.
  A is visited, but it's just the PARENT edge → not a cycle.

  DFS vs BFS for cycle detection:
  Both work equally well.
  DFS uses recursion stack (implicit stack).
  BFS uses explicit queue.
```

---
## Appraoch
### Algorithm:
```
  dfs(node, parent, adj, visited):
    visited[node] = 1

    For each neighbor of node:
      If NOT visited[neighbor]:
          If dfs(neighbor, node, ...) == true → return true
      Else if neighbor ≠ parent:
          return true  ← CYCLE FOUND!

    return false

  isCycle(V, adj):
    visited[] = all 0

    For each vertex i (0 to V-1):
        If NOT visited[i]:
            If dfs(i, parent=-1, ...) → return true

    return false
```

---

### Code:
```cpp
#include <bits/stdc++.h>
using namespace std;

class Solution {
public:
    // DFS to detect cycle from 'node' with given 'parent'
    bool dfs(int node, int parent, vector<int> adj[], vector<int>& visited) {
        // Mark current node as visited
        visited[node] = 1;

        // Explore all neighbors
        for (int neighbor : adj[node]) {
            if (!visited[neighbor]) {
                // Unvisited → go deeper
                if (dfs(neighbor, node, adj, visited)) return true;
            }
            else if (neighbor != parent) {
                // Visited AND not parent → CYCLE!
                return true;
            }
            // Visited AND == parent → just the back edge, skip
        }

        return false; // No cycle from this path
    }

    bool isCycle(int V, vector<int> adj[]) {
        vector<int> visited(V, 0);

        // Handle disconnected components
        for (int i = 0; i < V; i++) {
            if (!visited[i]) {
                if (dfs(i, -1, adj, visited)) return true;
            }
        }
        return false;
    }
};

int main() {
    Solution sol;

    // Example 1: No cycle (path graph 0-1-2-3)
    int V1 = 4;
    vector<int> adj1[4];
    adj1[0] = {1}; adj1[1] = {0,2};
    adj1[2] = {1,3}; adj1[3] = {2};
    cout << "Example 1: " << (sol.isCycle(V1, adj1) ? "True" : "False") << endl;

    // Example 2: Cycle exists (triangle 0-1-2-0)
    int V2 = 3;
    vector<int> adj2[3];
    adj2[0] = {1,2}; adj2[1] = {0,2}; adj2[2] = {0,1};
    cout << "Example 2: " << (sol.isCycle(V2, adj2) ? "True" : "False") << endl;

    // Example 3: Cycle 4→5→6→4
    int V3 = 5;
    vector<int> adj3[V3];
    adj3[1] = {2}; adj3[2] = {1};
    adj3[4] = {5,6}; adj3[5] = {4,6}; adj3[6] = {4,5};
    cout << "Example 3: " << (sol.isCycle(V3, adj3) ? "True" : "False") << endl;

    return 0;
}
```

## Step-by-Step Trace

### No Cycle Case
```
  Graph: 0 ─── 1 ─── 2 ─── 3
  adj[0]=[1], adj[1]=[0,2], adj[2]=[1,3], adj[3]=[2]

  isCycle: i=0, not visited → dfs(0, -1)

  dfs(0, parent=-1):
    visited[0]=1
    neighbor 1: not visited → dfs(1, parent=0)

      dfs(1, parent=0):
        visited[1]=1
        neighbor 0: visited, 0==parent(0) → SKIP
        neighbor 2: not visited → dfs(2, parent=1)

          dfs(2, parent=1):
            visited[2]=1
            neighbor 1: visited, 1==parent(1) → SKIP
            neighbor 3: not visited → dfs(3, parent=2)

              dfs(3, parent=2):
                visited[3]=1
                neighbor 2: visited, 2==parent(2) → SKIP
                return false ←────── no cycle

            dfs(3) returned false
            return false ←────── no cycle

          dfs(2) returned false
          return false ←────── no cycle

        dfs(1) returned false
        return false ←────── no cycle

  All visited, return false → No cycle ✓
```

### Cycle Case:
```
  Graph: 0─1─2─0 (triangle)
  adj[0]=[1,2], adj[1]=[0,2], adj[2]=[0,1]

  dfs(0, parent=-1):
    visited[0]=1
    neighbor 1: not visited → dfs(1, parent=0)

      dfs(1, parent=0):
        visited[1]=1
        neighbor 0: visited, 0==parent(0) → SKIP
        neighbor 2: not visited → dfs(2, parent=1)

          dfs(2, parent=1):
            visited[2]=1
            neighbor 0: visited, 0≠parent(1) → CYCLE! return true ✓

        dfs(2) returned true → return true ✓
    dfs(1) returned true → return true ✓

  Cycle detected! ✓
```

### DFS Recursion Tree Visualization:
```
  Triangle 0─1─2─0:

  dfs(0, -1)
  ├── dfs(1, 0)
  │   ├── skip 0 (visited, == parent)
  │   └── dfs(2, 1)
  │       ├── see 0: visited, 0≠parent(1) → CYCLE! ✓
  │       └── (would skip 1 as parent)
  └── (would skip 2 if reached from 0)

  Path 0─1─2─3 (no cycle):

  dfs(0, -1)
  └── dfs(1, 0)
      ├── skip 0 (parent)
      └── dfs(2, 1)
          ├── skip 1 (parent)
          └── dfs(3, 2)
              └── skip 2 (parent)
              return false
          return false
      return false
  return false → No cycle ✓
```

### BFS vs DFS Cycle Detection:
```
  Property           BFS                    DFS
  ─────────────────────────────────────────────────────
  Data Structure     Queue + pairs          Recursion stack
  Tracks parent via  Queue entries          Function parameter
  Implementation     Iterative              Recursive
  Stack overflow?    No (explicit queue)    Possible (deep recursion)
  Same logic?        YES ← both track       YES ← parent to avoid
                     parent to avoid        back-edge confusion
                     back-edge confusion

  Both: visited AND ≠ parent → CYCLE ✓

  Choose BFS: large graphs (avoid stack overflow)
  Choose DFS: cleaner recursive code, natural for trees
```

---

## Disconnected Graph
```
  Graph:
  Component 1: 0─1─2 (no cycle)
  Component 2: 3─4─5─3 (has cycle)

  isCycle loop:
  i=0: not visited → dfs(0,-1) → false (component 1 explored)
  i=3: not visited → dfs(3,-1) → TRUE (cycle in component 2)
  return true ✓

  Without the outer loop:
  Would only check from i=0 → miss component 2 → WRONG!
```

---

### Edge Cases:
```
  V=1, no edges:          V=2, one edge:
  dfs(0,-1)               dfs(0,-1):
  no neighbors              neighbor 1: dfs(1,0)
  return false ✓              neighbor 0: visited, 0==parent → SKIP
                              return false
                            return false → No cycle ✓

  Self-loop (adj[0]=[0]):  Multiple edges (multigraph):
  dfs(0,-1):               adj[0]=[1,1]
    neighbor 0: visited,   Two edges between 0 and 1
    0 ≠ -1 → CYCLE! ✓      dfs from 0: push 1 (parent=0)
                            From 1: neighbor 0=parent(skip)
                            But second neighbor 0: still parent→skip
                            Depends on implementation
```

### Complexity Analysis:

**Time Complexity:** O(V + E)
- Each vertex visited exactly once: O(V)
- Each edge explored at most twice (undirected): O(E)
- Total: **O(V + E)**

**Space Complexity:** O(V + E)
- Adjacency list: O(V + E)
- Visited array: O(V)
- Recursion call stack: O(V) worst case (linear graph)
- Total: **O(V + E)**
```
  Recursion depth worst case:
  Linear graph: 0─1─2─3─...─(V-1)
  DFS goes all the way to depth V
  Stack frames = V → O(V) space

  For very large V, use BFS to avoid stack overflow!
```
