## Accounts Merge - DSU

### Problem Statement:

Given a list of `accounts` where each element `accounts[i]` is a list of strings, where the first element `accounts[i][0]` is a name, and the rest of the elements are emails representing emails of the account.

**Merge** accounts that share at least one common email. Return each user's name followed by their **sorted, unique emails**.

> **Note:** Accounts can have the same name but belong to different people if they don't share any common email.

---

## Key Insight

```
Problem: Group accounts by shared emails

Observation 1: Email sharing creates connectivity

If accounts A and B share email e1:
→ They belong to same person
→ Connect A ↔ B

If accounts B and C share email e2:
→ Connect B ↔ C
→ Transitively: A ↔ B ↔ C all belong to same person!

This forms connected components!

Observation 2: DSU is perfect for this

DSU tracks connected components efficiently
→ Union accounts that share emails
→ Group all emails under ultimate parent
→ Time: O(N × α(N)) ≈ O(N)

Strategy:
1. Map: email → account index
2. When same email appears in different accounts:
   → Union those account indices
3. Group emails by ultimate parent
4. Format result with name + sorted emails
```

---

## Algorithm

```
Step 1: Build email-to-account mapping

For each account i:
  For each email in account i:
    If email not seen before:
      map[email] = i
    Else:
      union(i, map[email])  ← Accounts share this email!

Step 2: Group emails by ultimate parent

For each (email, accountIndex) in map:
  ultimateParent = find(accountIndex)
  Add email to mergedMails[ultimateParent]

Step 3: Build final result

For each account i:
  If mergedMails[i] is not empty:
    Sort mergedMails[i]
    Create result: [name] + sorted emails
    Add to answer

Step 4: Return sorted answer
```

## Code

```cpp
#include <bits/stdc++.h>
using namespace std;

class DisjointSet {
    vector<int> rank, parent, size;
    
public:
    DisjointSet(int n) {
        rank.resize(n + 1, 0);
        parent.resize(n + 1);
        size.resize(n + 1, 1);
        
        for (int i = 0; i <= n; i++) {
            parent[i] = i;
        }
    }
    
    int findUPar(int node) {
        if (node == parent[node])
            return node;
        return parent[node] = findUPar(parent[node]);
    }
    
    void unionBySize(int u, int v) {
        int ulp_u = findUPar(u);
        int ulp_v = findUPar(v);
        
        if (ulp_u == ulp_v) return;
        
        if (size[ulp_u] < size[ulp_v]) {
            parent[ulp_u] = ulp_v;
            size[ulp_v] += size[ulp_u];
        }
        else {
            parent[ulp_v] = ulp_u;
            size[ulp_u] += size[ulp_v];
        }
    }
};

class Solution {
public:
    vector<vector<string>> accountsMerge(vector<vector<string>>& accounts) {
        int n = accounts.size();
        DisjointSet ds(n);
        
        // Map: email -> account index
        unordered_map<string, int> emailToNode;
        
        // Step 1: Union accounts with common emails
        for (int i = 0; i < n; i++) {
            for (int j = 1; j < accounts[i].size(); j++) {
                string email = accounts[i][j];
                
                if (emailToNode.find(email) == emailToNode.end()) {
                    emailToNode[email] = i;
                }
                else {
                    ds.unionBySize(i, emailToNode[email]);
                }
            }
        }
        
        // Step 2: Group emails by ultimate parent
        vector<string> mergedMails[n];
        for (auto& [email, node] : emailToNode) {
            int ultimateParent = ds.findUPar(node);
            mergedMails[ultimateParent].push_back(email);
        }
        
        // Step 3: Build final result
        vector<vector<string>> ans;
        for (int i = 0; i < n; i++) {
            if (mergedMails[i].empty()) continue;
            
            // Sort emails
            sort(mergedMails[i].begin(), mergedMails[i].end());
            
            vector<string> temp;
            temp.push_back(accounts[i][0]);  // Name
            
            for (string& email : mergedMails[i]) {
                temp.push_back(email);
            }
            
            ans.push_back(temp);
        }
        
        return ans;
    }
};

int main() {
    vector<vector<string>> accounts = {
        {"John", "j1@com", "j2@com", "j3@com"},
        {"John", "j4@com"},
        {"Raj", "r1@com", "r2@com"},
        {"John", "j1@com", "j5@com"},
        {"Raj", "r2@com", "r3@com"},
        {"Mary", "m1@com"}
    };
    
    Solution obj;
    vector<vector<string>> result = obj.accountsMerge(accounts);
    
    for (auto& acc : result) {
        cout << acc[0] << ": ";
        for (int i = 1; i < acc.size(); i++) {
            cout << acc[i] << " ";
        }
        cout << endl;
    }
    
    return 0;
}
```

**Output:**
```
John: j1@com j2@com j3@com j5@com 
John: j4@com 
Mary: m1@com 
Raj: r1@com r2@com r3@com 
```

## Why This Works

```
Key property: Email sharing is transitive

If account A shares email with B,
and B shares email with C,
then A, B, C all belong to same person!

Example:
A: [e1@, e2@]
B: [e2@, e3@]  ← shares e2@ with A
C: [e3@, e4@]  ← shares e3@ with B

Process A:
  e1@ → map to A
  e2@ → map to A

Process B:
  e2@ → already mapped to A
        → union(A, B)
  e3@ → map to B

Process C:
  e3@ → already mapped to B
        → union(C, B)
        → But B is connected to A
        → So C connects to A transitively!

DSU handles this transitivity automatically! ✓

Final: A, B, C all have same ultimate parent
→ All emails grouped together
→ All belong to same person! ✓
```

Not valid per problem          Handled as strings ✓

## Why Use Account Index as Node?

```
Question: Why not use emails as nodes in DSU?

Using account index as nodes:
→ Fixed number of nodes (N accounts)
→ Simple integer operations
→ Easy to access account name later
→ parent[i] = account index

vs. Using email strings as nodes:
→ Variable number of nodes (unknown # emails)
→ String operations (slower)
→ Need separate mapping back to account
→ More complex implementation

Example:
Account 0: [John, e1@, e2@]
Account 1: [John, e1@, e3@]

Approach 1 (account index):
unite(0, 1) via shared e1@
Simple! ✓

Approach 2 (email as node):
unite(e1@, e2@)
unite(e1@, e3@)
Then need to map back to accounts
More complex! ✗

Account index is cleaner! ✓
```

## Complexity Analysis

**Time Complexity:** `O(N × K × α(N)) + O(E log E)`
- N = number of accounts
- K = average emails per account
- E = total number of emails
- Process all emails: O(N × K × α(N))
- Sort emails for each group: O(E log E)
- Total: **O(N × K + E log E)**

**Space Complexity:** `O(N + E)`
- DSU arrays: O(N)
- Email map: O(E)
- Merged mails: O(E)
- Answer: O(E)
- Total: **O(N + E)**

```
Detailed breakdown:

Processing accounts:
→ N accounts
→ K emails per account (average)
→ Each email: map lookup/insert + union
→ Union: O(α(N)) ≈ O(1)
→ Total: O(N × K × α(N)) ≈ O(N × K)

Grouping emails:
→ Iterate E emails
→ Each find(): O(α(N)) ≈ O(1)
→ Total: O(E)

Sorting emails:
→ For each group, sort emails
→ Total emails = E
→ Worst case: all in one group
→ Sort: O(E log E)

Building result:
→ O(N + E) to construct answer

Total: O(N × K + E log E)

Space:
→ parent, rank, size: O(N)
→ emailToNode map: O(E)
→ mergedMails: O(E)
→ Answer: O(E)
→ Total: O(N + E)
```

## Alternative Approach: DFS

```cpp
class Solution {
public:
    vector<vector<string>> accountsMerge(vector<vector<string>>& accounts) {
        int n = accounts.size();
        
        // Build graph: email -> list of account indices
        unordered_map<string, vector<int>> emailToAccounts;
        for (int i = 0; i < n; i++) {
            for (int j = 1; j < accounts[i].size(); j++) {
                emailToAccounts[accounts[i][j]].push_back(i);
            }
        }
        
        // DFS to find connected components
        vector<bool> visited(n, false);
        vector<vector<string>> result;
        
        for (int i = 0; i < n; i++) {
            if (visited[i]) continue;
            
            // DFS from account i
            set<string> emails;  // Use set for automatic sorting
            queue<int> q;
            q.push(i);
            visited[i] = true;
            
            while (!q.empty()) {
                int acc = q.front();
                q.pop();
                
                for (int j = 1; j < accounts[acc].size(); j++) {
                    string email = accounts[acc][j];
                    emails.insert(email);
                    
                    for (int neighbor : emailToAccounts[email]) {
                        if (!visited[neighbor]) {
                            visited[neighbor] = true;
                            q.push(neighbor);
                        }
                    }
                }
            }
            
            vector<string> merged;
            merged.push_back(accounts[i][0]);
            for (string email : emails) {
                merged.push_back(email);
            }
            result.push_back(merged);
        }
        
        return result;
    }
};
```

**DSU vs DFS:**
- DSU: O(N × K × α(N)) ≈ O(N × K), cleaner
- DFS: O(N × K × E), more code
- Both work, DSU preferred! ✓
