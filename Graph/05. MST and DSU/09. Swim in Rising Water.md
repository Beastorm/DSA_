## Swim in Rising Water

### Problem Statement:

Given an `n × n` matrix `grid` where `grid[i][j]` is the unique elevation of cell `(i, j)`. Rain starts at `t=0`. At time `t`, every cell with elevation `≤ t` is flooded. You can move 4-directionally between adjacent cells instantly **if both elevations ≤ t**. Find the **minimum time `t`** to reach from `(0,0)` to `(n-1, n-1)`.

---

## Examples

**Example 1:**
- **Input:** `grid = [[0,2],[1,3]]`
- **Output:** `3`
```
  Grid:         t=0:    t=1:    t=2:    t=3:
  0  2          ●       ●       ●  ●    ●  ●
  1  3          ×       ●       ●       ●  ●

  t=0: only (0,0) accessible (elev=0)
  t=1: (0,0),(1,0) accessible (elev≤1)
  t=2: (0,0),(0,1),(1,0) accessible (elev≤2)
  t=3: ALL cells accessible → can reach (1,1) ✓
  Answer = 3
```

**Example 2:**
- **Input:** `5×5 spiral grid`
- **Output:** `16`
```
   0  1  2  3  4
  24 23 22 21  5
  12 13 14 15 16
  11 17 18 19 20
  10  9  8  7  6

  Optimal path must cross elevation 16 (at cell (2,4))
  → minimum t = 16 ✓
```

---

## Key Insight
```
  The cost to travel a path = MAXIMUM elevation on that path.
  (We wait until t is high enough for the hardest cell to cross)

  Goal: Find path from (0,0) to (n-1,n-1) with
        MINIMUM possible maximum elevation.

  This is the "Minimax Path" problem!

  Solution: Modified Dijkstra with Min-Heap
  → Always expand the cell with LOWEST elevation first
  → Track the MAXIMUM elevation encountered so far on path
  → When we reach destination → that max = answer!

  Why max(current_elevation, grid[nr][nc])?
  → We can only move when t = max(elevations on path)
  → Path cost = max elevation seen so far
```

---

## Algorithm
```
  swimInWater(grid):

  1. Min-heap = {(elevation, row, col)}
     Push {grid[0][0], 0, 0}
     visited[0][0] = true

  2. While heap not empty:
       (elev, r, c) = heap.top(); pop

       If (r,c) == destination → return elev

       For each 4-direction neighbor (nr, nc):
           If valid AND not visited:
               visited[nr][nc] = true
               heap.push({max(elev, grid[nr][nc]), nr, nc})

  3. Return -1 (unreachable)

  Key: max(elev, grid[nr][nc])
  → elev = max elevation seen so far on this path
  → grid[nr][nc] = elevation of next cell
  → max of both = time needed to cross this path
```

## Code
```cpp
#include <bits/stdc++.h>
using namespace std;

class Solution {
public:
    int swimInWater(vector<vector<int>>& grid) {
        int n = grid.size();

        // Min-heap: {max_elevation_so_far, row, col}
        priority_queue<vector<int>,
                       vector<vector<int>>,
                       greater<vector<int>>> minHeap;

        vector<vector<int>> visited(n, vector<int>(n, 0));

        // Start from top-left
        minHeap.push({grid[0][0], 0, 0});
        visited[0][0] = 1;

        // 4 directions
        vector<pair<int,int>> dirs = {{0,1},{1,0},{0,-1},{-1,0}};

        while (!minHeap.empty()) {
            auto curr = minHeap.top();
            minHeap.pop();

            int elevation = curr[0]; // Max elevation on current path
            int r = curr[1];
            int c = curr[2];

            // Reached destination!
            if (r == n-1 && c == n-1) return elevation;

            // Explore neighbors
            for (auto& [dr, dc] : dirs) {
                int nr = r + dr;
                int nc = c + dc;

                if (nr >= 0 && nc >= 0 && nr < n && nc < n
                    && !visited[nr][nc]) {

                    visited[nr][nc] = 1;
                    // Key: track max elevation on path
                    minHeap.push({max(elevation, grid[nr][nc]), nr, nc});
                }
            }
        }

        return -1;
    }
};

int main() {
    Solution sol;

    vector<vector<int>> grid1 = {{0,2},{1,3}};
    cout << "Example 1: " << sol.swimInWater(grid1) << endl; // 3

    vector<vector<int>> grid2 = {
        { 0, 1, 2, 3, 4},
        {24,23,22,21, 5},
        {12,13,14,15,16},
        {11,17,18,19,20},
        {10, 9, 8, 7, 6}
    };
    cout << "Example 2: " << sol.swimInWater(grid2) << endl; // 16

    return 0;
}
```

```

  n=2, destination=(1,1)

  Init:
  visited=[[1,0],[0,0]]
  heap=[(0, 0,0)]

  ── Pop (0, 0,0): elev=0 ──
  Not destination (0,0)≠(1,1)
  Neighbors:
    RIGHT (0,1): valid, not vis → push (max(0,2)=2, 0,1), vis[0][1]=1
    DOWN  (1,0): valid, not vis → push (max(0,1)=1, 1,0), vis[1][0]=1
  heap=[(1,1,0),(2,0,1)]

  ── Pop (1, 1,0): elev=1 ──
  Not destination (1,0)≠(1,1)
  Neighbors:
    UP    (0,0): visited → skip
    RIGHT (1,1): valid, not vis → push (max(1,3)=3, 1,1), vis[1][1]=1
    DOWN  (2,0): out of bounds → skip
    LEFT  (1,-1): out of bounds → skip
  heap=[(2,0,1),(3,1,1)]

  ── Pop (2, 0,1): elev=2 ──
  Not destination (0,1)≠(1,1)
  Neighbors:
    LEFT  (0,0): visited → skip
    DOWN  (1,1): visited → skip
    UP    (-1,1): OOB → skip
    RIGHT (0,2): OOB → skip
  heap=[(3,1,1)]

  ── Pop (3, 1,1): elev=3 ──
  DESTINATION (1,1)! Return 3 ✓
```

## Why Dijkstra (Not BFS)?
```
  BFS = unit weight (each step costs 1)
  Here: different cells have DIFFERENT "costs" (elevations)

  We want MINIMUM MAXIMUM elevation path.
  → Not minimum sum, but minimum MAX.

  Min-heap ensures we always explore
  the path with SMALLEST maximum elevation first.
  → Greedy: once destination popped, answer is optimal ✓

  Alternative: Binary search + BFS
  Binary search on answer t, BFS to check reachability
  → Same time complexity, more complex code
```

---

## The `max(elevation, grid[nr][nc])` Explained
```
  elevation = maximum elevation seen so far on this path
  grid[nr][nc] = elevation of next cell to visit

  When we push to heap: max(elevation, grid[nr][nc])

  This means: "To travel this path AND enter this new cell,
  we need water level t = max of all cells on path"

  Example:
  Path 0→1→3→2, elevations = 0,1,3,2
  After visiting cell with elev=3: path_max = 3
  Next cell has elev=2: max(3,2)=3 (still 3)
  → We already needed t=3, adding cell elev=2 doesn't change it ✓
```

## Edge Cases
```
  Single cell:                  n=1
  grid=[[5]]
  Push (5,0,0), pop → destination!
  Return 5 ✓

  Already reachable at t=0:     Path forced through high elevation:
  grid=[[0,0],[0,0]]            Must cross mountain peak
  Push (0,0,0), neighbors 0     Min-heap finds min-max path ✓
  All 0 → return 0 ✓
```

## Complexity Analysis

**Time Complexity:** O(N² log N)
- Total cells = N²
- Each cell pushed to min-heap at most once
- Each heap operation: O(log N²) = O(2 log N) = O(log N)
- Total: **O(N² log N)**

**Space Complexity:** O(N²)
- `visited` matrix: O(N²)
- Min-heap: O(N²) worst case (all cells)
- Total: **O(N²)**
```
  N = grid side length
  Total cells = N²

  Similar to Dijkstra on a grid:
  O(V log V) where V = N²
  = O(N² log N²)
  = O(N² × 2 log N)
  = O(N² log N) ✓
```
