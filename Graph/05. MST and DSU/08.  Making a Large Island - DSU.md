## Making a Large Island - DSU

### Problem Statement:

Given an `n × n` binary matrix `grid`, you are allowed to change **at most one 0 to 1**.

A group of connected 1s forms an **island**, where two 1s are connected if they share a side (4-directional).

**Return** the size of the **largest island** in the grid after applying this operation.

---

## Examples

**Example 1:**

**Input:**
```
grid = [[1, 0],
        [0, 1]]
```

**Output:** `3`

```
Before:          After changing [0,1] to 1:
1 0              1 1
0 1              0 1

Two separate islands (size 1 each)
→ Connect them by changing one 0
→ Largest island = 3 ✓

Alternative: Change [1,0] to 1
1 0              1 0
0 1      →       1 1
→ Also creates island of size 3 ✓
```

**Example 2:**

**Input:**
```
grid = [[1, 1],
        [1, 1]]
```

**Output:** `4`

```
Grid is already all 1s
→ No 0 to change
→ Largest island = 4 (entire grid) ✓
```

**Example 3:**

**Input:**
```
grid = [[1, 0, 1],
        [0, 0, 0],
        [1, 0, 1]]
```

**Output:** `3`

```
Before:
1 0 1
0 0 0
1 0 1

Islands: 4 separate islands of size 1 each

Best change: [1,0] or [0,1] or [1,2] or [2,1]
Let's try [1,1] (center):
1 0 1
0 1 0  ← Changed
1 0 1

Now [1,1] connects with: [0,1]=0, [1,0]=0, [1,2]=0, [2,1]=0
None are land! Still size 1 island

Try [0,1]:
1 1 1
0 0 0
1 0 1

[0,1] connects [0,0] and [0,2]
→ Island size = 1 + 1 + 1 = 3 ✓
```

## Key Insight

```
Problem: Find max island size after flipping AT MOST one 0 to 1

Observation 1: What happens when we flip a 0?

The cell [i,j] = 0 has up to 4 neighbors
Each neighbor might be part of different islands
Flipping [i,j] to 1 could CONNECT multiple islands!

New island size = sum of sizes of all connected islands + 1

Example:
  Island A (size 3)    Island B (size 2)
     |                    |
     0  ← flip this
     |
  Island C (size 4)

New size = 3 + 2 + 4 + 1 = 10 ✓

Observation 2: Must avoid double counting!

If two neighbors belong to SAME island:
→ Don't count that island twice!
→ Use ultimate parent to identify unique islands
→ Store parents in SET for uniqueness

Observation 3: Edge case

If grid is all 1s (no 0s):
→ Can't flip anything
→ Answer = size of entire island
→ Must handle separately!

Strategy:
1. Build islands using DSU
2. For each 0: calculate potential size if flipped
3. Track maximum
4. Handle all-1s edge case
```

---

## Algorithm

```
Step 1: Build initial islands using DSU

For each cell (i, j):
  If grid[i][j] == 1:
    For each neighbor that is also 1:
      union(cell, neighbor)

Step 2: Consider flipping each 0

For each cell (i, j):
  If grid[i][j] == 0:
    Find unique islands among 4 neighbors
    (using SET of ultimate parents)
    
    potentialSize = 1 (the flipped cell)
    For each unique neighboring island:
      potentialSize += size of that island
    
    maxSize = max(maxSize, potentialSize)

Step 3: Handle edge case (no 0s)

For each cell:
  maxSize = max(maxSize, size of component)

Return maxSize
```

---

## Node Numbering

```
Convert 2D coordinates to 1D node number:

nodeNumber = row × n + col

Example: n = 3 grid
  col: 0  1  2
row 0: 0  1  2
row 1: 3  4  5
row 2: 6  7  8

Cell [1,2] → node = 1×3 + 2 = 5 ✓
Cell [2,0] → node = 2×3 + 0 = 6 ✓

Reverse:
row = nodeNumber / n
col = nodeNumber % n
```

## Code

```cpp
#include <bits/stdc++.h>
using namespace std;

class DisjointSet {
public:
    vector<int> parent, size;
    
    DisjointSet(int n) {
        parent.resize(n);
        size.resize(n, 1);
        for (int i = 0; i < n; i++) {
            parent[i] = i;
        }
    }
    
    int findUPar(int node) {
        if (node == parent[node])
            return node;
        return parent[node] = findUPar(parent[node]);
    }
    
    void unionBySize(int u, int v) {
        int ulp_u = findUPar(u);
        int ulp_v = findUPar(v);
        
        if (ulp_u == ulp_v) return;
        
        if (size[ulp_u] < size[ulp_v]) {
            parent[ulp_u] = ulp_v;
            size[ulp_v] += size[ulp_u];
        }
        else {
            parent[ulp_v] = ulp_u;
            size[ulp_u] += size[ulp_v];
        }
    }
};

class Solution {
private:
    int delRow[4] = {-1, 0, 1, 0};
    int delCol[4] = {0, 1, 0, -1};
    
    bool isValid(int row, int col, int n) {
        return row >= 0 && row < n && col >= 0 && col < n;
    }
    
public:
    int largestIsland(vector<vector<int>>& grid) {
        int n = grid.size();
        DisjointSet ds(n * n);
        
        // Step 1: Build initial islands
        for (int row = 0; row < n; row++) {
            for (int col = 0; col < n; col++) {
                if (grid[row][col] == 0) continue;
                
                // Check 4 neighbors
                for (int i = 0; i < 4; i++) {
                    int newRow = row + delRow[i];
                    int newCol = col + delCol[i];
                    
                    if (isValid(newRow, newCol, n) && 
                        grid[newRow][newCol] == 1) {
                        int nodeNo = row * n + col;
                        int adjNodeNo = newRow * n + newCol;
                        ds.unionBySize(nodeNo, adjNodeNo);
                    }
                }
            }
        }
        
        int ans = 0;
        
        // Step 2: Try flipping each 0
        for (int row = 0; row < n; row++) {
            for (int col = 0; col < n; col++) {
                if (grid[row][col] == 1) continue;
                
                // Set to store unique island parents
                set<int> components;
                
                // Check 4 neighbors
                for (int i = 0; i < 4; i++) {
                    int newRow = row + delRow[i];
                    int newCol = col + delCol[i];
                    
                    if (isValid(newRow, newCol, n) && 
                        grid[newRow][newCol] == 1) {
                        int adjNodeNo = newRow * n + newCol;
                        components.insert(ds.findUPar(adjNodeNo));
                    }
                }
                
                // Calculate potential size
                int sizeTotal = 1;  // The flipped cell
                for (int parent : components) {
                    sizeTotal += ds.size[parent];
                }
                
                ans = max(ans, sizeTotal);
            }
        }
        
        // Step 3: Edge case - handle all 1s grid
        for (int cellNo = 0; cellNo < n * n; cellNo++) {
            ans = max(ans, ds.size[ds.findUPar(cellNo)]);
        }
        
        return ans;
    }
};

int main() {
    vector<vector<int>> grid = {
        {1, 0},
        {0, 1}
    };
    
    Solution sol;
    int ans = sol.largestIsland(grid);
    
    cout << "Largest island size: " << ans << endl;
    
    return 0;
}
```

**Output:**
```
Largest island size: 3
```

## Why Use SET for Components?

```
Problem: Same island might appear multiple times!

Example:
  1-1
  | |
  1-1
  |
  0  ← flip this

Cell [2,0] has neighbors:
  Up [1,0]: land, parent = A
  Right [2,1]: not shown, assume water
  
But what if:
  1-1
  | |
  1-1
  |\
  0-1  ← [2,1] is also land

Cell [2,0] has neighbors:
  Up [1,0]: land, parent = A
  Right [2,1]: land, parent = A (same island!)

Without SET:
sizeTotal = 1 + size[A] + size[A] = 1 + 4 + 4 = 9 ✗ WRONG!

With SET:
components = {A}  (duplicate ignored!)
sizeTotal = 1 + size[A] = 1 + 4 = 5 ✓ CORRECT!

SET ensures each island counted once! ✓
```

## Edge Case: All 1s Grid

```
grid = [[1, 1],
        [1, 1]]

After Step 1: One island of size 4

Step 2: Try flipping each 0
  → No 0s exist!
  → This loop contributes nothing to ans

Without Step 3:
  ans = 0 ✗ WRONG!

With Step 3:
  for (int cellNo = 0; cellNo < n*n; cellNo++) {
      ans = max(ans, ds.size[ds.findUPar(cellNo)]);
  }
  
  Checks size of existing islands
  → ans = max(0, 4) = 4 ✓

Edge case handled! ✓

Another edge case: All 0s grid
grid = [[0, 0],
        [0, 0]]

Step 2: Each 0 has no land neighbors
  sizeTotal = 1 for each (just the flipped cell)
  ans = 1

Correct! Flipping one 0 creates island of size 1 ✓
```

## Complexity Analysis

**Time Complexity:** `O(n²)`
- Step 1 (Build islands): O(n² × α(n²)) ≈ O(n²)
  - Visit each cell once
  - Each union: O(α(n²)) ≈ O(1)
- Step 2 (Try flipping): O(n²)
  - Visit each cell once
  - 4 neighbors: O(4) = O(1)
  - Set operations: O(4) = O(1)
- Step 3 (Edge case): O(n²)
- Total: **O(n²)**

**Space Complexity:** `O(n²)`
- DSU arrays (parent, size): O(n²)
- Set for components: O(4) = O(1) per cell
- Total: **O(n²)**

```
Detailed breakdown:

Building islands:
→ n² cells × 4 neighbors
→ n² × 4 × O(α(n²)) unions
→ ≈ O(n²)

Trying flips:
→ At most n² cells to check
→ Each: 4 neighbors + set insert + sum sizes
→ O(1) per cell
→ Total: O(n²)

Edge case:
→ n² cells
→ Each: O(α(n²)) ≈ O(1)
→ Total: O(n²)

Space:
→ parent array: n² entries
→ size array: n² entries
→ Set per cell: max 4 entries
→ Total: O(n²)
```

## Alternative: BFS/DFS Approach

```cpp
class Solution {
public:
    int largestIsland(vector<vector<int>>& grid) {
        int n = grid.size();
        
        // Step 1: Label each island with unique ID
        int islandId = 2;  // Start from 2 (0=water, 1=unvisited land)
        unordered_map<int, int> islandSize;
        
        for (int i = 0; i < n; i++) {
            for (int j = 0; j < n; j++) {
                if (grid[i][j] == 1) {
                    islandSize[islandId] = bfs(grid, i, j, islandId, n);
                    islandId++;
                }
            }
        }
        
        // Step 2: Try flipping each 0
        int ans = 0;
        
        for (int i = 0; i < n; i++) {
            for (int j = 0; j < n; j++) {
                if (grid[i][j] == 0) {
                    set<int> neighbors;
                    // Check 4 neighbors
                    if (i > 0 && grid[i-1][j] > 1) neighbors.insert(grid[i-1][j]);
                    if (i < n-1 && grid[i+1][j] > 1) neighbors.insert(grid[i+1][j]);
                    if (j > 0 && grid[i][j-1] > 1) neighbors.insert(grid[i][j-1]);
                    if (j < n-1 && grid[i][j+1] > 1) neighbors.insert(grid[i][j+1]);
                    
                    int size = 1;
                    for (int id : neighbors) {
                        size += islandSize[id];
                    }
                    ans = max(ans, size);
                }
            }
        }
        
        // Edge case: all 1s
        for (auto& [id, size] : islandSize) {
            ans = max(ans, size);
        }
        
        return ans;
    }
    
private:
    int bfs(vector<vector<int>>& grid, int r, int c, int id, int n) {
        queue<pair<int,int>> q;
        q.push({r, c});
        grid[r][c] = id;
        int size = 0;
        
        while (!q.empty()) {
            auto [i, j] = q.front();
            q.pop();
            size++;
            
            int dr[] = {-1, 0, 1, 0};
            int dc[] = {0, 1, 0, -1};
            
            for (int d = 0; d < 4; d++) {
                int ni = i + dr[d], nj = j + dc[d];
                if (ni >= 0 && ni < n && nj >= 0 && nj < n && grid[ni][nj] == 1) {
                    grid[ni][nj] = id;
                    q.push({ni, nj});
                }
            }
        }
        
        return size;
    }
};
```

**Comparison:**
- BFS/DFS: O(n²), modifies grid (or needs copy)
- DSU: O(n²), doesn't modify grid
- Both work, DSU is cleaner for this problem! ✓

## Summary

```
Problem: Largest island after flipping at most one 0 to 1

Key insight:
→ Flipping 0 can connect multiple islands
→ Need to find unique adjacent islands
→ Sum their sizes + 1 (flipped cell)

Algorithm:
1. Build islands using DSU
2. For each 0:
   - Find unique neighboring islands (using SET)
   - Calculate potential size
   - Track maximum
3. Handle edge case (no 0s)

Data structures:
→ DSU for connectivity
→ SET for unique island parents

Time: O(n²)
Space: O(n²)

Key tricks:
→ Use SET to avoid double-counting
→ Check existing island sizes for all-1s grid
→ 2D→1D mapping: node = row × n + col

Perfect DSU application! ✓
```
