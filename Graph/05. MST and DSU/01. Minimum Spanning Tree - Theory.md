## 1. What is a Spanning Tree?

A **spanning tree** of a graph is a tree that:

- Contains **all N nodes** present in the original graph
- Has exactly **N-1 edges**
- All nodes are **reachable from each other** (connected)
- Has **no cycles**

**What is a Minimum Spanning Tree (MST)?**

> Among all possible spanning trees of a graph, the **Minimum Spanning Tree (MST)** is the one for which the **sum of all edge weights is minimum**.

## Key Insight

```
Spanning Tree vs MST vs Shortest Path

Concept            Goal                    Edges    Algorithm
────────────────────────────────────────────────────────────────
Spanning Tree      Connect all nodes       N-1      DFS/BFS
MST                Connect all with        N-1      Prim's/Kruskal's
                   minimum total weight
Shortest Path      Minimize path from      Variable Dijkstra/Bellman
                   source to destination

Key difference:

MST: Minimizes TOTAL edge weight of tree
     (global optimization)

Shortest Path: Minimizes PATH weight from A to B
              (local optimization)

Example:

     10
A ────── B
|        |
5        5
|        |
C ────── D
     1

MST: C-D (1), A-C (5), D-B (5) → Total = 11
Shortest A to B: A-B = 10 (not in MST!)

MST ≠ Shortest path tree! ✓
```

## Intuition

```
Imagine you are building a road network to connect all villages:

→ Villages = nodes
→ Possible roads = edges with construction costs
→ Goal: Connect every village so people can travel between any two
→ Constraint: Spend the LEAST total money

You cannot build all possible roads — that would be too expensive.
You need just enough roads so every village is reachable.

The MST is the cheapest set of roads that connects everyone!

Why exactly N-1 edges?

Think of it like connecting dots with lines:
→ 2 dots need 1 line
→ 3 dots need 2 lines
→ 4 dots need 3 lines
→ N dots need N-1 lines

Any more lines and you create a loop (cycle).
Any fewer lines and someone gets disconnected.

Why avoid cycles?

A cycle means you built an extra road you don't need.
If villages A-B-C-A form a triangle, you can remove any one
road and everyone is still connected. Removing the most
expensive road saves money — that is what MST does! ✓
```

## Algorithm

```
Two main algorithms to find MST:

1. Prim's Algorithm:
   → Start from any node
   → Grow the tree by adding the cheapest edge
     that connects a visited node to an unvisited node
   → Like a spider web spreading from the center
   → Uses priority queue
   → Time: O(E log V)
   → Best for dense graphs

2. Kruskal's Algorithm:
   → Sort all edges by weight (cheapest first)
   → Pick the cheapest edge that doesn't form a cycle
   → Like picking cards from a sorted deck
   → Uses Union-Find (Disjoint Set)
   → Time: O(E log E)
   → Best for sparse graphs

Both always produce a correct MST! ✓
```

### Mathematical Properties

```
Property 1: Cut Property

If you cut the graph into two groups,
the lightest edge crossing the cut
MUST be in the MST.

Group 1: {A, B}    Group 2: {C, D}
Crossing edges: A-C (3), B-D (5), A-D (7)
Lightest: A-C (3) → Must be in MST! ✓

Property 2: Cycle Property

The heaviest edge in any cycle
is NOT in the MST (unless there's a tie).

Cycle: A-B (1), B-C (5), C-A (3)
Heaviest: B-C (5) → Not in MST! ✓

Property 3: Uniqueness

If all edge weights are distinct,
the MST is unique.
If some weights are equal,
multiple MSTs may exist.

Property 4: Edge Count

MST always has exactly N-1 edges
(minimum edges to connect N nodes)

Property 5: Subgraph

MST of a connected graph is always
a subgraph of that graph.
```

## Step-by-Step Trace

```
Find the MST for the following graph:

Given Graph:

     3       5
 0 ────── 1 ────── 2
 |        |        |
 2        4        6
 |        |        |
 5 ────── 4 ────── 3
      1        7

Edges:
0-1: 3, 1-2: 5, 0-5: 2, 1-4: 4
2-3: 6, 5-4: 1, 4-3: 7

Step-by-step MST construction (Kruskal's approach):

Sort edges by weight:

5-4: 1
0-5: 2
0-1: 3
1-4: 4
1-2: 5
2-3: 6
4-3: 7

Pick edges (avoid cycles):

1. 5-4 (1) ✓ → {5, 4}

2. 0-5 (2) ✓ → {0, 5, 4}

3. 0-1 (3) ✓ → {0, 1, 5, 4}

4. 1-4 (4) ✗ → Cycle! (0-1-4-5-0)

5. 1-2 (5) ✓ → {0, 1, 2, 5, 4}

6. 2-3 (6) ✓ → {0, 1, 2, 3, 5, 4} All connected!

MST edges: {5-4, 0-5, 0-1, 1-2, 2-3}
MST weight: 1 + 2 + 3 + 5 + 6 = 17 ✓

MST Visualization:

     3       5
 0 ────── 1 ────── 2
 |                  |
 2                  6
 |                  |
 5 ────── 4         3
      1

Sum = 17 ✓
N-1 = 5 edges ✓
All 6 nodes connected ✓
No cycles ✓
```

## Edge Cases

```
Single node:             Disconnected graph:
N = 1                   N = 4, edges only connect 2 pairs
MST has 0 edges         No spanning tree exists!
Trivial ✓               MST requires connected graph ✗

All edges same weight:   Complete graph:
Any spanning tree is    Many possible spanning trees
an MST!                 Pick any N-1 edges without cycle

Negative weights:        Zero-weight edges:
Allowed! ✓              Allowed! ✓
MST still works         Just means free connection
(just pick least total)

Self-loops:              Parallel edges:
Never in MST ✗          Keep only the lighter one
Waste of an edge
```

## Complexity Analysis

**Prim's Algorithm:**

- Time: **O(E log V)** with binary heap
- Space: **O(V)** for the priority queue and visited set

**Kruskal's Algorithm:**

- Time: **O(E log E)** for sorting edges
- Space: **O(V)** for the Union-Find data structure

```
When to use which:

Prim's:                  Kruskal's:
→ Dense graphs           → Sparse graphs
→ Graph given as         → Graph given as
  adjacency matrix         edge list
→ Already have a         → Easier to implement
  starting node            with Union-Find

Both produce correct MST! ✓
```

## Interview Questions

```
Q1: What is the difference between a spanning tree and an MST?

A: A spanning tree is any set of roads that connects all villages
   with no loops. An MST is the CHEAPEST such set. Think of it
   like buying groceries — a spanning tree is any full shopping
   cart, but the MST is the cart that costs the least.

──────────────────────────────────────────────────────────────

Q2: Why does an MST have exactly N-1 edges?

A: Imagine connecting islands with bridges. Two islands need one
   bridge. Three islands need two bridges. If you add one more
   bridge, you create a loop you don't need. If you remove one,
   an island gets cut off. So N islands need exactly N-1 bridges.

──────────────────────────────────────────────────────────────

Q3: Can a graph have more than one MST?

A: Yes, when two roads cost the same. Imagine two bridges to an
   island, both costing $5. You can pick either one and the total
   cost stays the same. But if every road has a different price,
   there is only one MST.

──────────────────────────────────────────────────────────────

Q4: Is the MST the same as the shortest path tree?

A: No. MST makes the TOTAL cost of all roads as small as possible.
   Shortest path tree makes the cost from ONE starting village to
   all others as small as possible. They solve different problems.
   The road from A to B in an MST might not be the fastest way.

──────────────────────────────────────────────────────────────

Q5: Why can't an MST have cycles?

A: A cycle means you built a road you don't need. If three
   villages form a triangle, you can knock down any one road and
   everyone can still reach each other. Knock down the most
   expensive one and you save money. MST never wastes money.

──────────────────────────────────────────────────────────────

Q6: What is the cut property?

A: Draw a line splitting the villages into two groups. The cheapest
   bridge crossing that line MUST be in the MST. Why? Without it,
   the two groups are cut off from each other. Any other bridge
   across would cost more.

──────────────────────────────────────────────────────────────

Q7: What is the cycle property?

A: In any loop of roads, the most expensive one is never in the
   MST. You can always kick it out and use a cheaper road in the
   same loop instead. The only exception is when two roads in the
   loop cost the same — then either one works.

──────────────────────────────────────────────────────────────

Q8: When should I use Prim's vs Kruskal's?

A: Use Prim's when the graph is packed with edges (dense) and you
   start from one village and grow outward like a spider web.
   Use Kruskal's when the graph has few edges (sparse) — you sort
   all edges by cost and pick the cheapest ones one by one.

──────────────────────────────────────────────────────────────

Q9: Can MST handle negative edge weights?

A: Yes. Negative weight just means someone pays YOU to build that
   road. MST still picks the set of roads with the smallest total
   cost. The algorithms work the same way — they just like
   "negative cost" roads even more.

──────────────────────────────────────────────────────────────

Q10: What happens if the graph is disconnected?

A: Then a spanning tree is impossible. Some villages are on
    separate islands with no bridges at all. You get a "forest"
    instead — one small tree for each group of connected villages.
    Normal MST algorithms assume everyone is reachable somehow.

──────────────────────────────────────────────────────────────

Q11: Does MST always include the cheapest edge?

A: Yes! The cheapest road is always helpful. If you skip it, you
    must use a more expensive road to connect those same two
    villages. That makes the total cost higher, so MST always
    grabs the cheapest edge.

──────────────────────────────────────────────────────────────

Q12: Does MST always skip the most expensive edge?

A: Not always. If the most expensive road is the ONLY bridge to a
    remote village, you have no choice — you must build it. The
    rule is: skip the most expensive road ONLY if it is part of a
    loop where a cheaper road can do the same job.

──────────────────────────────────────────────────────────────

Q13: What are real-world uses of MST?

A: Everyday examples:
    → Phone company laying cables to connect all neighborhoods
      at the lowest cost
    → City planners building roads between towns with minimum
      construction spending
    → Water department designing pipes to all houses with the
      least amount of pipe
    → Delivery companies planning routes that visit all stops
      with minimum driving

──────────────────────────────────────────────────────────────

Q14: Can we use BFS or DFS to find MST?

A: No. BFS and DFS are blind to costs. They find ANY set of roads
    that connects everyone, but they don't care if it is cheap or
    expensive. To find the cheapest set, you need Prim's or
    Kruskal's, which always look at the price tag.

──────────────────────────────────────────────────────────────

Q15: What are the most common mistakes?

A: Common mistakes and fixes:

    Mistake 1: Thinking MST gives the fastest route between two places
    → Fix: MST cares about total cost of ALL roads, not one route

    Mistake 2: Believing there is only one possible MST
    → Fix: When two edges cost the same, swapping them gives another MST

    Mistake 3: Forgetting that MST must have exactly N-1 edges
    → Fix: Count the edges — there should always be one less than nodes

    Mistake 4: Accidentally keeping a loop in the answer
    → Fix: If you can remove an edge and everyone is still connected,
           that edge should not be in the MST

    Mistake 5: Trying to use BFS or DFS for MST
    → Fix: Use Prim's or Kruskal's instead

    Mistake 6: Running MST on a graph with disconnected parts
    → Fix: Make sure every node can reach every other node first
```
