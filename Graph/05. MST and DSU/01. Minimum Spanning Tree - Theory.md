## Minimum Spanning Tree - Theory

### What is a Spanning Tree?

A **spanning tree** of a graph is a tree that:
- Contains **all N nodes** present in the original graph
- Has exactly **N-1 edges**
- All nodes are **reachable from each other** (connected)
- Has **no cycles**

---

## Understanding Spanning Trees

```
Given: Undirected weighted graph with N=5 nodes and M=6 edges

Original Graph:
    1
0 ────── 1
|  \     |
4   3    2
|    \   |
3 ──── 2
    5

(Multiple edges connecting 5 nodes with various weights)
```

### Multiple Spanning Trees Are Possible

```
Spanning Tree 1:          Spanning Tree 2:          Spanning Tree 3:
    1                         1                         1
0 ────── 1              0 ────── 1              0 ────── 1
|        |                       |              |
4        2              3        2              4        2
|        |              |        |              |        |
3        2              3        2              3        2
                  
Sum = 1+4+2+5 = 12      Sum = 1+3+2+5 = 11      Sum = 1+4+2+3 = 10

A graph can have MANY spanning trees!
Each has N-1 = 4 edges connecting all 5 nodes.
```

## What is a Minimum Spanning Tree?

> Among all possible spanning trees of a graph, the **Minimum Spanning Tree (MST)** is the one for which the **sum of all edge weights is minimum**.

```
All possible spanning trees and their edge weight sums:

Tree 1: sum = 18
Tree 2: sum = 24
Tree 3: sum = 18
Tree 4: sum = 16 ← MINIMUM! ✓
Tree 5: sum = 20
...

MST = Tree with minimum total edge weight

Properties of MST:
→ Contains all N nodes ✓
→ Has exactly N-1 edges ✓
→ All nodes connected ✓
→ No cycles ✓
→ Minimum total edge weight ✓
```

```
Note: There may exist MULTIPLE minimum spanning trees
for a graph, just like a graph can have multiple spanning trees.

Example:
    1       1
A ───── B ───── C
|               |
1               1
|               |
D ──────────── E
        1

MSTs (both with sum = 4):
MST 1: A-B, B-C, A-D, D-E
MST 2: A-B, B-C, C-E, A-D

Both are valid MSTs! ✓
```

## Why MST Matters

```
Real-world applications:

1. Network Design:
   Connect all computers with minimum cable length
   
   Computers: A, B, C, D
   Cable costs:
   A-B: $100, A-C: $200, B-C: $50, C-D: $80
   
   MST: B-C ($50), C-D ($80), A-B ($100)
   Total: $230 (minimum cost to connect all!)

2. Road Network:
   Connect all villages with minimum road construction
   → MST gives optimal road layout

3. Electrical Grid:
   Wire all houses with minimum wiring
   → MST minimizes total wire length

4. Cluster Analysis:
   Group similar data points
   → Remove heaviest edges from MST

5. Approximation Algorithms:
   TSP (Travelling Salesman) approximation
   → MST provides 2-approximation
```

## Spanning Tree vs MST vs Shortest Path

```
Concept            Goal                    Edges    Algorithm
────────────────────────────────────────────────────────────────
Spanning Tree      Connect all nodes       N-1      DFS/BFS
MST                Connect all with        N-1      Prim's/Kruskal's
                   minimum total weight
Shortest Path      Minimize path from      Variable Dijkstra/Bellman
                   source to destination

Key difference:

MST: Minimizes TOTAL edge weight of tree
     (global optimization)

Shortest Path: Minimizes PATH weight from A to B
              (local optimization)

Example:
     10
A ────── B
|        |
1        1
|        |
C ────── D
     1

MST: A-C (1), C-D (1), D-B (1) → Total = 3
Shortest A to B: A-C-D-B = 3

But consider:
     10
A ────── B
|        |
5        5
|        |
C ────── D
     1

MST: C-D (1), A-C (5), D-B (5) → Total = 11
Shortest A to B: A-B = 10 (not in MST!)

MST ≠ Shortest path tree! ✓
```

## Mathematical Properties

```
Property 1: Cut Property
If you cut the graph into two groups,
the lightest edge crossing the cut
MUST be in the MST.

Group 1: {A, B}    Group 2: {C, D}
Crossing edges: A-C (3), B-D (5), A-D (7)
Lightest: A-C (3) → Must be in MST! ✓

Property 2: Cycle Property
The heaviest edge in any cycle
is NOT in the MST (unless there's a tie).

Cycle: A-B (1), B-C (5), C-A (3)
Heaviest: B-C (5) → Not in MST! ✓

Property 3: Uniqueness
If all edge weights are distinct,
the MST is unique.

If some weights are equal,
multiple MSTs may exist.

Property 4: Edge Count
MST always has exactly N-1 edges
(minimum edges to connect N nodes)

Property 5: Subgraph
MST of a connected graph is always
a subgraph of that graph.
```

## Algorithms for Finding MST

```
Two main algorithms:

1. Prim's Algorithm:
   → Greedy approach
   → Start from any node
   → Keep adding cheapest edge connecting
     visited set to unvisited set
   → Uses priority queue
   → Time: O(E log V)
   → Best for dense graphs

2. Kruskal's Algorithm:
   → Greedy approach
   → Sort all edges by weight
   → Pick cheapest edge that doesn't form cycle
   → Uses Union-Find (Disjoint Set)
   → Time: O(E log E)
   → Best for sparse graphs

Comparison:
Property              Prim's              Kruskal's
──────────────────────────────────────────────────────────
Approach              Vertex-based         Edge-based
Start                 Any vertex           Lightest edge
Data Structure        Priority Queue       Union-Find
Time                  O(E log V)           O(E log E)
Best for              Dense graphs         Sparse graphs
Implementation        BFS-like             Sort + DSU

Both always produce correct MST! ✓
(Will be discussed in detail in upcoming articles)
```

## Common Misconceptions

```
❌ "MST gives shortest path between any two nodes"
✓ MST minimizes TOTAL weight, not individual paths

❌ "MST is unique for every graph"
✓ Multiple MSTs possible if edge weights are not distinct

❌ "Removing any edge from MST disconnects the graph"
✓ This is CORRECT! Every edge in MST is a bridge

❌ "Adding any edge to MST creates exactly one cycle"
✓ This is CORRECT! Since MST has N-1 edges, N edges = 1 cycle

❌ "MST always contains the shortest edge of the graph"
✓ This is CORRECT! Lightest edge is always in MST

❌ "MST never contains the heaviest edge"
✓ WRONG! Heaviest edge may be in MST if it's a bridge
   (only non-bridge heaviest edge is excluded)

❌ "Prim's and Kruskal's always give the same MST"
✓ They give MSTs with same total weight
   But actual edges may differ if multiple MSTs exist
```

## Summary

```
Spanning Tree:
→ Subgraph connecting all N nodes with N-1 edges
→ No cycles, all nodes reachable
→ Multiple possible for any graph

Minimum Spanning Tree (MST):
→ Spanning tree with minimum total edge weight
→ May not be unique (multiple MSTs possible)
→ Found using Prim's or Kruskal's algorithm

Key Takeaways:
1. MST has N nodes, N-1 edges
2. MST minimizes total weight (not individual paths)
3. Lightest edge always in MST
4. Heaviest edge in a cycle never in MST
5. Every MST edge is a bridge (removing disconnects)
6. Two algorithms: Prim's (vertex-based) & Kruskal's (edge-based)
```
