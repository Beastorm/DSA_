## 1. What is a Spanning Tree?

A **spanning tree** of a graph is a tree that:

- Contains **all N nodes** present in the original graph
- Has exactly **N-1 edges**
- All nodes are **reachable from each other** (connected)
- Has **no cycles**

**What is a Minimum Spanning Tree (MST)?**

> Among all possible spanning trees of a graph, the **Minimum Spanning Tree (MST)** is the one for which the **sum of all edge weights is minimum**.

## Key Insight

```
Spanning Tree vs MST vs Shortest Path

Concept            Goal                    Edges    Algorithm
────────────────────────────────────────────────────────────────
Spanning Tree      Connect all nodes       N-1      DFS/BFS
MST                Connect all with        N-1      Prim's/Kruskal's
                   minimum total weight
Shortest Path      Minimize path from      Variable Dijkstra/Bellman
                   source to destination

Key difference:

MST: Minimizes TOTAL edge weight of tree
     (global optimization)

Shortest Path: Minimizes PATH weight from A to B
              (local optimization)

Example:

     10
A ────── B
|        |
5        5
|        |
C ────── D
     1

MST: C-D (1), A-C (5), D-B (5) → Total = 11
Shortest A to B: A-B = 10 (not in MST!)

MST ≠ Shortest path tree! ✓
```

## Intuition

```
Imagine you are building a road network to connect all villages:

→ Villages = nodes
→ Possible roads = edges with construction costs
→ Goal: Connect every village so people can travel between any two
→ Constraint: Spend the LEAST total money

You cannot build all possible roads — that would be too expensive.
You need just enough roads so every village is reachable.

The MST is the cheapest set of roads that connects everyone!

Why exactly N-1 edges?

Think of it like connecting dots with lines:
→ 2 dots need 1 line
→ 3 dots need 2 lines
→ 4 dots need 3 lines
→ N dots need N-1 lines

Any more lines and you create a loop (cycle).
Any fewer lines and someone gets disconnected.

Why avoid cycles?

A cycle means you built an extra road you don't need.
If villages A-B-C-A form a triangle, you can remove any one
road and everyone is still connected. Removing the most
expensive road saves money — that is what MST does! ✓
```

## Algorithm

```
Two main algorithms to find MST:

1. Prim's Algorithm:
   → Start from any node
   → Grow the tree by adding the cheapest edge
     that connects a visited node to an unvisited node
   → Like a spider web spreading from the center
   → Uses priority queue
   → Time: O(E log V)
   → Best for dense graphs

2. Kruskal's Algorithm:
   → Sort all edges by weight (cheapest first)
   → Pick the cheapest edge that doesn't form a cycle
   → Like picking cards from a sorted deck
   → Uses Union-Find (Disjoint Set)
   → Time: O(E log E)
   → Best for sparse graphs

Both always produce a correct MST! ✓
```

### Mathematical Properties

```
Property 1: Cut Property

If you cut the graph into two groups,
the lightest edge crossing the cut
MUST be in the MST.

Group 1: {A, B}    Group 2: {C, D}
Crossing edges: A-C (3), B-D (5), A-D (7)
Lightest: A-C (3) → Must be in MST! ✓

Property 2: Cycle Property

The heaviest edge in any cycle
is NOT in the MST (unless there's a tie).

Cycle: A-B (1), B-C (5), C-A (3)
Heaviest: B-C (5) → Not in MST! ✓

Property 3: Uniqueness

If all edge weights are distinct,
the MST is unique.
If some weights are equal,
multiple MSTs may exist.

Property 4: Edge Count

MST always has exactly N-1 edges
(minimum edges to connect N nodes)

Property 5: Subgraph

MST of a connected graph is always
a subgraph of that graph.
```

## Step-by-Step Trace

```
Find the MST for the following graph:

Given Graph:

     3       5
 0 ────── 1 ────── 2
 |        |        |
 2        4        6
 |        |        |
 5 ────── 4 ────── 3
      1        7

Edges:
0-1: 3, 1-2: 5, 0-5: 2, 1-4: 4
2-3: 6, 5-4: 1, 4-3: 7

Step-by-step MST construction (Kruskal's approach):

Sort edges by weight:

5-4: 1
0-5: 2
0-1: 3
1-4: 4
1-2: 5
2-3: 6
4-3: 7

Pick edges (avoid cycles):

1. 5-4 (1) ✓ → {5, 4}

2. 0-5 (2) ✓ → {0, 5, 4}

3. 0-1 (3) ✓ → {0, 1, 5, 4}

4. 1-4 (4) ✗ → Cycle! (0-1-4-5-0)

5. 1-2 (5) ✓ → {0, 1, 2, 5, 4}

6. 2-3 (6) ✓ → {0, 1, 2, 3, 5, 4} All connected!

MST edges: {5-4, 0-5, 0-1, 1-2, 2-3}
MST weight: 1 + 2 + 3 + 5 + 6 = 17 ✓

MST Visualization:

     3       5
 0 ────── 1 ────── 2
 |                  |
 2                  6
 |                  |
 5 ────── 4         3
      1

Sum = 17 ✓
N-1 = 5 edges ✓
All 6 nodes connected ✓
No cycles ✓
```

## Edge Cases

```
Single node:             Disconnected graph:
N = 1                   N = 4, edges only connect 2 pairs
MST has 0 edges         No spanning tree exists!
Trivial ✓               MST requires connected graph ✗

All edges same weight:   Complete graph:
Any spanning tree is    Many possible spanning trees
an MST!                 Pick any N-1 edges without cycle

Negative weights:        Zero-weight edges:
Allowed! ✓              Allowed! ✓
MST still works         Just means free connection
(just pick least total)

Self-loops:              Parallel edges:
Never in MST ✗          Keep only the lighter one
Waste of an edge
```

## Complexity Analysis

**Prim's Algorithm:**

- Time: **O(E log V)** with binary heap
- Space: **O(V)** for the priority queue and visited set

**Kruskal's Algorithm:**

- Time: **O(E log E)** for sorting edges
- Space: **O(V)** for the Union-Find data structure

```
When to use which:

Prim's:                  Kruskal's:
→ Dense graphs           → Sparse graphs
→ Graph given as         → Graph given as
  adjacency matrix         edge list
→ Already have a         → Easier to implement
  starting node            with Union-Find

Both produce correct MST! ✓
```

## Interview Questions

```
Q1: What is the difference between a spanning tree and an MST?

A: A spanning tree connects all nodes with N-1 edges and no cycles.
   An MST is a spanning tree with the smallest possible total edge
   weight. Think of it like building roads between villages — a
   spanning tree is any valid road network, but the MST is the
   cheapest one.

──────────────────────────────────────────────────────────────

Q2: Why does an MST have exactly N-1 edges?

A: Think of connecting dots with strings. Two dots need one string.
   Three dots need two strings. N dots need N-1 strings. Any more
   creates a loop. Any fewer leaves someone disconnected.

──────────────────────────────────────────────────────────────

Q3: Can a graph have more than one MST?

A: Yes, if some edges have the same weight. Imagine two different
   roads between the same villages that cost the same. You can swap
   them and the total cost stays the same. If all weights are
   different, the MST is unique.

──────────────────────────────────────────────────────────────

Q4: Is the MST the same as the shortest path tree?

A: No! MST connects ALL nodes as cheaply as possible overall.
   Shortest path tree connects ONE source to all others as cheaply
   as possible. The path between two nodes in an MST is not
   always the shortest path between them.

──────────────────────────────────────────────────────────────

Q5: Why can't an MST have cycles?

A: A cycle means you built an extra road you don't need. If
   villages A-B-C form a triangle, you can remove any one road
   and everyone is still connected. Removing the most expensive
   one saves money. MST always avoids waste.

──────────────────────────────────────────────────────────────

Q6: What is the cut property?

A: If you draw a line dividing the graph into two groups, the
   cheapest road crossing that line MUST be in the MST. Why?
   Because without it, the two groups would be disconnected, and
   any other crossing road would cost more.

──────────────────────────────────────────────────────────────

Q7: What is the cycle property?

A: In any loop of roads, the most expensive one is never in the
   MST (unless there is a tie). You can always replace it with a
   cheaper road in the loop and save money.

──────────────────────────────────────────────────────────────

Q8: When should I use Prim's vs Kruskal's?

A: Use Prim's for dense graphs (lots of edges) where the graph is
   given as a matrix. Use Kruskal's for sparse graphs (few edges)
   where the graph is given as a list of edges. Both give the
   same total weight.

──────────────────────────────────────────────────────────────

Q9: Can MST handle negative edge weights?

A: Yes. MST simply picks the set of edges with the smallest total
   weight. Negative weights are fine — they just mean some
   connections are "profitable" or free. The algorithms work the
   same way.

──────────────────────────────────────────────────────────────

Q10: What happens if the graph is disconnected?

A: A spanning tree cannot exist because some nodes are unreachable
    from others. You would get a "spanning forest" instead — a
    separate tree for each connected group. Standard MST algorithms
    assume the graph is connected.

──────────────────────────────────────────────────────────────

Q11: Does MST always include the lightest edge in the graph?

A: Yes! The cheapest road is always useful. If you don't include
    it, you must use a more expensive road to connect those two
    villages, which makes the total cost higher.

──────────────────────────────────────────────────────────────

Q12: Does MST always exclude the heaviest edge?

A: Not always. If the heaviest edge is the ONLY way to connect a
    village to the rest, it must be included. The rule is: the
    heaviest edge in a CYCLE is excluded, not the heaviest edge
    overall.

──────────────────────────────────────────────────────────────

Q13: What are real-world uses of MST?

A: Common uses include:
    → Laying cables to connect all buildings at minimum cost
    → Building roads between towns with minimum construction
    → Designing efficient water pipe networks
    → Approximating solutions for the travelling salesman problem
    → Clustering data points in machine learning

──────────────────────────────────────────────────────────────

Q14: Can we use BFS or DFS to find MST?

A: No. BFS and DFS ignore edge weights. They find any spanning
    tree, not the cheapest one. To find the MST, you need Prim's
    or Kruskal's which specifically consider weights.

──────────────────────────────────────────────────────────────

Q15: What are the most common mistakes?

A: Common mistakes and fixes:

    Mistake 1: Thinking MST gives shortest paths between all pairs
    → Fix: MST minimizes total weight, not individual paths

    Mistake 2: Assuming MST is always unique
    → Fix: Multiple MSTs exist when weights tie

    Mistake 3: Forgetting N-1 edges rule
    → Fix: MST always has exactly N-1 edges

    Mistake 4: Including a cycle in the MST
    → Fix: Reject any edge that forms a cycle

    Mistake 5: Using BFS/DFS for MST
    → Fix: Use Prim's or Kruskal's

    Mistake 6: Not checking if graph is connected
    → Fix: MST only exists for connected graphs
```
