## Number of Islands - II - Online Queries - DSU

### Problem Statement:

You are given an `n × m` grid (2D matrix) and an array of `k` operations. Initially, the grid is all **water (0)**. Each operation converts a cell from water to **land (1)** at position `[row, col]`.

After each operation, **count the number of islands** in the grid.

> **Note:** An island is a group of 1s (land) connected by 4-directional adjacency (up, down, left, right).

Return an array of size `k` showing the number of islands after each operation.

## Key Insight

```
Problem: Dynamic island counting with online queries

Challenge:
→ After each operation, need to know island count
→ Can't recount entire grid each time (too slow!)
→ Need to update incrementally

Naive approach:
After each operation, run DFS/BFS to count islands
Time: O(k × n × m) → Too slow!

DSU approach:
→ Each cell is a node
→ When adding land, check 4 neighbors
→ If neighbor is land: union them
→ Track island count dynamically
Time: O(k × α(n×m)) ≈ O(k) → Efficient! ✓

Key observations:
1. Adding land initially creates new island (+1)
2. If new land connects to existing island:
   → Merge them (union)
   → If they were separate: islands count -1
3. DSU tracks connected components efficiently
```

---

## Algorithm

```
Initialize:
→ DSU of size n × m
→ vis[n][m] = all 0 (no land yet)
→ cnt = 0 (island count)
→ ans = [] (result array)

For each operation [row, col]:

  Step 1: Check if already land
    If vis[row][col] == 1:
      → Already land, no change
      → ans.push_back(cnt)
      → continue

  Step 2: Add new land
    vis[row][col] = 1
    cnt++ (new island created)

  Step 3: Check 4 neighbors
    For each direction (up, down, left, right):
      If neighbor is valid and is land:
        nodeNo = row × m + col
        adjNodeNo = adjRow × m + adjCol
        
        If find(nodeNo) != find(adjNodeNo):
          → Different components, merge them
          → union(nodeNo, adjNodeNo)
          → cnt-- (two islands merged into one)

  Step 4: Record current count
    ans.push_back(cnt)

Return ans
```

## Node Numbering: 2D to 1D

```
Why convert 2D coordinates to 1D?

DSU works with integer node IDs
Grid has 2D coordinates [row, col]
Need mapping: (row, col) → nodeID

Formula: nodeID = row × m + col

Example: n=4, m=5 grid

  col: 0  1  2  3  4
row 0: 0  1  2  3  4
row 1: 5  6  7  8  9
row 2: 10 11 12 13 14
row 3: 15 16 17 18 19

Cell [1,2] → node 1×5 + 2 = 7 ✓
Cell [3,4] → node 3×5 + 4 = 19 ✓

Reverse (if needed):
row = nodeID / m
col = nodeID % m

This linearizes the 2D grid for DSU! ✓
```

## Code

```cpp
#include <bits/stdc++.h>
using namespace std;

class DisjointSet {
    vector<int> rank, parent, size;
    
public:
    DisjointSet(int n) {
        rank.resize(n + 1, 0);
        parent.resize(n + 1);
        size.resize(n + 1, 1);
        
        for (int i = 0; i <= n; i++) {
            parent[i] = i;
        }
    }
    
    int findUPar(int node) {
        if (node == parent[node])
            return node;
        return parent[node] = findUPar(parent[node]);
    }
    
    void unionBySize(int u, int v) {
        int ulp_u = findUPar(u);
        int ulp_v = findUPar(v);
        
        if (ulp_u == ulp_v) return;
        
        if (size[ulp_u] < size[ulp_v]) {
            parent[ulp_u] = ulp_v;
            size[ulp_v] += size[ulp_u];
        }
        else {
            parent[ulp_v] = ulp_u;
            size[ulp_u] += size[ulp_v];
        }
    }
};

class Solution {
private:
    bool isValid(int row, int col, int n, int m) {
        return row >= 0 && row < n && col >= 0 && col < m;
    }
    
public:
    vector<int> numOfIslands(int n, int m, vector<vector<int>>& operators) {
        DisjointSet ds(n * m);
        vector<vector<int>> vis(n, vector<int>(m, 0));
        
        int cnt = 0;
        vector<int> ans;
        
        // Direction arrays for 4 neighbors
        int dr[] = {-1, 0, 1, 0};
        int dc[] = {0, 1, 0, -1};
        
        for (auto& op : operators) {
            int row = op[0];
            int col = op[1];
            
            // If already land, no change
            if (vis[row][col] == 1) {
                ans.push_back(cnt);
                continue;
            }
            
            // Mark as land
            vis[row][col] = 1;
            cnt++;  // New island created
            
            // Check 4 neighbors
            for (int i = 0; i < 4; i++) {
                int adjRow = row + dr[i];
                int adjCol = col + dc[i];
                
                if (isValid(adjRow, adjCol, n, m) && vis[adjRow][adjCol] == 1) {
                    int nodeNo = row * m + col;
                    int adjNodeNo = adjRow * m + adjCol;
                    
                    // If different components, merge
                    if (ds.findUPar(nodeNo) != ds.findUPar(adjNodeNo)) {
                        cnt--;  // Merging two islands
                        ds.unionBySize(nodeNo, adjNodeNo);
                    }
                }
            }
            
            ans.push_back(cnt);
        }
        
        return ans;
    }
};

int main() {
    int n = 4, m = 5;
    vector<vector<int>> operators = {
        {0, 0}, {0, 0}, {1, 1}, {1, 0}, {0, 1},
        {0, 3}, {1, 3}, {0, 4}, {3, 2}, {2, 2},
        {1, 2}, {0, 2}
    };
    
    Solution obj;
    vector<int> ans = obj.numOfIslands(n, m, operators);
    
    cout << "Island count after each operation: ";
    for (int count : ans) {
        cout << count << " ";
    }
    cout << endl;
    
    return 0;
}
```

**Output:**
```
Island count after each operation: 1 1 2 1 1 2 2 2 3 3 1 1
```

## Merging Logic

```
When does merging happen?

When new land cell has land neighbors!

Example scenario:
Current islands: A and B (separate)

Grid:
1 0 0 1    A: cells {0,0}
0 0 0 0    B: cells {0,3}
0 0 0 0

Add land at [0,1]:
1 1 0 1    
0 0 0 0

Check neighbors of [0,1]:
→ Left [0,0]: land, component A
→ Right [0,2]: water
→ Others: water/out of bounds

Merge [0,1] with A:
→ Before: cnt = 2 (A and B)
→ Add [0,1]: cnt = 3
→ Merge with A: cnt = 2
→ Final: cnt = 2 ✓

Add land at [0,2]:
1 1 1 1
0 0 0 0

Check neighbors of [0,2]:
→ Left [0,1]: land, component A
→ Right [0,3]: land, component B

Merge with [0,1] (component A):
→ cnt = 3 (new land)
→ Merge with A: cnt = 2
Merge with [0,3] (component B):
→ find([0,1]) and find([0,3]) now different
→ Merge A and B: cnt = 1
→ Final: cnt = 1 ✓

All islands merged into one!
```

## Why Check Ultimate Parents?

```
Critical check:
if (ds.findUPar(nodeNo) != ds.findUPar(adjNodeNo)) {
    cnt--;
    ds.unionBySize(nodeNo, adjNodeNo);
}

Why not just check if neighbor is land?

Example showing why:

Grid:
1 1 0
1 0 0
0 0 0

All three 1s already connected (component A)

Add land at [1,1]:
1 1 0
1 1 0
0 0 0

Check neighbors:
→ Up [0,1]: land, component A
→ Left [1,0]: land, component A

Without ultimate parent check:
cnt = 1 (initially one island)
Add [1,1]: cnt = 2
Merge with [0,1]: cnt = 1
Merge with [1,0]: cnt = 0 ✗ WRONG!

With ultimate parent check:
find([1,1]) after first merge = A
find([1,0]) = A
→ Same ultimate parent!
→ Don't decrement again ✓
→ cnt stays 1 ✓

Ultimate parent check prevents double-counting! ✓
```

## Complexity Analysis

**Time Complexity:** `O(k × α(n×m)) ≈ O(k)`
- k operations
- Each operation: check 4 neighbors
- Each neighbor check: 2 find() + 1 union()
- find() and union(): O(α(n×m)) ≈ O(1)
- Total: **O(k × α(n×m)) ≈ O(k)**

**Space Complexity:** `O(n × m)`
- DSU arrays: O(n × m)
- Visited array: O(n × m)
- Answer array: O(k)
- Total: **O(n × m + k)**

```
Detailed breakdown:

Per operation:
→ Check if already land: O(1)
→ Mark as land: O(1)
→ Check 4 neighbors: O(4) = O(1)
→ For each land neighbor:
  - 2 × find(): O(α(n×m))
  - 1 × union(): O(α(n×m))
→ Total per operation: O(α(n×m))

Total time: k × O(α(n×m)) ≈ O(k)

Compare with naive approach:
→ After each operation, DFS/BFS entire grid
→ Time: O(k × n × m)
→ For k=10000, n=1000, m=1000: 10^13 operations!
→ DSU approach: only 10^4 operations! ✓

Space:
→ parent, rank, size: 3 × (n×m) = O(n×m)
→ vis[][]: n×m = O(n×m)
→ ans[]: k = O(k)
→ Total: O(n×m + k)
```

```

## Common Mistakes

```
❌ Mistake 1: Not handling duplicates
vis[row][col] = 1;
cnt++;  // WRONG! Even if already land

✓ Correct:
if (vis[row][col] == 1) {
    ans.push_back(cnt);
    continue;
}

❌ Mistake 2: Not checking ultimate parents
if (vis[adjRow][adjCol] == 1) {
    cnt--;  // WRONG! Might merge same component twice

✓ Correct:
if (ds.findUPar(nodeNo) != ds.findUPar(adjNodeNo)) {
    cnt--;
    ds.unionBySize(nodeNo, adjNodeNo);
}

❌ Mistake 3: Wrong node numbering
int nodeNo = row + col;  // WRONG!

## Why DSU is Perfect Here

```
Problem requirements:
1. Online queries (process one at a time)
2. Dynamic connectivity (islands merge/form)
3. Need count after EACH operation

Why DSU wins:

✓ Incremental updates:
  → Add land: O(1)
  → Check/merge neighbors: O(α(n×m)) ≈ O(1)
  → Total per operation: O(1)

✓ Dynamic connectivity:
  → Union connects components
  → Find checks if connected
  → Perfect for merging islands!

✓ Efficient counting:
  → Track count directly
  → Increment when adding
  → Decrement when merging
  → No need to recount!

Alternative (DFS/BFS after each op):
✗ Time: O(k × n × m)
✗ Must recount entire grid each time
✗ Too slow for online queries!

DSU is THE solution for this problem! ✓
```

## Summary

```
Problem: Count islands after each online operation

Key challenges:
→ Online queries (can't batch process)
→ Dynamic connectivity (islands merge)
→ Need efficient count updates

Solution: Disjoint Set Union (DSU)

Algorithm:
1. For each operation [row, col]:
   - If duplicate: skip
   - Mark as land: cnt++
   - Check 4 neighbors
   - If neighbor is land from different component:
     → Merge components: cnt--
   - Record current count

Data structures:
→ DSU for connectivity
→ vis[][] for land tracking
→ cnt for island count

Time: O(k × α(n×m)) ≈ O(k)
Space: O(n × m)

Why DSU?
→ Efficient online updates
→ Natural merging mechanism
→ Direct component counting
→ Perfect for this problem! ✓
```
