## Prim's Algorithm - Minimum Spanning Tree

### Problem Statement:

Given a **weighted, undirected, and connected graph** of `V` vertices and `E` edges, find the **sum of weights** of the edges of the Minimum Spanning Tree (MST).

Sometimes you may also need to find the actual MST edges, stored as pairs `{u, v}`.

---

## Prim's Algorithm - Key Idea

```
Greedy Strategy: Grow the MST one edge at a time

Start from any node (typically node 0)
At each step:
  → Among all edges connecting MST to non-MST nodes
  → Pick the edge with MINIMUM weight
  → Add that edge and node to MST

Repeat until all nodes are in MST

Why does this work?
→ Cut Property: Minimum weight edge crossing any cut
  must be in the MST
→ At each step, we're finding minimum edge crossing
  the cut between {MST nodes} and {non-MST nodes}
→ Greedy choice is always optimal! ✓
```

---

## Algorithm

```
Data Structures Needed:

1. Priority Queue (Min-Heap):
   Stores: {weight, node} or {weight, node, parent}
   → Always gives us the minimum weight edge
   → Helps make greedy choice efficiently

2. Visited Array:
   Tracks which nodes are already in MST
   → Prevents cycles
   → Ensures we don't revisit nodes

3. Sum Variable:
   Accumulates total MST weight

4. MST Array (optional):
   Stores actual MST edges as {parent, node} pairs
   → Only needed if we want the tree structure
   → Not needed if only sum is required

Algorithm Steps:

1. Initialize:
   visited[V] = all false
   sum = 0
   pq.push({0, 0}) or {0, 0, -1}
   (weight=0, start node=0, parent=-1)

2. While priority queue is not empty:
   a. Pop {wt, node} with minimum weight
   
   b. If node is already visited: continue
   
   c. Mark node as visited
   
   d. Add wt to sum
   
   e. (Optional) Store {parent, node} in MST
   
   f. For each unvisited neighbor:
      → Push {edgeWeight, neighbor} to pq

3. Return sum (and MST if needed)
```

## Code

```cpp
#include <bits/stdc++.h>
using namespace std;

class Solution {
public:
    // Function to find the sum of weights of MST edges
    int spanningTree(int V, vector<vector<int>> adj[]) {
        
        // Min-heap: {weight, node}
        priority_queue<pair<int, int>, vector<pair<int, int>>, greater<pair<int, int>>> pq;
        
        // Visited array
        vector<int> vis(V, 0);
        
        // Start from node 0 with weight 0
        pq.push({0, 0});
        int sum = 0;
        
        while (!pq.empty()) {
            auto it = pq.top();
            pq.pop();
            
            int wt = it.first;
            int node = it.second;
            
            // Skip if already in MST
            if (vis[node] == 1) continue;
            
            // Add to MST
            vis[node] = 1;
            sum += wt;
            
            // Explore neighbors
            for (auto it : adj[node]) {
                int adjNode = it[0];
                int edW = it[1];
                
                if (!vis[adjNode]) {
                    pq.push({edW, adjNode});
                }
            }
        }
        
        return sum;
    }
};

int main() {
    int V = 5;
    vector<vector<int>> edges = {
        {0, 1, 2}, {0, 2, 1}, {1, 2, 1},
        {2, 3, 2}, {3, 4, 1}, {4, 2, 2}
    };
    
    // Build adjacency list
    vector<vector<int>> adj[V];
    for (auto it : edges) {
        vector<int> tmp(2);
        tmp[0] = it[1];
        tmp[1] = it[2];
        adj[it[0]].push_back(tmp);
        
        tmp[0] = it[0];
        tmp[1] = it[2];
        adj[it[1]].push_back(tmp);
    }
    
    Solution obj;
    int sum = obj.spanningTree(V, adj);
    
    cout << "Sum of MST edge weights: " << sum << endl;
    
    return 0;
}
```

**Output:** `Sum of MST edge weights: 5`

## Code with MST Edge Tracking

```cpp
#include <bits/stdc++.h>
using namespace std;

class Solution {
public:
    // Returns {sum, MST edges}
    pair<int, vector<pair<int, int>>> spanningTreeWithEdges(int V, vector<vector<int>> adj[]) {
        // Min-heap: {weight, node, parent}
        priority_queue<tuple<int, int, int>, vector<tuple<int, int, int>>, greater<tuple<int, int, int>>> pq;
        
        vector<int> vis(V, 0);
        vector<pair<int, int>> mstEdges;
        
        // Start: weight=0, node=0, parent=-1
        pq.push({0, 0, -1});
        int sum = 0;
        
        while (!pq.empty()) {
            auto [wt, node, parent] = pq.top();
            pq.pop();
            
            if (vis[node]) continue;
            
            vis[node] = 1;
            sum += wt;
            
            // Store edge (skip first node with parent=-1)
            if (parent != -1) {
                mstEdges.push_back({parent, node});
            }
            
            // Explore neighbors
            for (auto it : adj[node]) {
                int adjNode = it[0];
                int edW = it[1];
                
                if (!vis[adjNode]) {
                    pq.push({edW, adjNode, node});
                }
            }
        }
        
        return {sum, mstEdges};
    }
};

int main() {
    int V = 5;
    vector<vector<int>> edges = {
        {0, 1, 2}, {0, 2, 1}, {1, 2, 1},
        {2, 3, 2}, {3, 4, 1}, {4, 2, 2}
    };
    
    vector<vector<int>> adj[V];
    for (auto it : edges) {
        vector<int> tmp(2);
        tmp[0] = it[1];
        tmp[1] = it[2];
        adj[it[0]].push_back(tmp);
        
        tmp[0] = it[0];
        tmp[1] = it[2];
        adj[it[1]].push_back(tmp);
    }
    
    Solution obj;
    auto [sum, mstEdges] = obj.spanningTreeWithEdges(V, adj);
    
    cout << "MST sum: " << sum << endl;
    cout << "MST edges:\n";
    for (auto [u, v] : mstEdges) {
        cout << u << " - " << v << endl;
    }
    
    return 0;
}
```

**Output:**
```
MST sum: 5
MST edges:
0 - 2
1 - 2
2 - 3
3 - 4
```

## Why Greedy Works

```
Proof using Cut Property:

Cut Property states:
"For any cut dividing vertices into two sets,
 the minimum weight edge crossing the cut
 must be in the MST."

Prim's algorithm:
At each step, MST nodes form one set,
non-MST nodes form another set.

We always pick minimum weight edge crossing this cut!
→ This edge MUST be in MST (by cut property)
→ Greedy choice is optimal! ✓

Example visualization:

Step 1:
MST set: {0}
Non-MST: {1, 2, 3, 4}
Cut edges: 0-1(2), 0-2(1)
Minimum: 0-2(1) ← Pick this! ✓

Step 2:
MST set: {0, 2}
Non-MST: {1, 3, 4}
Cut edges: 2-1(1), 2-3(2), 2-4(2), 0-1(2)
Minimum: 2-1(1) ← Pick this! ✓

And so on...
Each greedy choice is guaranteed optimal! ✓
```

## Priority Queue Duplicates

```
Why can same node appear multiple times in PQ?

Example:
       2
  0 ──── 1
   \    /
    1  1
     \/
      2

From node 0:
→ Add {2, 1} (via edge 0-1)
→ Add {1, 2} (via edge 0-2)

From node 2 (processed before node 1):
→ Add {1, 1} (via edge 2-1)

Priority queue now has:
{1, 1}, {2, 1}
 ↑       ↑
Same node 1, different weights!

When we pop {1, 1}:
→ Mark node 1 as visited
→ Add weight 1 to sum

When we pop {2, 1}:
→ Node 1 already visited
→ Skip! (continue) ✓

This is why we check vis[node] before processing!

Duplicates are OK:
→ We always process best (minimum) weight first
→ Later duplicates are skipped
→ No incorrect behavior ✓
```

## Starting Node Choice

```
Question: Can we start from any node?

Answer: YES! ✓

MST is unique (if all weights distinct)
regardless of starting point.

Example: Start from node 0 vs node 2

Start from 0:           Start from 2:
Step 1: Add 0-2 (1)    Step 1: Add 2-0 (1)
Step 2: Add 2-1 (1)    Step 2: Add 2-1 (1)
Step 3: Add 2-3 (2)    Step 3: Add 2-3 (2)
Step 4: Add 3-4 (1)    Step 4: Add 3-4 (1)

Same edges, same sum! ✓

Why does this work?
→ MST is a property of the graph, not the algorithm
→ Greedy choice (minimum edge) is independent of start
→ All starting points lead to same MST ✓

Convention:
→ Usually start from node 0 for simplicity
→ But any node works! ✓
```

## Edge Cases

```
Single node:              Disconnected graph:
V = 1, edges = []        Graph has multiple components
sum = 0                  MST doesn't exist!
MST = {}                 (Problem states "connected graph")

Complete graph:           All edges same weight:
n(n-1)/2 edges           Multiple valid MSTs
Pick any n-1 edges       Algorithm picks one arbitrarily
Sum is well-defined ✓    Sum is same for all ✓

Self-loops:              Parallel edges:
Edge u→u                 Multiple edges between u-v
Ignored (creates cycle)  Pick minimum weight edge ✓

Zero-weight edges:       Negative weights:
Allowed! ✓               Not typical for MST problems
MST includes them        But algorithm still works ✓
```

## Complexity Analysis

**Time Complexity:** `O(E log E)`
- Priority queue operations: O(E) insertions × O(log E) each
- Each edge added to PQ at most once (per endpoint)
- Maximum PQ size: O(E)
- Total: **O(E log E)**

**Space Complexity:** `O(V + E)`
- Adjacency list: O(E)
- Visited array: O(V)
- Priority queue: O(E) maximum
- MST edges (optional): O(V-1) = O(V)
- Total: **O(V + E)**

```
Detailed analysis:

E = number of edges
V = number of vertices

Best case: O(E log V)
  Sparse graph, few PQ operations

Worst case: O(E log E)
  Dense graph, many duplicates in PQ
  
Average case: O(E log E)
  Since E ≤ V² in connected graph
  log E ≈ log V² = 2 log V
  So O(E log E) ≈ O(E log V)

Space breakdown:
→ adj[]: O(2E) = O(E) for undirected
→ vis[]: O(V)
→ pq: O(E) worst case (duplicates)
→ Total: O(V + E)

Optimization:
Using Fibonacci heap: O(E + V log V)
But complex implementation, rarely used in practice.
```

## Prim's vs Kruskal's

```
Property            Prim's              Kruskal's
────────────────────────────────────────────────────────
Approach            Vertex-based         Edge-based
Start               Any vertex           Lightest edge
Grows               Connected tree       Forest → Tree
Data Structure      Priority Queue       Union-Find (DSU)
Time                O(E log E)           O(E log E)
Best for            Dense graphs         Sparse graphs
                    (many edges)         (few edges)
Implementation      Similar to Dijkstra  Sort + DSU

Example preference:

Dense graph (E ≈ V²):
→ Prim's slightly better
→ PQ operations on fewer elements

Sparse graph (E ≈ V):
→ Kruskal's slightly better
→ Sorting fewer edges

Both produce correct MST! ✓
Choice depends on graph density and preference.
```

## When to Use Prim's Algorithm

```
✓ Use Prim's when:
  → Finding Minimum Spanning Tree
  → Graph is dense (many edges)
  → Graph is stored as adjacency list/matrix
  → Need to grow MST from specific starting point
  → Implementation simplicity preferred

✗ Don't use when:
  → Graph has negative cycles (MST undefined)
  → Need shortest paths (use Dijkstra instead)
  → Graph is sparse → Kruskal's may be better

Similar problems:
→ Minimum Cost to Connect All Points
→ Min Cost to Connect All Cities
→ Network Cable Installation
→ Connecting Cities With Minimum Cost

Real-world applications:
→ Network design (minimize cable length)
→ Circuit design (minimize wire length)
→ Cluster analysis
→ Approximation for TSP
→ Image segmentation

Key insight:
"Connect all nodes with minimum total weight" = MST = Prim's! ✓
```

## Summary

```
Prim's Algorithm:
→ Greedy approach for Minimum Spanning Tree
→ Grows MST one vertex at a time
→ Always picks minimum weight edge to unvisited node
→ Uses priority queue for efficiency

Time: O(E log E)
Space: O(V + E)

Key points:
1. Start from any node (typically 0)
2. Use min-heap priority queue
3. Mark visited nodes to avoid cycles
4. Always pick minimum weight edge crossing cut
5. Repeat until all nodes in MST

Guarantees:
→ Produces minimum spanning tree ✓
→ Works for any connected undirected graph ✓
→ Handles duplicate edges ✓
→ Sum is always optimal ✓
```
