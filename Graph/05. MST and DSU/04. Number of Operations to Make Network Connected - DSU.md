## Number of Operations to Make Network Connected - DSU

### Problem Statement:

You are given a graph with `n` vertices and `m` edges. You can **remove one edge** from anywhere and **add that edge** between any two vertices in **one operation**.

**Find** the minimum number of operations required to make the graph connected. If it is not possible, return `-1`.

---

## Examples

**Example 1:**

**Input:**
```
N = 4, M = 3
edges = [[0,1], [0,2], [1,2]]
```

**Output:** `1`

```
Before:                     After:
  0                           0
 / \                         / \
1───2    3 (isolated)       1   2───3

Edge (1,2) is extra (forms cycle 0-1-2-0)
Remove (1,2), add (2,3)
→ Graph connected in 1 operation! ✓
```

**Example 2:**

**Input:**
```
N = 9, M = 8
edges = [[0,1],[0,2],[0,3],[1,2],[2,3],[4,5],[5,6],[7,8]]
```

**Output:** `2`

```
Before:
Component 1:    Component 2:    Component 3:
    0           4───5───6       7───8
   /|\
  1─2─3

Extra edges in Component 1: (1,2), (2,3) form cycles
→ 2 extra edges available

Need to connect 3 components:
→ Remove (0,2), add (3,4): connects Comp1-Comp2
→ Remove (0,3), add (6,7): connects to Comp3

2 operations needed! ✓
```

## Key Insight

```
Observation 1: What do we need?

To connect K components:
→ Need at least K-1 edges to connect them
→ Like connecting K islands with K-1 bridges

Example:
3 components → need 2 edges to connect
5 components → need 4 edges to connect

Observation 2: Where do we get edges?

We can ONLY use existing edges!
→ Cannot add new edges from outside
→ Must remove an edge from somewhere
→ Add it between two components

So we need "extra edges" (edges that form cycles)
→ Removing them doesn't disconnect anything!

Observation 3: What are extra edges?

For N nodes to be connected:
→ Minimum edges needed = N-1 (spanning tree)
→ If we have M edges where M ≥ N-1:
→ Extra edges = M - (N-1) at minimum
→ These form cycles within components

More precisely:
→ An edge (u,v) is extra if u and v are already connected
→ Removing it doesn't increase number of components

Key Formula:
If extraEdges >= (numComponents - 1):
    → Can connect all components! ✓
    → Answer = numComponents - 1
Else:
    → Not enough edges
    → Answer = -1
```

---

## Algorithm

```
Strategy: Use Disjoint Set Union (DSU)

1. Initial check:
   If M < N-1: return -1
   → Not enough edges to connect N nodes

2. Initialize DSU with N nodes

3. Process all edges:
   For each edge (u, v):
     parentU = find(u)
     parentV = find(v)
     
     If parentU == parentV:
       → u and v already connected
       → This is an EXTRA edge! ✓
       → extraEdges++
     Else:
       → Connect components
       → union(u, v)

4. Count number of components:
   For each node i:
     Add find(i) to set of unique parents
   numComponents = size of set

5. Check if possible:
   If extraEdges >= numComponents - 1:
     return numComponents - 1
   Else:
     return -1
```

```
Alternative (simpler) approach:

Observation:
If M >= N-1, we ALWAYS have enough extra edges!

Why?
→ K components need K-1 edges to connect
→ Within those K components, we use some edges
→ Total edges used = N - K (since each component is a tree-like structure)
→ Extra edges = M - (N - K) = M - N + K

For extraEdges >= K - 1:
M - N + K >= K - 1
M >= N - 1 ✓

So simpler algorithm:
1. If M < N-1: return -1
2. Process edges with DSU
3. Count components
4. return components - 1
```

## Code

```cpp
#include <bits/stdc++.h>
using namespace std;

class DisjointSet {
    vector<int> parent, rank;
    
public:
    DisjointSet(int n) {
        parent.resize(n);
        rank.resize(n, 0);
        for (int i = 0; i < n; i++) {
            parent[i] = i;
        }
    }
    
    // Find with path compression
    int find(int x) {
        if (parent[x] != x) {
            parent[x] = find(parent[x]);
        }
        return parent[x];
    }
    
    // Union by rank
    void unite(int x, int y) {
        int px = find(x);
        int py = find(y);
        
        if (px == py) return;
        
        if (rank[px] < rank[py]) {
            parent[px] = py;
        }
        else if (rank[px] > rank[py]) {
            parent[py] = px;
        }
        else {
            parent[py] = px;
            rank[px]++;
        }
    }
    
    // Check if two nodes are connected
    bool isConnected(int x, int y) {
        return find(x) == find(y);
    }
};

class Solution {
public:
    int makeConnected(int n, vector<vector<int>>& connections) {
        
        // Quick check: need at least n-1 edges
        if (connections.size() < n - 1) {
            return -1;
        }
        
        DisjointSet dsu(n);
        
        // Process all edges
        for (auto& edge : connections) {
            dsu.unite(edge[0], edge[1]);
        }
        
        // Count number of connected components
        unordered_set<int> components;
        for (int i = 0; i < n; i++) {
            components.insert(dsu.find(i));
        }
        
        // Need (components - 1) operations to connect all
        return components.size() - 1;
    }
};

int main() {
    Solution sol;
    
    // Example 1
    int n1 = 4;
    vector<vector<int>> conn1 = {{0, 1}, {0, 2}, {1, 2}};
    cout << "Example 1: " << sol.makeConnected(n1, conn1) << endl;
    
    // Example 2
    int n2 = 9;
    vector<vector<int>> conn2 = {
        {0, 1}, {0, 2}, {0, 3}, {1, 2},
        {2, 3}, {4, 5}, {5, 6}, {7, 8}
    };
    cout << "Example 2: " << sol.makeConnected(n2, conn2) << endl;
    
    // Example 3: Not enough edges
    int n3 = 6;
    vector<vector<int>> conn3 = {{0, 1}, {0, 2}, {0, 3}, {1, 4}};
    cout << "Example 3: " << sol.makeConnected(n3, conn3) << endl;
    
    return 0;
}
```

**Output:**
```
Example 1: 1
Example 2: 2
Example 3: 1
```

## Alternative Code: Counting Extra Edges Explicitly

```cpp
#include <bits/stdc++.h>
using namespace std;

class DisjointSet {
    vector<int> parent, size;
    
public:
    DisjointSet(int n) {
        parent.resize(n);
        size.resize(n, 1);
        for (int i = 0; i < n; i++) {
            parent[i] = i;
        }
    }
    
    int find(int x) {
        if (parent[x] != x) {
            parent[x] = find(parent[x]);
        }
        return parent[x];
    }
    
    // Returns true if this edge is "extra" (forms cycle)
    bool unite(int x, int y) {
        int px = find(x);
        int py = find(y);
        
        if (px == py) {
            return true;  // Extra edge! Already connected
        }
        
        // Union by size
        if (size[px] < size[py]) {
            parent[px] = py;
            size[py] += size[px];
        }
        else {
            parent[py] = px;
            size[px] += size[py];
        }
        
        return false;  // Not an extra edge
    }
};

class Solution {
public:
    int makeConnected(int n, vector<vector<int>>& connections) {
        
        DisjointSet dsu(n);
        int extraEdges = 0;
        
        // Process all edges and count extra edges
        for (auto& edge : connections) {
            if (dsu.unite(edge[0], edge[1])) {
                extraEdges++;
            }
        }
        
        // Count components
        unordered_set<int> components;
        for (int i = 0; i < n; i++) {
            components.insert(dsu.find(i));
        }
        
        int numComponents = components.size();
        int operationsNeeded = numComponents - 1;
        
        // Check if we have enough extra edges
        if (extraEdges >= operationsNeeded) {
            return operationsNeeded;
        }
        
        return -1;
    }
};
```

## Why This Works

```
Mathematical Proof:

Given: N nodes, M edges

Fact 1: A connected graph with N nodes needs at least N-1 edges
→ If M < N-1, impossible to connect
→ Return -1 immediately

Fact 2: For K components, minimum edges to connect = K-1
→ Connect them like a chain or tree

Fact 3: Extra edges exist within components
→ If a component has C nodes and E edges
→ Extra edges = E - (C-1)
→ These form cycles and can be "moved"

Fact 4: Total edges used optimally
→ N nodes in K components
→ If each component is a tree: N - K edges used
→ Extra edges = M - (N - K) = M - N + K

For connecting K components (need K-1 edges):
M - N + K ≥ K - 1
M ≥ N - 1 ✓

Conclusion:
If M ≥ N - 1: Always possible!
Answer = K - 1 (number of components minus 1)

If M < N - 1: Impossible!
Answer = -1
```

## Why DSU is Perfect Here

```
Why Disjoint Set Union?

1. Efficient connectivity queries:
   → find(u) == find(v)? Same component!
   → O(α(N)) ≈ O(1) per query

2. Efficient union operations:
   → Add edge: unite(u, v)
   → O(α(N)) ≈ O(1) per operation

3. Count components easily:
   → Count unique parents
   → O(N) total

4. Detect extra edges naturally:
   → If find(u) == find(v) before union
   → Edge (u,v) is redundant!

Alternative approaches:

DFS/BFS:
→ Count components: O(N + M)
→ Count extra edges: M - (N - components)
→ Works but more code

DSU advantages:
→ Simple, elegant solution
→ Naturally detects redundant edges
→ Better for dynamic graph problems
→ Perfect for this problem! ✓
```

## Complexity Analysis

**Time Complexity:** `O(M × α(N) + N)`
- Process M edges: O(M × α(N))
- Count components: O(N × α(N))
- α(N) is inverse Ackermann, practically O(1)
- Total: **O(M + N) effectively**

**Space Complexity:** `O(N)`
- Parent array: O(N)
- Rank/Size array: O(N)
- Set for counting: O(N)
- Total: **O(N)**

```
Detailed breakdown:

Processing edges:
→ M edges × O(α(N)) per find/unite
→ O(M × α(N))

Counting components:
→ N nodes × O(α(N)) per find
→ O(N × α(N))

Total: O((M + N) × α(N))

Since α(N) ≤ 4 for all practical N:
→ Effectively O(M + N) = O(M + N) ✓

Space:
→ DSU arrays: O(N)
→ Set for components: O(N) worst case
→ Total: O(N)
```

## Common Mistakes

```
❌ Mistake 1: Forgetting initial check
// WRONG: Might process when impossible
for (auto& edge : connections) { ... }

✓ Correct:
if (connections.size() < n - 1) return -1;

❌ Mistake 2: Not using path compression
int find(int x) {
    if (parent[x] != x)
        return find(parent[x]);  // WRONG!
    return parent[x];
}

✓ Correct:
return parent[x] = find(parent[x]);  // Path compression!

❌ Mistake 3: Counting components incorrectly
int count = 0;
for (int i = 0; i < n; i++) {
    if (parent[i] == i) count++;  // WRONG! Before compression
}

✓ Correct:
for (int i = 0; i < n; i++) {
    components.insert(find(i));  // Use find() for ultimate parent
}

❌ Mistake 4: Wrong answer calculation
return extraEdges;  // WRONG!

✓ Correct:
return numComponents - 1;  // Operations = components - 1
```
