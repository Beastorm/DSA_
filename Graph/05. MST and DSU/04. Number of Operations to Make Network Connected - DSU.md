## 1. Problem Statement

You are given a graph with `n` vertices and `m` edges. You can **remove one edge** from anywhere and **add that edge** between any two vertices in **one operation**.

**Find** the minimum number of operations required to make the graph connected. If it is not possible, return `-1`.

**Input Format:**
- `n` = number of vertices
- `connections` = list of undirected edges `[u, v]`

**Output:**
- Minimum operations to make graph connected, or `-1`

---

## Examples

**Example 1:**

- **Input:**

  ```
  N = 4, M = 3
  edges = [[0,1], [0,2], [1,2]]
  ```

- **Graph:**

  ```
  Before:                     After:

    0                           0
   / \                         / \
  1───2    3 (isolated)       1   2───3
  ```

- **Output:** `1`

  ```
  Edge (1,2) is extra (forms cycle 0-1-2-0)
  Remove (1,2), add (2,3)
  → Graph connected in 1 operation! ✓
  ```

**Example 2:**

- **Input:**

  ```
  N = 9, M = 8
  edges = [[0,1],[0,2],[0,3],[1,2],[2,3],[4,5],[5,6],[7,8]]
  ```

- **Output:** `2`

  ```
  Before:

  Component 1:    Component 2:    Component 3:

      0           4───5───6       7───8
     /|\
    1─2─3

  Extra edges in Component 1: (1,2), (2,3) form cycles
  → 2 extra edges available

  Need to connect 3 components:
  → Remove (0,2), add (3,4): connects Comp1-Comp2
  → Remove (0,3), add (6,7): connects to Comp3

  2 operations needed! ✓
  ```

## Key Insight

```
Observation 1: What do we need?

To connect K components:
→ Need at least K-1 edges to connect them
→ Like connecting K islands with K-1 bridges

Example:
3 components → need 2 edges to connect
5 components → need 4 edges to connect

Observation 2: Where do we get edges?

We can ONLY use existing edges!
→ Cannot add new edges from outside
→ Must remove an edge from somewhere
→ Add it between two components

So we need "extra edges" (edges that form cycles)
→ Removing them doesn't disconnect anything!

Observation 3: What are extra edges?

For N nodes to be connected:
→ Minimum edges needed = N-1 (spanning tree)
→ If we have M edges where M ≥ N-1:
→ Extra edges = M - (N-1) at minimum
→ These form cycles within components

More precisely:
→ An edge (u,v) is extra if u and v are already connected
→ Removing it doesn't increase number of components

Key Formula:

If extraEdges >= (numComponents - 1):
    → Can connect all components! ✓
    → Answer = numComponents - 1
Else:
    → Not enough edges
    → Answer = -1
```

---

## Intuition

```
Imagine you have several groups of islands with bridges between
some of them. You want to connect ALL islands into one big group.

But you cannot build new bridges — you can only:
→ Remove a bridge from somewhere
→ Place it between two different groups

Where do you get bridges to move?

Look for "useless" bridges — bridges that form a loop. If three
islands A-B-C-A are connected in a triangle, you can remove any
one bridge and everyone is still connected. That bridge is free
to move elsewhere!

How many bridges do you need to move?

If you have K separate groups of islands, you need K-1 bridges
to connect them all. Like connecting 3 groups needs 2 bridges.

The whole problem becomes:
1. Count how many separate groups you have
2. Check if you have enough "spare" bridges (cycle edges)
3. If yes, answer = groups - 1
4. If no, answer = -1
```

## Algorithm

```
makeConnected(n, connections):

1. Quick check:
   If connections.size() < n - 1:
     return -1
   → Not enough edges to ever connect all nodes

2. Initialize DSU with n nodes

3. Process all edges:
   For each edge (u, v):
     If find(u) == find(v):
       → Already connected → extra edge!
       → extraEdges++
     Else:
       → unite(u, v)

4. Count components:
   For each node i:
     Add find(i) to a set of unique parents
   numComponents = size of set

5. Return numComponents - 1

Why is this enough?

If M >= N-1, we ALWAYS have enough extra edges!
Proof:
→ K components need K-1 edges to connect
→ Total edges used within components = N - K
→ Extra edges = M - (N - K) = M - N + K

For extraEdges >= K - 1:
M - N + K >= K - 1
M >= N - 1 ✓

So if M >= N-1, answer is simply components - 1!
```

## Code

```cpp
#include <bits/stdc++.h>
using namespace std;

class DisjointSet {
    vector<int> parent, size;
    
public:
    DisjointSet(int n) {
        parent.resize(n);
        size.resize(n, 1);
        for (int i = 0; i < n; i++) {
            parent[i] = i;
        }
    }
    
    int find(int x) {
        if (parent[x] != x) {
            parent[x] = find(parent[x]);
        }
        return parent[x];
    }
    
    void unite(int x, int y) {
        int px = find(x);
        int py = find(y);
        
        if (px == py) return;
        
        if (size[px] < size[py]) swap(px, py);
        parent[py] = px;
        size[px] += size[py];
    }
};

class Solution {
public:
    int makeConnected(int n, vector<vector<int>>& connections) {
        
        // Quick check: need at least n-1 edges
        if (connections.size() < n - 1) {
            return -1;
        }
        
        DisjointSet dsu(n);
        
        // Process all edges
        for (auto& edge : connections) {
            dsu.unite(edge[0], edge[1]);
        }
        
        // Count number of connected components
        unordered_set<int> components;
        for (int i = 0; i < n; i++) {
            components.insert(dsu.find(i));
        }
        
        // Need (components - 1) operations to connect all
        return components.size() - 1;
    }
};

int main() {
    Solution sol;
    
    // Example 1
    int n1 = 4;
    vector<vector<int>> conn1 = {{0, 1}, {0, 2}, {1, 2}};
    cout << "Example 1: " << sol.makeConnected(n1, conn1) << endl;
    
    // Example 2
    int n2 = 9;
    vector<vector<int>> conn2 = {
        {0, 1}, {0, 2}, {0, 3}, {1, 2},
        {2, 3}, {4, 5}, {5, 6}, {7, 8}
    };
    cout << "Example 2: " << sol.makeConnected(n2, conn2) << endl;
    
    // Example 3: Not enough edges
    int n3 = 6;
    vector<vector<int>> conn3 = {{0, 1}, {0, 2}, {0, 3}, {1, 4}};
    cout << "Example 3: " << sol.makeConnected(n3, conn3) << endl;
    
    return 0;
}
```

**Output:**

```
Example 1: 1
Example 2: 2
Example 3: 1
```

### Alternative: Counting Extra Edges Explicitly

```cpp
class SolutionExplicit {
public:
    int makeConnected(int n, vector<vector<int>>& connections) {
        
        // Quick check
        if (connections.size() < n - 1) {
            return -1;
        }
        
        DisjointSet dsu(n);
        int extraEdges = 0;
        
        // Process all edges and count extra edges
        for (auto& edge : connections) {
            int u = edge[0], v = edge[1];
            
            if (dsu.find(u) == dsu.find(v)) {
                // Already connected → extra edge
                extraEdges++;
            } else {
                dsu.unite(u, v);
            }
        }
        
        // Count components
        unordered_set<int> components;
        for (int i = 0; i < n; i++) {
            components.insert(dsu.find(i));
        }
        
        int numComponents = components.size();
        int operationsNeeded = numComponents - 1;
        
        // Check if we have enough extra edges
        if (extraEdges >= operationsNeeded) {
            return operationsNeeded;
        }
        
        return -1;
    }
};
```

## Step-by-Step Trace

```
Example 1: N=4, edges=[[0,1], [0,2], [1,2]]

── Initialization ──

parent = [0, 1, 2, 3]
size   = [1, 1, 1, 1]

Each node is its own component

── Process Edges ──

Edge (0, 1):
  find(0) = 0, find(1) = 1
  Different components → unite
  parent = [0, 0, 2, 3]

Edge (0, 2):
  find(0) = 0, find(2) = 2
  Different components → unite
  parent = [0, 0, 0, 3]

Edge (1, 2):
  find(1) = 0, find(2) = 0
  SAME component! ← Extra edge detected! ✓
  No union needed

── Count Components ──

find(0) = 0
find(1) = 0
find(2) = 0
find(3) = 3

Unique parents: {0, 3} → 2 components

── Result ──

Components = 2
Operations needed = 2 - 1 = 1 ✓
```

```
Example 2: N=9, edges=[[0,1],[0,2],[0,3],[1,2],[2,3],[4,5],[5,6],[7,8]]

── Process Edges ──

(0,1): unite → parent[1] = 0
(0,2): unite → parent[2] = 0
(0,3): unite → parent[3] = 0
(1,2): find(1)=0, find(2)=0 → EXTRA EDGE ✓
(2,3): find(2)=0, find(3)=0 → EXTRA EDGE ✓
(4,5): unite → parent[5] = 4
(5,6): unite → parent[6] = 4
(7,8): unite → parent[8] = 7

Extra edges: 2

── Count Components ──

Nodes 0,1,2,3 → parent 0
Nodes 4,5,6   → parent 4
Nodes 7,8     → parent 7

Components: {0, 4, 7} → 3 components

── Result ──

Operations needed = 3 - 1 = 2
Extra edges = 2 ≥ 2 ✓

Answer: 2 ✓
```

## Edge Cases

```
Single node, no edges:       All nodes isolated:
N = 1, M = 0                N = 5, M = 0
Components = 1              Not enough edges!
Answer = 0 ✓                M < N-1 → Answer = -1 ✗

Already connected:           Complete graph:
N = 3                       N = 4
edges = [[0,1],[1,2]]      All pairs connected
Components = 1              Many extra edges
Answer = 0 ✓                Answer = 0 ✓

Minimum edges needed:       Self-loops (if allowed):
N = 4, M = 3               Don't affect connectivity
Exactly N-1 edges          Count as extra edges
If connected: Answer = 0
If not: can't fix ✗

Linear chain:              Multiple extra edges:
0─1─2─3─4                  Lots of cycles
Components = 1             Many edges to redistribute
Answer = 0 ✓               Answer = components - 1 ✓
```

## Complexity Analysis

**Time Complexity:** `O(M × α(N) + N)` ≈ `O(M + N)`

- Process M edges: O(M × α(N))
- Count components: O(N × α(N))
- α(N) is inverse Ackermann, practically O(1)
- Total: **O(M + N) effectively**

**Space Complexity:** `O(N)`

- Parent array: O(N)
- Size/Rank array: O(N)
- Set for counting: O(N)
- Total: **O(N)**

## Interview Questions

```
Q1: What is this problem asking?

A: We have a network of computers. Some are connected, some are
   not. We can move one existing cable at a time to connect
   different groups. We want to know the minimum number of cable
   moves to make everyone connected. If it is impossible, say -1.

──────────────────────────────────────────────────────────────

Q2: Why check if M < N-1 at the start?

A: To connect N computers, you need at least N-1 cables. If you
   have fewer than that, it is physically impossible to connect
   everyone. Like trying to connect 5 islands with only 3 bridges
   — someone will always be left out.

──────────────────────────────────────────────────────────────

Q3: Why is the answer simply (components - 1)?

A: If you have K separate groups, you need K-1 bridges to connect
   them into one. Think of it like linking K chains — you need
   K-1 links to join them into one long chain.

   And if M >= N-1, you are guaranteed to have enough spare
   cables (cycle edges) to make these K-1 moves.

──────────────────────────────────────────────────────────────

Q4: How does DSU help solve this?

A: DSU tells us two things quickly:
   → How many separate groups exist (count unique parents)
   → Which edges are "extra" (both ends already connected)

   Without DSU, we would need BFS/DFS to find groups, which is
   slower and more code.

──────────────────────────────────────────────────────────────

Q5: What is an "extra edge"?

A: An edge that connects two nodes already in the same group.
   It forms a loop, so removing it does not disconnect anyone.
   These are the edges we can "move" to connect other groups.

──────────────────────────────────────────────────────────────

Q6: Can we always connect if M >= N-1?

A: Yes! The math guarantees it. With N-1 or more edges, there
   are always enough extra edges (loops) to redistribute and
   connect all components. The only impossible case is having
   too few edges overall.

──────────────────────────────────────────────────────────────

Q7: What if there are no extra edges?

A: If M = N-1 and the graph is already a tree, there are no
   extra edges. But if the graph is already connected (one
   component), the answer is 0 — no moves needed. If it is not
   connected, then M < N-1, which we catch at the start.

──────────────────────────────────────────────────────────────

Q8: Why not just count extra edges directly?

A: You can, but it is unnecessary. If M >= N-1, the math proves
   extraEdges >= components - 1. So just counting components
   and returning components - 1 is enough. The explicit extra
   edge counting is more intuitive but not required.

──────────────────────────────────────────────────────────────

Q9: What if the graph is already connected?

A: Then DSU will find only 1 component. The answer is 1 - 1 = 0.
   No moves needed — everyone is already connected!

──────────────────────────────────────────────────────────────

Q10: Can this problem be solved with BFS/DFS instead?

A: Yes, but DSU is cleaner. BFS/DFS can count components, but
    detecting extra edges is trickier. DSU naturally handles both
    in one pass. For this problem, DSU is the standard solution.

──────────────────────────────────────────────────────────────

Q11: What happens with self-loops?

A: A self-loop (u, u) connects a node to itself. It is always an
    extra edge because the node is already connected to itself.
    It can be moved elsewhere if needed.

──────────────────────────────────────────────────────────────

Q12: What if multiple edges exist between the same pair?

A: Only one edge is needed to connect them. The rest are extra
    edges that can be moved. DSU handles this naturally — after
    the first edge unites them, the rest are detected as extra.

──────────────────────────────────────────────────────────────

Q13: What are real-world applications?

A: Common scenarios:
    → Reorganizing a computer network with limited cables
    → Rewiring a building with existing wires
    → Optimizing road networks with fixed construction budget
    → Any situation where you can relocate existing connections

──────────────────────────────────────────────────────────────

Q14: How does this relate to MST?

A: Both problems connect a graph with minimum cost. But here we
    can only MOVE existing edges, not add new ones. An MST finds
    the cheapest way to build from scratch. This problem finds
    the cheapest way to reorganize what you already have.

──────────────────────────────────────────────────────────────

Q15: What are the most common mistakes?

A: Common mistakes and fixes:

    Mistake 1: Forgetting the M < N-1 check
    → Fix: Always check first — impossible cases return -1

    Mistake 2: Counting parent[i] == i for components
    → Fix: Use find(i) because path compression changes parents

    Mistake 3: Returning extraEdges instead of components - 1
    → Fix: Answer is how many bridges you need, not how many
           spare bridges you have

    Mistake 4: Not using path compression in DSU
    → Fix: Always compress paths for near-O(1) find

    Mistake 5: Off-by-one in component count
    → Fix: Components - 1, not components

    Mistake 6: Thinking extraEdges could be less than needed
    → Fix: If M >= N-1, math guarantees enough extra edges
```
