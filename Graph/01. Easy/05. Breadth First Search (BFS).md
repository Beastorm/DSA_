## Breadth First Search (BFS)

**Problem Statement:**   
Given an undirected graph, return a vector of all nodes by traversing the graph using **Breadth First Search (BFS)**.

---

>### Examples:

**Example 1:**
- **Input:** `V=5`, `adj=[[2,3],[1,4,5],[1],[2,5],[4,5]]`
- **Output:** `1 2 3 4 5`
```
  Graph (1-indexed):
  1 ─── 2 ─── 4
  |     |     |
  3     └─ 5 ─┘

  BFS from 1 (level by level):
  Level 0: 1
  Level 1: 2, 3        (neighbors of 1)
  Level 2: 4, 5        (neighbors of 2 not yet visited)
  Output: 1 2 3 4 5 ✓
```

**Example 2:**
- **Input:** `V=4`, `adj=[[2,3],[1,4],[1],[2]]`
- **Output:** `1 2 3 4`
```
  Graph:
  1 ─── 2 ─── 4
  |
  3

  BFS from 1:
  Level 0: 1
  Level 1: 2, 3
  Level 2: 4
  Output: 1 2 3 4 ✓
```

---

### What is BFS?
```
  BFS = Explore ALL neighbors at current level
        BEFORE going to the next level.

  Like ripples in water:
  → Start at the center (source)
  → Spread outward one level at a time

  Uses a QUEUE (FIFO).

  DFS (go deep):             BFS (go wide):
  1→2→4→(back)→5→3          Level 0: 1
                             Level 1: 2, 3
                             Level 2: 4, 5
```

---

### Key Insight:
```
  BFS Traversal:
  1. Start at source → push to queue, mark visited
  2. While queue not empty:
       Pop front node
       Add to result
       For each unvisited neighbor:
           Mark visited
           Push to queue

  WHY mark visited when PUSHING (not popping)?
  → Prevents same node being added to queue multiple times!
  → If we wait until popping, duplicates enter queue.

  ✓ Correct:          ✗ Wrong:
  Mark when pushing   Mark when popping
  Prevents duplicates Duplicates in queue!
```

---
## Approach
### Algorithm:
```
  BFS(start, adj, V):
  ┌─────────────────────────────────────────────┐
  │ 1. visited[start] = true                    │
  │ 2. queue.push(start)                        │
  │ 3. while queue not empty:                   │
  │      node = queue.front(); queue.pop()      │
  │      result.push_back(node)                 │
  │      for each neighbor of node:             │
  │          if NOT visited[neighbor]:           │
  │              visited[neighbor] = true        │
  │              queue.push(neighbor)            │
  │ 4. return result                            │
  └─────────────────────────────────────────────┘
```

---

### C++ Code:
```cpp
#include <bits/stdc++.h>
using namespace std;

class Solution {
public:
    vector<int> bfsOfGraph(int V, vector<vector<int>>& adj) {
        vector<bool> visited(V + 1, false);
        vector<int> result;
        queue<int> q;

        // Step 1: Start from node 1
        visited[1] = true;
        q.push(1);

        // Step 2: Process until the queue is empty
        while (!q.empty()) {
            // Get front node
            int node = q.front();
            q.pop();

            // Add to result
            result.push_back(node);

            // Step 3: Push all unvisited neighbors
            for (int neighbor : adj[node]) {
                if (!visited[neighbor]) {
                    visited[neighbor] = true; // Mark NOW (when pushing)
                    q.push(neighbor);
                }
            }
        }

        return result;
    }
};

int main() {
    Solution sol;

    // Example 1
    int V1 = 5;
    vector<vector<int>> adj1(V1 + 1);
    adj1[1] = {2, 3};
    adj1[2] = {1, 4, 5};
    adj1[3] = {1};
    adj1[4] = {2, 5};
    adj1[5] = {2, 4};

    cout << "Example 1: ";
    for (int x : sol.bfsOfGraph(V1, adj1)) cout << x << " ";
    cout << endl; // 1 2 3 4 5

    // Example 2
    int V2 = 4;
    vector<vector<int>> adj2(V2 + 1);
    adj2[1] = {2, 3};
    adj2[2] = {1, 4};
    adj2[3] = {1};
    adj2[4] = {2};

    cout << "Example 2: ";
    for (int x : sol.bfsOfGraph(V2, adj2)) cout << x << " ";
    cout << endl; // 1 2 3 4

    return 0;
}
```

  node = 2, result = [1, 2]

  node = 4, result = [1, 2, 3, 4]
  Neighbors of 4: [2]
    → 2: visited → skip
  queue = []

  Final result: [1, 2, 3, 4] ✓

### BFS for Disconnected Graphs:
### C++ Code:
```cpp
vector<int> bfsAll(int V, vector<vector<int>>& adj) {
    vector<bool> visited(V + 1, false);
    vector<int> result;

    for (int v = 1; v <= V; v++) {
        if (!visited[v]) {
            // Start new BFS component
            queue<int> q;
            visited[v] = true;
            q.push(v);

            while (!q.empty()) {
                int node = q.front(); q.pop();
                result.push_back(node);

                for (int neighbor : adj[node]) {
                    if (!visited[neighbor]) {
                        visited[neighbor] = true;
                        q.push(neighbor);
                    }
                }
            }
        }
    }
    return result;
}
```
```
  Disconnected: 1─2─3,  4─5,  6(isolated)

  v=1: BFS → visits {1,2,3}
  v=4: BFS → visits {4,5}
  v=6: BFS → visits {6}

  Result: [1,2,3,4,5,6] ✓
```

### BFS vs DFS:
```
  Property           BFS                    DFS
  ───────────────────────────────────────────────────
  Data Structure     Queue (FIFO)           Stack/Recursion
  Traversal Order    Level by level         Depth first
  Space (worst)      O(W) width             O(H) height
  Shortest Path      ✓ Guaranteed (unwtd)   ✗ Not guaranteed
  Cycle Detection    ✓ Yes                  ✓ Yes
  Connected Comp.    ✓ Yes                  ✓ Yes
  Implementation     Iterative (queue)      Recursive or iterative

  Choose BFS when:              Choose DFS when:
  - Shortest path needed        - Topological sort
  - Level-order traversal       - Detect cycles
  - Nearest neighbor            - Explore all paths
  - Social network distance     - Maze solving
```

---

### Edge Cases:
```
  Single node:              No edges:
  V=1, adj[1]=[]            V=4, all adj empty

  Push 1, pop 1             BFS(1): visits just 1
  No neighbors              Need outer loop for all nodes
  Result=[1] ✓              Result=[1,2,3,4] with loop ✓

  All connected:            Star graph (1 center):
  Complete graph            1─2, 1─3, 1─4, 1─5

  BFS visits all in         Level 0: {1}
  O(V+E) ✓                  Level 1: {2,3,4,5}
                            Result: [1,2,3,4,5] ✓
```

---

### Complexity Analysis:

**Time Complexity:** O(V + E)
- Each vertex dequeued exactly once: O(V)
- Each edge examined at most twice (undirected): O(E)
- Total: **O(V + E)**

**Space Complexity:** O(V)
- `visited[]` array: O(V)
- Queue holds at most O(V) nodes: O(V)
- `result[]` array: O(V)
- Total: **O(V)**
```
  Unlike DFS, BFS does NOT risk stack overflow
  because it uses an explicit queue.

  BFS queue max size = width of graph (widest level)
  For balanced tree: O(N/2) = O(N) at the last level.
  For star graph:    O(N-1) = O(N) neighbors of center.
```
