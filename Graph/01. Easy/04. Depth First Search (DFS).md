## Depth First Search (DFS)

**Problem Statement:**   
Given an undirected graph, return a vector of all nodes by traversing the graph using **Depth First Search (DFS)**.

---

>### Examples:

**Example 1:**
- **Input:** `V=5`, `adj=[[2,3],[1,4,5],[1],[2,5],[4,5]]`
- **Output:** `1 2 4 5 3`
```
  Graph (1-indexed):
  1 ─── 2 ─── 4
  |     |     |
  3     5 ─── ┘

  adj[1]=[2,3], adj[2]=[1,4,5], adj[3]=[1]
  adj[4]=[2,5], adj[5]=[4,5? wait...
  adj[4]=[2,5], adj[5]=[2,4]

  DFS from 1:
  1 → 2 → 4 → 5 → (backtrack) → 3
  Output: 1 2 4 5 3 ✓
```

**Example 2:**
- **Input:** `V=4`, `adj=[[2,3],[1,4],[1],[2]]`
- **Output:** `1 2 4 3`
```
  Graph:
  1 ─── 2 ─── 4
  |
  3

  DFS from 1:
  1 → 2 → 4 → (backtrack to 2) → (backtrack to 1) → 3
  Output: 1 2 4 3 ✓
```

---

## What is DFS?
```
  DFS = Go as DEEP as possible before backtracking.

  Think of it like exploring a maze:
  → Pick a direction and keep going until dead end.
  → Backtrack and try another direction.

  DFS uses a STACK (or recursion = implicit stack).

  BFS (comparison):          DFS:
  Explore level by level     Explore depth first
  Uses Queue                 Uses Stack/Recursion
  1→2,3→4,5,6               1→2→4→(back)→5→(back)→3
```

---

### Key Insight:
```
  DFS Traversal:
  1. Start from source node
  2. Mark it visited
  3. Add to result
  4. For each unvisited neighbor:
       Recursively call DFS

  "Go deep first, backtrack when stuck"

  Example tree of DFS calls (graph: 1-2-4-5, 1-3):

  DFS(1)
  ├── DFS(2)
  │   ├── DFS(4)
  │   │   └── DFS(5) ← go deeper
  │   │       └── (all neighbors visited → backtrack)
  │   └── (5 already visited → skip)
  └── DFS(3) ← backtrack to 1, try next neighbor
```

---
## Approach
### Algorithm:
```
  Initialize:
    visited[] = all false
    result = []

  DFS(node, adj, visited, result):
    visited[node] = true
    result.push_back(node)

    for each neighbor in adj[node]:
        if NOT visited[neighbor]:
            DFS(neighbor, adj, visited, result)

  Main:
    Call DFS(startNode, ...)
    Return result
```

---

### C++  Code (Recursive DFS):
```cpp
#include <bits/stdc++.h>
using namespace std;

class Solution {
private:
    void dfs(int node, vector<vector<int>>& adj, vector<bool>& visited, vector<int>& result) {

        // Mark current node as visited
        visited[node] = true;

        // Add to result
        result.push_back(node);

        // Visit all unvisited neighbors (go deep)
        for (int neighbor : adj[node]) {
            if (!visited[neighbor]) {
                dfs(neighbor, adj, visited, result);
            }
        }
        // Backtrack happens automatically when a function returns
    }

public:
    vector<int> dfsOfGraph(int V, vector<vector<int>>& adj) {
        vector<bool> visited(V + 1, false);
        vector<int> result;

        // Start DFS from node 1 (1-indexed)
        dfs(1, adj, visited, result);

        return result;
    }
};

int main() {
    Solution sol;

    // Example 1: V=5
    // adj[1]=[2,3], adj[2]=[1,4,5], adj[3]=[1], adj[4]=[2,5], adj[5]=[2,4]
    int V1 = 5;
    vector<vector<int>> adj1(V1 + 1);
    adj1[1] = {2, 3};
    adj1[2] = {1, 4, 5};
    adj1[3] = {1};
    adj1[4] = {2, 5};
    adj1[5] = {2, 4};

    vector<int> res1 = sol.dfsOfGraph(V1, adj1);
    cout << "Example 1: ";
    for (int x : res1) cout << x << " ";
    cout << endl; // 1 2 4 5 3

    // Example 2: V=4
    int V2 = 4;
    vector<vector<int>> adj2(V2 + 1);
    adj2[1] = {2, 3};
    adj2[2] = {1, 4};
    adj2[3] = {1};
    adj2[4] = {2};

    vector<int> res2 = sol.dfsOfGraph(V2, adj2);
    cout << "Example 2: ";
    for (int x : res2) cout << x << " ";
    cout << endl; // 1 2 4 3

    return 0;
}
```

### C++ Code (Iterative DFS using Stack):
```cpp
#include <bits/stdc++.h>
using namespace std;

class Solution {
public:
    vector<int> dfsOfGraph(int V, vector<vector<int>>& adj) {
        vector<bool> visited(V + 1, false);
        vector<int> result;
        stack<int> st;

        // Push starting node
        st.push(1);
        visited[1] = true;

        while (!st.empty()) {
            int node = st.top();
            st.pop();

            result.push_back(node);

            // Push neighbors in REVERSE order
            // so smaller numbered neighbor is processed first
            for (int i = adj[node].size() - 1; i >= 0; i--) {
                int neighbor = adj[node][i];
                if (!visited[neighbor]) {
                    visited[neighbor] = true;
                    st.push(neighbor);
                }
            }
        }

        return result;
    }
};
```

### Why in reverse?
| Scenario | Action Needed |
|----------|---------------|
| Order doesn't matter | No sorting, no reverse needed |
| Need lexicographical order | Sort adj list + Reverse push |
| Already sorted input | Just reverse push |

### Step-by-Step Trace:
```
  V=4, adj[1]=[2,3], adj[2]=[1,4], adj[3]=[1], adj[4]=[2]

  Graph:
  1 ─── 2 ─── 4
  |
  3

  visited = [F, F, F, F, F]  (index 0 unused)
  result = []

  ── DFS(1) ──
  visited[1] = true
  result = [1]
  Neighbors of 1: [2, 3]
    → neighbor 2: not visited → call DFS(2)

    ── DFS(2) ──
    visited[2] = true
    result = [1, 2]
    Neighbors of 2: [1, 4]
      → neighbor 1: visited → skip
      → neighbor 4: not visited → call DFS(4)

      ── DFS(4) ──
      visited[4] = true
      result = [1, 2, 4]
      Neighbors of 4: [2]
        → neighbor 2: visited → skip
      No more neighbors → RETURN (backtrack to DFS(2))

    Back in DFS(2): no more unvisited neighbors → RETURN (backtrack to DFS(1))

  Back in DFS(1):
    → neighbor 3: not visited → call DFS(3)

    ── DFS(3) ──
    visited[3] = true
    result = [1, 2, 4, 3]
    Neighbors of 3: [1]
      → neighbor 1: visited → skip
    No more neighbors → RETURN

  DFS(1) complete. result = [1, 2, 4, 3] ✓
```

```
  └── DFS(3) ←── explore another branch
      │ result=[1,2,4,3]
      │ (no unvisited neighbors)
      └── RETURN ←── backtrack

  DFS(1) complete!
  Final result: [1, 2, 4, 3] ✓
```

### DFS Tree vs Graph:
```
  Original Graph:           DFS Tree (from node 1):
  1 ─── 2 ─── 4              1
  |     |     |             / \
  3     └─── 5             2   3
  │           |           / \
  └───────────┘          4   (5 via back edge)
                        /
                       5

  Tree edges: edges used in DFS traversal
  Back edges: edges to already-visited nodes (shown as cycles)

  DFS helps detect:
  - Cycles (back edges exist)
  - Connected components
  - Topological ordering (for DAGs)
```

---

### DFS vs BFS Comparison:
```
  Property          DFS                    BFS
  ─────────────────────────────────────────────────────
  Data Structure    Stack (recursion)      Queue
  Order             Depth first            Level by level
  Space             O(H) (height)          O(W) (width)
  Shortest Path     ✗ Not guaranteed       ✓ Guaranteed
  Cycle Detection   ✓ Yes                  ✓ Yes
  Connected Comp.   ✓ Yes                  ✓ Yes
  Topological Sort  ✓ Yes                  ✗ (Kahn's algo)

  Use DFS when:                Use BFS when:
  - Need to explore all paths  - Need shortest path
  - Topological sort           - Level-order traversal
  - Detect cycles              - Nearest neighbor
  - Solve mazes/puzzles        - Connected components
```

### Edge Cases:
```
  Single node, no edges:    Disconnected graph:
  V=1, adj[1]=[]            V=4, adj[1]=[2], adj[3]=[4]

  DFS(1): visited[1]=true   DFS from 1: visits {1,2}
  result=[1]                Node 3,4 not visited!
  Return [1] ✓              Need outer loop for all components

  Self-loop:                Complete graph:
  adj[1]=[1,2]              All nodes connected to all

  1 is visited when          DFS visits all nodes in one call
  processing adj[1]:
  skip self-loop ✓
```

---

### For Disconnected Graphs:
```cpp
// If the graph might be disconnected, loop over all nodes
vector<int> dfsAll(int V, vector<vector<int>>& adj) {
    vector<bool> visited(V + 1, false);
    vector<int> result;

    for (int v = 1; v <= V; v++) {
        if (!visited[v]) {
            dfs(v, adj, visited, result);
        }
    }

    return result;
}
```

### Complexity Analysis:

**Time Complexity:** O(V + E)
- Each vertex visited exactly once: O(V)
- Each edge explored exactly twice (undirected): O(E)
- Total: **O(V + E)**

**Space Complexity:** O(V)
- Visited array: O(V)
- Recursion stack depth = height of DFS tree = O(V) worst case
- Result array: O(V)
- Total: **O(V)**
```
  Worst case stack depth:
  Linear graph 1─2─3─4─...─N
  DFS goes: 1→2→3→...→N
  Stack depth = N = O(V)

  For very deep graphs, use iterative DFS
  to avoid stack overflow! ✓
```
