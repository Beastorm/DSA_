# Introduction to Graph | Types | Different Conventions

## What is a Graph?

A **Graph** is a non-linear data structure consisting of:
- **Nodes (Vertices):** The entities in the graph.
- **Edges:** The connections between nodes.
```
  Simple Graph:
       1
      / \
     2   3
    / \
   4   5

  Nodes: {1, 2, 3, 4, 5}
  Edges: {(1,2), (1,3), (2,4), (2,5)}
```

## Types of Graphs

### 1. Undirected Graph

Edges have **no direction** — connection goes both ways.
```
  1 ─── 2         Edge (1,2) means:
  |     |         1 can reach 2 AND
  3 ─── 4         2 can reach 1

  adj[1] = [2,3]
  adj[2] = [1,4]
  adj[3] = [1,4]
  adj[4] = [2,3]
```

### 2. Directed Graph (Digraph)

Edges have a **specific direction**.
```
  1 ──→ 2         Edge (1→2) means:
  ↑     ↓         1 can reach 2 BUT
  4 ←── 3         2 CANNOT reach 1 directly

  adj[1] = [2]
  adj[2] = [3]
  adj[3] = [4]
  adj[4] = [1]
```

### 3. Weighted Graph

Edges have **weights/costs** associated with them.
```
  Undirected Weighted:    Directed Weighted:
    1 ─5─ 2                 1 ─5→ 2
    |     |                 ↑     ↓
    3     10                3 4 ← 3
    |     |
    3 ─8─ 4

  Used in: Shortest path (Dijkstra, Bellman-Ford)
```

### 4. Unweighted Graph

All edges have **unit weight** (or weight is irrelevant).
```
  1 ─── 2         All edges = weight 1
  |     |         Used in: BFS shortest path
  3 ─── 4
```

## More Graph Types

### Cyclic vs Acyclic
```
  Cyclic:                      Acyclic:
  1 ─── 2                        1
  |     |                       / \
  4 ─── 3  ← cycle: 1-2-3-4    2   3
                                   |
  Contains a cycle                 4
                           No cycle (Tree is acyclic)
```

### Connected vs Disconnected
```
  Connected:                 Disconnected:
  1 ─── 2 ─── 3             1 ─── 2     3 ─── 4
        |                   (component1) (component2)
        4
  Every node is reachable   Some nodes are unreachable
  from every other node     to others
```

### DAG (Directed Acyclic Graph)
```
  Directed + No cycles:
       A
      / \
     B   C
      \ /
       D

  Used in: Topological sort, task scheduling,
           build systems, dependency resolution
```

### Bipartite Graph
```
  Nodes are split into 2 groups.
  Edges ONLY between groups, NOT within.

  Group 1: {1,2,3}    Group 2: {4,5,6}

  1 ─── 4
  2 ─── 5
  3 ─── 6
  2 ─── 4

  No edge within {1,2,3} or within {4,5,6}
  Used in: Matching problems
```

### Complete Graph
```
  Every node is connected to every other node.
  N nodes → N(N-1)/2 edges (undirected)

  K4 (complete graph, 4 nodes):
      1
     /|\
    / | \
   2─── 3
    \ | /
     \|/
      4

  N=4: 4×3/2 = 6 edges
```

## Graph Conventions

### Convention 1: 0-indexed vs 1-indexed Nodes
```
  0-indexed (common in competitive programming):
  Nodes: 0, 1, 2, 3, ..., N-1
  Array size: adj[N]

  1-indexed (common in problems):
  Nodes: 1, 2, 3, ..., N
  Array size: adj[N+1]

  Always check the problem statement!
  Tip: Use 1-indexed to avoid confusion.
```

### Convention 2: Number of Edges
```
  Undirected graph:
  - Each edge counted ONCE in input
  - But stored TWICE in adjacency list
  - Space = O(2E)

  Directed graph:
  - Each edge counted ONCE in input
  - Stored ONCE in adjacency list
  - Space = O(E)

  Can m = 0? YES (no edges, just isolated nodes)
  Is there a max m? m ≤ N(N-1)/2 for undirected
                    m ≤ N(N-1) for directed
```

### Convention 3: Self-loops
```
  A self-loop is an edge from a node to itself.

       ┌─┐
       1 ←→ (self-loop)

  adj[1] = [1]  (1 points to itself)

  Some problems explicitly state "no self-loops"
  Always read constraints carefully!
```

### Convention 4: Multi-edges (Parallel Edges)
```
  Multiple edges between same pair of nodes:

  1 ═══ 2   (two edges between 1 and 2)

  Simple Graph: no multi-edges, no self-loops
  Multigraph:   allows multi-edges and self-loops

  Most competitive programming problems use simple graphs.
```

## Path and Cycle Definitions
```
  Path: sequence of nodes where consecutive nodes share an edge
  1 → 2 → 4 → 5  is a path

  Simple Path: no node repeated
  Cycle: path that starts and ends at same node
  1 → 2 → 3 → 1  is a cycle

  Length of path = number of edges in it
  Path 1→2→4: length = 2
```

---

## Degree of a Node
```
  Undirected:
  Degree = number of edges connected to node

       1 ─── 2 ─── 4
       |     |
       3 ─── ┘

  deg(1) = 2  (connected to 2,3)
  deg(2) = 3  (connected to 1,3,4)
  deg(3) = 2  (connected to 1,2)
  deg(4) = 1  (connected to 2 only → leaf)

  Sum of all degrees = 2 × E  (Handshaking Lemma)

  Directed:
  In-degree  = edges coming INTO node
  Out-degree = edges going OUT of node

       1 ──→ 2 ──→ 3
             ↑
             4

  in-deg(1)=0, out-deg(1)=1
  in-deg(2)=2, out-deg(2)=1
  in-deg(3)=1, out-deg(3)=0
  in-deg(4)=0, out-deg(4)=1
```

## Trees as Graphs
```
  A Tree is a special graph:
  - Undirected
  - Connected
  - Acyclic
  - N nodes → exactly N-1 edges

       1
      / \
     2   3
    / \
   4   5

  Properties:
  ✓ Connected (all nodes reachable)
  ✓ Acyclic (no cycles)
  ✓ N=5 nodes, E=4 edges = N-1 ✓
```

## Summary: Graph Types
```
  Type               Directed?  Weighted?  Cycles?    Example
  ──────────────────────────────────────────────────────────────
  Undirected Graph   No         No         Maybe      Social network
  Directed Graph     Yes        No         Maybe      Web links
  Weighted Graph     Maybe      Yes        Maybe      Road map
  Tree               No         No         No         File system
  DAG                Yes        Maybe      No         Task scheduler
  Bipartite          No         No         Even only  Job matching
  Complete Graph     No         No         Yes        Tournament

  Most Important for DSA:
  ┌────────────────────────────────────────────────┐
  │  Undirected + Unweighted → BFS, DFS            │
  │  Directed + Unweighted   → Topological Sort    │
  │  Weighted                → Dijkstra, Bellman   │
  │  DAG                     → DP on graphs        │
  └────────────────────────────────────────────────┘
```

## Input/Output Convention
```cpp
#include <bits/stdc++.h>
using namespace std;

int main() {
    int n, m;
    cin >> n >> m; // nodes, edges

    // Adjacency list (1-indexed, undirected, unweighted)
    vector<vector<int>> adj(n + 1);

    for (int i = 0; i < m; i++) {
        int u, v;
        cin >> u >> v;
        adj[u].push_back(v);
        adj[v].push_back(u); // Remove for directed
    }

    // Weighted version: store {neighbor, weight}
    vector<vector<pair<int,int>>> wadj(n + 1);
    // wadj[u].push_back({v, w});

    return 0;
}
```
```
  Key Takeaways:
  1. Graph = Nodes + Edges
  2. Undirected ↔ Directed (check problem!)
  3. Weighted ↔ Unweighted (check if weights given)
  4. Use Adjacency List for most problems (O(V+E) space)
  5. Use Adjacency Matrix only for dense graphs or O(1) edge check
  6. Check: 0-indexed or 1-indexed?
  7. Check: self-loops, multi-edges allowed?
```
