# Graph Representation

## Introduction

A graph consists of **nodes (vertices)** and **edges** connecting them. Before applying any graph algorithm, we need to store the graph efficiently.

---

## Input Format
```
  First line:  n m  (n = number of nodes, m = number of edges)
  Next m lines: u v  (edge between node u and node v)

  Example:
  5 6          ← 5 nodes, 6 edges
  1 2
  1 3
  2 4
  3 4
  3 5
  4 5
```

## Two Common Representations
```
  ┌──────────────────────────────────────────────────┐
  │  Adjacency Matrix  vs  Adjacency List             │
  │                                                   │
  │  Space: O(N²)          Space: O(2E) undirected   │
  │                                O(E)  directed    │
  │  Edge check: O(1)      Edge check: O(degree)     │
  │  Find neighbors: O(N)  Find neighbors: O(degree) │
  └──────────────────────────────────────────────────┘
```

---

## Representation 1: Adjacency Matrix

A 2D array of size `(N+1) × (N+1)`.
`adj[u][v] = 1` means there is an edge between `u` and `v`.
```
  Graph: 1-2, 1-3, 2-4, 3-4, 3-5, 4-5

       1  2  3  4  5
  1  [ 0  1  1  0  0 ]
  2  [ 1  0  0  1  0 ]
  3  [ 1  0  0  1  1 ]
  4  [ 0  1  1  0  1 ]
  5  [ 0  0  1  1  0 ]

  adj[3][5] = 1 → edge between 3 and 5 ✓
  adj[1][5] = 0 → no edge between 1 and 5 ✓
```

### Code (Undirected Graph):
```cpp
#include <bits/stdc++.h>
using namespace std;

int main() {
    int n, m;
    cin >> n >> m; // nodes, edges

    // Adjacency Matrix (1-indexed)
    int adj[n + 1][n + 1];
    memset(adj, 0, sizeof(adj)); // Initialize with 0

    for (int i = 0; i < m; i++) {
        int u, v;
        cin >> u >> v;
        adj[u][v] = 1;
        adj[v][u] = 1; // Remove for directed graph
    }

    return 0;
}
```

### Complexity:
```
  Space: O(N²) — stores N×N matrix
  Edge check (u,v): O(1) — direct lookup
  Find all neighbors of u: O(N) — scan entire row

  Drawback: N=10^4 → N²=10^8 → too much memory!
  Only suitable for dense graphs (many edges).
```

## Representation 2: Adjacency List

An array of lists where `adj[u]` stores all neighbors of node `u`.
```
  Graph: 1-2, 1-3, 2-4, 3-4, 3-5, 4-5

  adj[1] → [2, 3]
  adj[2] → [1, 4]
  adj[3] → [1, 4, 5]
  adj[4] → [2, 3, 5]
  adj[5] → [3, 4]

  Each edge appears TWICE in an undirected graph.
  Total space = 2×E entries.
```

### Code (Undirected Graph):
```cpp
#include <bits/stdc++.h>
using namespace std;

int main() {
    int n, m;
    cin >> n >> m;

    // Adjacency List
    vector<vector<int>> adj(n + 1);

    for (int i = 0; i < m; i++) {
        int u, v;
        cin >> u >> v;
        adj[u].push_back(v);
        adj[v].push_back(u); // Undirected: add both directions
    }

    // Print adjacency list
    for (int i = 1; i <= n; i++) {
        cout << i << " → ";
        for (int neighbor : adj[i]) {
            cout << neighbor << " ";
        }
        cout << endl;
    }

    return 0;
}
```

### Code (Directed Graph):
```cpp
#include <bits/stdc++.h>
using namespace std;

int main() {
    int n, m;
    cin >> n >> m;

    vector<vector<int>> adj(n + 1);

    for (int i = 0; i < m; i++) {
        int u, v;
        cin >> u >> v;
        adj[u].push_back(v); // Only u → v (directed)
        // NO adj[v].push_back(u)
    }

    return 0;
}
```
```
  Directed graph: 1→2, 1→3, 2→4

  adj[1] → [2, 3]
  adj[2] → [4]
  adj[3] → []
  adj[4] → []

  Edge from u to v only, NOT v to u ✓
```

### Complexity:
```
  Space: O(2E) undirected, O(E) directed
  Edge check (u,v): O(degree(u))
  Find all neighbors: O(degree(u)) ← much faster!

  Advantage: sparse graphs use much less space.
  N=10^5, E=10^5: adj list uses 2×10^5 vs 10^10 for matrix!
```

## Representation 3: Weighted Graph

For graphs with **edge weights**, store pairs `(neighbor, weight)`.
```
  Weighted graph:
  1 ─5─ 2
  |     |
  3    10
  |     |
  3 ─8─ 4

  adj[1] → [(2,5), (3,3)]
  adj[2] → [(1,5), (4,10)]
  adj[3] → [(1,3), (4,8)]
  adj[4] → [(2,10), (3,8)]
```

### Adjacency Matrix (Weighted):
```
  adj[u][v] = weight of edge (u,v)
  adj[u][v] = 0 or INF if no edge

       1    2    3    4
  1  [ 0    5    3    0  ]
  2  [ 5    0    0    10 ]
  3  [ 3    0    0    8  ]
  4  [ 0    10   8    0  ]
```

### Adjacency List (Weighted):
```cpp
#include <bits/stdc++.h>
using namespace std;

int main() {
    int n, m;
    cin >> n >> m;

    // pair<neighbor, weight>
    vector<vector<pair<int, int>>> adj(n + 1);

    for (int i = 0; i < m; i++) {
        int u, v, w;
        cin >> u >> v >> w;
        adj[u].push_back({v, w}); // u → v with weight w
        adj[v].push_back({u, w}); // v → u with weight w (undirected)
    }

    // Print weighted adjacency list
    for (int i = 1; i <= n; i++) {
        cout << i << " → ";
        for (auto [neighbor, weight] : adj[i]) {
            cout << "(" << neighbor << "," << weight << ") ";
        }
        cout << endl;
    }

    return 0;
}
```

## Comparison Summary
```
  Property           Adjacency Matrix    Adjacency List
  ──────────────────────────────────────────────────────
  Space              O(N²)               O(V + E)
  Add Edge           O(1)                O(1)
  Remove Edge        O(1)                O(E)
  Check Edge(u,v)    O(1)                O(degree)
  Find Neighbors     O(N)                O(degree)
  Best for           Dense graphs        Sparse graphs
  Memory (N=1000)    10^6 cells          2E cells

  ┌─────────────────────────────────────────────────┐
  │  In practice, the ADJACENCY LIST is preferred   │
  │  for most graph problems (BFS, DFS, Dijkstra)   │
  │  because real graphs are usually sparse.        │
  └─────────────────────────────────────────────────┘
```

## Types of Graphs
```
  Undirected:        Directed:          Weighted:
  1 ─── 2            1 ──→ 2            1 ─5─ 2
  |     |            ↑     ↓            |     |
  3 ─── 4            4 ←── 3            3 ─8─ 4

  adj[u].push_back(v)  adj[u].push_back(v)  adj[u].push_back({v,w})
  adj[v].push_back(u)  (no reverse)          adj[v].push_back({u,w})

  Space: O(2E)         Space: O(E)           Space: O(2E) pairs
```
