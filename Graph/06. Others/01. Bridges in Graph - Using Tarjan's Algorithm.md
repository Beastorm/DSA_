## Bridges in Graph - Using Tarjan's Algorithm

### Problem Statement:

There are `n` servers numbered from `0` to `n-1` connected by undirected connections. `connections[i] = [ai, bi]` represents a connection between servers `ai` and `bi`.

A **critical connection** (bridge) is a connection that, if removed, will make some servers unable to reach other servers.

**Return** all critical connections in the network in any order.

---

## Key Concepts

```
What is a Bridge?

A bridge is an edge whose removal INCREASES
the number of connected components.

Example:
    A---B---C
        |
        D

Edge B-C is a bridge:
→ Remove it: {A,B,D} and {C} become separate
→ Number of components: 1 → 2

Edge A-B is NOT a bridge:
→ Remove it: Still connected via A-B-D path
→ Wait, there's no such path here!
→ Actually A-B IS a bridge in this graph

Corrected example:
    A---B---C
       / \
      D   E

Edge B-C is a bridge (no alternate path)
Edge A-B is NOT a bridge (can reach A via A-D-B)

Key insight:
An edge (u,v) is a bridge if there's NO back edge
from v's subtree to u or u's ancestors!
```

## Tarjan's Algorithm Intuition

```
Two key concepts:

1. Discovery Time (tin[]):
   When we first visit a node during DFS
   
   Example DFS order: 0 → 1 → 2 → 3
   tin[0] = 1
   tin[1] = 2
   tin[2] = 3
   tin[3] = 4

2. Low Link Value (low[]):
   Lowest discovery time reachable from node's subtree
   (including back edges)
   
   low[v] = min(
     tin[v],                    // Discovery time
     low[child],                // Children's low values
     tin[back_edge_destination] // Back edges
   )

Bridge Detection Rule:
Edge (u, v) is a bridge if:
  low[v] > tin[u]

Why?
→ If low[v] > tin[u]: v's subtree can't reach u or ancestors
→ No back edge exists!
→ Removing (u,v) disconnects v's subtree
→ Therefore, (u,v) is a BRIDGE! ✓
```

---

## Algorithm

```
Initialize:
→ timer = 1 (global time counter)
→ vis[] = all 0 (visited array)
→ tin[] = discovery time array
→ low[] = low link value array
→ bridges[] = result array

DFS(node, parent):
  1. Mark node as visited
     vis[node] = 1
  
  2. Set discovery time and initial low value
     tin[node] = low[node] = timer
     timer++
  
  3. For each neighbor of node:
     
     If neighbor == parent:
       → Skip (don't go back to parent)
     
     If neighbor not visited:
       → DFS(neighbor, node)
       → Update low[node] = min(low[node], low[neighbor])
       
       → Check if bridge:
         If low[neighbor] > tin[node]:
           → Bridge found! Add [node, neighbor]
     
     If neighbor already visited:
       → Back edge! Update low[node]
       → low[node] = min(low[node], low[neighbor])

Call DFS(0, -1) to start
Return bridges[]
```

## Code

```cpp
#include <bits/stdc++.h>
using namespace std;

class Solution {
private:
    int timer = 1;
    
    void dfs(int node, int parent, vector<int>& vis,
             vector<int> adj[], int tin[], int low[],
             vector<vector<int>>& bridges) {
        
        // Mark visited
        vis[node] = 1;
        
        // Set discovery time and low value
        tin[node] = low[node] = timer;
        timer++;
        
        // Explore neighbors
        for (int neighbor : adj[node]) {
            // Skip parent edge
            if (neighbor == parent) continue;
            
            if (!vis[neighbor]) {
                // Unvisited neighbor: recurse
                dfs(neighbor, node, vis, adj, tin, low, bridges);
                
                // Update low value from child
                low[node] = min(low[node], low[neighbor]);
                
                // Check if bridge
                if (low[neighbor] > tin[node]) {
                    bridges.push_back({node, neighbor});
                }
            }
            else {
                // Already visited: back edge
                low[node] = min(low[node], low[neighbor]);
            }
        }
    }
    
public:
    vector<vector<int>> criticalConnections(int n, 
        vector<vector<int>>& connections) {
        
        // Build adjacency list
        vector<int> adj[n];
        for (auto& edge : connections) {
            adj[edge[0]].push_back(edge[1]);
            adj[edge[1]].push_back(edge[0]);
        }
        
        // Initialize arrays
        vector<int> vis(n, 0);
        int tin[n];
        int low[n];
        vector<vector<int>> bridges;
        
        // Run DFS from node 0
        dfs(0, -1, vis, adj, tin, low, bridges);
        
        return bridges;
    }
};

int main() {
    int n = 4;
    vector<vector<int>> connections = {
        {0, 1}, {1, 2}, {2, 0}, {1, 3}
    };
    
    Solution obj;
    vector<vector<int>> bridges = obj.criticalConnections(n, connections);
    
    cout << "Bridges: ";
    for (auto& bridge : bridges) {
        cout << "[" << bridge[0] << "," << bridge[1] << "] ";
    }
    cout << endl;
    
    return 0;
}
```

**Output:**
```
Bridges: [1,3]
```

## Visual Understanding

```
Example: Why [1,3] is a bridge

Graph with tin and low values:
    0 (tin=1, low=1)
   /|\
  / | \
 1  |  2
(2,1) (3,1)
 |   \/
 |
 3 (tin=4, low=4)

Bridge condition: low[child] > tin[parent]

Edge (1,3):
  parent = 1, child = 3
  low[3] = 4
  tin[1] = 2
  4 > 2? YES → BRIDGE! ✓

Why low[3] = 4?
→ Node 3 has no back edges
→ Can't reach any ancestor of 1
→ Only reachable time is its own (4)
→ Therefore, removing (1,3) disconnects 3!

Edge (0,1):
  parent = 0, child = 1
  low[1] = 1
  tin[0] = 1
  1 > 1? NO → NOT a bridge ✗

Why low[1] = 1?
→ From 1's subtree (including 2)
→ There's back edge 2→0
→ Can reach node 0 (time=1)
→ Therefore, alternate path exists!
```

## Why Skip Parent Edge?

```
Problem: Undirected graph has bidirectional edges

Example: Edge (0,1) appears as:
  adj[0] contains 1
  adj[1] contains 0

Without parent check:
DFS(0, parent=-1):
  Visit neighbor 1
  DFS(1, parent=0):
    Visit neighbor 0 ← Would go back immediately!
    But 0 is already visited
    Incorrectly treats 0-1 as back edge
    Updates low[1] = min(low[1], low[0])
    Hides the true bridge!

With parent check:
DFS(1, parent=0):
  For neighbor 0:
    if (neighbor == parent) continue; ✓
    Skips the edge back to parent
    Only processes actual back edges

This prevents:
1. Infinite loops (0→1→0→1...)
2. False back edge detection
3. Incorrect low value updates
```

## Complexity Analysis

**Time Complexity:** `O(V + E)`
- DFS visits each vertex once: O(V)
- Each edge explored twice (undirected): O(2E) = O(E)
- Total: **O(V + E)**

**Space Complexity:** `O(V + E)`
- Adjacency list: O(V + 2E) = O(V + E)
- vis, tin, low arrays: O(3V) = O(V)
- Recursion stack: O(V) in worst case
- Total: **O(V + E)**

```
Detailed breakdown:

DFS traversal:
→ Each vertex visited once: O(V)
→ Each edge checked twice (u→v and v→u): O(E)
→ Per vertex/edge: O(1) operations
→ Total: O(V + E)

Space:
→ Adjacency list: V vertices + 2E edges = O(V + E)
→ vis[]: V entries
→ tin[]: V entries
→ low[]: V entries
→ Recursion depth: O(V) worst case (linear graph)
→ Total: O(V + E)

Comparison with other bridge-finding methods:
- Naive (remove each edge, check connectivity): O(E × (V+E))
- Tarjan's: O(V + E) ✓
- Much more efficient!
```

## Common Mistakes

```
❌ Mistake 1: Not skipping parent
for (int neighbor : adj[node]) {
    if (!vis[neighbor]) {  // WRONG! Missing parent check
        dfs(neighbor, node, ...);
    }
}

✓ Correct:
if (neighbor == parent) continue;

❌ Mistake 2: Using tin instead of low for back edges
if (!vis[neighbor]) {
    // ...
} else {
    low[node] = min(low[node], tin[neighbor]);  // WRONG!
}

✓ Correct:
low[node] = min(low[node], low[neighbor]);

❌ Mistake 3: Wrong bridge condition
if (low[neighbor] >= tin[node]) {  // WRONG! Should be >
    bridges.push_back({node, neighbor});
}

✓ Correct:
if (low[neighbor] > tin[node]) {

❌ Mistake 4: Not initializing timer
int timer;  // WRONG! Uninitialized

✓ Correct:
int timer = 1;  // Start from 1 (or 0)

❌ Mistake 5: Forgetting to increment timer
tin[node] = low[node] = timer;
// WRONG! Forgot timer++

✓ Correct:
tin[node] = low[node] = timer;
timer++;
```

## Why `low[neighbor] > tin[node]` Detects Bridges

```
Mathematical proof:

Theorem: Edge (u,v) is a bridge iff low[v] > tin[u]

Proof:

(⇒) If (u,v) is a bridge:
  → Removing (u,v) disconnects v's subtree from u
  → No path from v's subtree to u or ancestors
  → Therefore, low[v] > tin[u] ✓

(⇐) If low[v] > tin[u]:
  → v's subtree can't reach u or any ancestor
  → No back edge from v's subtree to upper levels
  → Only connection to upper levels is edge (u,v)
  → Therefore, (u,v) is a bridge ✓

Example:
    u (tin=2)
    |
    v (tin=3, low=4)
    |
    w (tin=4, low=4)

low[v] = 4 > tin[u] = 2
→ v's subtree {v,w} can't reach u or above
→ Edge (u,v) is bridge! ✓

vs. Non-bridge:
    u (tin=2)
    |
    v (tin=3, low=1)
   /
  back edge to ancestor (tin=1)

low[v] = 1 ≤ tin[u] = 2
→ v's subtree CAN reach ancestors
→ Edge (u,v) is NOT a bridge ✗
```
