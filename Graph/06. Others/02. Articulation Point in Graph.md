## Articulation Point in Graph

### Problem Statement:

Find all **articulation points** (cut vertices) in an undirected graph. An articulation point is a vertex whose removal **increases the number of connected components**.

---

## Examples

**Example 1:**

**Graph:**
```
    0
   /|\
  1 | 3
   \|/
    2
```

**Output:** `[0, 2]`

```
Explanation:

Removing node 0:
  Component 1: {1, 2}
  Component 2: {3}
  → 2 components (increased from 1)
  → 0 is articulation point ✓

Removing node 2:
  Component 1: {0, 1}
  Component 2: {3}
  → 2 components
  → 2 is articulation point ✓

Removing node 1 or 3:
  → Still 1 component
  → Not articulation points ✗
```

**Example 2:**

**Graph:**
```
    0---1---2
        |   |
        4---3
```

**Output:** `[1, 4]`

```
Removing node 1:
  Component 1: {0}
  Component 2: {2, 3, 4}
  → 2 components
  → 1 is articulation point ✓

Removing node 4:
  Component 1: {0, 1, 2}
  Component 2: {3}
  → 2 components
  → 4 is articulation point ✓

Others don't disconnect the graph ✗
```

---

## Key Concepts

```
Articulation Point vs Bridge:

Bridge:
→ Edge whose removal disconnects graph
→ Example: edge (u,v)

Articulation Point:
→ Vertex whose removal disconnects graph
→ Example: vertex u

Relationship:
→ If (u,v) is a bridge, one of u or v might be articulation point
→ But not always! (consider leaf nodes)

Example:
    A---B---C
    
B is articulation point (removing disconnects A from C)
Both (A,B) and (B,C) are bridges
But A and C are NOT articulation points (leaves)
```

---

## Tarjan's Algorithm for Articulation Points

```
Similar to bridge detection, but with key differences:

1. Discovery Time (tin[]):
   When we first visit a node during DFS

2. Low Link Value (low[]):
   Lowest discovery time reachable from subtree

3. Articulation Point Condition:
   
   For non-root node u:
   → If any child v has: low[v] >= tin[u]
   → Then u is an articulation point!
   
   Why >= instead of > (like bridges)?
   → If low[v] == tin[u]: v can reach u but not beyond
   → Removing u still disconnects v's subtree from rest
   → So u is articulation point!

4. Special Case: Root Node
   → Root is articulation point if it has > 1 child in DFS tree
   → Why? Each child represents separate component
   → Removing root disconnects these components
```

---

## Algorithm

```
Initialize:
→ timer = 1
→ vis[] = all 0
→ tin[] = discovery time
→ low[] = low link value
→ mark[] = is articulation point?

DFS(node, parent):
  1. Mark visited
     vis[node] = 1
  
  2. Set times
     tin[node] = low[node] = timer++
  
  3. Count children (for root case)
     child = 0
  
  4. For each neighbor:
     
     If neighbor == parent:
       → Skip
     
     If neighbor not visited:
       → DFS(neighbor, node)
       → low[node] = min(low[node], low[neighbor])
       → child++
       
       → Articulation check (non-root):
         If low[neighbor] >= tin[node] AND parent != -1:
           mark[node] = 1  ← Articulation point!
     
     Else (visited):
       → low[node] = min(low[node], tin[neighbor])
  
  5. Root special case:
     If parent == -1 AND child > 1:
       mark[node] = 1  ← Root is articulation point!

Main:
  For each unvisited node:
    DFS(node, -1)
  
  Return all nodes where mark[] = 1
```

## Code

```cpp
#include <bits/stdc++.h>
using namespace std;

class Solution {
private:
    int timer = 1;
    
    void dfs(int node, int parent, vector<int>& vis,
             int tin[], int low[], vector<int>& mark,
             vector<int> adj[]) {
        
        // Mark visited
        vis[node] = 1;
        
        // Set discovery time and low value
        tin[node] = low[node] = timer++;
        
        int child = 0;  // Count children for root case
        
        // Explore neighbors
        for (int neighbor : adj[node]) {
            // Skip parent
            if (neighbor == parent) continue;
            
            if (!vis[neighbor]) {
                // Unvisited: recurse
                dfs(neighbor, node, vis, tin, low, mark, adj);
                
                // Update low value from child
                low[node] = min(low[node], low[neighbor]);
                
                // Articulation point check (non-root)
                if (low[neighbor] >= tin[node] && parent != -1) {
                    mark[node] = 1;
                }
                
                child++;
            }
            else {
                // Already visited: back edge
                low[node] = min(low[node], tin[neighbor]);
            }
        }
        
        // Root case: articulation if > 1 child
        if (parent == -1 && child > 1) {
            mark[node] = 1;
        }
    }
    
public:
    vector<int> articulationPoints(int n, vector<int> adj[]) {
        vector<int> vis(n, 0);
        vector<int> mark(n, 0);
        int tin[n];
        int low[n];
        
        // Handle disconnected components
        for (int i = 0; i < n; i++) {
            if (!vis[i]) {
                dfs(i, -1, vis, tin, low, mark, adj);
            }
        }
        
        // Collect articulation points
        vector<int> ans;
        for (int i = 0; i < n; i++) {
            if (mark[i] == 1) {
                ans.push_back(i);
            }
        }
        
        // Return -1 if no articulation points
        return ans.empty() ? vector<int>{-1} : ans;
    }
};

int main() {
    int n = 5;
    vector<vector<int>> edges = {
        {0, 1}, {1, 4}, {2, 4}, {2, 3}, {3, 4}
    };
    
    // Build adjacency list
    vector<int> adj[n];
    for (auto& e : edges) {
        adj[e[0]].push_back(e[1]);
        adj[e[1]].push_back(e[0]);
    }
    
    Solution sol;
    vector<int> result = sol.articulationPoints(n, adj);
    
    cout << "Articulation points: ";
    for (int v : result) {
        cout << v << " ";
    }
    cout << endl;
    
    return 0;
}
```

**Output:**
```
Articulation points: 1 4
```

## Why `low[v] >= tin[u]` for Articulation Points?

```
Comparison with bridges:

Bridges: low[v] > tin[u]
  → v's subtree CANNOT reach u or ancestors
  → Edge (u,v) is the only connection
  → Removing edge disconnects

Articulation Points: low[v] >= tin[u]
  → v's subtree can reach AT MOST u (not beyond)
  → Removing u disconnects v's subtree from rest
  → Node u is articulation point

Example showing the difference:

Graph:
    1 (tin=1)
    |
    2 (tin=2, low=2)
   / \
  3   4
   \ /
    5

For edge (1,2):
  low[2] = 2 (can reach itself via cycle 2-3-5-4-2)
  tin[1] = 1
  low[2] > tin[1]? 2 > 1? YES
  → (1,2) is a BRIDGE ✓

For node 1:
  low[2] >= tin[1]? 2 >= 1? YES
  → But node 2's subtree can still reach node 2
  → Removing node 1 disconnects 2's subtree
  → Node 1 is ARTICULATION POINT ✓

The >= captures the case where subtree
can reach the node but not beyond it!
```

## Root Node Special Case

```
Why is root handled differently?

Non-root node:
  → Has parent in DFS tree
  → Removing it might disconnect children from parent
  → Check: low[child] >= tin[node]

Root node:
  → Has NO parent in DFS tree
  → Different disconnection logic
  → If root has > 1 child in DFS tree:
    → Each child is separate subtree
    → Removing root disconnects these subtrees
    → Root is articulation point!

Example 1: Root with 2 children
    0 (root)
   / \
  1   2

DFS tree from 0:
  → Child 1: subtree {1, ...}
  → Child 2: subtree {2, ...}
  → child = 2 > 1
  → 0 is articulation point! ✓

Example 2: Root with 1 child
    0 (root)
    |
    1---2

DFS tree from 0:
  → Child 1: subtree {1, 2, ...}
  → child = 1
  → 0 is NOT articulation point ✗
  → Removing 0 just leaves {1,2,...} connected

Code:
if (parent == -1 && child > 1) {
    mark[node] = 1;
}
```

## Why Use `tin[neighbor]` for Back Edges?

```
Question: For visited neighbors (back edges),
why update low[node] = min(low[node], tin[neighbor])
instead of low[neighbor]?

Answer: Back edges point to ancestors!

Example:
    1 (tin=1, low=1)
    |
    2 (tin=2)
   /|
  3 |
   \|
    4 (tin=4)
    ↑ back edge to 2

When at node 4, checking neighbor 2:
  2 is already visited (ancestor)
  
  Using tin[2]:
    low[4] = min(low[4], tin[2])
           = min(4, 2) = 2 ✓
    Correctly shows 4 can reach ancestor 2
  
  Using low[2] (WRONG):
    low[2] might be 1 (if 2 can reach 1)
    low[4] = min(4, 1) = 1
    But this is misleading!
    The connection is directly to 2, not 1

tin[] represents the actual reachable ancestor
low[] might include indirect reachability

For back edges: use tin[] ✓
For tree edges: use low[] (after DFS returns) ✓
```

## Complexity Analysis

**Time Complexity:** `O(V + E)`
- DFS visits each vertex once: O(V)
- Each edge explored twice: O(E)
- Total: **O(V + E)**

**Space Complexity:** `O(V)`
- vis, mark arrays: O(2V) = O(V)
- tin, low arrays: O(2V) = O(V)
- Recursion stack: O(V)
- Total: **O(V)**

```
Detailed breakdown:

DFS traversal:
→ Each vertex visited once
→ Each edge checked twice (undirected)
→ Constant work per vertex/edge
→ Total: O(V + E)

Space:
→ vis[]: V entries
→ mark[]: V entries
→ tin[]: V entries
→ low[]: V entries
→ Recursion depth: V (worst case chain)
→ Total: O(V)

Note: Adjacency list not counted
(considered part of input)
```

## Common Mistakes

```
❌ Mistake 1: Using > instead of >=
if (low[neighbor] > tin[node] && parent != -1) {  // WRONG!
    mark[node] = 1;
}

✓ Correct:
if (low[neighbor] >= tin[node] && parent != -1) {

❌ Mistake 2: Forgetting root special case
// Only checking low[child] >= tin[node]
// WRONG! Misses root with multiple children

✓ Correct:
if (parent == -1 && child > 1) {
    mark[node] = 1;
}

❌ Mistake 3: Using low[] for back edges
low[node] = min(low[node], low[neighbor]);  // WRONG!

✓ Correct (for visited neighbor):
low[node] = min(low[node], tin[neighbor]);

❌ Mistake 4: Not handling disconnected components
dfs(0, -1, ...);  // WRONG! Only processes one component

✓ Correct:
for (int i = 0; i < n; i++) {
    if (!vis[i]) dfs(i, -1, ...);
}

❌ Mistake 5: Counting parent as child
for (auto neighbor : adj[node]) {
    if (!vis[neighbor]) {
        child++;  // WRONG! Counts before parent check
    }
}

✓ Correct:
if (neighbor == parent) continue;
if (!vis[neighbor]) {
    // ... dfs ...
    child++;  // Count after processing
}
```

## Articulation Points vs Bridges

```
Property              Articulation Point     Bridge
─────────────────────────────────────────────────────────────
Definition            Vertex removal         Edge removal
                      increases components   increases components

Condition             low[v] >= tin[u]       low[v] > tin[u]
                      (can reach at most u)  (cannot reach u)

Root handling         Special case           No special case
                      (child > 1)

Leaf nodes            Never articulation     Edge to leaf is bridge
                      (except single edge)   

Count in tree         n-2 (all non-leaves)   n-1 (all edges)

Example:
    1---2---3
    
Articulation points: {2}
Bridges: {(1,2), (2,3)}

Node 2 is articulation (removing disconnects 1 from 3)
Both edges are bridges (removing either disconnects graph)
But nodes 1 and 3 are NOT articulation points (leaves)
```

## Summary

```
Problem: Find articulation points in undirected graph

Articulation Point:
→ Vertex whose removal disconnects graph
→ Critical vertex for connectivity

Tarjan's Algorithm:
→ DFS-based, single pass
→ Track tin[] and low[]
→ Check low[child] >= tin[node]

Key differences from bridges:
1. Condition: >= instead of >
2. Root special case: child > 1
3. Mark vertices not edges

Implementation points:
→ Skip parent edge
→ Update low from children (DFS)
→ Update low from back edges (tin[])
→ Handle disconnected components

Time: O(V + E)
Space: O(V)

Applications:
```
