## 1. Problem Statement

Given an **undirected graph** with `n` vertices and `m` edges, determine whether a **Hamiltonian Path** exists in the graph.

- **Hamiltonian Path**: A path that visits every **vertex** exactly once.
- **Hamiltonian Cycle**: A cycle that visits every **vertex** exactly once and returns to the starting vertex.

**Return:** `true` if such a path/cycle exists, `false` otherwise.

> **Note:** This problem is **NP-Complete**. There is no known efficient algorithm for all cases. We use backtracking to try all possibilities.

## Key Insight

```
Observation 1: Hamiltonian = Visit Every Vertex Once

Not every edge — every VERTEX!

Visual comparison:

Euler Path (edges):
  0 ─── 1 ─── 2 ─── 3
  Walk: 0→1→2→3 (uses all 3 edges)
  Vertex 1 visited twice? OK for Euler!

Hamiltonian Path (vertices):
  0 ─── 1 ─── 2 ─── 3
  Walk: 0→1→2→3 (visits all 4 vertices)
  Each vertex exactly once! ✓

──────────────────────────────────────────────────────────────

Observation 2: Backtracking is Needed

There is no simple formula like Euler's odd-degree rule.
We must TRY different paths and backtrack when stuck.

Visual (backtracking):

  Graph:
        1
       / \
      2   3
      │   │
      4   5

  Try: 1 → 2 → 4 (stuck! 3 and 5 unvisited) ✗
  Backtrack to 2.
  Try: 1 → 2... no other neighbors.
  Backtrack to 1.
  Try: 1 → 3 → 5 (stuck! 2 and 4 unvisited) ✗
  Backtrack to 3.
  Try: 1 → 3... no other neighbors.
  Backtrack to 1.

  No path found! Return false.

──────────────────────────────────────────────────────────────

Observation 3: Try Every Starting Vertex

A Hamiltonian Path might exist only from certain starting points.

Visual:

  Graph: 1 ─── 2 ─── 3 ─── 4

  Start at 1: 1→2→3→4 ✓ (works!)
  Start at 2: 2→1... stuck, or 2→3→4... 1 unvisited ✗

  So we must try starting from every vertex!
```

## Intuition

```
Imagine you are a tourist visiting cities.

You want to visit EVERY city exactly once.
You don't care about visiting every road — just every city.

──────────────────────────────────────────────────────────────

Hamiltonian Path = Tourist Route:

You start at one city, visit every other city exactly once,
and stop at the last city. No need to return home.

Visual:

    Delhi ── Jaipur ── Agra ── Mumbai

    Route: Delhi → Jaipur → Agra → Mumbai
    Every city visited once. Done! ✓

──────────────────────────────────────────────────────────────

Hamiltonian Cycle = Round Trip:

You start at home, visit every city exactly once,
and return home at the end.

Visual:

    Delhi ── Jaipur
     │         │
    Mumbai ── Agra

    Route: Delhi → Jaipur → Agra → Mumbai → Delhi
    Every city visited once, back home! ✓

──────────────────────────────────────────────────────────────

Why Some Graphs Fail:

Star graph — like an airport hub with only flights to suburbs.

          City A
            │
            │
    City B──Hub──City C
            │
            │
          City D

You fly from City A to Hub. Now you must go to City B, C, or D.
Say you go to City B. You are stuck! To reach C and D,
you must return to Hub, but you already visited Hub.

The hub is a trap — once you leave, you cannot come back
without revisiting it. ✗

──────────────────────────────────────────────────────────────

Backtracking = Trying Different Routes:

Like planning a road trip with a map and eraser.

Step 1: Start at City 1, go to City 2.
Step 2: From City 2, try City 3.
Step 3: From City 3, dead end! All neighbors visited.
Step 4: Erase City 3. Back to City 2.
Step 5: From City 2, try City 4 instead.
Step 6: Continue... until all cities visited or all tries fail.
```

## Algorithm

```
Step 1: Build adjacency list

For each edge (u, v):
  adj[u].push_back(v)
  adj[v].push_back(u)

Visual:

  edges = [[1,2], [2,3], [3,4]]

  adj[1] = [2]
  adj[2] = [1, 3]
  adj[3] = [2, 4]
  adj[4] = [3]

──────────────────────────────────────────────────────────────

Step 2: Try each vertex as starting point

For start = 0 to n-1:
  visited = [false, false, ..., false]
  count = 0
  if dfs(start) returns true:
    return true

Return false

Visual (trying starts):

  Graph: 1 ─── 2 ─── 3 ─── 4

  Try start = 1:
    1 → 2 → 3 → 4 (count = 4 = n) ✓
    Return true!

  (No need to try other starts)

──────────────────────────────────────────────────────────────

Step 3: DFS with backtracking

Function dfs(node):
  visited[node] = true
  count++

  If count == n:
    return true  ← All vertices visited!

  For each neighbor of node:
    If neighbor not visited:
      If dfs(neighbor) returns true:
        return true  ← Found valid path!

  visited[node] = false  ← BACKTRACK!
  count--
  return false

Visual (backtracking in action):

  Graph:
        1
       / \
      2   3
      │
      4

  dfs(1): visit 1, count=1
    neighbor 2: dfs(2): visit 2, count=2
      neighbor 1: visited, skip
      neighbor 4: dfs(4): visit 4, count=3
        neighbor 2: visited, skip
        count(3) != n(4) → return false
      backtrack: unvisit 4, count=2
      no more neighbors → return false
    backtrack: unvisit 2, count=1
    neighbor 3: dfs(3): visit 3, count=2
      neighbor 1: visited, skip
      count(2) != n(4) → return false
    backtrack: unvisit 3, count=1
  backtrack: unvisit 1, count=0
  return false
```

## Code

```cpp
#include <iostream>
#include <vector>
using namespace std;

// DFS function to find the Hamiltonian Path using Backtracking
bool dfs(int node, vector<vector<int>>& adj, vector<bool>& visited, int& count, int n) {
    visited[node] = true;
    count++;

    if (count == n) {
        return true;
    }

    for (int i = 0; i < adj[node].size(); i++) {
        int neighbor = adj[node][i];
        if (!visited[neighbor]) {
            if (dfs(neighbor, adj, visited, count, n)) {
                return true;
            }
        }
    }

    visited[node] = false;
    count--;
    return false;
}

// Function to check if a Hamiltonian Path exists
bool checkHamiltonianPath(int n, int m, vector<vector<int>>& edges) {
    vector<vector<int>> adj(n);
    for (int i = 0; i < m; i++) {
        int u = edges[i][0] - 1;
        int v = edges[i][1] - 1;
        adj[u].push_back(v);
        adj[v].push_back(u);
    }

    for (int i = 0; i < n; i++) {
        vector<bool> visited(n, false);
        int count = 0;
        if (dfs(i, adj, visited, count, n)) {
            return true;
        }
    }

    return false;
}

// DFS function for Hamiltonian Cycle
bool dfsCycle(int node, int start, vector<vector<int>>& adj, vector<bool>& visited, int& count, int n) {
    visited[node] = true;
    count++;

    if (count == n) {
        // Check if last node is connected back to start
        for (int neighbor : adj[node]) {
            if (neighbor == start) {
                return true;
            }
        }
        return false;
    }

    for (int i = 0; i < adj[node].size(); i++) {
        int neighbor = adj[node][i];
        if (!visited[neighbor]) {
            if (dfsCycle(neighbor, start, adj, visited, count, n)) {
                return true;
            }
        }
    }

    visited[node] = false;
    count--;
    return false;
}

// Function to check if a Hamiltonian Cycle exists
bool checkHamiltonianCycle(int n, int m, vector<vector<int>>& edges) {
    vector<vector<int>> adj(n);
    for (int i = 0; i < m; i++) {
        int u = edges[i][0] - 1;
        int v = edges[i][1] - 1;
        adj[u].push_back(v);
        adj[v].push_back(u);
    }

    for (int i = 0; i < n; i++) {
        vector<bool> visited(n, false);
        int count = 0;
        if (dfsCycle(i, i, adj, visited, count, n)) {
            return true;
        }
    }

    return false;
}

// ==================== DRIVER CODE ====================
int main() {
    int n = 4, m = 4;
    vector<vector<int>> edges = {{1,2}, {2,3}, {3,4}, {4,1}};

    if (checkHamiltonianPath(n, m, edges))
        cout << "Hamiltonian Path Exists" << endl;
    else
        cout << "No Hamiltonian Path" << endl;

    if (checkHamiltonianCycle(n, m, edges))
        cout << "Hamiltonian Cycle Exists" << endl;
    else
        cout << "No Hamiltonian Cycle" << endl;

    return 0;
}
```

**Output:**

```
Hamiltonian Path Exists
Hamiltonian Cycle Exists
```

### Common Mistakes

```
❌ Mistake 1: Confusing with Euler Path
  Euler = visit every EDGE once
  Hamiltonian = visit every VERTEX once

  Visual:
    Graph: 1 ─── 2 ─── 3

    Euler Path: 1→2→3 (uses both edges) ✓
    Hamiltonian Path: 1→2→3 (visits all 3 vertices) ✓

    But in general, they are DIFFERENT problems!

✓ Correct: Remember — Hamiltonian = vertices!

──────────────────────────────────────────────────────────────

❌ Mistake 2: Forgetting to backtrack
  visited[node] = true;
  count++;
  // ... try neighbors ...
  // Forgot to unmark! ✗

  This means once you visit a node, it stays visited
  forever, preventing other paths from using it.

✓ Correct:
  visited[node] = false;  // BACKTRACK!
  count--;

  Visual:
    Try: 1 → 2 → 3 (fails)
    Backtrack: unvisit 3, then 2
    Try: 1 → 4 → 5 (might work!)

──────────────────────────────────────────────────────────────

❌ Mistake 3: Not trying all starting vertices
  Only try start = 0
  If dfs(0) fails, return false ✗

  The path might only exist from a different start!

✓ Correct:
  for (int i = 0; i < n; i++) {
      if (dfs(i)) return true;
  }

──────────────────────────────────────────────────────────────

❌ Mistake 4: Off-by-one in cycle check
  In cycle, after visiting all n vertices,
  check if LAST vertex connects to START.

  if (count == n) {
      return true;  // WRONG! Must check edge to start!
  }

✓ Correct:
  if (count == n) {
      for (int neighbor : adj[node])
          if (neighbor == start) return true;
      return false;
  }

──────────────────────────────────────────────────────────────

❌ Mistake 5: Thinking there is a fast formula
  "All degrees >= n/2 → Hamiltonian exists" (Dirac's theorem)
  This is a SUFFICIENT condition, not NECESSARY!

  Some graphs with small degrees still have Hamiltonian paths.
  There is no simple check like Euler's odd-degree rule.

✓ Correct: Backtracking is the general approach.
```

## Complexity Analysis

**Time Complexity:** `O(N!)`

- In the worst case (complete graph), we try all permutations of vertices.
- From first node: (N-1) choices
- From second node: (N-2) choices
- ...and so on
- Total: `(N-1) × (N-2) × ... × 1 = (N-1)!`
- We try each starting node: `N × (N-1)! = N!`

**Space Complexity:** `O(N)`

- `visited` array: O(N)
- Recursion stack: O(N) (path length)
- Adjacency list: O(N + E)
- Total: **O(N + E)**

```
Visual breakdown:

Time:
  Complete graph with N vertices:

  Start at vertex 0:
    0 → choose from (N-1) neighbors
      → choose from (N-2) neighbors
        → choose from (N-3) neighbors
          ...
          → 1 choice left

  Total paths from 0: (N-1)!

  Try all N starting vertices:
    N × (N-1)! = N!

  For N = 10: 10! = 3,628,800 paths
  For N = 15: 15! ≈ 1.3 trillion paths!

  This is why Hamiltonian is NP-Complete —
  no known fast algorithm for all cases.

Space:
  visited array: N booleans
  recursion stack: at most N levels deep
  adjacency list: N + E entries

  Overall: O(N + E)
```

## Interview Questions

```
Q1: What is the difference between Hamiltonian and Euler paths?

A: Hamiltonian visits every VERTEX exactly once.
   Euler visits every EDGE exactly once.

   Visual:

   Graph:
       1 ──── 2 ──── 3
              │
              4

   Euler Path: 1→2→4→2→3 (uses edges 1-2, 2-4, 4-2, 2-3)
   Wait, edge 2-4 used twice! Not valid.
   Actually: 4→2→1, then stuck. Or 1→2→3, edge 2-4 unused.
   This graph has no Euler Path!

   Hamiltonian Path: 1→2→4 (3 vertices) or 1→2→3 (3 vertices)
   Or 4→2→3 (3 vertices). All visit each vertex once! ✓

   They solve completely different problems!

──────────────────────────────────────────────────────────────

Q2: Why is the Hamiltonian problem NP-Complete?

A: There is no known shortcut to know if a path exists without
   basically trying all possibilities. Unlike Euler (where we just
   count odd degrees), Hamiltonian has no simple formula.

   Think of it like a maze with N rooms. To know if a path through
   all rooms exists, you might need to try every possible route.
   The number of routes grows as N factorial — incredibly fast!

   Visual:

   N = 5:  5!  = 120 paths     (manageable)
   N = 10: 10! = 3.6 million   (slow)
   N = 20: 20! ≈ 2.4 quintillion (impossible!)

──────────────────────────────────────────────────────────────

Q3: What is backtracking?

A: Backtracking is like exploring a maze with a pencil and eraser.
   You try one path. If it leads to a dead end, you erase your
   steps and try a different path.

   Visual:

        Start
       /     \
      A       B
     / \       \
    C   D       E

   Try: Start → A → C (dead end!)
   Erase C. Back to A.
   Try: Start → A → D (dead end!)
   Erase D, erase A. Back to Start.
   Try: Start → B → E (success!)

   In code: mark visited, recurse, then unmark (backtrack).

──────────────────────────────────────────────────────────────

Q4: Why do we need to try every starting vertex?

A: A Hamiltonian Path might only work from certain starting points.

   Visual:

   Graph: 1 ─── 2 ─── 3 ─── 4

   Start at 1: 1→2→3→4 ✓ (works!)
   Start at 2: 2→1... stuck! Or 2→3→4... 1 unvisited ✗

   If we only tried start = 2, we would miss the valid path!

──────────────────────────────────────────────────────────────

Q5: How do you check for a Hamiltonian Cycle vs Path?

A: For a cycle, after visiting all N vertices, check if the last
   vertex has an edge back to the START vertex.

   Visual (Cycle):

   Graph:
       1 ──── 2
       │      │
       │      │
       4 ──── 3

   Path: 1→2→3→4 (all 4 visited)
   Check: is 4 connected to 1? YES! ✓
   → Hamiltonian Cycle exists!

   Visual (Path only):

   Graph: 1 ─── 2 ─── 3 ─── 4

   Path: 1→2→3→4 (all 4 visited)
   Check: is 4 connected to 1? NO! ✗
   → Only Hamiltonian Path, not Cycle.

──────────────────────────────────────────────────────────────

Q6: What is Dirac's Theorem?

A: Dirac's Theorem gives a SUFFICIENT (but not necessary) condition:
   If every vertex has degree >= N/2, then a Hamiltonian Cycle exists.

   Visual:

   Graph with 6 vertices where every vertex has degree >= 3:

       1 ──── 2
      /|\    /|\
     / | \  / | \
    6──3──4──5

   Every vertex connected to at least 3 others.
   By Dirac's theorem: Hamiltonian Cycle guaranteed! ✓

   But the reverse is NOT true:
   Some graphs with small degrees still have Hamiltonian paths.

──────────────────────────────────────────────────────────────

Q7: Why does the star graph have no Hamiltonian Path?

A: The center is a bottleneck. Once you leave the center for a leaf,
   you cannot return without revisiting the center.

   Visual:

          2
          │
    3 ─── 1 ─── 4
          │
          5

   Start at 2: 2→1. Now choose 3, 4, or 5.
   Say 3: 2→1→3. Stuck! 4 and 5 unreachable.
   Because from 3, only neighbor is 1 (already visited).

   Start at 1: 1→2. Now at 2, only neighbor is 1 (visited).
   Stuck immediately!

   No matter where you start, you get stuck with unvisited leaves.

──────────────────────────────────────────────────────────────

Q8: Can a disconnected graph have a Hamiltonian Path?

A: No! If the graph has two separate pieces, you cannot jump from
   one piece to the other. So you cannot visit all vertices.

   Visual:

   1 ─── 2        3 ─── 4

   After visiting 1 and 2, how do you reach 3 and 4?
   There is no bridge between the islands! ✗

──────────────────────────────────────────────────────────────

Q9: What are real-world uses of Hamiltonian paths?

A: Real-world examples:

   Traveling Salesman Problem:
   → Visit every city exactly once, minimize total distance
   → Hamiltonian Cycle with minimum cost!

   DNA Sequencing:
   → Reconstruct DNA from short fragments
   → Each fragment must appear exactly once in order

   Circuit Board Drilling:
   → Drill holes at every required point exactly once
   → Minimize movement time

   School Bus Routing:
   → Pick up students from every stop exactly once
   → Most efficient route

──────────────────────────────────────────────────────────────

Q10: What is the most common mistake in backtracking?

A: Forgetting to unmark visited nodes when backtracking!

   Visual (WRONG — no backtrack):

   Try: 1 → 2 → 3 (fails)
   visited = {1, 2, 3} permanently marked ✗
   Now try: 1 → 4 → 5
   But 1 is still marked! Cannot start! ✗

   Visual (CORRECT — with backtrack):

   Try: 1 → 2 → 3 (fails)
   Unmark 3, unmark 2. visited = {1}
   Try: 1 → 4 → 5 (might work!) ✓

   Always unmark before returning false!

──────────────────────────────────────────────────────────────

Q11: Can we optimize the backtracking?

A: Yes! Several optimizations:

   1. Pruning: If a vertex has no unvisited neighbors and
      count < n, stop exploring that branch early.

   2. Start from high-degree vertices: More likely to find
      a path from well-connected vertices.

   3. Warnsdorff's Rule (for knight's tour): Always move to
      the neighbor with the fewest onward moves.

   But even with optimizations, worst case is still exponential.

──────────────────────────────────────────────────────────────

Q12: What is the Knight's Tour problem?

A: A famous Hamiltonian Path problem! On a chessboard, can a
   knight visit every square exactly once?

   Visual (5x5 board):

   Start at (0,0). Knight moves in L-shape.
   Try all possible moves, backtrack when stuck.

   If knight returns to start after visiting all squares:
   → Closed Knight's Tour (Hamiltonian Cycle!)

   This is exactly Hamiltonian Path on a special graph where
   each square is a vertex and each valid knight move is an edge.

──────────────────────────────────────────────────────────────

Q13: How does the code handle 1-based vs 0-based indexing?

A: The input edges are 1-based (as in the problem), but arrays
   in C++ are 0-based. So we subtract 1 from each vertex.

   Visual:

   Input edge: [1, 2]  (vertices 1 and 2)
   Code: u = 1-1 = 0, v = 2-1 = 1
   Stored as: adj[0].push_back(1), adj[1].push_back(0)

   Output uses original vertex numbers implicitly through
   the order in the result.

──────────────────────────────────────────────────────────────

Q14: What is the difference between DFS and Backtracking?

A: DFS explores all paths without undoing choices.
   Backtracking is DFS + undoing choices when they lead to dead ends.

   Visual (DFS without backtrack):

   Graph: 1 ─── 2 ─── 3
              │
              4

   DFS from 1: 1→2→3, then 2→4
   Visited: {1,2,3,4}. Done!

   Visual (Backtracking):

   Same graph, but we need EXACTLY N vertices in ONE path.
   Try 1→2→3 (3 vertices, need 4). Dead end? No, but stuck.
   Backtrack from 3. Try 1→2→4 (3 vertices, need 4). Stuck.
   Backtrack. No path from 1 exists! ✗

   Backtracking = DFS with the ability to change your mind!

──────────────────────────────────────────────────────────────

Q15: What are the most common mistakes?

A: Common mistakes and fixes:

   Mistake 1: Confusing Hamiltonian with Euler
   → Fix: Hamiltonian = vertices, Euler = edges

   Mistake 2: Forgetting to backtrack (unmark visited)
   → Fix: Always set visited[node] = false before returning

   Mistake 3: Not trying all starting vertices
   → Fix: Loop through all vertices as start

   Mistake 4: Cycle check missing edge to start
   → Fix: After count==n, verify edge to start exists

   Mistake 5: Expecting a fast formula
   → Fix: Accept that backtracking is needed for general case
```
