## Euler Path and Euler Circuit in a Graph

---

### Problem Statement

Given an **undirected graph**, determine whether it has an **Euler Path**, an **Euler Circuit**, or **neither**.

- **Euler Path**: A path that visits every **edge** exactly once.
- **Euler Circuit**: A circuit (closed path) that visits every **edge** exactly once and returns to the starting vertex.

**Return Values:**
- `0` → Neither Euler Path nor Euler Circuit exists
- `1` → Euler Path exists (but not Circuit)
- `2` → Euler Circuit exists

---

### Examples

**Example 1: Euler Circuit Exists**

```
Input:
n = 4, m = 4
edges = [[0,1], [1,2], [2,3], [3,0]]

Graph: 0 — 1 — 2 — 3 — 0 (Cycle)

All degrees = 2 (even)
Output: 2 (Euler Circuit exists)
```

**Example 2: Euler Path Exists (No Circuit)**

```
Input:
n = 4, m = 3
edges = [[0,1], [1,2], [2,3]]

Graph: 0 — 1 — 2 — 3

Degrees: 0 and 3 have odd degree (exactly 2)
Output: 1 (Euler Path exists)
```

**Example 3: Neither Exists**

```
Input:
n = 4, m = 3
edges = [[0,1], [0,2], [0,3]]

Graph: Star with center 0

Degrees: 0 has degree 3, others have degree 1 → 4 odd degrees
Output: 0 (Neither exists)
```

**Example 4: Disconnected Graph**

```
Input:
n = 5, m = 2
edges = [[0,1], [2,3]]

Graph has two separate components with edges

Output: 0 (Graph is not connected)
```

---

### Intuition

**Euler Path / Circuit Conditions:**

- An **Euler Circuit** exists only if **all vertices have even degree**.
- An **Euler Path** exists if **exactly 0 or 2 vertices have odd degree**.
- The graph must be **connected** (or have one non-trivial component).

**Why this works:**

- If more than 2 vertices have odd degree → impossible to traverse all edges exactly once.
- If the graph is disconnected (with edges in multiple components) → we cannot traverse all edges in one path/circuit.
- We use DFS to check connectivity after confirming the degree condition.

---

### Algorithm Steps

1. **Count the degree** of every vertex.
2. **Count the number of odd-degree vertices**.
3. If odd-degree vertices are **not 0 or 2** → return `0` (impossible).
4. Perform **DFS** from any vertex that has at least one edge.
5. Check if all vertices with degree > 0 are visited.
   - If any unvisited vertex with edges exists → graph is disconnected → return `0`.
6. If everything passes:
   - Return `2` if all degrees are even → **Euler Circuit**
   - Return `1` if exactly 2 odd degrees → **Euler Path**

---

### Code (C++)

```cpp
#include <bits/stdc++.h>

using namespace std;

void dfs(vector<vector<int>> &adj, vector<bool> &visited, int node)
{
    visited[node] = true;

    for (auto &&i : adj[node])
    {
        if (!visited[i])
            dfs(adj, visited, i);
    }
}

int eulerCircuitOrEulerPath(vector<vector<int>> &adj, vector<int> &degree, vector<bool> &visited, int n, int m)
{
    int noOfODDegreeNodes = 0;
    for (int i = 0; i < n; i++)
    {
        degree[i] = adj[i].size();

        if (degree[i] % 2 != 0)
        {
            noOfODDegreeNodes++;
        }
    }

    if (noOfODDegreeNodes != 0 && noOfODDegreeNodes != 2)
        return 0;

    for (int i = 0; i < n; i++)
    {
        if (degree[i])
        {
            dfs(adj, visited, i);
            break;
        }
    }

    for (int i = 0; i < n; i++)
    {
        if (degree[i] && !visited[i])
        {
            return 0;
        }
    }

    if (noOfODDegreeNodes == 0)
    {
        return 2; // Euler Circuit
    }
    else
    {
        return 1; // Euler Path
    }
}

// ==================== DRIVER CODE ====================
int main(int argc, char const *argv[])
{
    int n = 4, m = 4;
    vector<vector<int>> adj(n);

    // Build graph: 0-1, 1-2, 2-3, 3-0
    adj[0] = {1, 3};
    adj[1] = {0, 2};
    adj[2] = {1, 3};
    adj[3] = {2, 0};

    vector<bool> visited(n, 0);
    vector<int> degree(n, 0);

    int ans = eulerCircuitOrEulerPath(adj, degree, visited, n, m);

    cout << "Result: " << ans << endl;

    return 0;
}
```

**Output:**
```
Result: 2
```

## Euler Path and Euler Circuit in a Graph

---

### Problem Statement

Given an **undirected graph**, determine whether it has an **Euler Path** or an **Euler Circuit**.

- **Euler Path**: A path that visits every **edge** exactly once.
- **Euler Circuit**: A circuit (closed path) that visits every **edge** exactly once and returns to the starting vertex.

---

### Examples

**Example 1: Euler Circuit Exists**

```
Graph: 0 — 1 — 2 — 0
       |         |
       3 — 4 — 3

Degrees: All vertices have even degree.
Output: Euler Circuit Exists
```

**Example 2: Euler Path Exists (but not Circuit)**

```
Graph: 0 — 1 — 2 — 3

Degrees: 0 and 3 have odd degree (2 vertices with odd degree).
Output: Euler Path Exists
```

**Example 3: Neither Exists**

```
Graph: A star with 3 leaves (center degree 3, leaves degree 1)
Degrees: More than 2 vertices have odd degree.
Output: Neither Euler Path nor Circuit Exists
```

---

### Conditions for Existence (Undirected Graph)

| Condition                          | Euler Circuit | Euler Path     |
|------------------------------------|---------------|----------------|
| All vertices have **even degree**  | Yes           | Yes            |
| Exactly **2 vertices** have odd degree | No         | Yes            |
| More than **2 vertices** have odd degree | No      | No             |

**Note:** The graph must also be **connected** (or have exactly one non-trivial connected component).

---

### Approach: Hierholzer’s Algorithm

Hierholzer’s algorithm is an efficient way to find the **Eulerian Path** or **Circuit**:

1. Check the degree conditions.
2. Start DFS from a vertex with odd degree (for path) or any vertex (for circuit).
3. Keep removing edges as we traverse.

### Time & Space Complexity

**Time Complexity: O(V + E)**

- Counting degrees: We iterate over all vertices and their adjacency lists → **O(V + E)**
- DFS traversal: In the worst case, we visit every vertex and every edge once → **O(V + E)**
- Checking connectivity: We iterate over all vertices again → **O(V)**
- **Total = O(V + E)**

**Space Complexity: O(V + E)**

- Adjacency list stores all edges → **O(V + E)**
- `visited[]` array → **O(V)**
- `degree[]` array → **O(V)**
- Recursion stack in DFS (worst case) → **O(V)**
- **Total = O(V + E)**

---

### Notes

- This code only **detects** whether an Euler Path or Euler Circuit exists.
- It does **not** construct the actual path.
- Returns `2` for Euler Circuit, `1` for Euler Path, and `0` if neither exists.
- The graph must be connected (except for isolated vertices with degree 0).
