## 1. Problem Statement

Given an **undirected graph**, determine whether it has an **Euler Path**, an **Euler Circuit**, or **neither**.

- **Euler Path**: A path that visits every **edge** exactly once.
- **Euler Circuit**: A circuit (closed path) that visits every **edge** exactly once and returns to the starting vertex.

**Return Values:**
- `0` → Neither Euler Path nor Euler Circuit exists
- `1` → Euler Path exists (but not Circuit)
- `2` → Euler Circuit exists

## Key Insight

```
The Secret: Count Odd-Degree Vertices

Think of each edge as a bridge. Every bridge connects two land
pieces (vertices). Each time a bridge touches a land piece, it
counts as one connection.

Key observation:
→ Each edge adds 1 to the degree of TWO vertices
→ So the sum of all degrees = 2 × number of edges (always even!)

──────────────────────────────────────────────────────────────

Euler Circuit Condition:

You can start walking, use every bridge exactly once,
and END where you STARTED.

Visual:

    Start here
        ↓
    0 ───── 1
    │       │
    │       │
    3 ───── 2
        ↑
    End here (same as start!)

For this to work:
→ Every time you ENTER a vertex, you must also LEAVE it
→ Each visit uses 2 bridges (one in, one out)
→ So every vertex must have an EVEN number of bridges!

Rule: All vertices have even degree → Euler Circuit! ✓

──────────────────────────────────────────────────────────────

Euler Path Condition:

You can start walking and use every bridge exactly once,
but you END at a DIFFERENT place.

Visual:

    Start here          End here
        ↓                   ↓
    0 ───── 1 ───── 2 ───── 3

For this to work:
→ At the START vertex: you leave one more time than you enter
→ At the END vertex: you enter one more time than you leave
→ At all other vertices: enter = leave (even degree)

So exactly 2 vertices have odd degree (start and end)!

Rule: Exactly 0 or 2 odd-degree vertices → Euler Path possible! ✓

──────────────────────────────────────────────────────────────

Neither Condition:

If there are more than 2 odd-degree vertices,
it is impossible to walk every bridge exactly once.

Visual (4 odd degrees):

         1
         │
    3 ───0─── 2

Vertex 0: 3 bridges (odd)
Vertex 1: 1 bridge (odd)
Vertex 2: 1 bridge (odd)
Vertex 3: 1 bridge (odd)

No matter where you start, you will get stuck
with unused bridges! ✗

Rule: More than 2 odd degrees → Impossible! ✗

──────────────────────────────────────────────────────────────

Connectivity Check:

Even if degrees look perfect, the graph must be connected!

Visual (disconnected):

    0 ─── 1        2 ─── 3

Component 1: 0-1 (degrees even)
Component 2: 2-3 (degrees even)

All degrees even, but two separate islands!
You cannot walk from island 1 to island 2.

Rule: Must check connectivity! ✓
```

## Intuition

```
Imagine you are a mailman delivering mail along streets.

Each street is an edge. Each intersection is a vertex.
You want to walk every street exactly once.

──────────────────────────────────────────────────────────────

Euler Circuit = Perfect Mail Route:

You start at the post office, deliver to every street,
and return to the post office.

Visual:

        Post Office
             │
      Street A│Street B
             │
        House 1 ─── Street C ─── House 2
             │
      Street D│Street E
             │
        House 3

At every intersection, you arrive and leave.
Arrive = 1 street, Leave = 1 street.
Total streets at each intersection must be EVEN.

If any intersection has an odd number of streets,
you will enter but cannot leave (or vice versa).

──────────────────────────────────────────────────────────────

Euler Path = One-Way Mail Route:

You start at home, deliver to every street,
and end at a friend's house. No need to return home.

Visual:

    Your Home ── Street 1 ── Shop ── Street 2 ── Friend's House

At Your Home: you leave once, never arrive → odd degree!
At Friend's House: you arrive once, never leave → odd degree!
At Shop: you arrive once, leave once → even degree!

Exactly 2 odd intersections: start and end!

──────────────────────────────────────────────────────────────

Why More Than 2 Odd Vertices Fails:

Imagine 4 intersections each with 3 streets:

         North
          │
    West──┼──East
          │
         South

You start at North. Walk to center. Now you have 2 choices.
Pick West. Walk to center. Now you have 1 choice left.
Pick East. Walk to center. Now South is unreachable!

You are stuck with South's street unused. ✗

With more than 2 odd intersections, you will always
get stuck before using all streets.
```

## Algorithm

```
Step 1: Count degrees of all vertices

For each vertex i:
  degree[i] = number of edges connected to i

Visual:

    0 ───── 1 ───── 2 ───── 3

  degree[0] = 1
  degree[1] = 2
  degree[2] = 2
  degree[3] = 1

──────────────────────────────────────────────────────────────

Step 2: Count odd-degree vertices

oddCount = 0
For each vertex i:
  if degree[i] is odd:
    oddCount++

Visual:

  degree[0] = 1 → odd ✓
  degree[1] = 2 → even
  degree[2] = 2 → even
  degree[3] = 1 → odd ✓

  oddCount = 2

If oddCount is not 0 and not 2:
  return 0 (impossible!)

──────────────────────────────────────────────────────────────

Step 3: Check connectivity

Find any vertex with degree > 0:
  Run DFS from that vertex

For all vertices with degree > 0:
  If not visited:
    return 0 (disconnected!)

Visual (connected graph):

    DFS from vertex 0:
    0 → 1 → 2 → 3

    All vertices with edges visited? YES ✓

Visual (disconnected graph):

    DFS from vertex 0:
    0 → 1

    Vertex 2 has edges but not visited! NO ✗

──────────────────────────────────────────────────────────────

Step 4: Return result

If oddCount == 0:
  return 2 (Euler Circuit!)
Else:
  return 1 (Euler Path!)
```

## Code

```cpp
#include <bits/stdc++.h>
using namespace std;

void dfs(vector<vector<int>> &adj, vector<bool> &visited, int node) {
    visited[node] = true;
    for (auto &&i : adj[node]) {
        if (!visited[i])
            dfs(adj, visited, i);
    }
}

int eulerCircuitOrEulerPath(vector<vector<int>> &adj, vector<int> &degree,
                             vector<bool> &visited, int n, int m) {
    int noOfODDegreeNodes = 0;

    // Step 1: Count degrees
    for (int i = 0; i < n; i++) {
        degree[i] = adj[i].size();
        if (degree[i] % 2 != 0) {
            noOfODDegreeNodes++;
        }
    }

    // Step 2: Check odd degree count
    if (noOfODDegreeNodes != 0 && noOfODDegreeNodes != 2)
        return 0;

    // Step 3: Check connectivity
    for (int i = 0; i < n; i++) {
        if (degree[i]) {
            dfs(adj, visited, i);
            break;
        }
    }

    for (int i = 0; i < n; i++) {
        if (degree[i] && !visited[i]) {
            return 0;
        }
    }

    // Step 4: Return result
    if (noOfODDegreeNodes == 0)
        return 2; // Euler Circuit
    else
        return 1; // Euler Path
}

int main(int argc, char const *argv[]) {
    int n = 4, m = 4;
    vector<vector<int>> adj(n);

    // Build graph: 0-1, 1-2, 2-3, 3-0
    adj[0] = {1, 3};
    adj[1] = {0, 2};
    adj[2] = {1, 3};
    adj[3] = {2, 0};

    vector<bool> visited(n, 0);
    vector<int> degree(n, 0);

    int ans = eulerCircuitOrEulerPath(adj, degree, visited, n, m);
    cout << "Result: " << ans << endl;

    return 0;
}
```

**Output:**

```
Result: 2
```

## Step-by-Step Trace

```
Example 1: Euler Circuit

Graph:
    0 ───── 1
    │       │
    │       │
    3 ───── 2

Adjacency list:
  adj[0] = {1, 3}
  adj[1] = {0, 2}
  adj[2] = {1, 3}
  adj[3] = {2, 0}

── Step 1: Count degrees ──

  degree[0] = adj[0].size() = 2 (even)
  degree[1] = adj[1].size() = 2 (even)
  degree[2] = adj[2].size() = 2 (even)
  degree[3] = adj[3].size() = 2 (even)

── Step 2: Count odd degrees ──

  noOfODDegreeNodes = 0

  0 is not 2, but it IS 0 → condition passes! ✓

── Step 3: Check connectivity ──

  Find first vertex with degree > 0: vertex 0

  DFS from vertex 0:
    visit 0 → mark visited[0] = true
      neighbor 1: not visited → dfs(1)
        visit 1 → mark visited[1] = true
          neighbor 0: visited, skip
          neighbor 2: not visited → dfs(2)
            visit 2 → mark visited[2] = true
              neighbor 1: visited, skip
              neighbor 3: not visited → dfs(3)
                visit 3 → mark visited[3] = true
                  neighbor 2: visited, skip
                  neighbor 0: visited, skip
                return
            return
        return
    return

  visited: [true, true, true, true]

  Check all vertices with degree > 0:
    degree[0]=2, visited[0]=true ✓
    degree[1]=2, visited[1]=true ✓
    degree[2]=2, visited[2]=true ✓
    degree[3]=2, visited[3]=true ✓

  All connected! ✓

── Step 4: Return result ──

  noOfODDegreeNodes == 0 → return 2 (Euler Circuit!) ✓
```

```
Example 2: Euler Path

Graph:
    0 ───── 1 ───── 2 ───── 3

Adjacency list:
  adj[0] = {1}
  adj[1] = {0, 2}
  adj[2] = {1, 3}
  adj[3] = {2}

── Step 1: Count degrees ──

  degree[0] = 1 (odd)
  degree[1] = 2 (even)
  degree[2] = 2 (even)
  degree[3] = 1 (odd)

── Step 2: Count odd degrees ──

  noOfODDegreeNodes = 2

  2 is allowed! → condition passes! ✓

  DFS from vertex 0:
    visit 0 → neighbor 1 → dfs(1)
      visit 1 → neighbor 0 (visited), neighbor 2 → dfs(2)
        visit 2 → neighbor 1 (visited), neighbor 3 → dfs(3)
          visit 3 → neighbor 2 (visited)

  visited: [true, true, true, true]

  All vertices with edges are visited! ✓

  noOfODDegreeNodes == 2 → return 1 (Euler Path!) ✓

### Common Mistakes

```
❌ Mistake 1: Forgetting connectivity check
  if (oddCount == 0) return 2;  // WRONG! What if disconnected?

  Visual:
    0 ─── 1        2 ─── 3
    │     │        │     │
    └─────┘        └─────┘

    All degrees even, but two separate cycles!
    → Must check connectivity!

✓ Correct: Always run DFS/BFS to verify connectivity!

──────────────────────────────────────────────────────────────

❌ Mistake 2: Counting isolated vertices as odd
  Isolated vertex has degree 0 (even) → ignore it!

  Visual:
    0 ─── 1          2 (alone)

    degree[2] = 0 → even, not odd!
    Don't count it in oddCount.

✓ Correct: Only vertices with edges matter for Euler check.

──────────────────────────────────────────────────────────────

❌ Mistake 3: Running DFS from isolated vertex
  for (int i = 0; i < n; i++) {
      dfs(adj, visited, i);  // WRONG! Might start at isolated vertex
  }

  Visual:
    0 (alone)      1 ─── 2

    DFS from 0 → visits only 0
    Then check 1: degree > 0 but not visited → disconnected! ✗

✓ Correct:
  for (int i = 0; i < n; i++) {
      if (degree[i] > 0) {
          dfs(adj, visited, i);
          break;
      }
  }

──────────────────────────────────────────────────────────────

❌ Mistake 4: Confusing Euler Path with Hamiltonian Path
  Euler Path = visit every EDGE exactly once
  Hamiltonian Path = visit every VERTEX exactly once

  Visual (Euler):
    0 ─── 1 ─── 2 ─── 3
    Walk: 0→1→2→3 (uses all 3 edges) ✓

  Visual (Hamiltonian):
    Same graph, same walk visits all 4 vertices ✓
    But in general, they are DIFFERENT problems!

✓ Correct: Euler = edges, Hamiltonian = vertices. Don't mix them up!

──────────────────────────────────────────────────────────────

❌ Mistake 5: Not handling multiple edges or self-loops
  Multiple edges between same vertices: degree counts each edge!

  Visual:
    0 ═════ 1   (two edges between 0 and 1)

    degree[0] = 2, degree[1] = 2 → even → Euler Circuit! ✓

  Self-loop at vertex 0: adds 2 to degree[0]!

  Visual:
    0 ───┐
         │ (self loop)
    0 ───┘

    degree[0] = 2 → even!
```

## Complexity Analysis

**Time Complexity:** `O(V + E)`

- Counting degrees: iterate all vertices and edges → **O(V + E)**
- DFS traversal: visit every vertex and edge once → **O(V + E)**
- Checking connectivity: iterate all vertices → **O(V)**
- Total: **O(V + E)**

**Space Complexity:** `O(V + E)`

- Adjacency list stores all edges → **O(V + E)**
- `visited[]` array → **O(V)**
- `degree[]` array → **O(V)**
- Recursion stack in DFS (worst case) → **O(V)**
- Total: **O(V + E)**

```
Visual breakdown:

Time:
  Count degrees:
    for each vertex: check adjacency list
    → O(sum of all adjacency list sizes) = O(2E) = O(E)
    → Plus O(V) for the loop
    → Total: O(V + E)

  DFS:
    Each vertex visited once
    Each edge explored once
    → Total: O(V + E)

  Connectivity check:
    for each vertex: O(V)

  Overall: O(V + E) ✓

Space:
  Adjacency list: O(V + E)
  visited array: O(V)
  degree array: O(V)
  DFS stack: O(V) worst case

  Overall: O(V + E) ✓
```

## Interview Questions

```
Q1: What is the difference between an Euler Path and an Euler Circuit?

A: An Euler Path is like a one-way hike through every trail in a
   park — you start at one parking lot, walk every trail exactly
   once, and end at a different parking lot.

   An Euler Circuit is like a loop trail — you start at your car,
   walk every trail exactly once, and end up back at your car.

   Visual:

   Euler Path:
     Start ──Trail── Trail── Trail── End
     (different start and end)

   Euler Circuit:
     Start ──Trail── Trail── Trail── Start
     (back where you began!)

──────────────────────────────────────────────────────────────

Q2: How do you check if an Euler Circuit exists?

A: Two conditions must be true:

   1. Every intersection must have an EVEN number of trails.
      Why? Every time you enter, you must also leave.

   2. All trails must be in one connected piece.
      You cannot jump from one park to another!

   Visual (Euler Circuit possible):

       0 ──── 1
       │      │
       │      │
       3 ──── 2

       All degrees = 2 (even) ✓
       All connected ✓

──────────────────────────────────────────────────────────────

Q3: How do you check if an Euler Path exists (but not Circuit)?

A: Two conditions:

   1. Exactly TWO intersections have an ODD number of trails.
      These are your start and end points.

   2. All trails are connected.

   Visual (Euler Path possible):

       0 ──── 1 ──── 2 ──── 3

       Degrees: 1, 2, 2, 1
       Odd degrees at 0 and 3 → start and end! ✓

──────────────────────────────────────────────────────────────

Q4: Why does more than 2 odd degrees make it impossible?

A: Think of each odd intersection as a dead end. With 2 dead ends,
   you start at one and finish at the other. With 4 dead ends,
   you start at one, finish at another, but two dead ends are
   left unreachable!

   Visual (4 odd degrees — impossible):

            1
            │
       3 ───0─── 2

       Start at 1, walk to 0.
       From 0, you have 3 choices.
       No matter which you pick, you leave one branch unused! ✗

──────────────────────────────────────────────────────────────

Q5: Why must the graph be connected?

A: Imagine two separate islands with bridges on each. You can walk
   all bridges on island 1, but you have no way to swim to island 2
   to walk its bridges. So you cannot walk ALL bridges.

   Visual:

       0 ─── 1        2 ─── 3

       Island 1       Island 2

       Walk 0→1. Now stuck! Cannot reach 2 or 3. ✗

──────────────────────────────────────────────────────────────

Q6: What is the relationship between sum of degrees and edges?

A: Every edge connects two vertices. So each edge adds 1 to the
   degree of TWO vertices. Therefore:

   Sum of all degrees = 2 × number of edges

   This is always EVEN!

   Visual:

       0 ───── 1

       edge (0,1) adds 1 to degree[0] and 1 to degree[1]
       Sum of degrees = 2 = 2 × 1 edge ✓

   This means the number of odd-degree vertices is ALWAYS even!
   (You cannot have 1 or 3 odd-degree vertices — impossible!)

──────────────────────────────────────────────────────────────

Q7: Can a graph have 0 odd-degree vertices but still no Euler Circuit?

A: Yes! If the graph is disconnected. All degrees could be even,
   but if there are two separate pieces, you cannot walk between
   them.

   Visual:

       0 ─── 1        2 ─── 3
       │     │        │     │
       └─────┘        └─────┘

       All degrees = 2 (even) ✓
       But two separate cycles! ✗

   That is why we MUST check connectivity!

──────────────────────────────────────────────────────────────

Q8: What is the difference between Euler Path and Hamiltonian Path?

A: Euler Path visits every EDGE exactly once.
   Hamiltonian Path visits every VERTEX exactly once.

   Visual:

   Graph:
       0 ──── 1 ──── 2
              │
              3

   Euler Path: 0→1→3→1→2 (visits edge 1-3 and 1-2, edge 0-1)
   Wait — edge 1-3 is used once, edge 0-1 once, edge 1-2 once.
   But vertex 1 is visited twice! That's fine for Euler.

   Hamiltonian Path: 0→1→3 (visits vertices 0,1,3 — misses 2!)
   Or: 3→1→2 (misses 0!)
   Or: 0→1→2 (misses 3!)

   They solve completely different problems!

Q9: Does a single edge have an Euler Path?

A: Yes! Two vertices, one edge.

   Visual:
       0 ─── 1

   Degrees: 1, 1 → exactly 2 odd degrees!
   Path: 0 → 1

   It has an Euler Path but NOT an Euler Circuit
   (cannot return to 0 without reusing the edge).

Q10: What are real-world uses of Euler Paths?

A: Real-world examples:

   Mail delivery:
   → Walk every street exactly once, return to post office
   → Euler Circuit = most efficient route!

   Garbage collection:
   → Drive every road exactly once
   → Save fuel and time!

   Snow plowing:
   → Clear every street exactly once
   → Minimize driving on already-cleared roads

   DNA sequencing:
   → Scientists use Euler paths to reconstruct DNA from fragments
   → Each fragment is an edge, reconstruct the full sequence!

Q11: What happens if the graph has self-loops?

A: A self-loop is an edge from a vertex to itself.

   Visual:
       0 ───┐
            │ (loop)
       0 ───┘

   It adds 2 to the degree of that vertex!
   (One "in" and one "out" — both at the same vertex)

   So a self-loop always contributes an EVEN number to degree.
   It does not change the odd/even count!

Q12: What about multiple edges between the same vertices?

A: Multiple edges are allowed! Each edge is counted separately.

   Visual:
       0 ═════ 1   (two parallel edges)

       degree[0] = 2 (even)
       degree[1] = 2 (even)

   Two parallel edges between 0 and 1:
   → Euler Circuit exists! Walk one edge, walk back on the other.

Q13: Can you find the actual Euler Path, not just check if it exists?

A: Yes! Hierholzer's Algorithm finds the actual path:

   1. Start at any vertex (odd-degree vertex for Euler Path)
   2. Follow unused edges until stuck
   3. When stuck, backtrack and insert cycles
   4. Continue until all edges are used

   Visual:

       0 ──── 1 ──── 2 ──── 3

       Start at 0 (odd degree):
       0 → 1 → 2 → 3 (stuck!)
       All edges used → Path found!

   For this problem, we only CHECK if it exists.
   Finding the actual path is a follow-up question!

Q14: Why is the number of odd-degree vertices always even?

A: Every edge adds 2 to the total degree count (1 for each end).
   So sum of all degrees = 2 × E, which is always even.

   If we had an odd number of odd-degree vertices,
   their sum would be odd (odd + odd + ... = odd).
   Adding even-degree vertices (which sum to even)
   would give odd + even = odd total.

   But total degree MUST be even!
   So odd-degree vertices must come in pairs!

   Visual proof:

       Odd + Odd = Even
       Odd + Odd + Odd + Odd = Even
       (Any even number of odds sums to even)

       Odd + Odd + Odd = Odd ✗ (impossible!)

Q15: What are the most common mistakes?

A: Common mistakes and fixes:

   Mistake 1: Forgetting connectivity check
   → Fix: Always run DFS/BFS from a vertex with edges

   Mistake 2: Confusing Euler with Hamiltonian
   → Fix: Euler = edges, Hamiltonian = vertices

   Mistake 3: Counting isolated vertices
   → Fix: Vertices with degree 0 don't matter

   Mistake 4: Starting DFS from isolated vertex
   → Fix: Find first vertex with degree > 0

   Mistake 5: Thinking 0 odd degrees always means circuit
   → Fix: Must also check graph is connected!
