## Strongly Connected Components - Kosaraju's Algorithm

### Problem Statement:

Given a **Directed Graph** with `V` vertices (numbered from 0 to V-1) and `E` edges, find the **number of strongly connected components** (SCCs) in the graph.

---

## What is a Strongly Connected Component?

```
Definition:
A Strongly Connected Component (SCC) is a maximal subgraph
where every vertex is reachable from every other vertex.

Key property:
In an SCC, for any two vertices u and v:
→ There exists a path from u to v
→ There exists a path from v to u

Example:
    0 → 1
    ↑   ↓
    3 ← 2

Vertices {0, 1, 2, 3} form an SCC:
- Can reach 1 from 0: 0→1 ✓
- Can reach 0 from 1: 1→2→3→0 ✓
- Can reach any vertex from any other ✓
```

## Kosaraju's Algorithm Intuition

```
Why can't we just use DFS?

Problem: DFS finds reachable nodes, not mutual reachability

Example:
    0 → 1 → 2
    
From 0: can reach {1, 2}
But can {1, 2} reach 0? NO!
So they're not in same SCC

Kosaraju's Insight:
Use FINISHING TIMES from DFS + REVERSED GRAPH!

Key observations:
1. In DFS, nodes with later finishing times
   are in "higher" SCCs in condensation graph
   
2. If we reverse all edges:
   - SCCs remain the same (mutual reachability preserved)
   - But topological order of SCC graph reverses
   
3. Process nodes in decreasing finishing time on reversed graph:
   - Each DFS gives exactly one SCC!
```

---

## Algorithm Steps

```
Step 1: DFS on original graph (fill stack by finish time)
  For each unvisited node:
    DFS(node)
    After exploring all neighbors, push node to stack
  
  Stack now has nodes in order of finishing times
  (top = latest finishing time)

Step 2: Transpose the graph
  For each edge (u → v):
    Add reverse edge (v → u) to transposed graph
  
  Create adjT[] (transposed adjacency list)

Step 3: DFS on transposed graph (process stack order)
  Reset visited array
  While stack is not empty:
    Pop node from stack
    If not visited:
      DFS(node) on transposed graph  ← This is 1 SCC!
      Increment SCC count
  
  Return SCC count
```

---

## Why This Works

```
Mathematical Proof:

Lemma 1: If C1 and C2 are different SCCs and there's
         an edge from C1 to C2, then finishing time of
         any node in C1 > finishing time of any node in C2

Proof: DFS reaches C1 first, explores to C2, finishes C2,
       then returns and finishes C1. ✓

Lemma 2: In transposed graph, SCCs remain the same
         (reversing edges preserves mutual reachability)

Lemma 3: If we process nodes in decreasing finish time
         on transposed graph:
         - Start from "sink" SCC in original graph
         - This becomes "source" in transposed graph
         - DFS from source in transposed graph visits
           exactly that SCC (can't escape to other SCCs)

Combining these:
→ Each DFS in Step 3 visits exactly one SCC
→ Number of DFS calls = number of SCCs ✓
```

## Code

```cpp
#include <bits/stdc++.h>
using namespace std;

class Solution {
private:
    // Step 1: DFS to fill stack by finishing time
    void dfs(int node, vector<int>& vis, vector<int> adj[], 
             stack<int>& st) {
        vis[node] = 1;
        
        // Visit all neighbors
        for (int neighbor : adj[node]) {
            if (!vis[neighbor]) {
                dfs(neighbor, vis, adj, st);
            }
        }
        
        // Push to stack after finishing all neighbors
        st.push(node);
    }
    
    // Step 3: DFS on transposed graph
    void dfsTranspose(int node, vector<int>& vis, 
                      vector<int> adjT[]) {
        vis[node] = 1;
        
        for (int neighbor : adjT[node]) {
            if (!vis[neighbor]) {
                dfsTranspose(neighbor, vis, adjT);
            }
        }
    }
    
public:
    int kosaraju(int V, vector<int> adj[]) {
        // Step 1: Fill stack with finishing times
        vector<int> vis(V, 0);
        stack<int> st;
        
        for (int i = 0; i < V; i++) {
            if (!vis[i]) {
                dfs(i, vis, adj, st);
            }
        }
        
        // Step 2: Build transposed graph
        vector<int> adjT[V];
        for (int u = 0; u < V; u++) {
            for (int v : adj[u]) {
                adjT[v].push_back(u);  // Reverse edge
            }
        }
        
        // Step 3: DFS on transposed graph in stack order
        fill(vis.begin(), vis.end(), 0);  // Reset visited
        int sccCount = 0;
        
        while (!st.empty()) {
            int node = st.top();
            st.pop();
            
            if (!vis[node]) {
                sccCount++;
                dfsTranspose(node, vis, adjT);
            }
        }
        
        return sccCount;
    }
};

int main() {
    int V = 5;
    vector<int> adj[V];
    
    // Example edges: creating the graph
    adj[1].push_back(0);
    adj[0].push_back(2);
    adj[2].push_back(1);
    adj[0].push_back(3);
    adj[3].push_back(4);
    
    Solution obj;
    int sccCount = obj.kosaraju(V, adj);
    
    cout << "Number of SCCs: " << sccCount << endl;
    
    return 0;
}
```

**Output:**
```
Number of SCCs: 3
```

## Why Transpose?

```
Question: Why reverse the graph?

Answer: To process SCCs in correct order!

Consider:
  SCC1 → SCC2 (edge between SCCs)

In original graph:
  - DFS from SCC1 would visit SCC2
  - Can't isolate SCCs!

In transposed graph:
  - Edge direction: SCC1 ← SCC2
  - DFS from SCC2 doesn't reach SCC1
  - Perfect isolation! ✓

Example:
Original:
  [0→1→2 cycle] → [3] → [4]
   SCC1           SCC2   SCC3

DFS from 0 reaches everything → can't separate!

Transposed:
  [0←1←2 cycle] ← [3] ← [4]

DFS from 0: reaches {0,1,2} only ✓
DFS from 3: reaches {3} only ✓
DFS from 4: reaches {4} only ✓

Each DFS = one SCC! ✓
```

---

## Stack Order Importance

```
Why process in decreasing finishing time?

Finishing time = when node is pushed to stack
Later finish = deeper in recursion

Property: If SCC1 → SCC2 (edge exists):
  → Finishing time of SCC1 > finishing time of SCC2

Process order on transposed graph:
  1. Pop highest finishing time (SCC1)
  2. In transposed graph, SCC1 has no outgoing edges
  3. DFS visits only SCC1 nodes
  4. Mark them visited
  5. Pop next SCC, which is now isolated
  6. Repeat

Without correct order:
  - Might start from SCC2
  - In transposed graph: SCC2 → SCC1
  - DFS would visit both SCCs together
  - Wrong! ✗

With stack order:
  - Start from sink SCC (no outgoing in original)
  - This is source in transposed graph
  - Isolated correctly ✓
```

## Complexity Analysis

**Time Complexity:** `O(V + E)`
- Step 1 (DFS): O(V + E)
- Step 2 (Transpose): O(V + E)
- Step 3 (DFS on transpose): O(V + E)
- Total: **O(V + E)**

**Space Complexity:** `O(V + E)`
- Original adjacency list: O(V + E)
- Transposed adjacency list: O(V + E)
- Visited array: O(V)
- Stack: O(V)
- Recursion depth: O(V)
- Total: **O(V + E)**

```
Detailed breakdown:

Three DFS passes:
→ Each visits all vertices and edges once
→ 3 × O(V + E) = O(V + E)

Building transpose:
→ Iterate through all edges once
→ O(E)

Stack operations:
→ Push/pop each vertex once
→ O(V)

Total time: O(V + E)

Space:
→ Two adjacency lists: 2 × O(V + E)
→ Visited array: O(V)
→ Stack: O(V)
→ Recursion: O(V) depth
→ Total: O(V + E)
```

## Common Mistakes

```
❌ Mistake 1: Not resetting visited array
// After Step 1
for (int i = 0; i < V; i++) {
    if (!vis[node]) {  // WRONG! vis still has old values
        dfsTranspose(node, vis, adjT);
    }
}

✓ Correct:
fill(vis.begin(), vis.end(), 0);  // Reset!

❌ Mistake 2: Wrong transpose direction
for (int u = 0; u < V; u++) {
    for (int v : adj[u]) {
        adjT[u].push_back(v);  // WRONG! Same direction
    }
}

✓ Correct:
adjT[v].push_back(u);  // Reverse!

❌ Mistake 3: Pushing before visiting neighbors
void dfs(int node, ..., stack<int>& st) {
    vis[node] = 1;
    st.push(node);  // WRONG! Push too early
    for (neighbor : adj[node]) {
        if (!vis[neighbor]) dfs(neighbor, ...);
    }
}

✓ Correct:
st.push(node);  // Push AFTER visiting all neighbors

❌ Mistake 4: Not checking visited in Step 3
while (!st.empty()) {
    int node = st.top();
    st.pop();
    sccCount++;  // WRONG! Node might be visited
    dfsTranspose(node, vis, adjT);
}

✓ Correct:
if (!vis[node]) {
    sccCount++;
    dfsTranspose(node, vis, adjT);
}

❌ Mistake 5: Wrong DFS parameter
dfs(node, vis, adjT, st);  // WRONG! Passing stack to transpose DFS

✓ Correct:
dfsTranspose(node, vis, adjT);  // No stack needed
```

## Alternative: Tarjan's Algorithm

```cpp
class TarjanSCC {
private:
    int timer = 0;
    
    void dfs(int node, int parent, vector<int>& vis,
             vector<int> adj[], int tin[], int low[],
             stack<int>& st, vector<bool>& inStack,
             int& sccCount) {
        vis[node] = 1;
        tin[node] = low[node] = timer++;
        st.push(node);
        inStack[node] = true;
        
        for (int neighbor : adj[node]) {
            if (!vis[neighbor]) {
                dfs(neighbor, node, vis, adj, tin, low, st, inStack, sccCount);
                low[node] = min(low[node], low[neighbor]);
            }
            else if (inStack[neighbor]) {
                low[node] = min(low[node], tin[neighbor]);
            }
        }
        
        // If node is root of SCC
        if (low[node] == tin[node]) {
            sccCount++;
            while (true) {
                int top = st.top();
                st.pop();
                inStack[top] = false;
                if (top == node) break;
            }
        }
    }
    
public:
    int tarjanSCC(int V, vector<int> adj[]) {
        vector<int> vis(V, 0);
        int tin[V], low[V];
        stack<int> st;
        vector<bool> inStack(V, false);
        int sccCount = 0;
        
        for (int i = 0; i < V; i++) {
            if (!vis[i]) {
                dfs(i, -1, vis, adj, tin, low, st, inStack, sccCount);
            }
        }
        
        return sccCount;
    }
};
```

**Comparison:**
- Kosaraju: 2 DFS passes, transpose graph
- Tarjan: 1 DFS pass, uses low-link values
- Both O(V+E), Tarjan slightly faster in practice
- Kosaraju simpler to understand! ✓

## Summary

```
Problem: Find number of SCCs in directed graph

SCC Definition:
→ Maximal subgraph with mutual reachability
→ Every vertex reaches every other vertex

Kosaraju's Algorithm:
1. DFS on original graph → fill stack by finish time
2. Transpose graph (reverse all edges)
3. DFS on transpose in stack order
   → Each DFS = one SCC

Why it works:
→ Finish time respects SCC topological order
→ Transpose makes sink SCCs into sources
→ Stack order ensures processing sink SCCs first
→ Each DFS on transpose visits exactly one SCC

Time: O(V + E)
Space: O(V + E)

Applications:
→ Web page ranking (PageRank)
→ Social network analysis
→ Compiler optimization
→ Recommendation systems

Classic algorithm - master it! ✓
```
