## Bridge in a Graph | Critical Connections in a Network

---

### Problem Statement

Given an undirected connected graph with `n` vertices and a list of connections (edges), find all **critical connections** (bridges).

A **critical connection** is an edge that, if removed, increases the number of connected components in the graph (i.e., disconnects the graph).

**Input:**  
- `n` = number of servers (vertices)  
- `connections` = list of edges `[ai, bi]`

**Output:**  
Return a list of all critical connections (bridges) in any order.

---

### Examples

**Example 1:**

```
Input: n = 4, connections = [[0,1],[1,2],[2,0],[1,3]]
Output: [[1,3]]
Explanation: Removing edge 1-3 disconnects the graph.
```

**Example 2:**

```
Input: n = 2, connections = [[0,1]]
Output: [[0,1]]
Explanation: The only edge is a bridge.
```

---

### Brute Force Approach

**Idea:**  
For every edge in the graph, temporarily remove it and check whether the graph remains connected using DFS or BFS. If removing the edge disconnects the graph, then that edge is a **bridge** (critical connection).

**Steps:**
1. For each edge `(u, v)` in the graph:
   - Remove the edge temporarily.
   - Run DFS/BFS from any node and check if all nodes are still reachable.
   - If not all nodes are visited → the edge is a bridge.
   - Add the edge back.

**Code Snippet (Brute Force)**

```cpp
bool isConnected(int n, vector<vector<int>>& adj, int skipU, int skipV) {
    vector<bool> visited(n, false);
    queue<int> q;
    q.push(0);
    visited[0] = true;
    int count = 1;

    while (!q.empty()) {
        int node = q.front(); q.pop();
        for (int neigh : adj[node]) {
            if ((node == skipU && neigh == skipV) || (node == skipV && neigh == skipU))
                continue; // skip the removed edge
            if (!visited[neigh]) {
                visited[neigh] = true;
                count++;
                q.push(neigh);
            }
        }
    }
    return count == n;
}

vector<vector<int>> findBridgesBruteForce(int n, vector<vector<int>>& connections) {
    vector<vector<int>> adj(n);
    for (auto& e : connections) {
        adj[e[0]].push_back(e[1]);
        adj[e[1]].push_back(e[0]);
    }

    vector<vector<int>> bridges;
    for (auto& edge : connections) {
        int u = edge[0], v = edge[1];
        if (!isConnected(n, adj, u, v)) {
            bridges.push_back({u, v});
        }
    }
    return bridges;
}
```

**Time Complexity:**  
**O(E × (V + E))**  
(For each of the `E` edges, we run DFS/BFS which takes `O(V + E)` time)

**Space Complexity:** O(V + E)

**Why it's inefficient:**  
Very slow for large graphs (especially when `E` is large).  
This is why we use **Tarjan’s Algorithm** which finds all bridges in **O(V + E)** time.

---

### Why Tarjan’s Algorithm?

We use **Tarjan’s DFS-based algorithm** to find bridges efficiently in **O(V + E)** time.

**Core Idea:**
- Perform DFS and keep track of two values for each node:
  - `disc[u]`: Discovery time of node `u`
  - `low[u]`: Earliest discovery time of nodes reachable from subtree rooted at `u`

**Bridge Condition:**
An edge `u → v` is a **bridge** if:
> `low[v] > disc[u]`

This means there is **no back edge** from the subtree of `v` to an ancestor of `u`.

---

### Approach (Tarjan’s Algorithm for Bridges)

1. Build the adjacency list from the given connections.
2. Initialize:
   - `disc[]` and `low[]` arrays with `-1`
   - `timer = 0`
   - `parent` array (to avoid going back to parent)
3. Run DFS from node `0`:
   - Update `disc[u]` and `low[u]`
   - For each adjacent node `v`:
     - If `v` is not visited → recurse on `v`, then update `low[u]`
     - If `low[v] > disc[u]` → edge `u-v` is a bridge
     - If `v` is visited and not parent → back edge → update `low[u] = min(low[u], disc[v])`
4. Collect all bridge edges.

---

### Code (C++)

```cpp
#include <bits/stdc++.h>
using namespace std;

class Solution {
private:
    int timer = 0;
    vector<int> disc, low;
    vector<vector<int>> adj;
    vector<vector<int>> bridges;

    void dfs(int node, int parent) {
        disc[node] = low[node] = timer++;
        
        for(int neigh : adj[node]) {
            if(neigh == parent) continue;
            
            if(disc[neigh] == -1) {
                // Not visited
                dfs(neigh, node);
                
                // Update low of current node
                low[node] = min(low[node], low[neigh]);
                
                // Bridge condition
                if(low[neigh] > disc[node]) {
                    bridges.push_back({node, neigh});
                }
            } 
            else {
                // Back edge
                low[node] = min(low[node], disc[neigh]);
            }
        }
    }

public:
    vector<vector<int>> criticalConnections(int n, vector<vector<int>>& connections) {
        adj.resize(n);
        disc.assign(n, -1);
        low.assign(n, -1);
        
        // Build adjacency list
        for(auto& edge : connections) {
            int u = edge[0], v = edge[1];
            adj[u].push_back(v);
            adj[v].push_back(u);
        }
        
        // Start DFS from node 0 (assuming graph is connected)
        dfs(0, -1);
        
        return bridges;
    }
};

// ==================== DRIVER CODE ====================
int main() {
    Solution sol;
    
    // Example 1
    vector<vector<int>> conn1 = {{0,1},{1,2},{2,0},{1,3}};
    vector<vector<int>> res1 = sol.criticalConnections(4, conn1);
    cout << "Example 1 Bridges: ";
    for(auto& e : res1) cout << "[" << e[0] << "," << e[1] << "] ";
    cout << endl;

    // Example 2
    vector<vector<int>> conn2 = {{0,1}};
    vector<vector<int>> res2 = sol.criticalConnections(2, conn2);
    cout << "Example 2 Bridges: ";
    for(auto& e : res2) cout << "[" << e[0] << "," << e[1] << "] ";
    cout << endl;

    return 0;
}
```

---

### Time & Space Complexity

- **Time Complexity:** O(V + E)
- **Space Complexity:** O(V + E)

---

### Notes

- The algorithm assumes the graph is **connected**. For disconnected graphs, run DFS from all unvisited nodes.
- `parent` parameter is crucial to avoid treating the parent as a back edge.
- This is the same concept used for finding **Articulation Points** (just the condition changes).
