## Course Schedule I and II | Prerequisite Tasks | Topological Sort

### Problem Statement:

**Course Schedule I:** Given `N` tasks (0 to N-1) and prerequisite pairs `[a, b]` (must finish `b` before `a`), determine if it's **possible to finish all tasks**.

**Course Schedule II:** Same setup, but return the **actual order** to complete all tasks. If impossible, return empty array.

---

## Examples

**Example 1:**
- **Input:** `N=4`, pairs=`[[1,0],[2,1],[3,2]]`
- **Output:** `Yes` / Order: `[0, 1, 2, 3]` or `[3, 2, 1, 0]`
```
  [1,0] → finish 0 before 1: 0→1
  [2,1] → finish 1 before 2: 1→2
  [3,2] → finish 2 before 3: 2→3

  Graph: 0→1→2→3 (linear chain, no cycle)
  Order: 0, 1, 2, 3 ✓
```

**Example 2:**
- **Input:** `N=4`, pairs=`[[1,2],[4,3],[2,4],[4,1]]`
- **Output:** `No` / `[]`
```
  1→2, 4→3, 2→4, 4→1
  1→2→4→1 CYCLE!
  Impossible ✗
```

---

## Key Insight
```
  Prerequisite structure = Directed Graph
  [a, b] means b→a (b must come before a)

  Possible to finish all tasks ↔ Graph is a DAG (no cycle)

  Course Schedule I:  Detect cycle → yes/no
  Course Schedule II: Topological sort → return order or []

  Both problems → SAME ALGORITHM (Kahn's BFS)!

  If topological sort processes all V nodes → valid order → YES
  If some nodes remain (in cycle) → impossible → NO / []
```

## Course Schedule I: Can We Finish?

### Algorithm
```
  1. Build directed graph from prerequisites:
     For each [a,b]: add edge b→a, indegree[a]++

  2. Push all courses with indegree=0 to queue

  3. BFS:
     Pop course → count++
     Reduce indegree of neighbors
     Push neighbors with indegree=0

  4. If count == N → YES (all finished)
     Else → NO (cycle exists)
```

### Code
```cpp
#include <bits/stdc++.h>
using namespace std;

class Solution {
public:
    bool canFinish(int N, vector<vector<int>>& prerequisites) {
        // Build graph and compute in-degrees
        vector<vector<int>> adj(N);
        vector<int> indegree(N, 0);

        for (auto& pre : prerequisites) {
            int a = pre[0], b = pre[1];
            adj[b].push_back(a); // b must be done before a
            indegree[a]++;
        }

        // Push all 0 in-degree nodes
        queue<int> q;
        for (int i = 0; i < N; i++) {
            if (indegree[i] == 0) q.push(i);
        }

        int count = 0;
        while (!q.empty()) {
            int node = q.front(); q.pop();
            count++;

            for (int neighbor : adj[node]) {
                indegree[neighbor]--;
                if (indegree[neighbor] == 0) q.push(neighbor);
            }
        }

        return count == N; // All processed → no cycle
    }
};

int main() {
    Solution sol;

    // Example 1: No cycle
    vector<vector<int>> pre1 = {{1,0},{2,1},{3,2}};
    cout << "Can finish: " << (sol.canFinish(4, pre1) ? "Yes" : "No") << endl;

    // Example 2: Cycle
    vector<vector<int>> pre2 = {{1,2},{4,3},{2,4},{4,1}};
    cout << "Can finish: " << (sol.canFinish(5, pre2) ? "Yes" : "No") << endl;

    return 0;
}
```

## Course Schedule II: Return Order

### Algorithm
```
  Same as Course Schedule I, but also collect
  the processing order!

  1. Build graph: [a,b] → edge b→a, indegree[a]++
  2. Queue: all indegree=0 courses
  3. BFS: pop course → add to ORDER → update neighbors
  4. If order.size() == N → return order
     Else → return [] (cycle, impossible)
```

### Code
```cpp
#include <bits/stdc++.h>
using namespace std;

class Solution {
public:
    vector<int> findOrder(int N, vector<vector<int>>& prerequisites) {
        // Build adjacency list and in-degree array
        vector<vector<int>> adj(N);
        vector<int> indegree(N, 0);

        for (auto& pre : prerequisites) {
            int a = pre[0], b = pre[1];
            adj[b].push_back(a); // Must do b before a
            indegree[a]++;
        }

        // Push courses with no prerequisites
        queue<int> q;
        for (int i = 0; i < N; i++) {
            if (indegree[i] == 0) q.push(i);
        }

        vector<int> order;

        // BFS (Kahn's Algorithm)
        while (!q.empty()) {
            int node = q.front(); q.pop();
            order.push_back(node); // Add to ordering

            for (int neighbor : adj[node]) {
                indegree[neighbor]--;
                if (indegree[neighbor] == 0) {
                    q.push(neighbor);
                }
            }
        }

        // If all courses processed → return order, else []
        if (order.size() == N) return order;
        return {}; // Cycle exists
    }
};

int main() {
    Solution sol;

    // Example 1: Valid ordering exists
    vector<vector<int>> pre1 = {{1,0},{2,1},{3,2}};
    vector<int> order1 = sol.findOrder(4, pre1);
    cout << "Order: ";
    for (int x : order1) cout << x << " ";
    cout << endl; // 0 1 2 3

    // Example 2: Cycle — impossible
    vector<vector<int>> pre2 = {{1,2},{2,4},{4,1}};
    vector<int> order2 = sol.findOrder(5, pre2);
    if (order2.empty()) cout << "Impossible (cycle detected)" << endl;

    return 0;
}
```

## Step-by-Step Trace

### Example 1: N=4, pairs=[[1,0],[2,1],[3,2]]
```
  Build Graph:
  [1,0] → adj[0]=[1], indeg[1]++
  [2,1] → adj[1]=[2], indeg[2]++
  [3,2] → adj[2]=[3], indeg[3]++

  Graph: 0→1→2→3
  indegree = [0, 1, 1, 1]
              0  1  2  3

  Push indegree=0: queue=[0]

  ── BFS ──

  Pop 0: order=[0], count=1
    neighbor 1: indeg[1]=1→0 → push 1
    queue=[1]

  Pop 1: order=[0,1], count=2
    neighbor 2: indeg[2]=1→0 → push 2
    queue=[2]

  Pop 2: order=[0,1,2], count=3
    neighbor 3: indeg[3]=1→0 → push 3
    queue=[3]

  Pop 3: order=[0,1,2,3], count=4
    no neighbors
    queue=[]

  order.size()=4 == N=4 → return [0,1,2,3] ✓
```

### Example 2: Cycle [[1,2],[2,4],[4,1]]
```
  Build Graph:
  [1,2] → adj[2]=[1], indeg[1]++
  [2,4] → adj[4]=[2], indeg[2]++
  [4,1] → adj[1]=[4], indeg[4]++

  indegree = [0, 1, 1, 0, 1]
              0  1  2  3  4

  Push indegree=0: queue=[0,3]

  Pop 0: order=[0], no outgoing edges
  Pop 3: order=[0,3], no outgoing edges
  queue=[]

  order.size()=2 ≠ N=5 → return [] (cycle: 1→2? wait)

  Actually: 1,2,4 are stuck in cycle:
  1 needs 2 (indeg stays 1)
  2 needs 4 (indeg stays 1)
  4 needs 1 (indeg stays 1)
  → Never reach 0 → never pushed → CYCLE! ✓
```

## Real-World Analogy
```
  Course schedule:
  CS101 → CS201 → CS301 → CS401

  Must take CS101 before CS201, etc.

  In-degree = number of courses you must take first.
  CS101 has no prereqs → in-degree=0 → take FIRST!
  CS201 has 1 prereq → take SECOND!
  ...

  Circular dependency (cycle):
  "Must take A before B, B before C, C before A"
  → Impossible! ✗
```

## Problem Mapping
```
  Course Schedule I:             Course Schedule II:
  ─────────────────────          ──────────────────────────
  canFinish(N, prereqs)          findOrder(N, prereqs)
  return: bool                   return: vector<int>

  count == N → true              order.size()==N → order
  count <  N → false             order.size()<N  → {}

  SAME algorithm, different output ✓
```

## Complexity Analysis

**Time Complexity:** O(V + E)
- V = N (number of courses)
- E = number of prerequisite pairs
- Building graph: O(E)
- BFS: each course visited once O(V), each edge once O(E)
- Total: **O(V + E)**

**Space Complexity:** O(V + E)
- Adjacency list: O(V + E)
- In-degree array: O(V)
- Queue: O(V) worst case
- Order array: O(V)
- Total: **O(V + E)**
