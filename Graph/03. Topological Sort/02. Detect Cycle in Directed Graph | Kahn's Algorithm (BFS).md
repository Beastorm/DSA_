# Detect Cycle in Directed Graph | Kahn's Algorithm (BFS)

### Problem Statement:

Given a Directed Graph with `V` vertices and `E` edges, check whether it contains any cycle or not using **BFS (Kahn's Algorithm)**.

---

## Examples

**Example 1:**
- **Input:** `V=5, E=5`
- **Output:** `True`
- **Explanation:** `2→3→4→2` is a cycle.
```
  0 ──→ 1 ──→ 2
              ↓   ↑
              3 ──→ 4 ──┐
              ↑          │
              └──────────┘  ← CYCLE: 2→3→4→2
```

**Example 2:**
- **Input:** `V=4, E=3`
- **Output:** `False`
- **Explanation:** DAG — no cycle.
```
  0 ──→ 1 ──→ 3
  0 ──→ 2 ──→ 3

  No back edges → No cycle ✓
```

---

## Key Insight
```
  Topological Sort is ONLY possible for DAGs (no cycles).

  Kahn's Algorithm performs topological sort by repeatedly
  removing nodes with in-degree = 0.

  If a cycle exists:
  → Nodes in the cycle ALWAYS have in-degree ≥ 1
  → They are NEVER added to the queue
  → They are NEVER processed
  → count < V at the end!

  Conclusion:
  ┌──────────────────────────────────────────────┐
  │ If count of processed nodes == V → NO cycle  │
  │ If count of processed nodes < V → CYCLE!     │
  └──────────────────────────────────────────────┘

  Example (cycle 2→3→4→2):
  In-degrees: indeg[2]≥1, indeg[3]≥1, indeg[4]≥1
  → nodes 2,3,4 never reach in-degree 0
  → never processed → count = 2 (only 0,1 processed) < 5 = V
  → CYCLE DETECTED ✓
```

---

## Algorithm
```
  hasCycle(V, adj):

  Step 1: Compute in-degrees
    indegree[v]++ for every edge u→v

  Step 2: Enqueue all nodes with in-degree = 0
    (these have no prerequisites → safe to process)

  Step 3: BFS
    count = 0
    While queue not empty:
        node = queue.front(); pop
        count++
        For each neighbor of node:
            indegree[neighbor]--
            If indegree[neighbor] == 0: push to queue

  Step 4: Check
    If count == V → no cycle (all nodes processed)
    If count <  V → cycle exists (some nodes stuck)
```

## Code
```cpp
#include <bits/stdc++.h>
using namespace std;

class Solution {
public:
    bool hasCycle(int V, vector<vector<int>>& adj) {

        // Step 1: Compute in-degree of each node
        vector<int> indegree(V, 0);
        for (int u = 0; u < V; u++) {
            for (int v : adj[u]) {
                indegree[v]++;
            }
        }

        // Step 2: Push all 0 in-degree nodes
        queue<int> q;
        for (int i = 0; i < V; i++) {
            if (indegree[i] == 0) q.push(i);
        }

        // Step 3: BFS — process nodes level by level
        int count = 0;
        while (!q.empty()) {
            int node = q.front();
            q.pop();
            count++; // One more node processed

            for (int neighbor : adj[node]) {
                indegree[neighbor]--;
                if (indegree[neighbor] == 0) {
                    q.push(neighbor);
                }
            }
        }

        // Step 4: If not all nodes processed → cycle exists
        return count != V;
    }
};

int main() {
    Solution obj;

    // Example 1: Cycle exists (2→3→4→2)
    int V1 = 5;
    vector<vector<int>> adj1 = {
        {1},    // 0→1
        {2},    // 1→2
        {3},    // 2→3
        {4},    // 3→4
        {2}     // 4→2 ← back edge!
    };
    cout << "Example 1: " << (obj.hasCycle(V1, adj1) ? "Cycle detected" : "No cycle") << endl;

    // Example 2: No cycle (DAG)
    int V2 = 4;
    vector<vector<int>> adj2 = {
        {1, 2},  // 0→1, 0→2
        {3},     // 1→3
        {3},     // 2→3
        {}       // 3 (terminal)
    };
    cout << "Example 2: " << (obj.hasCycle(V2, adj2) ? "Cycle detected" : "No cycle") << endl;

    return 0;
}
```

## Step-by-Step Trace

### Cycle Case: 2→3→4→2
```
  Graph: 0→1, 1→2, 2→3, 3→4, 4→2
  V=5

  ── Step 1: Compute In-Degrees ──
  0→1: indeg[1]++
  1→2: indeg[2]++
  2→3: indeg[3]++
  3→4: indeg[4]++
  4→2: indeg[2]++

  indegree = [0, 1, 2, 1, 1]
              0  1  2  3  4

  ── Step 2: Push in-degree=0 nodes ──
  Only node 0 has indegree=0
  queue = [0]

  ── Step 3: BFS ──

  Pop 0: count=1
    neighbor 1: indeg[1]=1→0 → push 1
    queue=[1]

  Pop 1: count=2
    neighbor 2: indeg[2]=2→1 (not 0, don't push)
    queue=[]

  Queue empty! count=2

  ── Step 4: Check ──
  count(2) ≠ V(5) → CYCLE EXISTS ✓

  Nodes 2,3,4 stuck in cycle → never processed!
```

### No Cycle Case (DAG)
```
  Graph: 0→{1,2}, 1→3, 2→3
  V=4

  ── Step 1: In-Degrees ──
  indegree = [0, 1, 1, 2]
              0  1  2  3

  ── Step 2: Push in-degree=0 nodes ──
  queue=[0]

  ── Step 3: BFS ──

  Pop 0: count=1
    neighbor 1: indeg[1]=1→0 → push 1
    neighbor 2: indeg[2]=1→0 → push 2
    queue=[1,2]

  Pop 1: count=2
    neighbor 3: indeg[3]=2→1
    queue=[2]

  Pop 2: count=3
    neighbor 3: indeg[3]=1→0 → push 3
    queue=[3]

  Pop 3: count=4
    no neighbors
    queue=[]

  ── Step 4: Check ──
  count(4) == V(4) → NO CYCLE ✓
```

## Visualization: Why Cycle Nodes Get Stuck
```
  Cycle: 2→3→4→2

  indegree initially:      After processing 0,1:
  2: indeg=2               2: indeg=1  (1 reduced it)
  3: indeg=1               3: indeg=1  (unchanged)
  4: indeg=1               4: indeg=1  (unchanged)

  Cycle's in-degrees can NEVER reach 0:
  2 needs 4 to reduce it → but 4 needs 3 → but 3 needs 2
  → DEADLOCK → nodes stuck → never processed → count < V ✓

  Cycle as a "deadlock":
  2 ──→ 3       Each waits for
  ↑      ↓       previous to be
  4 ←── ↵        processed first
  → circular dependency = cycle!
```

## Comparison: Three Cycle Detection Methods
```
  Method              Graph Type    Data Structure    Key Idea
  ──────────────────────────────────────────────────────────────
  BFS (Undirected)    Undirected    Queue + parent    vis + ≠ parent
  DFS (Directed)      Directed      Recursion         vis + pathVis
  Kahn's (BFS)        Directed      Queue + indegree  count < V

  Kahn's Advantage:
  → Simultaneously gives topological sort (if no cycle)
  → No extra arrays like pathVis needed
  → Easy to implement iteratively ✓
```

---

## Kahn's Dual Purpose
```
  Same code:

  hasCycle: return count != V (true = cycle)

  topologicalSort: return topo[] (the processed order)

  if (topo.size() == V) → valid topo sort (no cycle)
  if (topo.size() < V)  → cycle detected!

  Code reuse: cycle detection = incomplete topo sort check ✓
```

## Edge Cases
```
  No edges:               Self-loop:
  V=3, adj=[[],[],[]]     adj[0]=[0]

  indegree=[0,0,0]        indeg[0]=1 (self-loop)
  Push all → count=3      Push nothing → count=0 < 1
  count==V → No cycle ✓   CYCLE DETECTED ✓

  All disconnected:        Single node, no edges:
  V=4, no edges            V=1, adj=[[]]

  All indegree=0           Push 0, count=1=V
  count=4=V → No cycle ✓  No cycle ✓
```

## Complexity Analysis

**Time Complexity:** O(V + E)
- In-degree calculation: O(E)
- BFS: each vertex enqueued/dequeued once: O(V)
- Each edge processed once: O(E)
- Total: **O(V + E)**

**Space Complexity:** O(V + E)
- Adjacency list: O(V + E)
- In-degree array: O(V)
- BFS queue: O(V) worst case
- Total: **O(V + E)**
```
  Elegant property of Kahn's Algorithm:
  count == V → valid topological sort exists → NO cycle
  count <  V → topological sort impossible → CYCLE! ✓
```
