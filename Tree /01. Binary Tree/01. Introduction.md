## Tree Data Structures: Core Concepts

### 1. Hierarchical vs. Linear Data Structures
Unlike arrays, stacks, queues, and linked lists—which are all **linear data structures**—trees are **hierarchical**. 

> **Analogy:** A great way to visualize this hierarchy is to think about **computer folders**. You have an initial folder (root) that contains another folder, which might contain another folder inside of it, and so on.

### 2. The Meaning of "Binary"
The term "binary" is associated with 0 and 1. In this context, it means that any given node in the tree structure can have a **maximum of two children**. 
* A node can have **0, 1, or 2** children.
* It can never have more than two.

### 3. Important Tree Terminology
In addition to the **Root** and **Leaf** nodes, you should know these key relational terms:

* **Children:** The nodes that are located directly beneath another node.
* **Subtree:** If you select a specific node, that node and the entire portion of nodes beneath it make up its subtree.
* **Ancestors:** All the nodes directly above a specific node in its lineage. This includes a node's parent, grandparent, great-grandparent, and so on up to the root.

### 4. Extra Detail on Balanced Binary Trees
When calculating the maximum height of a **balanced binary tree**, the complexity is $log(N)$, where $N$ is the number of nodes. 

**Key Detail:** This is specifically a **base 2 log**.

**Example:**
If your tree has $N = 8$ nodes:
$$\log_2(8) = 3$$
This means the height of that tree cannot be greater than **3**.

## Binary Tree 
A **Binary Tree** is a hierarchical data structure where every "node" (or point) can have a maximum of two children (a left child and a right child)
. The very top node is called the root, and the nodes at the bottom with no children are called leaf nodes
.
Here is an easy breakdown of the 5 main types of binary trees, complete with simple text diagrams:
### 1. Full Binary Tree
In a full binary tree, every node has either 0 or exactly 2 children. No node is allowed to have just one child.   

**Diagram:**
```
      O       <- Root (has 2 children)
     / \
    O   O     <- Left node is a leaf (0 children). The right node has 2 children.
       / \
      O   O   <- Leaves (0 children)

```      
### 2. Complete Binary Tree
In this tree, all levels must be completely filled with nodes, except possibly the very last level. If the last level is not completely filled, all of its nodes must be placed as far left as possible.    
**Diagram:**
```
        O
      /   \
     O     O    <- Level 1 is full
    / \   /
   O   O O      <- Last level is not full, but all nodes are pushed to the LEFT.
```

### 3. Perfect Binary Tree
A perfect binary tree is perfectly symmetrical. All of the leaf nodes are on the exact same level, meaning every level of the tree is completely full.    
**Diagram:**
```
        O
      /   \
     O     O
    / \   / \
   O   O O   O  <- All leaf nodes are exactly on this same bottom level.
```

### 4. Balanced Binary Tree
A balanced binary tree is kept relatively short and wide rather than tall and skinny. Specifically, the maximum height of the tree is kept around log(N), where N is the total number of nodes. This structure is used to make searching for data much faster and more efficient.

### 5. Degenerate (or Skewed) Tree
This is a tree where every single node has only one child. Because it just forms a single straight line, it essentially looks and acts exactly like a linked list.   
**Diagram:**
```
   O      <- Root
    \
     O    <- 1 child
      \
       O  <- 1 child
```
